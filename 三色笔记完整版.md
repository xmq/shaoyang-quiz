## Office软件操作

> > 只用于考前速记：🔴核心必背；🔵公式、步骤、快捷键与踩分词；⚫条件、版本差异与判断陷阱。Microsoft Office 与 WPS Office 采用共性主线；题干指定软件、版本或操作系统时，以指定环境为准。

### 一、Office 共性基础

- 🔴 组件对应：Microsoft Word ↔ WPS 文字，Microsoft Excel ↔ WPS 表格，Microsoft PowerPoint ↔ WPS 演示；核心对象和操作逻辑相近，但界面入口和特色功能不必完全相同。
- 🔴 两套软件均可处理常见 `.docx/.xlsx/.pptx` 并导出 PDF；复杂宏、域、字体、动画、图表、公式和插件跨软件时可能不完全兼容。
- 🔴 Office 文档的基本操作是新建、打开、保存、另存为、关闭、打印和导出；“保存”更新当前文件，“另存为”可改变位置、名称或格式。
- 🔴 Word、Excel、PowerPoint 的现代默认格式分别是 `.docx`、`.xlsx`、`.pptx`；旧格式分别常见 `.doc`、`.xls`、`.ppt`。
- 🔴 含 VBA 宏的现代格式常见 `.docm`、`.xlsm`、`.pptm`；模板常见 `.dotx/.dotm`、`.xltx/.xltm`、`.potx/.potm`。
- 🔵 高频快捷键：`Ctrl+N` 新建，`Ctrl+O` 打开，`Ctrl+S` 保存，`F12` 常用于另存为，`Ctrl+P` 打印，`Ctrl+Z/Y` 撤销/恢复，`Ctrl+X/C/V` 剪切/复制/粘贴，`Ctrl+A` 全选，`Ctrl+F/H` 查找/替换。
- 🔵 跨软件答题：写功能目标和对象 → 写共性步骤 → 按题干替换为 Word/Excel/PowerPoint 或 WPS 文字/表格/演示的菜单名称 → 检查兼容与输出。
- ⚫ 自动保存、自动恢复与手动保存不是同一概念；恢复文件不能代替规范保存和备份。
- ⚫ PDF 通常用于保持版式和分发，不等同于可完整保留源文档所有编辑结构；兼容模式可能限制新功能。
- ⚫ Microsoft 365、Office 2016/2019/2021 与 WPS 个人版/教育版/专业版的默认值、云协作、宏和高级功能存在差异；不能把某版按钮位置写成绝对结论。

### 二、Word：编辑与格式

- 🔴 字符格式包括字体、字号、字形、颜色、字符间距和文字效果；段落格式包括对齐、缩进、段前段后、行距、制表位和项目符号/编号。
- 🔴 `Backspace` 通常删插入点左侧字符，`Delete` 通常删右侧字符；文本已选中时二者均可删除所选内容。
- 🔴 格式刷复制格式而非正文；单击通常使用一次，双击可连续应用，按 `Esc` 退出。
- 🔵 常用快捷键：`Ctrl+B/I/U` 加粗/倾斜/下划线，`Ctrl+L/E/R/J` 左/居中/右/两端对齐，`Ctrl+Enter` 插入分页符，`Shift+Enter` 插入手动换行符。
- 🔵 清除直接格式后可回到样式定义；统一长文档应优先使用“样式”，而非逐段手工改字体。
- ⚫ 中文字号中“初号”大于“一号”，“二号”大于“四号”；以磅为单位时数值越大字体越大，两套序列不能按数字直接比较。
- ⚫ 固定值行距可能遮挡过大的文字；多倍行距会随字体等因素计算。

### 三、Word：页面、节与页眉页脚

- 🔴 分页符只强制从下一页开始；分节符建立新“节”，不同节可分别设置纸张方向、页边距、分栏、页码格式和页眉页脚。
- 🔴 分节符常见“下一页、连续、奇数页、偶数页”；“连续”分节可在同页改变分栏等设置。
- 🔵 设置不同页眉页脚：插入分节符 → 进入后一节页眉/页脚 → 关闭“链接到前一节” → 分别编辑 → 必要时设置页码起始值。
- 🔵 只需同一节“首页不同”或“奇偶页不同”时，勾选相应选项即可，不必一定新增分节符。
- 🔴 页面设置高频项：纸张大小、方向、页边距、装订线、文字方向、分栏和应用范围。
- ⚫ 删除分节符会使相邻节合并，并可能改变前一部分版式；分页符不能代替分节符。

### 四、Word：对象、表格与长文档

- 🔴 嵌入型图片按字符参与排版；四周型、紧密型、衬于文字下方、浮于文字上方等属于浮动环绕。
- 🔴 组合对象通常要求对象为浮动型；图片的“裁剪”改变显示区域，不等于删除源图像数据。
- 🔴 表格中按 `Delete` 通常只清除内容；删除单元格、行、列或整表才改变结构。
- 🔵 Word 表格可用 `=SUM(ABOVE)`、`=AVERAGE(LEFT)` 等公式；源数据改变后常需选中结果按 `F9` 更新域。
- 🔴 标题样式是自动目录、大纲导航和多级编号的基础；手工加粗放大不等于应用标题样式。
- 🔵 长文档流程：定义样式 → 应用标题层级 → 插入分页/分节 → 设置题注和交叉引用 → 插入目录 → 全选后更新域。
- 🔴 邮件合并由主文档、数据源和合并域组成，可批量生成信函、标签、证书等。
- 🔴 修订记录正文增删和格式变化；批注用于讨论。删除批注不等于接受修订。
- ⚫ 自动目录内容或页码变化后要“更新目录”；不要手工逐条改目录。

### 五、Excel：对象、输入与引用

- 🔴 工作簿是文件，工作表是工作簿内的表页；单元格地址先列后行，如 `B3`。
- 🔴 Excel 2007 及以后每表通常为 1,048,576 行、16,384 列，末列 `XFD`；旧版 Excel 2003 为 65,536 行、256 列。
- 🔴 公式以 `=` 开头；长学号、身份证号等应按文本输入，可先设文本格式或在前面输入英文单引号。
- 🔴 相对引用 `A1` 随复制改变；绝对引用 `$A$1` 行列均锁定；混合引用 `$A1` 锁列、`A$1` 锁行。
- 🔵 编辑公式时 `F4` 可循环切换引用形式；跨表引用示例：`'期末成绩'!$B$2`。
- 🔵 填充公式前先问两件事：哪个量随位置变化，哪个量必须锁定；再决定 `$` 放在行、列还是二者。
- ⚫ `######` 最常见于列宽不足，也可能因负日期/时间等；不能一概判定数据丢失。
- ⚫ 工作表默认数量和可添加数量随版本、设置与内存变化，“永远是 3 张”“最多 255 张”均不严谨。

### 六、Excel：公式、函数与错误值

- 🔴 运算顺序以括号最可靠；常见顺序为引用运算、负号/百分号、乘方、乘除、加减、连接 `&`、比较。
- 🔵 基本统计：`SUM` 求和，`AVERAGE` 平均，`MAX/MIN` 极值，`COUNT` 数值个数，`COUNTA` 非空个数。
- 🔵 条件函数：`IF(条件,真值,假值)`；`COUNTIF/COUNTIFS` 条件计数；`SUMIF/SUMIFS` 条件求和；`AVERAGEIF(S)` 条件平均。
- 🔵 常考函数：`ROUND` 四舍五入，`LEFT/RIGHT/MID` 截取文本，`LEN` 字符数，`TODAY/NOW` 日期时间，`RANK.EQ` 排名，`VLOOKUP/XLOOKUP` 查找。
- 🔵 多条件判断常用 `AND`、`OR` 嵌入 `IF`；文本条件要加英文双引号，如 `">=60"`。
- 🔵 查找踩分：明确“查找值、查找区域、返回列/数组、精确或近似匹配”；传统 `VLOOKUP` 精确匹配最后参数写 `FALSE` 或 `0`。
- 🔴 常见错误：`#DIV/0!` 除零，`#VALUE!` 类型不合，`#NAME?` 名称无法识别，`#REF!` 引用失效，`#N/A` 无可用匹配，`#SPILL!` 动态数组溢出区受阻。
- ⚫ `COUNT` 不统计普通文本；`COUNTIF` 条件区域与 `SUMIF` 求和区域含义不同；函数名称或功能可能随版本新增。

### 七、Excel：数据管理、图表与打印

- 🔴 排序可按多个关键字；必须扩展选择整条记录，否则会破坏同行字段对应关系。
- 🔴 筛选只是暂时隐藏不符合条件的记录，不删除数据；高级筛选可使用条件区域并把结果复制到其他位置。
- 🔴 传统分类汇总前先按分类字段排序；数据透视表用于按字段分组、汇总和交叉分析，源数据变化后通常要刷新。
- 🔴 数据验证可限制类型、范围或提供下拉列表；条件格式按规则改变显示格式，二者均不等于加密保护。
- 🔵 图表选择：类别比较用柱形/条形；时间趋势用折线；少量类别构成用饼图；两个数值变量关系用散点图。
- 🔵 图表基本元素：图表标题、坐标轴、图例、数据系列、数据标签、网格线；答操作题时先选数据区域再插入合适图表。
- 🔴 冻结窗格用于滚动时保持标题可见，不是禁止编辑；锁定单元格只有在保护工作表后才发挥限制作用。
- 🔵 打印检查：打印区域 → 纸张方向/页边距 → 缩放 → 分页 → 重复标题行/列 → 页眉页脚 → 打印预览。
- ⚫ 普通图表通常随既有源区域数据变化而更新，但新增记录若不在引用区域内，未必自动纳入。

### 八、PowerPoint：结构、视图与母版

- 🔴 普通视图用于编辑；幻灯片浏览视图适合调整顺序和整体检查；备注页用于编辑演讲者备注；阅读视图在窗口中放映。
- 🔴 主题控制整体颜色、字体和效果；版式规定占位符结构；母版统一控制多个版式及其默认格式。
- 🔵 全局统一格式：视图 → 幻灯片母版 → 修改母版/相应版式 → 关闭母版视图；不要逐页手工修改。
- 🔴 新建幻灯片常继承当前版式；复制幻灯片可选择使用目标主题或保留源格式。
- 🔴 占位符属于版式结构，可容纳文本或内容；文本框是单独插入的对象，二者不能完全等同。
- ⚫ 修改母版不会必然覆盖幻灯片上已直接设置的全部格式；直接格式可能优先显示。

### 九、PowerPoint：对象、动画与放映

- 🔴 幻灯片切换作用于页面之间；动画作用于页面内对象。一个对象可设置多个动画，但要管理顺序和触发方式。
- 🔴 动画类型通常分进入、强调、退出、动作路径；开始方式常见“单击时、与上一动画同时、上一动画之后”。
- 🔵 动画设置流程：选对象 → 添加动画 → 设置效果选项 → 调整动画窗格顺序 → 设置开始/持续时间/延迟 → 预览。
- 🔴 超链接可跳转到网页、文件、邮件地址或指定幻灯片；动作按钮还可在单击/鼠标移过时执行跳转或声音。
- 🔵 `F5` 从头放映，`Shift+F5` 从当前页放映，`Esc` 退出；排练计时记录每页时间，录制可包含旁白、墨迹和计时。
- 🔴 插入音视频后应检查文件是否嵌入、兼容、可播放以及放映起止方式；“压缩媒体”可减小文件但可能损失质量。
- 🔴 讲义母版控制打印讲义版式；备注母版控制备注页，不要与幻灯片母版混淆。
- 🔵 演示设计踩分词：一页一重点、层级清楚、对比充分、字体可读、图表服务结论、动画克制、来源标注。
- ⚫ 动画和转场越多不等于效果越好；自动换片与单击换片可同时设置，实际行为取决于放映设置。

### 十、综合操作题答题法

- 🔵 Word：先辨“字符/段落/页面/节/对象/样式”，再选命令；涉及不同页设置优先想到“分节”。
- 🔵 Excel：先写目标 → 确定数据区域 → 写公式 → 判断引用是否锁定 → 填充 → 检查边界值和错误值。
- 🔵 PowerPoint：先辨“单页对象”还是“全局版式”；全局改外观优先考虑主题/母版，页面间效果用切换，页内对象效果用动画。
- 🔵 操作题踩分表述：选定对象 → 打开对应选项卡/对话框 → 设置具体参数 → 确定/应用 → 检查应用范围。
- ⚫ 菜单位置会随版本和 WPS/Microsoft Office 改变；期末主观题优先写“功能名称和操作逻辑”，不要只背图标位置。

## 信息技术与教学论

> > 只用于考前速记：🔴核心必背；🔵模型、步骤与简答踩分词；⚫适用条件、辨析和教材口径。义务教育以《义务教育信息科技课程标准（2022年版）》为主，高中以《普通高中信息技术课程标准（2017年版2020年修订）》为主。

### 一、课程性质、理念与核心素养

- 🔴 义务教育课程名称为“信息科技”，具有基础性、实践性和综合性；高中课程名称为“信息技术”。
- 🔴 信息科技不是单纯的软件操作课，既学习数据、算法、网络和系统等原理，也发展利用技术解决问题的能力和社会责任。
- 🔴 四项核心素养：信息意识、计算思维、数字化学习与创新、信息社会责任。
- 🔵 信息意识踩分词：信息敏感度、价值判断、来源可靠性、时效性、主动获取与利用。
- 🔵 计算思维踩分词：分解、抽象、建模、算法设计、自动化、评价与优化。
- 🔵 数字化学习与创新踩分词：工具选择、资源管理、协同学习、作品创作、迁移创新。
- 🔵 信息社会责任踩分词：法律伦理、隐私安全、知识产权、文明交流、负责任发布。
- ⚫ 计算思维不等于只会编程；用了电脑不等于实现数字化学习；核心素养四项相互支持，不能割裂。

### 二、主要学习理论

- 🔴 行为主义关注刺激—反应、练习、强化和可观察行为，适合基础操作训练与即时反馈。
- 🔴 认知主义关注信息加工、认知结构、先备知识和策略，可用先行组织者、分块、概念图和精加工。
- 🔴 建构主义强调学习者在情境、协作、会话和资源支持下主动建构意义，教师提供支架与反馈。
- 🔴 人本主义重视完整人格、情感、需要、自我实现和有意义学习，强调尊重与选择。
- 🔴 社会学习理论强调观察学习、榜样、替代性强化和自我效能；情境学习强调知识与真实活动、共同体和文化情境相联系。
- 🔴 最近发展区：学生独立水平与在帮助下可达到水平之间的区域；支架应随能力提高逐步撤除。
- 🔵 理论匹配：操作熟练—行为主义；知识结构—认知主义；真实项目协作—建构主义；情感需要—人本主义。
- ⚫ “学生主体”不等于教师退出；发现学习也不是完全不给指导。

### 三、教学设计的系统观与模型

- 🔴 教学设计是以学习目标达成为中心，对学习者、内容、活动、资源、环境和评价进行系统规划并持续改进。
- 🔵 通用流程：需求/课标分析 → 学习者与内容分析 → 制定目标 → 设计活动、策略、资源和评价 → 开发资源 → 实施 → 反馈修订。
- 🔴 ADDIE：Analysis 分析 → Design 设计 → Development 开发 → Implementation 实施 → Evaluation 评价；评价可贯穿全过程。
- 🔴 教—学—评一致性：目标规定学什么，活动提供学习经历，评价收集目标达成证据，三者必须对应。
- 🔵 逆向设计：确定预期结果 → 确定可接受证据 → 设计学习经历与教学活动。
- 🔵 ASSURE 常见骨架：分析学习者 → 陈述目标 → 选择方法、媒体和材料 → 使用媒体材料 → 要求学习者参与 → 评价与修订。
- ⚫ 不同教材可能采用 ADDIE、迪克—凯瑞、肯普或本土流程，名称不同但都强调分析、目标、策略、实施、评价和修订。

### 四、学习者、内容与教学目标分析

- 🔴 学习者分析：先备知识技能、认知发展、动机兴趣、学习经验、个体差异、设备条件、特殊学习需要。
- 🔵 获取证据：前测、访谈、问卷、课堂观察、作品分析和学习数据；不能只凭教师印象。
- 🔴 内容分析要确定知识结构、重点难点、先后关系、任务类型及其与课标和生活情境的联系。
- 🔴 教学重点由课标要求、内容结构和育人价值决定；教学难点由内容特性与具体学习者基础共同决定。
- 🔴 目标应面向核心素养，并转化为可观察、可评价的学习表现。
- 🔵 ABCD：A 学习者，B 可观察行为，C 行为条件，D 达成标准；常用动词为识别、解释、比较、设计、调试、评价、创作。
- 🔵 目标表述示例骨架：“在给定……条件下，学生能……，达到……标准”。
- ⚫ “了解、掌握、提高、培养”若不说明行为证据，难直接评价；重点不必然是难点。

### 五、教学策略与常用方法

- 🔴 讲授法适合系统解释概念和原理，效率高；应结合提问、案例、可视化和练习。
- 🔴 演示练习法：教师分段示范并口述关键思路，学生模仿、独立练习，再迁移到新任务。
- 🔴 任务驱动：把知识技能融入有意义任务，学生在分析、实施和评价任务中学习。
- 🔵 任务驱动流程：创设情境 → 明确任务 → 分析并形成方案 → 自主/协作实施 → 展示交流 → 评价反思与迁移。
- 🔵 好任务踩分词：真实、目标一致、难度适切、层次递进、成果可评、安全可行、允许选择。
- 🔴 项目式学习围绕真实复杂问题持续开展规划、调研、设计、制作、测试、迭代和公开展示，周期通常长于单一任务。
- 🔵 项目流程：驱动性问题 → 规划分工 → 调研设计 → 制作测试 → 迭代改进 → 展示答辩 → 反思评价。
- 🔴 探究学习常经历提出问题、作出假设、收集分析证据、形成解释、交流评价。
- 🔴 合作学习要有共同目标、积极相互依赖、个体责任、促进性互动、合作技能和小组反思。
- 🔵 防“搭便车”：角色明确、个人任务可追踪、随机汇报、同伴互评、个人反思、过程记录。
- ⚫ 围坐在一起不等于合作；任务驱动不等于照着教师步骤操作；项目式学习不等于制作一个作品。

### 六、信息科技课堂教学组织

- 🔵 上机课流程：设备/账号检查 → 明确目标与安全规则 → 发布任务/分段示范 → 学生实践和教师巡视 → 保存提交 → 展示评价 → 退出账号并整理设备。
- 🔴 课堂管理同时关注人员、设备、账号、文件、网络、时间和数据安全。
- 🔵 故障处理：先保人身和数据安全 → 判断是否为个别/普遍故障 → 查电源、连接、账号、网络、软件和路径 → 启用离线资源/备用方案 → 记录复盘。
- 🔴 差异化教学可设置基础、进阶、挑战任务；为困难学生提供步骤卡、示例、半成品或同伴支架，为高水平学生提供开放拓展。
- 🔵 教师巡视看“过程证据”：任务理解、操作策略、调试记录、协作质量和安全规范，而不只看最后作品。
- ⚫ 技术故障不能简单转成无目标自由活动；教师演示过长会挤压学生实践时间。

### 七、信息化教学形态

- 🔴 WebQuest 是以网络资源为主要信息来源的探究活动，经典结构包括导言、任务、过程、资源、评价和结论。
- 🔴 微课围绕相对聚焦的目标，以短小数字资源为核心并配套学习活动；视频是常见载体，但微课不等于一段视频。
- 🔴 翻转课堂把部分知识获取前移，课堂更多用于问题解决、协作、反馈和个别指导。
- 🔴 混合式学习把线上与线下、同步与异步活动按目标进行有机整合，不是简单叠加平台和课堂。
- 🔴 移动学习突出便携、泛在、情境和碎片化支持；仍要防注意分散、数字鸿沟和隐私风险。
- 🔵 翻转设计：课前资源+检测 → 教师分析学情 → 课中问题解决/协作 → 即时反馈 → 课后巩固反思。
- ⚫ 微课“5～10 分钟”是常见经验而非绝对法规；翻转不等于课前看视频、课上只做题；线上资源多不等于混合质量高。

### 八、教学媒体与数字资源

- 🔴 媒体选择首先服从目标、内容、学习者、策略和环境，再考虑可获得性、成本、易用性和安全版权。
- 🔵 媒体选择踩分词：目标匹配、表征优势、认知负荷、交互反馈、无障碍、技术条件、成本效益。
- 🔴 多媒体学习应减少无关信息，突出关键线索，把相互关联的文字和图形邻近呈现，并允许适当分段和学习者控制。
- 🔴 资源评价：科学性、教育性、技术性、艺术性、交互性、可用性、安全性和版权合规。
- 🔵 资源开发流程：需求分析 → 教学设计 → 脚本/原型 → 素材采集制作 → 集成 → 测试评价 → 发布维护。
- ⚫ 动画、声音越多不一定越好；装饰性信息可能增加外在认知负荷；网络资源可以引用但必须核验来源和授权。

### 九、教学评价基本分类

- 🔴 诊断性评价用于教学前了解起点与困难；形成性评价用于教学中反馈改进；总结性评价用于阶段结束判定达成。
- 🔴 标准参照评价把表现与既定标准比较；常模参照评价与群体比较；个体内差异评价与本人过去比较。
- 🔴 定量评价用分数、频数等数据；定性评价用描述、观察和解释；应根据目标结合使用。
- 🔴 自评、互评、教师评价和社会/专家评价属于评价主体多元化。
- 🔵 信息科技证据：纸笔测验、上机任务、程序运行、作品、设计文档、过程日志、答辩和档案袋。
- ⚫ 同一测验因使用目的不同，可成为诊断、形成或总结性评价；不能只看形式下结论。

### 十、评价工具与测验质量

- 🔴 量规 Rubric 把评价维度及各水平表现描述清楚；维度必须对应目标，水平描述应可观察、可区分。
- 🔵 作品量规常用维度：功能正确、技术/算法合理、创意表达、规范安全、协作过程、说明反思。
- 🔴 信度是结果的一致性和稳定性；效度是是否测到预期目标；信度高是效度高的必要非充分条件。
- 🔴 难度指数若采用通过率 `P=答对人数/总人数`，P 越大题越容易；区分度反映题目区分不同水平学习者的能力。
- 🔵 客观题命制：题干独立清楚、一个最佳答案、干扰项同质且有迷惑性、避免暗示和无意线索。
- 🔵 形成性反馈应及时、具体、面向任务和改进策略，回答“哪里好、哪里需改、下一步怎么做”。
- ⚫ 不同教材对“难度值”可能采用相反定义，计算前先看公式；判断题中的“永远、完全、只能”常是陷阱但不能机械判错。

### 十一、案例分析与教学设计题

- 🔵 案例分析四步：判断做法 → 引用课标/理论依据 → 分析对学习的影响 → 提出可执行改进并说明评价证据。
- 🔵 教学设计答题骨架：课标与学情 → 素养目标 → 重点难点 → 情境任务 → 活动流程 → 资源支架 → 评价量规 → 安全伦理 → 反思改进。
- 🔵 高频得分词：学生主体、真实情境、问题解决、支架、合作与个体责任、分层任务、教—学—评一致、过程性证据。
- 🔴 技术伦理必须写：个人信息最小必要、账号权限、内容审核、知识产权、网络文明、数据备份与安全退出。
- ⚫ 只写“激发兴趣、活跃气氛、培养能力”而无具体行为和证据，通常难得高分；“教师包办操作、只评价成品外观、目标不可测”是高频反例。

### 十二、说课、听评课与教学研究

- 🔴 说课通常说明教什么、为什么教、怎么教、如何评价，内容包括教材/课标、学情、目标、重点难点、教法学法、过程、评价和反思。
- 🔴 听评课关注目标达成、学习活动、师生互动、资源使用、课堂生成、评价反馈和学生真实学习证据。
- 🔵 评课表述：先呈现观察证据 → 对照目标/理论解释 → 指出影响 → 给出具体建议；避免只给笼统好坏评价。
- 🔴 行动研究常见循环：计划 → 行动 → 观察 → 反思 → 再计划；用于教师在真实课堂中持续改进。
- ⚫ 教案是教学实施方案，说课是对方案依据和思路的解释，二者不能等同。

## 多媒体技术

> > 只用于考前速记：🔴核心必背；🔵公式、流程、参数与踩分词；⚫条件、格式差异和判断陷阱。容量计算默认按题干给出的进制；未说明时写清采用 `1 B=8 bit`、`1 KiB=1024 B`。

### 一、多媒体基本概念

- 🔴 媒体常见含义包括感觉媒体、表示媒体、显示媒体、存储媒体和传输媒体；“多媒体技术”主要研究文本、图形、图像、声音、动画和视频等数字媒体的获取、处理、集成与传播。
- 🔴 多媒体主要特征：多样性/多维性、集成性、交互性、数字化和实时性；交互性是区别于传统单向复合媒体的重要特征。
- 🔴 超文本用链接组织文本节点；超媒体把链接对象扩展到图像、音频、视频等多种媒体，结构通常为非线性网络。
- 🔴 多媒体系统层次：硬件平台 → 驱动/操作系统 → 媒体处理软件 → 创作工具 → 多媒体应用。
- 🔵 系统评价踩分词：信息表现力、交互性、响应时间、同步性、压缩效率、兼容性、可用性。
- ⚫ “多种媒体同时出现”不自动等于完整多媒体系统；还要看数字集成、处理和交互能力。

### 二、数字化基础：采样、量化、编码

- 🔴 模拟信号时间和幅值连续；数字信号用离散符号表示。数字化基本过程是采样 → 量化 → 编码。
- 🔴 采样使时间离散；量化使幅值离散；编码把量化结果表示成二进制码。
- 🔴 采样定理：为无失真恢复带限信号，采样频率应至少为最高频率的 2 倍，即 `f_s ≥ 2f_max`。
- 🔵 量化位数为 `n` 时量化级数 `L=2^n`；位数越高动态范围和精度通常越好，数据量也越大。
- 🔵 数字化通用数据率：`采样点数/秒 × 每点位数 × 通道数`；总容量：`数据率 × 时间 ÷ 8`。
- ⚫ 满足奈奎斯特条件只是理论重建条件，实际系统还需抗混叠滤波并留过渡带；采样率越高不代表听感或画质无限提升。

### 三、数字图像基础

- 🔴 位图由像素阵列构成，适合照片和复杂色调；矢量图由数学对象描述，适合标志、图形和工程图，缩放通常不失真。
- 🔴 图像基本参数：像素尺寸、空间分辨率、颜色深度、颜色模式和压缩方式。
- 🔵 未压缩位图容量：`宽度像素 × 高度像素 × 每像素位数 ÷ 8`。
- 🔵 可表示颜色数：颜色深度 `n bit` 时最多 `2^n` 种；24 位真彩色常为 R/G/B 各 8 位。
- 🔴 分辨率含义要看语境：像素尺寸是总像素行列；PPI 表示每英寸像素数；DPI 多用于输出设备墨点密度。
- 🔵 印刷像素估算：`像素数 ≈ 印刷尺寸(英寸) × PPI`；英寸换算 `1 inch=2.54 cm`。
- ⚫ 改 PPI 而不重采样，像素总数不变，只改变预期打印尺寸；重采样才会增删像素。

### 四、颜色模型与图像处理

- 🔴 RGB 是加色模型，主要用于显示设备；三基色全强为白，全零为黑。
- 🔴 CMYK 是减色印刷模型，主要用于印刷输出；C/M/Y 为青/品红/黄，K 为黑。
- 🔴 HSB/HSV 用色相、饱和度、亮度/明度描述，符合人工调色直觉；YCbCr/YUV 将亮度与色度分离，便于视频压缩。
- 🔵 常见处理：裁剪、缩放、旋转、色阶/曲线、亮度对比度、色彩平衡、锐化、模糊、去噪。
- 🔴 直方图反映各亮度等级像素数量；偏左常表示暗部多，偏右常表示亮部多，但不能仅凭直方图断言曝光一定错误。
- 🔵 插值：最近邻速度快且保硬边；双线性较平滑；双三次通常质量更高但计算量更大。
- ⚫ RGB 到 CMYK 存在色域差异，屏幕颜色不保证印刷完全一致；锐化不会恢复真正丢失的细节。

### 五、图像格式、压缩与 Photoshop 高频点

- 🔴 BMP 常为无压缩或简单压缩位图；JPEG 采用有损压缩，适合连续色调照片；PNG 使用无损压缩并支持透明；GIF 最多常见 256 色并支持简单动画；TIFF 适合高质量存档/出版；SVG 是矢量格式。
- 🔴 有损压缩删除或近似不敏感信息，压缩比高但不可完全恢复；无损压缩可恢复原数据，适合文本、程序、图标和中间素材。
- 🔵 压缩比常写为 `原始数据量/压缩后数据量`；码率越高通常质量越好、文件越大，但还受编码器和内容复杂度影响。
- 🔴 图层用于独立组织对象；选区限定编辑范围；蒙版用灰度控制显示与隐藏；通道存储颜色分量或选区信息；路径是矢量轮廓。
- 🔵 Photoshop 非破坏流程：保留原图 → 复制/智能对象 → 调整图层 → 图层蒙版 → 分层保存 PSD → 按用途导出。
- 🔴 图层蒙版中通常白色显示、黑色隐藏、灰色半透明；蒙版可逆，直接橡皮擦删除像素通常破坏性更强。
- 🔵 高频工具：移动、选框/套索、快速选择、裁剪、吸管、画笔、仿制图章、修复、渐变、文字、钢笔。
- ⚫ JPEG 不支持常规透明通道和图层；导出 PNG/JPEG 往往会合并图层，PSD 才适合保留编辑结构。

### 六、数字音频

- 🔴 音频质量主要受采样频率、量化位数、通道数和编码方式影响；常见 CD 规格为 44.1 kHz、16 bit、双声道。
- 🔵 未压缩 PCM 音频容量：`采样频率 × 量化位数 × 通道数 × 时间 ÷ 8`。
- 🔵 示例结构：`44 100 × 16 × 2 × 秒数 ÷ 8` 字节；换成 MiB 再除 `1024²`。
- 🔴 波形音频记录声音采样数据；MIDI 记录音符、力度、音色和控制事件，不直接记录真实声波。
- 🔴 WAV 是容器，常装 PCM 也可装压缩数据；MP3/AAC 常为有损编码；FLAC 为无损压缩；MIDI 文件通常很小但回放依赖音源。
- 🔵 音频处理：录制 → 降噪/剪辑 → 均衡 → 动态处理 → 混音 → 归一化/限制 → 按用途编码导出。
- 🔴 信噪比越大通常噪声相对越小；分贝是对数量，功率比 `10lg(P_2/P_1)`，幅度比在同阻抗下常用 `20lg(A_2/A_1)`。
- ⚫ 提高采样率不能把低质量源文件变成真实高保真；“WAV 必然无压缩”“MIDI 是录音”均错误。

### 七、动画技术

- 🔴 动画利用视觉暂留，通过连续画面产生运动感；基本参数有帧、帧率、时间轴、关键帧和补间。
- 🔴 逐帧动画每个关键画面分别制作，控制精细但工作量大；补间动画由系统生成中间状态。
- 🔴 二维补间常见运动补间和形状补间；形状补间用于矢量形状变化，运动补间用于对象位置、缩放、旋转和透明度变化。
- 🔵 动画时长：`总帧数/帧率`；总帧数：`时长×帧率`。
- 🔴 元件/符号可复用，实例是舞台上的具体引用；修改元件通常影响其多个实例，实例可有独立位置、缩放等属性。
- 🔵 动画设计流程：脚本/分镜 → 素材和角色 → 关键姿势 → 补间/逐帧 → 时间节奏 → 声画同步 → 测试导出。
- ⚫ 帧率越高运动通常越平滑，但计算和容量增加；“关键帧”不等于每一帧都有新图像。

### 八、数字视频

- 🔴 视频由按时间排列的图像帧及音频等组成；关键参数有分辨率、帧率、扫描方式、色度采样、位深、码率和编码格式。
- 🔵 未压缩视频近似容量：`宽×高×每像素位数×帧率×时间÷8`，再加音频；实际压缩视频通常按码率计算。
- 🔵 压缩视频容量：`总码率(bit/s)×时间(s)÷8`；音视频分别给码率时先相加。
- 🔴 帧内压缩利用单帧空间冗余；帧间压缩利用前后帧时间冗余，常见 I 帧可独立解码，P/B 帧参考其他帧。
- 🔴 编解码器如 H.264/AVC、H.265/HEVC、AV1 决定压缩方法；容器如 MP4、MKV、MOV、AVI 负责封装音视频、字幕和元数据。
- 🔴 流媒体边接收边播放，可采用点播或直播；关键指标有启动时延、缓冲、吞吐、抖动和自适应码率。
- 🔵 视频制作流程：策划脚本 → 拍摄/采集 → 素材整理 → 粗剪 → 精剪 → 字幕/特效/调色 → 混音 → 编码导出 → 质量检查。
- ⚫ `.mp4` 只说明容器，不唯一决定编码器；分辨率相同不代表质量或容量相同；码率、帧率、内容复杂度均有影响。

### 九、多媒体压缩原理

- 🔴 数据冗余：空间冗余、时间冗余、统计/编码冗余、结构冗余、知识冗余和视觉/听觉心理冗余。
- 🔴 无损常见思想：游程编码、霍夫曼编码、算术编码、字典编码、预测后对残差编码。
- 🔴 JPEG 基本思想：颜色变换/色度抽样 → 分块 → DCT → 量化 → 熵编码；量化是主要有损环节。
- 🔴 视频编码：预测 → 变换 → 量化 → 熵编码，并结合运动估计/补偿降低时间冗余。
- 🔵 评价压缩：压缩比、码率、失真/客观质量、主观质量、编码复杂度、解码时延和容错性。
- ⚫ 压缩比越大不必然越好，必须同时满足质量、实时性和兼容要求；重复有损转码会累积损伤。

### 十、多媒体创作与交互设计

- 🔴 多媒体创作工具常按组织方式分为基于页/卡片、基于图标/流程、基于时间轴和基于程序语言。
- 🔴 创作流程：需求分析 → 教学/内容设计 → 脚本与结构设计 → 素材制作 → 集成开发 → 测试评价 → 发布维护。
- 🔵 脚本要说明页面编号、画面内容、解说/声音、交互行为、跳转关系、素材来源和时长。
- 🔴 导航结构常见线性、层次、网状和复合结构；应提供当前位置、返回、首页和退出等清晰反馈。
- 🔴 人机交互原则：一致性、可见性、及时反馈、用户控制、容错、减少记忆负担、无障碍和最少必要操作。
- 🔵 测试类型：功能、内容、兼容、性能、可用性、安全、安装发布；发现问题后记录“环境—步骤—现象—期望结果”。
- ⚫ 技术效果必须服务信息表达；过多跳转、自动播放和装饰动画会增加认知负荷并损害可用性。

### 十一、存储、传输与版权安全

- 🔴 多媒体系统需要处理大容量、实时连续和多流同步问题；存储与传输设计要考虑带宽、时延、抖动、缓存和容错。
- 🔵 声画同步：时间戳 → 缓冲 → 按播放时钟调度；实时系统出现数据不足时可能卡顿，缓存过大则增加延迟。
- 🔴 元数据描述文件属性、作者、拍摄时间、编码参数等；内容管理还要维护版本、标签、检索和权限。
- 🔴 使用网络图片、字体、音乐和视频必须确认授权范围，按要求署名；个人信息和肖像应取得合法授权并最小化处理。
- 🔵 发布检查：版权/隐私 → 字体与媒体打包 → 链接有效 → 编码兼容 → 清晰度/码率 → 字幕与无障碍 → 多设备测试。
- ⚫ “网上能下载”不等于可自由商用或修改；压缩、改色、截取也不会自动消除著作权义务。

### 十二、计算题统一写法

- 🔵 第一步找对象：图像看像素和位深；音频看采样率、位深、通道、时长；视频看未压缩参数或总码率。
- 🔵 第二步统一单位：bit 与 B 相差 8；秒、分钟先换算；KB/MiB 按题干采用 1000 或 1024。
- 🔵 第三步写公式和代入，保留单位；第四步做数量级检查并说明是否未压缩/忽略文件头。
- ⚫ 压缩后容量若只给“压缩比”，先确认定义是 `原/压缩后` 还是节省百分比；若给码率，通常不再乘像素和位深。

## 操作系统原理

> > 定位：考前速记，不代替零基础讲义。🔴背结论与术语；🔵背计算、算法和大题步骤；⚫看成立条件、比较与易错点。本文件不放习题。

### 1 操作系统总览

<span style="color:#c62828;font-weight:700">🔴【定义】操作系统是管理计算机软硬件资源、控制程序执行并为用户和应用提供服务的系统软件。</span>

<span style="color:#c62828;font-weight:700">🔴【四类功能】处理机管理、存储器管理、文件管理、设备管理；常再答“提供用户接口”。</span>

<span style="color:#c62828;font-weight:700">🔴【基本特征】并发、共享、虚拟、异步。并发是宏观上同时推进，并行是同一时刻真正同时执行；共享分互斥共享与同时访问。</span>

<span style="color:#c62828;font-weight:700">🔴【内核态与用户态】特权指令只能在内核态执行。系统调用是用户程序主动进入内核取得服务的接口；中断/异常是进入内核的机制。</span>

⚫【辨析】库函数不一定产生系统调用；系统调用不是普通函数调用。缺页是异常，I/O完成通常是外部中断。

### 2 进程、线程与处理机调度

<span style="color:#c62828;font-weight:700">🔴【进程】程序的一次执行过程，是资源分配和保护的基本单位；PCB是进程存在的唯一标志，保存标识、状态、现场、调度及资源信息。</span>

<span style="color:#c62828;font-weight:700">🔴【线程】进程内的执行单元，是处理机调度的基本单位。同进程线程共享地址空间和打开文件，但各自拥有程序计数器、寄存器和栈。</span>

<span style="color:#c62828;font-weight:700">🔴【三状态】就绪：只缺CPU；运行：正在占用CPU；阻塞：等待事件。就绪→运行靠调度，运行→就绪因时间片到/被抢占，运行→阻塞因等待事件，阻塞→就绪因事件完成。</span>

⚫【绝不会】阻塞态不能直接变运行态；调度只能从就绪队列选进程。进程切换要保存/恢复上下文，开销通常大于同进程线程切换。

<span style="color:#c62828;font-weight:700">🔴【调度层次】高级调度接纳作业；中级调度负责挂起/换入；低级调度从就绪队列选择进程。</span>

<span style="color:#1565c0;font-weight:700">🔵【时间指标】</span>

- 周转时间＝完成时刻－到达时刻；带权周转时间＝周转时间÷服务时间。
- 等待时间＝周转时间－实际运行时间（无额外阻塞时）；响应时间＝首次获得CPU时刻－到达时刻。
- 吞吐量是单位时间完成的作业数。平均值均为各进程指标之和除以进程数。

<span style="color:#c62828;font-weight:700">🔴【调度算法】FCFS按到达顺序、简单但有护航效应；SJF选服务时间最短，平均等待时间较优但可能饥饿；SRTF是抢占式最短剩余时间；优先级法可能饥饿，可用老化改善；RR按时间片轮转，兼顾响应，时间片过大退化为FCFS，过小则切换开销大。</span>

<span style="color:#1565c0;font-weight:700">🔵【调度计算五步】①按到达时刻维护就绪队列；②在每个到达/完成/时间片到期点重新决策；③画甘特图；④列首次运行和完成时刻；⑤计算周转、等待、响应并求平均。抢占式算法不能一次排到最后。</span>

### 3 同步、互斥与经典模型

<span style="color:#c62828;font-weight:700">🔴【临界资源】同一时刻只允许一个进程使用的资源；访问临界资源的代码段是临界区。同步强调先后制约，互斥强调不能同时进入。</span>

<span style="color:#c62828;font-weight:700">🔴【临界区原则】空闲让进、忙则等待、有限等待、让权等待。</span>

<span style="color:#c62828;font-weight:700">🔴【信号量】整型信号量只做初始化、P(wait)和V(signal)。P申请资源，V释放资源；记录型信号量可阻塞/唤醒进程，避免忙等。</span>

<span style="color:#1565c0;font-weight:700">🔵【PV题统一步骤】①找进程与动作先后；②列互斥资源和资源数量；③互斥量mutex初值1，资源计数按实际容量初始化；④P写在使用/等待之前，V写在释放/通知之后；⑤逐路径检查P、V配对及是否可能相互等待。</span>

<span style="color:#c62828;font-weight:700">🔴【生产者—消费者】大小为n的缓冲区：empty=n、full=0、mutex=1。生产者先P(empty)，再P(mutex)，放入后V(mutex)、V(full)；消费者先P(full)，再P(mutex)，取出后V(mutex)、V(empty)。</span>

⚫【次序陷阱】先P(mutex)再P(empty/full)可能“占锁等待”而死锁；V(mutex)与V资源数的先后一般不影响正确性，但按标准模板书写最稳。

<span style="color:#c62828;font-weight:700">🔴【读者—写者】读者可并发，写者与任何读写互斥。常用readcount统计读者数，用一个互斥量保护readcount，第一个读者锁写资源、最后一个读者释放。题目若要求写优先，还需增加控制新读者进入的信号量。</span>

<span style="color:#c62828;font-weight:700">🔴【管程】把共享数据和对它的操作封装，并保证管程内过程互斥执行；条件变量用于等待与唤醒，不等同于计数信号量。</span>

### 4 死锁

<span style="color:#c62828;font-weight:700">🔴【死锁】一组进程因竞争资源而永久等待，若无外力无法推进。</span>

<span style="color:#c62828;font-weight:700">🔴【四个必要条件】互斥、请求并保持、不剥夺、循环等待。破坏任一条件即可预防死锁。</span>

<span style="color:#c62828;font-weight:700">🔴【四种处理】预防：破坏必要条件；避免：运行中只进入安全状态；检测：允许发生后查找；解除：剥夺资源、撤销或回退进程。</span>

<span style="color:#c62828;font-weight:700">🔴【安全状态】存在一个安全序列，使每个进程都能依次获得其尚需资源并完成。安全状态一定无死锁；不安全状态不等于已经死锁，但可能导致死锁。</span>

<span style="color:#1565c0;font-weight:700">🔵【银行家算法】Need＝Max－Allocation；Work初值为Available。反复找未完成且Need≤Work的进程，假设其完成并令Work＝Work＋Allocation，标记Finish。全部完成则安全，选取顺序即安全序列。处理新请求时先查Request≤Need、Request≤Available，再“试分配+安全性检查”。</span>

⚫【向量比较】Need≤Work要求每一种资源分量都满足；Work回收的是已分配量Allocation，不是Need。

### 5 内存管理与地址变换

<span style="color:#c62828;font-weight:700">🔴【逻辑/物理地址】CPU生成逻辑地址，经MMU变为物理地址。重定位分装入时静态重定位和运行时动态重定位。</span>

<span style="color:#c62828;font-weight:700">🔴【连续分配】首次适应从低址查找，循环首次适应从上次位置继续，最佳适应找最小可用分区，最坏适应找最大分区。外部碎片是空闲区不连续；固定分区/页内未用空间属于内部碎片。</span>

<span style="color:#c62828;font-weight:700">🔴【分页】逻辑地址＝页号p＋页内偏移d，物理地址＝页框号f＋偏移d。页面与页框等大，页表完成p→f映射；分页有内部碎片而基本无外部碎片。</span>

<span style="color:#1565c0;font-weight:700">🔵【分页地址计算】页大小为2^k字节：偏移占k位；p＝逻辑地址÷页大小取整，d＝逻辑地址 mod 页大小；查页表得f，物理地址＝f×页大小＋d。</span>

<span style="color:#c62828;font-weight:700">🔴【分段】逻辑地址＝段号＋段内偏移，段长可变并反映程序逻辑。先查段表，再判断偏移<段长，最后物理地址＝段基址＋偏移。分页对用户透明、分段便于共享保护；段页式先分段再分页。</span>

⚫【越界】分页主要检查页号是否合法；分段必须检查段号及段内偏移，偏移等于段长也越界。

<span style="color:#c62828;font-weight:700">🔴【快表TLB】高速相联存储器，缓存最近页表项，利用局部性减少地址转换时间；TLB未命中不等于缺页，页表仍可能表明该页已在内存。</span>

<span style="color:#1565c0;font-weight:700">🔵【有效访问时间】严格按题目时序列概率树。若先查TLB，命中耗时t_TLB+t_M；未命中且页在内存常为t_TLB+2t_M；若题目允许并行查找或给出缺页率，必须按其模型另列。</span>

### 6 虚拟内存与页面置换

<span style="color:#c62828;font-weight:700">🔴【虚拟存储器】基于局部性，只把当前需要的页/段装入内存，使逻辑地址空间可大于物理内存。请求分页需要页表状态位、缺页中断、页面置换和外存后备。</span>

<span style="color:#c62828;font-weight:700">🔴【局部性】时间局部性：近期访问内容可能再次访问；空间局部性：邻近地址可能很快访问。</span>

<span style="color:#1565c0;font-weight:700">🔵【缺页处理】访问发现不在内存→硬件产生缺页异常→系统判断地址合法→找空闲框或选牺牲页（脏页先写回）→从外存调页→修改页表/TLB→重新执行被中断指令。</span>

<span style="color:#c62828;font-weight:700">🔴【置换算法】OPT淘汰未来最长时间不用页，只作理论最优；FIFO淘汰最早进入页，可能出现Belady异常；LRU淘汰最长时间未使用页，依据过去近似未来；CLOCK用访问位循环扫描，访问位1则清0并跳过，遇0淘汰。</span>

<span style="color:#1565c0;font-weight:700">🔵【置换题步骤】固定画“每次访问一列、每个页框一行”；逐列只更新当前状态，标出缺页；FIFO维护进入顺序，LRU维护最近访问次序；最后分别统计缺页次数与缺页率。初始装入空页框也算缺页。</span>

<span style="color:#c62828;font-weight:700">🔴【抖动】分给进程的页框不足，频繁缺页换页导致CPU利用率下降。可用工作集、缺页率控制或减少并发进程数改善。</span>

### 7 文件系统

<span style="color:#c62828;font-weight:700">🔴【文件】具有符号名的一组相关信息；文件控制块FCB记录属性和位置，Unix类系统目录项常保存文件名与inode号，inode不保存文件名。</span>

<span style="color:#c62828;font-weight:700">🔴【目录】单级简单但重名；两级按用户隔离；树形目录最常见；有向无环图支持共享。绝对路径从根开始，相对路径从当前目录开始。</span>

<span style="color:#c62828;font-weight:700">🔴【物理分配】连续分配访问快但有外部碎片且扩展难；链接分配扩展易、随机访问差；索引分配把数据块号集中到索引块，支持随机访问但有索引开销。</span>

<span style="color:#1565c0;font-weight:700">🔵【索引文件最大长度】先确定每个索引块可存“块大小÷块号长度”个地址，再分别计算直接、一级、二级、三级间接可寻址的数据块数，最后乘数据块大小。inode自身和索引块通常不计入文件数据长度。</span>

<span style="color:#c62828;font-weight:700">🔴【空闲空间】位示图便于查连续块，空闲链表实现简单。位图中0/1代表空闲还是占用必须按题设。</span>

<span style="color:#c62828;font-weight:700">🔴【文件共享】硬链接是多个目录项指向同一inode，通常不能跨文件系统；符号链接保存目标路径，可跨文件系统但目标删除后可能悬空。</span>

### 8 I/O与磁盘调度

<span style="color:#c62828;font-weight:700">🔴【I/O控制】程序查询由CPU反复轮询；中断方式由设备完成后通知CPU；DMA由控制器在主存与设备之间成块传输，CPU负责初始化和结束处理，并非完全不参与。</span>

<span style="color:#c62828;font-weight:700">🔴【缓冲】缓和CPU与设备速度不匹配，减少中断、提高并行性；单缓冲、双缓冲和循环缓冲并不改变设备本身速度。</span>

<span style="color:#c62828;font-weight:700">🔴【磁盘访问时间】寻道时间＋旋转延迟＋传输时间，机械磁盘通常寻道占比较大。</span>

<span style="color:#c62828;font-weight:700">🔴【磁盘调度】FCFS公平；SSTF选当前最近磁道，平均寻道短但远端可能饥饿；SCAN像电梯到一端再反向；LOOK只到当前方向最远请求；C-SCAN只单向服务，回程不服务，等待更均匀；C-LOOK同理但只到最远请求。</span>

<span style="color:#1565c0;font-weight:700">🔵【磁盘题步骤】写明初始磁头、方向和端点规则→按算法列访问序列→相邻磁道号差的绝对值求和→若求平均再除请求数。SCAN是否走到物理端点必须服从题干，LOOK则不走无请求端点。</span>

### 9 高频简答答题骨架

<span style="color:#1565c0;font-weight:700">🔵【概念比较】先各写定义，再按“单位/目的—资源是否共享—开销—适用场景”逐项比较，最后给结论。只写“前者好、后者差”不得分。</span>

<span style="color:#1565c0;font-weight:700">🔵【算法分析】先列初始状态与关键数据结构，再逐轮写“选择—状态更新—终止条件”，最后给结果并验证。</span>

<span style="color:#1565c0;font-weight:700">🔵【设计同步关系】先用自然语言写先后约束，再设置信号量初值，最后写每个进程P/V位置；用反例检查死锁、重复进入和资源计数越界。</span>

⚫【书写规范】调度、置换、银行家、磁盘题一定保留中间表或序列；同名算法有不同教材变体时，先在答案中声明采用的规则。

## 数据库技术

> > 定位：大学期末速记。🔴背定义、性质和SQL语义；🔵背建模、计算与设计步骤；⚫看前提、反例和易错点。本文件不放习题。

### 1 数据库系统与三级模式

<span style="color:#c62828;font-weight:700">🔴【基本概念】数据库DB是长期存储、有组织、可共享的数据集合；DBMS负责定义、操纵、控制和维护；数据库系统DBS＝数据库＋DBMS＋应用程序＋人员/环境。</span>

<span style="color:#c62828;font-weight:700">🔴【数据模型三要素】数据结构、数据操作、完整性约束。关系模型用二维表表达，逻辑结构简单，建立在集合论和谓词逻辑上。</span>

<span style="color:#c62828;font-weight:700">🔴【三级模式】外模式面向用户视图；模式描述全局逻辑结构；内模式描述物理存储。外模式/模式映像保证逻辑独立性，模式/内模式映像保证物理独立性。</span>

⚫【数量】一个数据库只有一个模式和一个内模式，可以有多个外模式。数据独立性不是“应用完全不改”，而是模式变化时尽量少改应用。

### 2 概念结构设计：E-R模型

<span style="color:#c62828;font-weight:700">🔴【元素】实体、属性、联系；属性可分简单/复合、单值/多值、存储/派生，候选码能唯一标识实体。</span>

<span style="color:#c62828;font-weight:700">🔴【联系基数】1:1、1:n、m:n。弱实体依赖所有者实体存在，通常具有部分码和标识联系。</span>

<span style="color:#1565c0;font-weight:700">🔵【E-R建模步骤】①从需求找实体和标识属性；②把可独立存在且有多个属性的概念建实体，描述性信息建属性；③找业务动词形成联系；④标注基数和参与约束；⑤检查冗余联系、多值属性和遗漏约束；⑥画图后用典型业务实例验证。</span>

<span style="color:#1565c0;font-weight:700">🔵【E-R转关系模式】实体各转一表；1:1可把任一方主码作为另一方外码，优先放在全参与方；1:n把“1”端主码放入“n”端作外码；m:n单独建联系表，主码通常是两端主码组合并加入联系属性；多值属性单独建表；弱实体表包含所有者主码。</span>

⚫【易错】外码应放在多端，不是“一端”；m:n若直接把一端主码放另一端，会丢失多值关系。联系表的组合主码是否含更多属性取决于业务唯一性。

### 3 关系模型、键与完整性

<span style="color:#c62828;font-weight:700">🔴【术语】关系是一张满足关系模型约束的表；元组是行，属性是列，域是属性取值集合。关系中元组无序、属性理论上无序，不能有完全相同元组。</span>

<span style="color:#c62828;font-weight:700">🔴【键】超码能唯一标识元组；候选码是无多余属性的最小超码；选定候选码为主码；候选码中属性是主属性；外码引用本表或其他表的候选码/主码。</span>

<span style="color:#c62828;font-weight:700">🔴【三类完整性】实体完整性：主码唯一且非空；参照完整性：外码取空值或取被参照表中已存在的键值；用户定义完整性：类型、范围和业务规则。</span>

⚫【NULL】NULL表示未知/缺失，不等于0或空串。与NULL普通比较结果为UNKNOWN，应使用IS NULL/IS NOT NULL；COUNT(列)忽略NULL，COUNT(*)统计行。

### 4 关系代数

<span style="color:#c62828;font-weight:700">🔴【基本运算】选择σ按行筛选；投影π按列选择并按集合语义去重；并、差、笛卡尔积要求并相容（并/差）；更名ρ解决同名和自连接。</span>

<span style="color:#c62828;font-weight:700">🔴【连接】θ连接＝笛卡尔积后按条件选择；等值连接的θ为等号；自然连接按所有同名属性等值连接并只保留一份同名列；外连接保留未匹配元组并补NULL。</span>

<span style="color:#c62828;font-weight:700">🔴【除法】R(X,Y)÷S(Y)得到X值：该X与S中每一个Y都在R中配对，典型语义是“选修了全部指定课程”。</span>

<span style="color:#1565c0;font-weight:700">🔵【代数表达步骤】①明确最终输出列；②从结果语义倒推涉及关系；③先选择尽早缩小行，先投影减少列但保留后续连接属性；④按键写连接条件；⑤最后投影目标属性；“全部/每一个”优先考虑除法或双重不存在。</span>

⚫【自然连接陷阱】同名属性不一定语义相同，不能无脑自然连接；笛卡尔积若漏写连接条件会使行数爆炸。

### 5 SQL查询与数据定义

<span style="color:#c62828;font-weight:700">🔴【逻辑执行顺序】FROM/JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT/OFFSET（产品语法略有差异）。</span>

<span style="color:#c62828;font-weight:700">🔴【DDL】CREATE定义表和约束；ALTER修改结构；DROP删除对象；TRUNCATE清空表结构通常保留。DML包括SELECT、INSERT、UPDATE、DELETE。</span>

<span style="color:#1565c0;font-weight:700">🔵【标准查询骨架】</span>

```sql
SELECT [DISTINCT] 表达式
FROM 表1 [JOIN 表2 ON 连接条件]
WHERE 行条件
GROUP BY 分组列
HAVING 组条件
ORDER BY 排序表达式 [ASC|DESC];
```

<span style="color:#c62828;font-weight:700">🔴【连接】INNER JOIN只保留匹配；LEFT JOIN保留左表全部行；RIGHT JOIN反之；FULL OUTER JOIN保留两侧（并非所有DBMS支持）。自连接必须给别名。</span>

⚫【外连接条件】想保留左表无匹配行时，针对右表的过滤通常写在ON；写进WHERE可能把补出的NULL行过滤掉，使左连接退化为内连接。

<span style="color:#c62828;font-weight:700">🔴【聚合】COUNT、SUM、AVG、MAX、MIN。WHERE在分组前筛行，不能直接用聚合条件；HAVING在分组后筛组。SELECT中的非聚合列原则上必须出现在GROUP BY。</span>

<span style="color:#c62828;font-weight:700">🔴【子查询】IN判断集合成员；EXISTS判断是否存在行；标量子查询必须至多一行；相关子查询引用外层列，对外层候选行反复求值。</span>

<span style="color:#1565c0;font-weight:700">🔵【“全部”SQL模板】用NOT EXISTS找不到反例：不存在一个目标集合元素，使该对象与它不存在对应关系，即双重NOT EXISTS。也可分组后用HAVING COUNT(DISTINCT 已满足项)=目标总数，但必须处理重复和目标集合为空。</span>

⚫【NOT IN与NULL】子查询结果含NULL时，NOT IN可能整体变UNKNOWN，反连接优先用NOT EXISTS并写清关联条件。

<span style="color:#c62828;font-weight:700">🔴【集合运算】UNION去重，UNION ALL保留重复；INTERSECT求交，EXCEPT/MINUS求差，列数和对应类型必须兼容。</span>

<span style="color:#c62828;font-weight:700">🔴【视图】由查询定义的虚表，提供逻辑独立性与安全隔离；复杂连接、聚合、DISTINCT等视图通常不可直接更新。物化视图实际保存结果并需刷新。</span>

<span style="color:#c62828;font-weight:700">🔴【索引】用额外空间和维护开销换查询速度。B+树适合等值与范围查询；哈希适合等值，不适合范围。高选择性、常用于连接/排序/筛选的列更值得索引。</span>

⚫【索引误区】索引不会提高所有操作；INSERT/UPDATE/DELETE需维护索引。对索引列做函数、隐式类型转换或前导通配符可能使普通索引难以利用。

### 6 函数依赖、键与闭包

<span style="color:#c62828;font-weight:700">🔴【函数依赖】X→Y表示任意合法关系中，X值相同则Y值必相同。平凡依赖满足Y⊆X；完全函数依赖表示去掉X任何属性都不再决定Y，部分依赖则存在真子集可决定Y；传递依赖常指X→Y、Y→Z且Y不能决定X时X→Z。</span>

<span style="color:#c62828;font-weight:700">🔴【Armstrong公理】自反、增广、传递；常用推论有合并、分解、伪传递，推导是可靠且完备的。</span>

<span style="color:#1565c0;font-weight:700">🔵【属性闭包X⁺】初值X⁺=X；反复扫描依赖，若某依赖左部⊆X⁺，就把右部加入；直到不再变化。若X⁺包含关系全部属性，则X是超码；再检查任一真子集都不是超码，才是候选码。</span>

<span style="color:#1565c0;font-weight:700">🔵【找候选码】不出现在任何依赖右部的属性通常必须在每个候选码中；从这些属性起求闭包，不足则尝试加入可导出剩余属性的组合；得到超码后逐属性做最小化。不要凭“属性最多/最少”猜键。</span>

<span style="color:#1565c0;font-weight:700">🔵【最小依赖集】①右部拆成单属性；②对每条依赖逐个检查左部属性是否多余（用其余依赖求闭包）；③逐条删除依赖，检查删除后左部闭包是否仍含右部；④可合并同左部依赖作展示。</span>

⚫【计算纪律】检查某条依赖是否冗余时必须暂时删除该依赖；检查左部多余属性时依赖集处理方式要按标准算法，不能直接看原关系中的偶然数据。

### 7 规范化与模式分解

<span style="color:#c62828;font-weight:700">🔴【1NF】属性值不可再分；【2NF】在1NF上，非主属性完全依赖每个候选码，消除对候选码的部分依赖；【3NF】在2NF上，非主属性不传递依赖候选码。形式化地，对每个非平凡X→A，X是超码或A是主属性。</span>

<span style="color:#c62828;font-weight:700">🔴【BCNF】每个非平凡函数依赖X→Y的决定因素X都是超码。BCNF比3NF严格，消除更多冗余，但分解后可能不能保持全部依赖。</span>

<span style="color:#c62828;font-weight:700">🔴【多值依赖与4NF】若在固定X下Y的取值与其余属性独立组合，则X↠Y；4NF要求每个非平凡多值依赖的左部为超码。多数基础期末重点到3NF/BCNF。</span>

<span style="color:#1565c0;font-weight:700">🔵【判断范式】①写出候选码与主属性；②找每条非平凡FD的决定因素；③先查1NF，再查非主属性是否部分依赖判2NF，再查3NF条件；④最后看所有决定因素是否超码判BCNF。必须说明违反哪条依赖。</span>

<span style="color:#c62828;font-weight:700">🔴【分解目标】无损连接保证分解后自然连接不产生伪元组、不丢信息；保持依赖保证可在子模式内检查原依赖。</span>

<span style="color:#1565c0;font-weight:700">🔵【二元无损判定】R分为R1、R2，对函数依赖集F，若(R1∩R2)→R1或(R1∩R2)→R2可由F推出，则分解无损。多个子模式用追赶法/表格法判定。</span>

<span style="color:#1565c0;font-weight:700">🔵【3NF综合法】求最小依赖集；每个X→A建模式XA并合并同左部；删除被其他模式包含的模式；若没有任何模式包含候选码，再加入一个候选码模式。所得通常无损且保持依赖。</span>

⚫【范式误区】范式越高不代表运行越快；工程中可能为性能有控制地反规范化，但要配套一致性维护。

### 8 事务、并发控制与恢复

<span style="color:#c62828;font-weight:700">🔴【ACID】原子性：全做或全不做；一致性：事务把数据库从一个一致状态带到另一个；隔离性：并发效果如某种串行执行；持久性：提交结果永久保存。</span>

<span style="color:#c62828;font-weight:700">🔴【并发异常】丢失修改、脏读、不可重复读、幻读。不可重复读关注同一行值变化，幻读关注满足谓词的行集合增删。</span>

<span style="color:#c62828;font-weight:700">🔴【调度】冲突操作是不同事务对同一数据项且至少一个为写。冲突可串行化等价于能通过交换不冲突操作变成串行调度。</span>

<span style="color:#1565c0;font-weight:700">🔵【优先图判定】每个事务一个结点；按调度先后扫描冲突操作，若Ti操作在Tj冲突操作前则加边Ti→Tj；图无环则冲突可串行化，拓扑序给等价串行顺序；有环则不是。</span>

<span style="color:#c62828;font-weight:700">🔴【封锁】共享锁S允许多个读，不允许写；排他锁X只允许持有者读写。两段锁2PL先增长只加锁，后收缩只解锁，可保证冲突可串行化；严格2PL把排他锁持有到提交/回滚，避免级联回滚并简化恢复。</span>

⚫【2PL】两段锁可能死锁；“可串行化”不代表无死锁。死锁可用等待图检测、超时或预防策略处理。

<span style="color:#c62828;font-weight:700">🔴【隔离级别】读未提交可能脏读；读已提交避免脏读；可重复读还避免不可重复读（幻读处理依DBMS）；串行化最强但并发度最低。SQL实现细节以产品和题设为准。</span>

<span style="color:#c62828;font-weight:700">🔴【日志恢复】WAL要求修改数据页写盘前，相关日志先写稳定存储；提交成功前提交日志需落盘。延迟更新通常只需REDO，立即更新通常需UNDO未提交事务并REDO已提交事务，具体以日志协议为准。</span>

<span style="color:#1565c0;font-weight:700">🔵【恢复题步骤】①从检查点/日志确定故障时已提交、未提交事务；②按协议确定REDO集合与UNDO集合；③REDO按日志正序，UNDO按逆序；④写出恢复后值。不要仅凭“最后一条日志”判断提交。</span>

### 9 数据库设计、安全与优化

<span style="color:#c62828;font-weight:700">🔴【设计阶段】需求分析→概念结构设计（E-R）→逻辑结构设计（关系模式/规范化）→物理设计（文件、索引、分区）→实施→运行维护。</span>

<span style="color:#c62828;font-weight:700">🔴【安全】身份鉴别确认是谁；授权控制能做什么；审计记录做过什么。GRANT授权，REVOKE收回；角色便于成组管理最小权限。</span>

<span style="color:#c62828;font-weight:700">🔴【完整性实现】PRIMARY KEY、FOREIGN KEY、UNIQUE、NOT NULL、CHECK；复杂跨行/跨表约束可用触发器或事务逻辑，但触发器会增加隐式行为和维护成本。</span>

<span style="color:#1565c0;font-weight:700">🔵【SQL设计题得分点】先声明主码/外码和数据类型→写约束→给查询主体→明确连接条件→处理重复、NULL和并列结果→必要时说明索引。先保证语义正确，再谈性能。</span>

<span style="color:#1565c0;font-weight:700">🔵【查询优化直觉】选择、投影尽量下推；小中间结果优先；避免无条件笛卡尔积；依据选择率和表规模选择索引/连接顺序；最终计划由优化器结合统计信息决定。</span>

## 数据结构与算法

> > 定位：大学期末算法速记。🔴背定义、性质和复杂度；🔵背算法过程、手算步骤与伪代码骨架；⚫看边界、稳定性和易错点。本文件不放习题。

### 1 基本概念与复杂度

<span style="color:#c62828;font-weight:700">🔴【数据结构】逻辑结构分集合、线性、树形、图状；存储结构常见顺序、链式、索引、散列。算法五特性：有穷性、确定性、可行性、输入、输出。</span>

<span style="color:#c62828;font-weight:700">🔴【复杂度】时间复杂度按基本操作执行次数的数量级，空间复杂度按额外空间数量级。通常考最坏或平均情况，必须注明分析口径。</span>

<span style="color:#c62828;font-weight:700">🔴【增长顺序】O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)。</span>

<span style="color:#1565c0;font-weight:700">🔵【循环分析】顺序语句相加取最大阶；嵌套循环相乘；变量每次乘/除常数通常是log n；递归先写递推式再展开或用主定理。</span>

<span style="color:#1565c0;font-weight:700">🔵【常见递推】T(n)=T(n/2)+O(1)为O(log n)；T(n)=2T(n/2)+O(n)为O(n log n)；T(n)=T(n−1)+O(n)为O(n²)。</span>

⚫【易错】O表示渐进上界，不等于精确运行时间；二分查找O(log n)以有序且支持随机访问为前提；递归空间还要算调用栈。

### 2 线性表

<span style="color:#c62828;font-weight:700">🔴【顺序表】地址连续、随机访问O(1)、按位查找快；插入删除需移动元素，平均O(n)，容量扩展可能搬迁。</span>

<span style="color:#c62828;font-weight:700">🔴【链表】结点离散、按位查找O(n)；已知目标结点前驱时插入删除O(1)。单链表结点含data和next；双链表再含prior；循环链表尾指针可使访问首尾方便。</span>

<span style="color:#1565c0;font-weight:700">🔵【单链表插入】在p后插s：`s->next=p->next; p->next=s;`。删除p后继q：`q=p->next; p->next=q->next; free(q);`。修改指针必须先保存将要丢失的后继。</span>

<span style="color:#1565c0;font-weight:700">🔵【建表】头插法生成顺序与输入相反，尾插法保持输入顺序并维护尾指针。设头结点可统一首结点的插删边界，但头结点不存有效数据。</span>

<span style="color:#1565c0;font-weight:700">🔵【逆置单链表】用pre、cur、next三指针循环：先保存next，再令cur->next=pre，随后pre和cur前移；最终头指针指向pre。时间O(n)，额外空间O(1)。</span>

⚫【删除结点】“已知结点p即可O(1)删除”对尾结点不成立，且复制后继数据会改变结点身份；考试应按题设结构和语义回答。

### 3 栈、队列与递归

<span style="color:#c62828;font-weight:700">🔴【栈】后进先出LIFO，只在栈顶插删；用于递归、括号匹配、表达式求值、DFS。顺序栈top常约定指向栈顶或下一空位，判空/判满公式随定义变化。</span>

<span style="color:#c62828;font-weight:700">🔴【队列】先进先出FIFO，队尾入队、队头出队；用于层序遍历、BFS、缓冲和调度。</span>

<span style="color:#1565c0;font-weight:700">🔵【循环队列】数组长m，常牺牲一个单元：空`front==rear`；满`(rear+1)%m==front`；队长`(rear-front+m)%m`；入队后`rear=(rear+1)%m`，出队后`front=(front+1)%m`。</span>

⚫【容量】上述约定下实际容量m−1；若另设size或tag可用满m个单元，公式必须按题设。

<span style="color:#1565c0;font-weight:700">🔵【表达式】中缀转后缀：操作数直接输出；左括号入栈；右括号弹到左括号；运算符弹出栈中优先级不低于它的运算符后入栈；末尾弹尽。后缀求值：遇操作数入栈，遇运算符弹右操作数b再弹左操作数a，算a op b后入栈。</span>

<span style="color:#c62828;font-weight:700">🔴【递归】必须有终止条件和规模缩小过程；系统用调用栈保存参数、局部变量和返回地址。可用显式栈改写，尾递归在部分环境可优化但不能默认。</span>

### 4 串与模式匹配

<span style="color:#c62828;font-weight:700">🔴【串】由零个或多个字符组成的有限序列。朴素匹配最坏O(nm)，失配时主串回退。</span>

<span style="color:#c62828;font-weight:700">🔴【KMP】利用模式串自身相同前后缀，主串指针不回退；next/前缀函数描述失配后模式应退到的位置，时间O(n+m)。</span>

<span style="color:#1565c0;font-weight:700">🔵【前缀函数求法】对模式每个前缀，求其“最长的、真前缀且又是真后缀”的长度；失配时把已匹配长度回退到前一个前缀函数值，直到可匹配或为0。不同教材next下标和初值有−1/0/1多种版本，先声明版本。</span>

⚫【真前后缀】不能等于字符串自身；next数组不是简单记录相同字符位置。

### 5 树与二叉树

<span style="color:#c62828;font-weight:700">🔴【术语】结点度是孩子数，树的度是最大结点度；深度从根向下，结点高度从叶向上。含n个结点的树有n−1条边。</span>

<span style="color:#c62828;font-weight:700">🔴【二叉树性质】第i层最多2^(i−1)个结点；高度h最多2^h−1个结点；非空二叉树叶结点数n0与度2结点数n2满足n0=n2+1。</span>

<span style="color:#c62828;font-weight:700">🔴【完全二叉树顺序编号】编号i的左孩子2i、右孩子2i+1、双亲⌊i/2⌋（存在时）；含n个结点，最后一个非叶结点为⌊n/2⌋。</span>

<span style="color:#c62828;font-weight:700">🔴【遍历】先序根左右，中序左根右，后序左右根，层序按层。递归遍历时间O(n)，栈空间O(h)。</span>

<span style="color:#1565c0;font-weight:700">🔵【由遍历还原】先序首元素/后序末元素确定根；在中序中定位根后划分左右子树，再递归。先序+后序通常不能唯一确定普通二叉树；必须有中序或额外结构条件。</span>

<span style="color:#c62828;font-weight:700">🔴【线索二叉树】把空指针改存前驱/后继，并用标志区分孩子与线索；中序线索树可在无栈递归下顺序遍历。</span>

<span style="color:#c62828;font-weight:700">🔴【二叉排序树BST】左子树键值小于根、右子树大于根（重复键策略另定）；中序遍历递增。查找平均与树高相关，最坏退化为链表O(n)。删除双孩子结点常以中序前驱/后继替换。</span>

<span style="color:#c62828;font-weight:700">🔴【AVL】任一结点左右子树高度差绝对值≤1。插入后从失衡点按LL、RR、LR、RL旋转：LL右旋，RR左旋，LR先左后右，RL先右后左。</span>

<span style="color:#1565c0;font-weight:700">🔵【AVL判型】从新结点向上找第一个平衡因子绝对值为2的结点A；看插入路径由A向下前两步方向，判四型并旋转；旋转后重新计算相关高度。</span>

<span style="color:#c62828;font-weight:700">🔴【哈夫曼树】带权路径长度WPL最小的最优二叉树。反复选权值最小的两棵树合并，父权为和；n个叶结点有2n−1个总结点。编码左0右1可互换，但必须前缀无歧义。</span>

<span style="color:#1565c0;font-weight:700">🔵【WPL】叶权×从根到叶的边数后求和；也等于每次合并权值之和。权值相同可能有不同树形和编码，但最小WPL相同。</span>

<span style="color:#c62828;font-weight:700">🔴【堆】完全二叉树且每个结点不大于（小根堆）或不小于（大根堆）孩子。建堆从最后一个非叶结点向前下滤，时间O(n)；单次插入/删除堆顶O(log n)。</span>

### 6 图的表示与遍历

<span style="color:#c62828;font-weight:700">🔴【图】无向图度数和＝2|E|；有向图入度和＝出度和＝|E|。连通无向图至少n−1条边，强连通是有向图任意两点互相可达。</span>

<span style="color:#c62828;font-weight:700">🔴【存储】邻接矩阵空间O(n²)，判边O(1)，适合稠密图；邻接表空间O(n+e)，遍历邻边高效，适合稀疏图。无向图一条边在邻接表存两次。</span>

<span style="color:#c62828;font-weight:700">🔴【DFS/BFS】DFS用递归或栈，沿深度回溯；BFS用队列逐层扩展。邻接表下均为O(n+e)，矩阵下为O(n²)。非连通图要从每个未访问顶点重新启动。</span>

<span style="color:#1565c0;font-weight:700">🔵【遍历序列】严格按给定顶点/邻接点次序；写visited，逐步维护栈/递归路径或队列。没有规定邻接次序时，合法序列可能不唯一。</span>

### 7 最小生成树、最短路与拓扑排序

<span style="color:#c62828;font-weight:700">🔴【生成树】连通无向图的生成树含全部n个顶点和n−1条边且无环。最小生成树MST使边权和最小；权值互异时MST唯一，存在同权不一定不唯一。</span>

<span style="color:#c62828;font-weight:700">🔴【Prim】从一个顶点集合出发，每次选连接集合内外的最小权边，适合稠密图。🔴【Kruskal】边按权从小到大，若加入不成环则选，用并查集判环，适合稀疏图。</span>

<span style="color:#1565c0;font-weight:700">🔵【MST手算】Prim维护已选顶点集和每个外部顶点的最小连接边；Kruskal先稳定排序全部边，再逐边用连通分量判断。选满n−1条边即停，最后核对覆盖全部顶点且无环。</span>

<span style="color:#c62828;font-weight:700">🔴【Dijkstra】求非负权图单源最短路。每轮确定当前未确定且dist最小顶点u，再对u的出边松弛：若dist[u]+w(u,v)<dist[v]则更新dist[v]和pre[v]。</span>

⚫【负权】Dijkstra不适用于含负权边的普通情形；Bellman-Ford可处理负权并检测可达负环。

<span style="color:#c62828;font-weight:700">🔴【Floyd】多源最短路，动态规划：`D[i][j]=min(D[i][j], D[i][k]+D[k][j])`，k必须作为最外层，时间O(n³)、空间O(n²)。</span>

<span style="color:#c62828;font-weight:700">🔴【拓扑排序】只适用于有向无环图DAG。反复选择入度0顶点输出并删除其出边；若输出少于n个则有环。序列可能不唯一。</span>

<span style="color:#c62828;font-weight:700">🔴【关键路径AOE网】事件为顶点、活动为边。先拓扑正序求事件最早发生时间ve（取最大），再逆拓扑求最迟发生时间vl（取最小）；活动e=ve[起点]，l=vl[终点]−工期，l−e=0的活动为关键活动。</span>

<span style="color:#1565c0;font-weight:700">🔵【关键路径步骤】拓扑排序→正向取max算ve→终点ve初始化全部vl→逆向取min算vl→逐边算e、l与余量→写关键路径及总工期。多条关键路径时，缩短一条路径上的单个活动未必缩短工期。</span>

### 8 查找与散列表

<span style="color:#c62828;font-weight:700">🔴【顺序查找】无序/有序均可，平均O(n)。🔴【折半查找】要求有序顺序表，比较后取左/右半区，时间O(log n)，链表不适合。</span>

<span style="color:#1565c0;font-weight:700">🔵【折半判定树】mid取整规则决定具体树形；成功ASL＝各结点层数之和/n（若比较次数从1计）；失败ASL按外部失败结点计算，必须采用题定口径。</span>

<span style="color:#c62828;font-weight:700">🔴【B树/B+树】均为多路平衡查找树。B树关键字和记录可在各结点；B+树记录通常只在叶层，内部结点作索引，叶结点有序相连，适合数据库范围查询。</span>

<span style="color:#c62828;font-weight:700">🔴【散列】用哈希函数把关键字映射到地址；冲突指不同关键字映射同地址。处理有开放定址（线性/二次探测、双散列）和拉链法。</span>

<span style="color:#1565c0;font-weight:700">🔵【散列表手算】按插入顺序计算初址；冲突后严格按探测序列找空位；查找沿同一序列直到命中或遇从未占用位置。删除开放定址元素应设“删除标记”，不能直接置空。</span>

<span style="color:#c62828;font-weight:700">🔴【装填因子】α＝表中记录数/表长。开放定址α越大冲突通常越多；拉链法α可大于1。</span>

### 9 内排序总表

| 算法 | 最好 | 平均 | 最坏 | 额外空间 | 稳定 | 关键特征 |
|---|---:|---:|---:|---:|:---:|---|
| 直接插入 | O(n) | O(n²) | O(n²) | O(1) | 是 | 基本有序时好 |
| 希尔 | 依增量 | 约低于O(n²) | 依增量 | O(1) | 否 | 分组插入 |
| 冒泡 | O(n) | O(n²) | O(n²) | O(1) | 是 | 相邻逆序交换 |
| 快速 | O(n log n) | O(n log n) | O(n²) | 平均O(log n)栈 | 否 | 枢轴分区 |
| 简单选择 | O(n²) | O(n²) | O(n²) | O(1) | 否 | 每趟选最小 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 否 | 完全二叉堆 |
| 归并 | O(n log n) | O(n log n) | O(n log n) | O(n) | 是 | 合并有序段 |
| 基数 | O(d(n+r)) | 同左 | 同左 | O(n+r) | 是 | 按位分配收集 |

<span style="color:#c62828;font-weight:700">🔴【稳定性】相等关键字排序后相对次序不变。稳定性是算法性质，不等同于结果数值有序程度；某次输入未改变相等项也不能证明算法稳定。</span>

<span style="color:#1565c0;font-weight:700">🔵【快速排序手算】先明确枢轴选择和分区版本；每趟只保证枢轴最终到位及两侧关系，不保证两侧内部有序；递归处理左右区间。近乎有序且取首元素为枢轴时可能退化。</span>

<span style="color:#1565c0;font-weight:700">🔵【堆排序】升序用大根堆：建堆→堆顶与末尾交换→有效堆长减1→根向下调整。建堆O(n)，整个排序O(n log n)。</span>

<span style="color:#1565c0;font-weight:700">🔵【归并排序】长度1子序列视为有序，两两合并；比较两个有序段当前最小元素，较小者进入临时区，最后复制剩余元素。若相等时先取左段可保持稳定。</span>

⚫【选择算法】数据基本有序可考虑插入；要求最坏O(n log n)且原地可用堆；要求稳定且可用O(n)空间可用归并；内部排序平均快速常用，但不可宣称任何输入都最快。

### 10 算法设计题得分骨架

<span style="color:#1565c0;font-weight:700">🔵【四段式】①说明输入、输出和边界；②写数据结构/辅助变量含义；③给伪代码并保证循环/递归终止；④证明正确性要点，分析时间与额外空间。</span>

<span style="color:#1565c0;font-weight:700">🔵【循环不变式】初始化前成立；每轮保持；终止时不变式推出目标。排序、查找、贪心证明常用。</span>

<span style="color:#1565c0;font-weight:700">🔵【递归设计】写清基例、子问题规模缩小、组合结果；复杂度包含递归树和合并工作。</span>

⚫【代码边界】空表、单元素、首尾结点、重复关键字、图不连通、权值相等、下标越界和内存释放是常见扣分点。

## 编程语言

> > 只用于考前速记：🔴核心必背；🔵语法、步骤、代码阅读与踩分词；⚫语言差异、边界和常见陷阱。课程主线统一为 C 语言与 Python 3：C 侧重底层数据表示、指针和文件，Python 侧重容器、函数与工程化基础。

### 一、程序、算法与翻译方式

- 🔴 程序是为解决问题而编写的指令序列；算法应具有有穷性、确定性、可行性、输入和输出。
- 🔴 算法基本结构：顺序、选择、循环；常用表示有自然语言、流程图、伪代码和程序设计语言。
- 🔴 编译通常把整个源程序翻译为目标程序后执行；解释通常边翻译边执行。现代语言实现可能混合编译、字节码和虚拟机。
- 🔴 语法错误违反语言规则；运行时错误在执行中发生；逻辑错误能运行但结果不符合要求。
- 🔵 程序开发：问题分析 → 建模/设计算法 → 编码 → 编译/运行 → 测试调试 → 文档维护。
- 🔵 算法题踩分：写输入、输出、初始化、循环/选择条件、更新语句、终止条件和边界处理。
- ⚫ “编译型/解释型”不是所有实现的绝对二分；Python 常先编译成字节码再由虚拟机执行。

### 二、标识符、变量、常量与数据类型

- 🔴 标识符一般由字母、数字、下划线等构成，不能以数字开头，不能使用保留字；是否区分大小写由语言决定。
- 🔴 变量具有名称、类型/值、作用域和生命周期；常量在规定范围内不可修改。
- 🔴 基本类型通常包括整数、实数/浮点、字符、布尔；复合类型包括数组、记录/结构体、容器和对象等。
- 🔵 类型选择：计数用整数，连续近似量用浮点，单字符用字符，真假条件用布尔；金融精确小数要关注十进制类型或整数化处理。
- 🔴 隐式转换由语言自动完成，显式转换由程序员指定；窄化转换可能溢出或损失精度。
- ⚫ 浮点数是有限二进制近似，`0.1+0.2` 未必精确等于 `0.3`；比较浮点数常使用容差。

### 三、运算符、表达式与输入输出

- 🔴 运算符常分算术、关系、逻辑、赋值、位运算和条件运算；表达式由操作数和运算符组成并产生值。
- 🔴 高优先级通常先算，优先级相同时再看结合性；考试不确定时用括号明确顺序。
- 🔵 逻辑短路：`AND/&&` 左侧为假时可能不算右侧；`OR/||` 左侧为真时可能不算右侧。
- 🔵 整除与余数：C 整数 `/` 截断小数，`%` 求余；Python `//` 向下取整除、`%` 满足整除关系。
- 🔴 输入要做类型转换和合法性检查；输出题要按实际执行顺序记录变量变化、换行和格式控制。
- ⚫ `=` 在 C/Python 中是赋值，不是比较；比较相等都用 `==`。Python 还能用 `is` 比较对象身份，但不能用它代替一般数值或字符串相等判断。

### 四、选择与循环结构

- 🔴 单分支、双分支、多分支根据互斥条件选择路径；多分支要注意条件覆盖、顺序和默认分支。
- 🔴 计数循环适合次数已知；条件循环适合次数未知；循环必须有初始化、条件判断、循环体和状态更新。
- 🔵 程序阅读表：每轮依次记录“循环变量—条件结果—关键变量—输出”，不要凭感觉心算。
- 🔵 循环设计：初始化 → 判断 → 处理 → 更新；验证 0 次、1 次、最后一次和越界后状态。
- 🔴 `break` 结束当前最近循环；`continue` 跳过本轮余下语句进入下一轮；`return` 结束函数并可返回值。
- ⚫ `while` 可能一次不执行；传统 `do...while` 至少执行一次；嵌套循环总次数通常是各层次数乘积，但内层边界可能依赖外层。

### 五、函数、参数、作用域与递归

- 🔴 函数把特定任务封装为可复用单元，具有名称、形参、函数体和返回值；过程可视为不强调返回值的子程序。
- 🔴 形参是定义时参数，实参是调用时传入的值/对象；参数传递语义要按具体语言判断。
- 🔴 局部变量只在所属块/函数可见；全局变量作用域更大但增加耦合，应谨慎使用。
- 🔵 函数题写法：明确输入参数 → 处理逻辑 → 返回结果；调用方接收返回值；检查所有执行路径是否返回。
- 🔴 递归必须有基本情况和递归推进，且每次调用应更接近基本情况；调用栈保存各层局部状态和返回位置。
- 🔵 递归跟踪：先写调用链到基本情况，再按返回顺序向上代值；区分“调用阶段”和“回归阶段”的输出。
- ⚫ C 默认值传递；要让函数修改调用者变量通常传地址。Python 是对象引用传递语义，可变对象可能原地改变，重新绑定形参不等于修改调用者变量。

### 六、数组、字符串与常用算法

- 🔴 C 数组和 Python 序列通常从 0 开始；C 数组元素类型固定，Python 列表可保存不同类型对象。
- 🔴 顺序查找逐个比较，时间复杂度通常 `O(n)`；二分查找要求数据有序，每次折半，时间复杂度 `O(log n)`。
- 🔵 二分查找：`low/high` 初始化 → 当区间有效时求 `mid` → 比较 → 保留左半或右半 → 找到或区间为空。
- 🔴 冒泡排序重复比较相邻元素并交换逆序对；选择排序每轮选出最值放到目标位置；插入排序把当前元素插入前方有序区。
- 🔵 冒泡高频边界：第 `i` 轮后端已有序，内层上界应缩小；交换需要临时变量或安全的并行赋值。
- 🔴 字符串是字符序列；C 字符串以 `\0` 结束，字符数组容量要给终止符留位置。
- 🔵 字符串常考：长度、拼接、比较、查找、截取、大小写转换；注意函数返回新对象还是原地修改。
- ⚫ 二分查找不能直接用于无序数据；数组下标越界在 C 中可能导致未定义行为，Python 通常抛出 `IndexError`。

### 七、测试、调试与程序质量

- 🔴 测试是发现缺陷，调试是定位并修复缺陷；测试不能证明程序绝对无错。
- 🔵 黑盒边界值：最小值、略大于最小、正常值、略小于最大、最大值，以及刚好越界值。
- 🔵 调试步骤：稳定复现 → 缩小范围 → 观察变量/调用栈 → 提出原因 → 最小修改 → 回归测试。
- 🔴 良好程序强调正确性、可读性、健壮性、效率、可维护性和可移植性。
- 🔵 代码改错踩分：指出错误位置 → 说明错误类型/后果 → 给出正确代码 → 用边界数据验证。
- ⚫ “程序有输出”不代表正确；只测正常数据会漏掉空输入、零、负数、重复值和上下界问题。

### 八、C 语言：程序结构与数据类型

- 🔴 C 程序由函数组成，`main` 是常见入口；预处理指令如 `#include` 在编译前处理，语句多以分号结束。
- 🔴 C 区分大小写；合法标识符不能以数字开头；关键字不能作标识符。
- 🔴 常见类型：`char`、`short`、`int`、`long`、`float`、`double`；确切字节数与实现有关，只有相对范围受标准约束。
- 🔵 格式化输入输出：`printf` 输出，`scanf` 输入；常见格式 `%d` 整数、`%f` 浮点输出、`%lf` 双精度输入、`%c` 字符、`%s` 字符串。
- 🔵 `scanf` 普通变量通常传地址，如 `scanf("%d", &x)`；字符数组名用于 `%s` 时本身可退化为地址，通常不再加 `&`。
- ⚫ 输入字符前残留换行是高频陷阱；`scanf(" %c", &ch)` 前导空格可跳过空白，但是否应跳过取决于题意。

### 九、C 语言：指针、数组、字符串与结构体

- 🔴 指针保存地址；`&x` 取变量地址，`*p` 间接访问指针所指对象；使用前必须指向有效存储区域。
- 🔴 数组名在多数表达式中转换为首元素指针，但数组与指针不是完全相同类型；`a[i]` 等价于 `*(a+i)`。
- 🔵 交换调用者变量：函数形参接收指针 → 调用时传 `&变量` → 函数内用 `*指针` 修改。
- 🔴 常见字符串函数：`strlen` 长度、`strcpy/strncpy` 复制、`strcat` 连接、`strcmp` 比较；必须保证目标空间足够并关注终止符。
- 🔴 结构体用 `struct` 把不同类型字段组合；普通对象用 `.` 访问成员，结构体指针用 `->`。
- 🔴 动态内存常用 `malloc/calloc/realloc/free`；申请后检查是否成功，使用后释放并避免悬空指针、重复释放和越界。
- ⚫ `sizeof(数组)` 在数组定义所在作用域可得整个数组字节数，但数组传入函数后形参通常退化为指针；`strlen` 不包含 `\0`。

### 十、C 语言：函数、文件与预处理

- 🔴 函数调用前应有声明/原型；返回类型、参数类型和调用方式应一致。
- 🔴 存储类别与作用域要区分：局部 `static` 变量生命周期贯穿程序但作用域仍在函数/块内；`extern` 用于引用外部定义。
- 🔵 文件流程：`fopen` → 检查返回指针 → `fscanf/fprintf` 或 `fread/fwrite` 读写 → 检查状态 → `fclose`。
- 🔴 文本模式按字符解释，二进制模式按字节读写；文件打开模式 `r/w/a` 分别常用于读、覆盖写、追加，`+` 表示可读写。
- 🔴 宏是预处理文本替换；带参数宏应给参数和整体加括号，如 `#define SQR(x) ((x)*(x))`。
- ⚫ `w` 模式会截断已有文件；宏参数可能被计算多次，不能简单等同于函数。

### 十一、Python：语法、对象与容器

- 🔴 Python 用缩进表示代码块，区分大小写，变量名绑定对象且类型动态确定。
- 🔴 常见不可变对象：数字、字符串、元组；常见可变对象：列表、字典、集合。
- 🔵 输入转换：`input()` 返回字符串；数值输入常用 `int(input())`、`float(input())`。输出用 `print`，格式化优先掌握 f-string。
- 🔴 切片常写 `序列[start:stop:step]`，包含 start、不包含 stop；省略值使用默认边界，负下标从末尾计数。
- 🔴 列表有序可重复；元组有序通常不可变；集合无重复并支持集合运算；字典存储键值映射，键须可哈希。
- 🔵 高频方法：列表 `append/extend/insert/pop/sort`；字符串 `split/join/strip/find/replace`；字典 `get/keys/values/items`；集合 `add/remove/discard`。
- ⚫ `list.sort()` 原地排序并返回 `None`，`sorted()` 返回新列表；`append` 加一个对象，`extend` 逐项扩展；集合无固定位置下标。

### 十二、Python：控制、函数、异常与文件

- 🔵 分支：`if/elif/else`；循环：`for 变量 in 可迭代对象`、`while 条件`；`range(start,stop,step)` 不包含 stop。
- 🔴 函数用 `def` 定义，`return` 返回结果；未显式返回时结果为 `None`。
- 🔴 形参形式可含位置参数、关键字参数、默认参数、`*args` 和 `**kwargs`；默认参数在定义时求值，可变默认参数会跨调用共享。
- 🔴 LEGB 名称查找：Local 局部、Enclosing 嵌套、Global 全局、Built-in 内置；修改全局绑定需谨慎使用 `global`。
- 🔵 异常：`try` 放可能失败代码，`except` 处理特定异常，`else` 在无异常时执行，`finally` 通常用于清理资源。
- 🔵 文件推荐 `with open(path, mode, encoding="utf-8") as f:`，离开块自动关闭；文本读写常用 `read/readline/readlines/write`。
- 🔴 类用 `class` 定义，实例方法首参通常写 `self`；`__init__` 初始化实例状态；继承可复用并扩展父类行为。
- ⚫ `except:` 捕获过宽会掩盖错误；读文件模式不存在文件会失败，写模式会覆盖；列表复制要防共享嵌套对象。

### 十三、C 与 Python 高频对照

| 概念 | C | Python 3 |
|---|---|---|
| 🔴 类型系统 | 静态类型，变量声明类型 | 动态类型，名称绑定对象 |

## 计算机组成原理

> > 定位：大学期末速记，重点覆盖数据表示、指令、存储、Cache、CPU、流水线与I/O。🔴背结论；🔵背计算/设计步骤；⚫看条件和易错点。本文件不放习题。

### 1 计算机系统与性能

<span style="color:#c62828;font-weight:700">🔴【冯·诺依曼思想】指令和数据都以二进制存入存储器；采用存储程序与程序控制；硬件包括运算器、控制器、存储器、输入和输出五部分。</span>

<span style="color:#c62828;font-weight:700">🔴【层次】硬件/微体系结构实现机器指令；ISA是软件可见的指令、寄存器、数据类型和寻址规范；操作系统和编译器位于其上。体系结构回答“机器能做什么”，组成回答“怎样实现”。</span>

<span style="color:#c62828;font-weight:700">🔴【CPU】主要由运算器、控制器和寄存器组构成。PC保存下一条指令地址，IR保存当前指令，MAR保存访存地址，MDR保存访存数据，通用寄存器保存操作数和结果。</span>

<span style="color:#1565c0;font-weight:700">🔵【CPU时间】`CPU时间＝指令数IC×平均CPI×时钟周期＝IC×CPI/主频`；性能与执行时间成反比。`加速比＝改进前时间/改进后时间`。</span>

<span style="color:#1565c0;font-weight:700">🔵【Amdahl定律】若可改进部分占原时间比例f、该部分加速s，则总加速比`S=1/[(1−f)+f/s]`。无限加速后的上限为`1/(1−f)`。</span>

⚫【性能误区】主频高不必然更快，需同时考虑IC、CPI和程序；MIPS不适合跨不同ISA/指令功能直接比较。

### 2 数制、定点数与运算

<span style="color:#c62828;font-weight:700">🔴【进制转换】整数十转R用除R取余倒序；小数用乘R取整正序；二进制转八/十六分别三位/四位一组。</span>

<span style="color:#c62828;font-weight:700">🔴【无符号n位】范围0～2^n−1。🔴【补码n位】范围−2^(n−1)～2^(n−1)−1；正数补码同真值二进制，负数由绝对值按位取反加1。</span>

<span style="color:#1565c0;font-weight:700">🔵【补码求值】最高位0按无符号求值；最高位1可“按位取反加1得绝对值再加负号”，或按权展开，最高位权为−2^(n−1)。</span>

<span style="color:#c62828;font-weight:700">🔴【补码加减】`[x−y]补=[x]补+[−y]补`，符号位参与运算，最高位进位丢弃。</span>

<span style="color:#c62828;font-weight:700">🔴【溢出】两个同号数相加结果异号则溢出；相减时操作数异号且结果符号与被减数不同则溢出。也可判符号位进位与最高数值位进位异或为1。</span>

⚫【进位≠溢出】进位主要反映无符号超范围，溢出反映有符号真值超范围。负数相加最高位有进位也可能不溢出。

<span style="color:#c62828;font-weight:700">🔴【原码乘法】符号单独异或，绝对值做移位加法。🔴【Booth】依据乘数相邻位处理`01加被乘数、10减被乘数、00/11不加减`后算术右移，适合补码有符号乘法。</span>

<span style="color:#c62828;font-weight:700">🔴【除法】恢复余数法试减为负时加回除数，商0；非负则商1。不同教材寄存器初值和移位次序不同，手算必须跟题设微操作。</span>

### 3 浮点数

<span style="color:#c62828;font-weight:700">🔴【IEEE 754单精度】1位符号＋8位阶码＋23位小数，阶码偏置127；双精度1＋11＋52，偏置1023。规格化数真值`(−1)^S × 1.F × 2^(E−Bias)`。</span>

<span style="color:#c62828;font-weight:700">🔴【特殊值】阶码全0：±0或非规格化数；阶码全1且尾数0：±∞；阶码全1且尾数非0：NaN。</span>

<span style="color:#1565c0;font-weight:700">🔵【十进制转单精度】①转二进制；②规格化成`1.F×2^e`；③S看正负；④E=e+127写8位；⑤小数点后填23位并按要求舍入；⑥合并并可转十六进制。</span>

<span style="color:#1565c0;font-weight:700">🔵【浮点加减】对阶（小阶向大阶移）→尾数加减→结果规格化→舍入→判断阶码溢出/下溢。对阶右移会丢低位，是主要误差来源。</span>

⚫【范围与精度】阶码位数主要决定范围，尾数位数主要决定精度；浮点数不连续，许多十进制小数不能精确表示。

### 4 存储系统与主存扩展

<span style="color:#c62828;font-weight:700">🔴【层次】寄存器→Cache→主存→辅存：速度逐级降低、容量逐级增大、每位价格逐级降低。Cache—主存弥补速度差，主存—辅存构成虚拟存储。</span>

<span style="color:#c62828;font-weight:700">🔴【RAM】SRAM速度快、无需刷新、常作Cache；DRAM集成度高、需刷新、常作主存。ROM非易失，Flash可电擦写。</span>

<span style="color:#1565c0;font-weight:700">🔵【存储容量】芯片`字数×位数`。用aK×b位芯片组成cK×d位：字扩展倍数c/a，位扩展倍数d/b，总片数为二者乘积。</span>

<span style="color:#1565c0;font-weight:700">🔵【地址线】按字节编址、地址宽n位可寻址2^n B；按字编址还需乘每字字节数。容量为2^m个编址单元时片内地址线m根，剩余高位用于片选。</span>

<span style="color:#1565c0;font-weight:700">🔵【芯片扩展设计】①算字/位扩展倍数；②低位地址接各芯片片内地址；③数据线按位并联分组；④高位地址经译码产生各字组片选；⑤写地址范围，检查首末地址与是否重叠。</span>

⚫【混淆】字扩展增加地址空间，用片选区分；位扩展增加一个字的数据位数，同组芯片同时选通。地址译码不完整会产生地址重叠/镜像。

<span style="color:#c62828;font-weight:700">🔴【DRAM刷新】集中刷新在一段时间统一刷新会有“死时间”；分散刷新把刷新插入每个存取周期；异步刷新把各行均匀分布在刷新周期内，兼顾效率。</span>

### 5 Cache（高频综合题）

<span style="color:#c62828;font-weight:700">🔴【依据】时间局部性与空间局部性。主存与Cache按等长块交换，CPU地址通常拆成标记Tag、组/行索引Index、块内偏移Offset。</span>

<span style="color:#1565c0;font-weight:700">🔵【位数先算】块大小B=2^b字节，则块内偏移b位；Cache数据容量C、E路组相联，则组数S=C/(B×E)，组索引s=log2S位；物理地址m位时Tag=m−s−b位。若容量明确包含标记等开销，须按题设重算。</span>

<span style="color:#c62828;font-weight:700">🔴【直接映射】主存块j映射到Cache行`j mod 行数`，硬件简单、冲突多。地址拆Tag+行号+块内偏移。</span>

<span style="color:#c62828;font-weight:700">🔴【全相联】块可放任意行，无索引，冲突少但比较器多。🔴【组相联】块映射到固定组`j mod 组数`，组内任意路，是折中方案。</span>

<span style="color:#1565c0;font-weight:700">🔵【访问序列手算】①地址÷块大小得主存块号和偏移；②按映射算行/组；③比较有效位与Tag判命中；④未命中按空路或LRU/FIFO等置换；⑤写回法淘汰脏块先写主存；⑥逐次统计命中率。</span>

<span style="color:#c62828;font-weight:700">🔴【写策略】写直达Write-through同时写Cache和主存，常配非写分配；写回Write-back只写Cache并置脏位，淘汰时写主存，常配写分配。具体组合不是绝对，按题设。</span>

<span style="color:#1565c0;font-weight:700">🔵【平均访问时间AMAT】常用`命中时间 + 失效率×失效代价`。若题目把“失效代价”定义为完整主存访问时间且命中检查已含在前项，不能重复相加。</span>

⚫【失效分类】强制失效首次访问不可避免；容量失效因总容量不足；冲突失效因映射限制。增加相联度主要减少冲突失效，不消除强制失效。

### 6 指令系统与寻址方式

<span style="color:#c62828;font-weight:700">🔴【指令格式】操作码规定操作，地址码给操作数位置/寻址信息。定长操作码译码简单；扩展操作码可在固定字长中兼顾多类地址数，短格式空出的地址字段用于扩展操作码。</span>

<span style="color:#1565c0;font-weight:700">🔵【操作码位数】若定长操作码支持N种操作至少需`ceil(log2N)`位。扩展操作码题按各类指令数量逐级保留编码前缀，不能把已分配短码当成长码前缀再用。</span>

<span style="color:#c62828;font-weight:700">🔴【地址数】三地址`A1 op A2→A3`，二地址常覆盖一个操作数，一地址常用隐含累加器，零地址常用于栈式机器。地址越少，指令可能更短但完成表达式需更多指令。</span>

<span style="color:#c62828;font-weight:700">🔴【常考寻址】</span>

- 立即：操作数在指令中，速度快，不能作可修改存储目的地址。
- 直接：EA=A；间接：EA=M[A]，扩大范围但多次访存。
- 寄存器：操作数在寄存器；寄存器间接：EA=(R)。
- 基址：EA=(BR)+A，利于重定位；变址：EA=(IX)+A，利于数组；相对：EA=(PC)+A，利于转移和位置无关代码。
- 堆栈：操作数隐含在栈顶。

<span style="color:#1565c0;font-weight:700">🔵【有效地址题】先识别寻址方式，再明确括号表示寄存器内容还是存储内容，按层次求EA，最后才访存取操作数。间接寻址的EA与操作数不是同一概念。</span>

<span style="color:#c62828;font-weight:700">🔴【RISC/CISC】RISC指令规整、寻址较少、Load/Store、寄存器多，利于流水；CISC指令和寻址丰富、长度可变较常见。现代实现会融合技术，不能绝对说RISC一定更快。</span>

### 7 CPU数据通路与控制器

<span style="color:#c62828;font-weight:700">🔴【指令周期】通常含取指、间址（若需）、执行、中断周期；机器周期由多个时钟周期组成。不同指令的执行阶段可能不同。</span>

<span style="color:#1565c0;font-weight:700">🔵【取指公共微操作】典型为`MAR←PC`，发读命令；`MDR←M[MAR]`；`IR←MDR`；`PC←PC+指令长度`。若单总线数据通路，不能让多个源同时驱动总线，必要时用暂存寄存器。</span>

<span style="color:#c62828;font-weight:700">🔴【控制器】硬布线控制由组合/时序逻辑产生信号，速度快、修改难；微程序控制把控制信号编码为微指令，规整易扩展、通常较慢。</span>

<span style="color:#c62828;font-weight:700">🔴【微指令】水平型并行控制字段多、字长长、并行度高；垂直型编码程度高、字长短、译码后并行度较低。微程序由微指令组成，存于控制存储器，微地址形成决定下一微指令。</span>

<span style="color:#1565c0;font-weight:700">🔵【数据通路大题】按每个时钟写“源寄存器→总线/ALU→目的寄存器”，同步给出读写、ALU功能和寄存器装入控制；检查数据依赖和资源冲突。只写自然语言流程通常拿不到全部控制信号分。</span>

<span style="color:#c62828;font-weight:700">🔴【中断】CPU通常在一条指令结束处响应可屏蔽中断：保存断点/状态→识别中断源→转中断服务→恢复现场返回。异常由当前指令内部引起，中断多由外部事件引起。</span>

### 8 指令流水线（高频综合题）

<span style="color:#c62828;font-weight:700">🔴【基本思想】把指令执行分为若干阶段，多条指令重叠执行，提高吞吐率，不必然缩短单条指令延迟。</span>

<span style="color:#1565c0;font-weight:700">🔵【理想等长k段】每段时间Δt，完成n条指令时间`T=(k+n−1)Δt`；非流水`T0=nkΔt`；加速比`S=T0/T`；吞吐率`TP=n/T`，n很大时趋近`1/Δt`；效率`E=S/k=n/(k+n−1)`。</span>

<span style="color:#1565c0;font-weight:700">🔵【不等长阶段】流水时钟周期取最长阶段延迟加流水寄存器开销；总时间仍按流水图逐周期数，不能用平均阶段时间。</span>

<span style="color:#c62828;font-weight:700">🔴【结构冒险】多条指令争同一硬件资源，可复制/分离资源或停顿。🔴【数据冒险】RAW真相关最常考，可旁路/转发、编译调度或停顿；WAR、WAW在乱序执行中出现。🔴【控制冒险】分支改变PC，可预测、延迟分支或清空错误路径。</span>

<span style="color:#1565c0;font-weight:700">🔵【流水图步骤】①列每条指令及阶段；②逐周期左到右排；③遇RAW先看转发能在哪个阶段取得结果，否则插气泡；④分支按题设“在哪阶段决定”处理清空/停顿；⑤数最后完成周期，再算CPI/加速比。</span>

⚫【装入—使用】经典五段流水中，load数据通常到MEM末尾才可用，即使有转发，紧随的使用指令常仍需停1周期；具体看题设阶段时序。

⚫【加速比】实际流水有填充/排空、阶段不均、寄存器开销和冒险，通常小于段数k。

### 9 总线与I/O系统

<span style="color:#c62828;font-weight:700">🔴【总线】数据总线通常双向，宽度影响每次传输量；地址总线通常由主设备输出，宽度影响寻址范围；控制总线各线方向不一，传送读写、中断、请求/应答。</span>

<span style="color:#1565c0;font-weight:700">🔵【总线带宽】`每次传输字节数×每秒传输次数`。若w位、频率f、每周期r次传输，则`BW=(w/8)×f×r`，再按十进制/二进制单位换算。</span>

<span style="color:#c62828;font-weight:700">🔴【仲裁】集中式有链式查询、计数器定时查询、独立请求；分布式由各模块共同决定。链式优先级固定、连线少但低优先级可能饥饿且链故障敏感。</span>

<span style="color:#c62828;font-weight:700">🔴【I/O编址】统一编址把设备寄存器放入内存地址空间，可用访存指令；独立编址有独立I/O空间和专用指令。</span>

<span style="color:#c62828;font-weight:700">🔴【查询/中断/DMA】查询由CPU忙等；中断使CPU与设备并行但每次传输仍多由CPU参与；DMA控制器取得总线后在设备与主存间成块传送，结束时中断CPU。</span>

<span style="color:#c62828;font-weight:700">🔴【DMA传送】停止CPU访存、周期挪用（窃取）、交替访问等。DMA与CPU会竞争主存/总线，并非传输期间CPU任何工作都不受影响。</span>

<span style="color:#1565c0;font-weight:700">🔵【中断优先】响应优先级决定谁先被服务，屏蔽字可改变处理优先级；多重中断需保存现场并允许更高处理优先级中断嵌套。按题设屏蔽位含义绘时序。</span>

### 10 虚拟存储补充

<span style="color:#c62828;font-weight:700">🔴【页式虚拟存储】虚页号经页表/TLB变物理页框号，页内偏移不变；页表项含有效、权限、访问、脏等位。TLB命中不再访问页表，缺页则访问外存。</span>

<span style="color:#1565c0;font-weight:700">🔵【多级地址拆分】页大小决定页内偏移位数；剩余虚页号按各级页表项数拆分。页表大小＝页表项数×每项字节数，多级页表只为实际使用的地址区域分配下级表。</span>

⚫【三种事件】Cache未命中通常去主存；TLB未命中去查页表；缺页去辅存，代价差几个数量级，不能混为一谈。

## 计算机网络

> > 定位：大学期末速记，重点覆盖分层、差错控制、子网划分、路由、TCP和应用协议。🔴背概念；🔵背计算/报文分析/设计步骤；⚫看条件和易错点。本文件不放习题。

### 1 分层体系与性能

<span style="color:#c62828;font-weight:700">🔴【OSI七层】物理、数据链路、网络、运输、会话、表示、应用。TCP/IP常分网络接口、网际、运输、应用四层；教学五层模型为物理、链路、网络、运输、应用。</span>

<span style="color:#c62828;font-weight:700">🔴【PDU】应用层报文→运输层段/用户数据报→网络层IP数据报/分组→链路层帧→物理层比特。发送逐层封装，接收逐层解封装。</span>

<span style="color:#c62828;font-weight:700">🔴【地址】MAC标识本链路接口，IP实现跨网逻辑寻址，端口标识主机内进程，域名便于人使用并由DNS解析。</span>

<span style="color:#1565c0;font-weight:700">🔵【时延】总时延＝发送时延＋传播时延＋处理时延＋排队时延；发送时延＝数据长度(bit)/链路速率(bit/s)；传播时延＝链路长度/信号传播速率。</span>

<span style="color:#1565c0;font-weight:700">🔵【时延带宽积】传播时延×带宽，表示链路上可容纳的比特数。往返时间RTT不是单向传播时延；流水发送多分组时不能把每个分组端到端时延简单相乘。</span>

⚫【吞吐量】端到端吞吐通常受瓶颈链路及协议开销限制，不等于任一链路标称速率。

### 2 物理层

<span style="color:#c62828;font-weight:700">🔴【信道】单工只能单向；半双工可双向但不能同时；全双工可同时双向。基带传输直接传数字信号，频带传输先调制到载波。</span>

<span style="color:#1565c0;font-weight:700">🔵【奈奎斯特】无噪声低通信道最高码元率`2W Baud`；若每码元M种离散状态，最高比特率`C=2W log2M`。</span>

<span style="color:#1565c0;font-weight:700">🔵【香农】有噪声信道极限`C=W log2(1+S/N)` bit/s；分贝换线性比`S/N=10^(dB/10)`。香农给理论上限，不直接给调制方式。</span>

<span style="color:#c62828;font-weight:700">🔴【编码】不归零NRZ简单但同步差；曼彻斯特每比特中间跳变，自同步但占带宽；差分曼彻斯特用比特开始处是否跳变表示数据，中间总跳变。</span>

<span style="color:#c62828;font-weight:700">🔴【复用】FDM按频带、TDM按时隙、WDM是光波频分、CDM用正交码片。同步TDM即使用户无数据也可能占固定时隙，统计TDM动态分配。</span>

<span style="color:#c62828;font-weight:700">🔴【传输介质】双绞线经济常用于LAN；同轴抗干扰较强；光纤带宽高、衰减低、抗电磁干扰；无线传播受频段、遮挡和多径影响。</span>

### 3 数据链路层：成帧、检错与可靠传输

<span style="color:#c62828;font-weight:700">🔴【功能】成帧、差错检测、透明传输、流量控制和可靠交付（视链路协议）。点到点链路和广播链路关注点不同。</span>

<span style="color:#c62828;font-weight:700">🔴【透明传输】字节填充在特殊字节前插转义字节；比特填充在连续5个1后插0，接收端删除。</span>

<span style="color:#c62828;font-weight:700">🔴【检错】奇偶校验能力有限；CRC把比特串视为多项式；海明码可定位并纠正一定数量错误。检错不等于纠错或可靠交付。</span>

<span style="color:#1565c0;font-weight:700">🔵【CRC计算】生成多项式G阶数r→数据后补r个0→模2除法（异或、无借位）→r位余数作为FCS→接收端用同一G除，余数0表示“未检出错误”，不保证绝对无错。</span>

<span style="color:#c62828;font-weight:700">🔴【停止等待ARQ】发一帧等确认，需序号防重复、计时器处理丢失、ACK确认。利用率在长时延高速链路较低。</span>

<span style="color:#c62828;font-weight:700">🔴【滑动窗口】GBN累计确认，接收窗口通常1，超时重传出错帧及以后已发帧；SR逐帧确认并缓存失序帧，只重传缺失帧，但收发窗口与序号空间要避免新旧帧混淆。</span>

<span style="color:#1565c0;font-weight:700">🔵【窗口利用率近似】无差错、确认时间忽略时，停止等待利用率约`T发送/(T发送+RTT)`；窗口W可流水时约`min(1, W×T发送/(T发送+RTT))`。严格按题目是否含ACK发送时间处理。</span>

### 4 以太网与局域网

<span style="color:#c62828;font-weight:700">🔴【MAC】48位，交换机按源MAC自学习表项，按目的MAC转发：已知单播定向转发，未知单播/广播向除入端口外泛洪。</span>

<span style="color:#c62828;font-weight:700">🔴【CSMA/CD】传统共享半双工以太网先听后发、边发边检测冲突，冲突后截断并二进制指数退避；交换式全双工以太网不存在冲突，不用CSMA/CD。</span>

<span style="color:#c62828;font-weight:700">🔴【最小帧】经典以太网最小帧64B（从目的MAC到FCS），保证最远站发送完前能检测冲突；最大常规帧1518B（不含前导码，未计VLAN标签）。</span>

<span style="color:#c62828;font-weight:700">🔴【设备】集线器工作物理层，所有端口同一冲突域；交换机工作链路层，每端口独立冲突域，默认同一广播域；路由器工作网络层，隔离广播域。</span>

<span style="color:#c62828;font-weight:700">🔴【VLAN】逻辑划分广播域，跨VLAN通信需要三层转发；802.1Q标签携带VLAN ID，接入口通常属于单VLAN，中继口承载多个VLAN。</span>

⚫【生成树STP】二层冗余会产生广播风暴、重复帧和MAC表震荡；STP阻塞部分链路形成无环逻辑拓扑。

### 5 IPv4、子网与CIDR（必考计算）

<span style="color:#c62828;font-weight:700">🔴【IPv4】32位，点分十进制。CIDR前缀/n表示前n位为网络前缀；同一子网判断必须把IP与掩码逐位AND比较网络地址。</span>

<span style="color:#1565c0;font-weight:700">🔵【已知前缀求范围】主机位h=32−n；地址总数2^h；传统可用主机数2^h−2；网络地址主机位全0，广播地址全1，可用范围二者之间。/31点到点、/32主机路由等特殊用途不套“减2”。</span>

<span style="color:#1565c0;font-weight:700">🔵【快速找块】在掩码第一个非255字节，块大小＝256−该字节；该字节网络边界为“IP字节÷块大小取整×块大小”，下一边界减1为广播。</span>

<span style="color:#1565c0;font-weight:700">🔵【定长子网FLSM】需要至少s个子网，借位k满足2^k≥s；新前缀＝原前缀+k；再检查剩余主机位满足每网主机需求。列网络地址、广播和可用范围。</span>

<span style="color:#1565c0;font-weight:700">🔵【VLSM】把主机需求从大到小排序；每个需求H选最小块2^h使`2^h−2≥H`；前缀32−h；从母网低地址起按块边界依次分配，最后核对不重叠且不越界。</span>

<span style="color:#1565c0;font-weight:700">🔵【路由聚合】把连续、大小适当且边界对齐的网络地址写二进制，找最长公共前缀；聚合前缀后的范围不能夹入不应包括的网段。不能只看十进制相同段。</span>

⚫【掩码】合法掩码必须连续1后连续0；子网数和主机数的“减2”是旧式规则，现代CIDR题按题设，主机数通常仍扣网络和广播地址。

### 6 IP数据报、ARP、ICMP与路由

<span style="color:#c62828;font-weight:700">🔴【转发】主机先判断目的是否同子网；同网直接ARP目的IP，异网ARP默认网关。每跳链路层源/目的MAC会变，通常端到端源/目的IP不变（NAT除外）。</span>

<span style="color:#c62828;font-weight:700">🔴【ARP】在局域网用目标IPv4查MAC；请求广播、应答通常单播，结果缓存。ARP不跨路由器。</span>

<span style="color:#c62828;font-weight:700">🔴【ICMP】报告差错和诊断；ping用回显请求/应答，traceroute利用TTL逐跳减到0产生超时报文（实现可能用UDP/ICMP等探测）。ICMP差错报文本身通常不再引发ICMP差错。</span>

<span style="color:#c62828;font-weight:700">🔴【IPv4首部】版本、首部长度、总长度、标识/标志/片偏移、TTL、协议、校验和、源/目的地址。首部校验和只校验首部；TTL每经路由器至少减1。</span>

<span style="color:#1565c0;font-weight:700">🔵【分片】每片总长度≤MTU；除最后片外数据长度应为8B倍数；片偏移单位8B，MF=1表示后面还有片，所有片标识相同。目的主机重组，IPv6路由器不分片。</span>

<span style="color:#c62828;font-weight:700">🔴【路由表】目的前缀、下一跳、接口和度量。转发采用最长前缀匹配，而非先出现或度量最小；默认路由0.0.0.0/0最后兜底。</span>

<span style="color:#c62828;font-weight:700">🔴【协议】RIP是距离向量、以跳数计量（16不可达），周期交换；OSPF是链路状态，在区域内泛洪LSA并用Dijkstra；BGP是自治系统间路径向量，重策略和可达性，不以最短物理路径为唯一目标。</span>

<span style="color:#1565c0;font-weight:700">🔵【Dijkstra路由题】初始化源到各点代价→每轮确定最小暂定距离结点→用其邻边松弛→记录前驱→得到最短路径树和下一跳。要求边权非负。</span>

### 7 运输层：UDP与TCP

<span style="color:#c62828;font-weight:700">🔴【端口】16位。套接字常由IP+端口标识端点；TCP连接由源/目的IP和源/目的端口四元组标识。</span>

<span style="color:#c62828;font-weight:700">🔴【UDP】无连接、尽最大努力、面向报文、首部8B，保留应用报文边界；适合实时、简单查询、应用自建可靠机制。UDP有校验和但不保证重传、有序和流控。</span>

<span style="color:#c62828;font-weight:700">🔴【TCP】面向连接、可靠、有序、全双工、面向字节流；用序号、确认、校验、重传、窗口实现。序号是该段第一个数据字节号，ACK值是下一个期望字节号，SYN和FIN各消耗一个序号。</span>

<span style="color:#1565c0;font-weight:700">🔵【三次握手】客户端SYN=x→服务器SYN=y, ACK=x+1→客户端ACK=y+1。三次用于确认双方收发能力并同步初始序号；不能缩写成只“请求、同意、确认”。</span>

<span style="color:#1565c0;font-weight:700">🔵【四次挥手】一方FIN→对方ACK；对方数据发送完再FIN→原方ACK。主动关闭方进入TIME_WAIT等待2MSL，防最后ACK丢失可重发且让旧报文消失。全双工两个方向独立关闭，所以常为四段。</span>

<span style="color:#1565c0;font-weight:700">🔵【序号确认计算】数据段长度L，则下一序号=本段SEQ+L（另加SYN/FIN占用）；累计ACK=N表示N之前字节均已收到。纯ACK若无SYN/FIN通常不消耗序号。</span>

<span style="color:#c62828;font-weight:700">🔴【流量控制】接收窗口rwnd防止发送方淹没接收缓存；滑动窗口按字节。零窗口后发送方用持续计时器探测，避免窗口更新丢失而死锁。</span>

<span style="color:#c62828;font-weight:700">🔴【拥塞控制】cwnd限制网络注入量，实际发送窗口=min(rwnd,cwnd)。慢开始cwnd按RTT近似指数增长；拥塞避免线性增长；超时通常把ssthresh减半、cwnd重置较小；三次重复ACK触发快速重传，配合快速恢复（具体版本按题设）。</span>

<span style="color:#1565c0;font-weight:700">🔵【TCP吞吐/窗口】忽略损耗时要充分利用链路，窗口至少约为带宽×RTT（字节单位一致）；若给MSS，窗口段数≈窗口/MSS。</span>

⚫【可靠范围】TCP只保证字节流送到对端TCP接收缓冲，不保证应用已处理，也不自带消息边界和端到端业务事务语义。

### 8 应用层

<span style="color:#c62828;font-weight:700">🔴【DNS】层次命名、分布式数据库；递归查询要求受托服务器替客户继续查，迭代查询返回下一步服务器。常用UDP 53，区传和大型响应等可用TCP。</span>

<span style="color:#c62828;font-weight:700">🔴【HTTP】请求—响应、无状态；HTTP/1.1默认持久连接并可流水化（现实支持有限），HTTP/2二进制分帧和多路复用，HTTP/3基于QUIC/UDP。HTTPS＝HTTP经TLS，提供加密、完整性和身份认证。</span>

<span style="color:#c62828;font-weight:700">🔴【缓存】强缓存依据Cache-Control/Expires；协商缓存用ETag/If-None-Match或Last-Modified/If-Modified-Since，未修改返回304。Cookie由服务器设置并由浏览器后续携带，Session常在服务器保存状态。</span>

<span style="color:#c62828;font-weight:700">🔴【邮件】SMTP负责发送/中继；POP3、IMAP用于收取，IMAP更适合服务器端同步；MIME支持非ASCII正文和附件。</span>

<span style="color:#c62828;font-weight:700">🔴【DHCP】动态分配网络参数，典型DORA：Discover广播、Offer、Request、ACK；客户端初始无IP时使用UDP 67/68。</span>

<span style="color:#c62828;font-weight:700">🔴【常见端口】FTP 20/21、SSH 22、Telnet 23、SMTP 25、DNS 53、DHCP 67/68、HTTP 80、POP3 110、IMAP 143、HTTPS 443。端口是默认约定，可配置改变。</span>

### 9 网络安全与排障

<span style="color:#c62828;font-weight:700">🔴【目标】机密性、完整性、可用性；认证确认身份，授权决定权限，不可否认防抵赖。</span>

<span style="color:#c62828;font-weight:700">🔴【密码】对称加密快但密钥分发难；非对称便于密钥交换和签名但慢；散列不可逆用于完整性；数字签名通常“私钥签、对应公钥验”，不提供内容保密。</span>

<span style="color:#c62828;font-weight:700">🔴【防护】防火墙按策略控制流量；IDS检测告警，IPS可阻断；VPN在不可信网络上构建受保护隧道，不能自动保证终端本身安全。</span>

<span style="color:#1565c0;font-weight:700">🔵【排障自下而上】①链路灯/无线/网卡；②`ipconfig/ifconfig/ip`检查IP、掩码、网关、DNS；③ping回环→本机地址→网关→公网IP→域名；④`arp -a`查邻居；⑤`tracert/traceroute`定位路径；⑥`nslookup/dig`查DNS；⑦`netstat/ss`查端口连接。防火墙禁ICMP时ping失败不等于主机必然离线。</span>

## 电路分析与电工技术

### 一、直流电路基础

#### 1. 电路与参考方向

<span style="color:#c62828;font-weight:700">🔴【基本量】：电流 `i=dq/dt`，电压是两点电势差，电功率在关联参考方向下 `p=ui`。</span>

<span style="color:#1565c0;font-weight:700">🔵【功率符号】：`p>0` 表示元件吸收功率，`p<0` 表示元件发出功率；先看参考方向，再看正负号。</span>

<span style="color:#c62828;font-weight:700">🔴【开路】：支路电流为0，端电压不一定为0。【短路】：端电压为0，电流不一定为0。</span>

⚫【单位】：`k=10³，M=10⁶，m=10⁻³，μ=10⁻⁶，n=10⁻⁹`。计算前先统一为V、A、Ω、F、H、Hz。

#### 2. 欧姆定律、KCL与KVL

<span style="color:#c62828;font-weight:700">🔴【欧姆定律】：电阻元件 `u=Ri`；电导 `G=1/R`。</span>

<span style="color:#c62828;font-weight:700">🔴【KCL】：任一节点电流代数和为0，本质是电荷守恒。</span>

<span style="color:#c62828;font-weight:700">🔴【KVL】：任一闭合回路电压代数和为0，本质是能量守恒。</span>

<span style="color:#1565c0;font-weight:700">🔵【列方程步骤】：标参考方向 → 选节点或回路 → 统一“流出为正”或统一绕行方向 → 写方程 → 负答案解释为实际方向相反。</span>

#### 3. 串并联与分压分流

<span style="color:#c62828;font-weight:700">🔴【串联】：电流相同，`Req=ΣR`；两电阻分压 `Uk=U·Rk/(R1+R2)`。</span>

<span style="color:#c62828;font-weight:700">🔴【并联】：电压相同，`1/Req=Σ(1/Rk)`；两支路分流 `I1=I·R2/(R1+R2)`。</span>

<span style="color:#1565c0;font-weight:700">🔵【快速验算】：串联等效电阻大于任一电阻；并联等效电阻小于最小支路电阻；`n`个相同电阻并联 `Req=R/n`。</span>

⚫【拓扑判断】：两元件串联要求中间节点无其他分支；并联要求两端分别接在同一对节点，不能只凭图形位置判断。

#### 4. 电源模型

<span style="color:#c62828;font-weight:700">🔴【理想电压源】：端电压固定，电流由外电路决定；理想电流源相反。</span>

<span style="color:#c62828;font-weight:700">🔴【源等效变换】：电压源 `Us` 串联 `Rs` ↔ 电流源 `Is=Us/Rs` 并联同一个 `Rs`。</span>

⚫【限制】：等效仅对外端口成立；理想电压源不能单独等效为理想电流源。

### 二、线性电路分析与网络定理

#### 5. 节点电压法与网孔电流法

<span style="color:#c62828;font-weight:700">🔴【节点法】：选参考节点，以其余节点电压为未知量列KCL，支路电流写成电压差/电阻。</span>

<span style="color:#1565c0;font-weight:700">🔵【通用节点式】：`本节点电导和×本节点电压 − 相邻互电导×相邻电压 = 注入节点的电流源代数和`。</span>

<span style="color:#c62828;font-weight:700">🔴【超级节点】：电压源跨两个非参考节点时，把两节点整体列KCL，再补电压约束式。</span>

<span style="color:#c62828;font-weight:700">🔴【网孔法】：平面电路给每个网孔设电流；共享电阻压降写为 `R(Ik−Ij)`。</span>

#### 6. 叠加、戴维宁与诺顿

<span style="color:#c62828;font-weight:700">🔴【叠加定理】：线性电路中总电压或总电流等于各独立源单独作用结果的代数和。</span>

<span style="color:#1565c0;font-weight:700">🔵【关源规则】：独立电压源置零→短路；独立电流源置零→开路；受控源始终保留。</span>

⚫【陷阱】：功率不能直接叠加，应先求总电压或总电流再算功率。

<span style="color:#c62828;font-weight:700">🔴【戴维宁】：任何线性二端网络可等效为 `Uth` 串联 `Rth`；`Uth` 是端口开路电压。</span>

<span style="color:#1565c0;font-weight:700">🔵【求Rth】：关闭独立源后从端口向内看；含受控源时加测试源，`Rth=Utest/Itest`。也可在条件允许时 `Rth=Uoc/Isc`。</span>

<span style="color:#c62828;font-weight:700">🔴【诺顿】：等效为 `In` 并联 `Rn`，其中 `In=Isc，Rn=Rth`，且 `Uth=InRth`。</span>

<span style="color:#c62828;font-weight:700">🔴【最大功率】：纯电阻直流网络中 `RL=Rth`，`Pmax=Uth²/(4Rth)`。</span>

### 三、一阶动态电路

#### 7. 储能元件与换路定律

<span style="color:#c62828;font-weight:700">🔴【电容】：`iC=C·duC/dt`，储能 `WC=CuC²/2`；直流稳态相当于开路。</span>

<span style="color:#c62828;font-weight:700">🔴【电感】：`uL=L·diL/dt`，储能 `WL=LiL²/2`；直流稳态相当于短路。</span>

<span style="color:#c62828;font-weight:700">🔴【连续性】：无冲激条件下 `uC(0+)=uC(0−)`，`iL(0+)=iL(0−)`。</span>

⚫【可突变量】：电容电流、电感电压可以在换路瞬间突变。

#### 8. 三要素法

<span style="color:#c62828;font-weight:700">🔴【统一形式】：`f(t)=f(∞)+[f(0+)−f(∞)]e^(−t/τ)`，`t≥0`。</span>

<span style="color:#1565c0;font-weight:700">🔵【三步】：求换路前稳态得初值 → 求换路后最终直流稳态 → 关闭独立源求储能元件看到的等效电阻。</span>

<span style="color:#c62828;font-weight:700">🔴【时间常数】：RC电路 `τ=ReqC`；RL电路 `τ=L/Req`。</span>

<span style="color:#1565c0;font-weight:700">🔵【记忆】：`t=τ` 完成总变化量约63.2%；约 `5τ` 可视为进入新稳态。</span>

### 四、正弦稳态交流电路

#### 9. 正弦量与相量

<span style="color:#c62828;font-weight:700">🔴【三要素】：最大值、角频率、初相位；`ω=2πf，T=1/f`。</span>

<span style="color:#c62828;font-weight:700">🔴【有效值】：正弦量 `U=Um/√2，I=Im/√2`；交流额定值和电表读数通常指有效值。</span>

<span style="color:#c62828;font-weight:700">🔴【相量法】：同频正弦量可用复数相量运算，但相量不是瞬时量，不能把时间变量写进相量。</span>

#### 10. 阻抗、相位与功率

<span style="color:#c62828;font-weight:700">🔴【阻抗】：`ZR=R`，`ZL=jωL=jXL`，`ZC=1/(jωC)=−jXC`。</span>

<span style="color:#c62828;font-weight:700">🔴【电抗】：`XL=2πfL` 随频率增大；`XC=1/(2πfC)` 随频率增大而减小。</span>

<span style="color:#1565c0;font-weight:700">🔵【相位口诀】：纯电阻同相；感性“电流滞后”；容性“电流超前”。</span>

<span style="color:#c62828;font-weight:700">🔴【串联RLC】：`Z=R+j(XL−XC)`；`|Z|=√[R²+(XL−XC)²]`；`I=U/|Z|`。</span>

<span style="color:#c62828;font-weight:700">🔴【功率】：`P=UIcosφ`，`Q=UIsinφ`，`S=UI`，且 `S²=P²+Q²`。</span>

⚫【单位】：有功W，无功var，视在VA；感性负载通常 `Q>0`。

#### 11. 谐振

<span style="color:#c62828;font-weight:700">🔴【串联谐振】：`XL=XC`，`ω0=1/√(LC)`，总阻抗最小且为R，电流最大，功率因数为1。</span>

⚫【陷阱】：谐振时电源端无功功率为0，但L、C两端电压可能很大，不能说电感电容“不工作”。

### 五、三相、电工控制与可靠性

#### 12. 对称三相电路

<span style="color:#c62828;font-weight:700">🔴【Y接】：`UL=√3Up，IL=Ip`。【Δ接】：`UL=Up，IL=√3Ip`。</span>

<span style="color:#c62828;font-weight:700">🔴【总有功功率】：`P=√3ULILcosφ`；线量公式与负载接法无关，但前提是三相对称。</span>

#### 13. 变压器与异步电动机

<span style="color:#c62828;font-weight:700">🔴【理想变压器】：`U1/U2=N1/N2`，`I1/I2=N2/N1`，阻抗折算 `Zin=(N1/N2)²ZL`。</span>

<span style="color:#c62828;font-weight:700">🔴【同步转速】：`n1=60f/p`，`p`为磁极对数；转差率 `s=(n1−n)/n1`，电动运行通常 `0<s<1`。</span>

⚫【陷阱】：正常异步电动机转子转速略低于同步转速；“四极”表示 `p=2`。

#### 14. 继电接触控制

<span style="color:#c62828;font-weight:700">🔴【自锁】：接触器常开辅助触点与启动按钮并联。【互锁】：正反转接触器的常闭辅助触点交叉串入对方线圈。</span>

<span style="color:#c62828;font-weight:700">🔴【保护】：熔断器/断路器—短路；热继电器—长期过载；接触器失电释放—失压保护。</span>

#### 15. 系统可靠性

<span style="color:#c62828;font-weight:700">🔴【功能串联】：任一失效系统即失效，`Rs=ΠRi`。</span>

<span style="color:#c62828;font-weight:700">🔴【功能并联】：至少一个成功系统就成功，`Rp=1−Π(1−Ri)`。</span>

⚫【前提与陷阱】：元件失效相互独立；可靠性逻辑的串并联不是电阻物理串并联，更不使用分压分流公式。

### 考前完整性清单

<span style="color:#c62828;font-weight:700">🔴能否独立完成：串并联、KCL/KVL、节点法、叠加、戴维宁、一阶三要素、阻抗与相位、交流功率、三相功率？</span>

<span style="color:#1565c0;font-weight:700">🔵是否会写完整过程：参考方向 → 原理/公式 → 代入统一单位 → 数值与单位 → 正负号或物理意义？</span>

## 模拟电子技术

### 一、二极管与整流电路

#### 1. PN结与二极管模型

<span style="color:#c62828;font-weight:700">🔴【单向导电】：正向偏置时势垒降低、导通；反向偏置时势垒升高、近似截止。</span>

<span style="color:#c62828;font-weight:700">🔴【温度】：温度升高，正向压降通常减小，反向饱和电流增大。</span>

<span style="color:#1565c0;font-weight:700">🔵【判管步骤】：假设导通/截止 → 按所选模型求电位与电流 → 检查阳极、阴极电位是否支持假设。</span>

<span style="color:#c62828;font-weight:700">🔴【常用模型】：理想模型导通压降0；恒压降模型硅管约0.7 V、锗管约0.2～0.3 V。</span>

⚫【击穿】：普通二极管反向击穿可能损坏；稳压管设计为在限流条件下工作于反向击穿区。

#### 2. 整流、滤波与限幅

<span style="color:#c62828;font-weight:700">🔴【平均值】：半波整流 `Uo≈0.45U2`；桥式全波整流 `Uo≈0.9U2`，均为忽略管压降的常用近似。</span>

<span style="color:#c62828;font-weight:700">🔴【桥式整流】：每半周有两只二极管导通；选管要检查平均电流和最高反向电压。</span>

<span style="color:#c62828;font-weight:700">🔴【电容滤波】：电容在峰值附近充电、其余时间向负载放电；增大C或负载电阻通常减小纹波。</span>

⚫【故障题】：桥中一管开路常退化为半波整流；电容开路导致纹波显著增大。

### 二、双极型三极管与基本放大器

#### 3. 三极管三个工作区

<span style="color:#c62828;font-weight:700">🔴【截止区】：发射结未正偏，`IB≈0，IC≈0`，开关近似断开。</span>

<span style="color:#c62828;font-weight:700">🔴【放大区】：发射结正偏、集电结反偏，`IC≈βIB`，用于线性放大。</span>

<span style="color:#c62828;font-weight:700">🔴【饱和区】：两结均正偏，`VCE≈VCE(sat)`，开关近似闭合；此时不能继续使用 `IC=βIB` 计算实际集电流。</span>

<span style="color:#1565c0;font-weight:700">🔵【判区流程】：先按放大区算 `IC=βIB` → 由外电路求 `VCE` → 若算得VCE过小或集电流超过负载允许值，则进入饱和；若基极无正向驱动则截止。</span>

#### 4. 静态工作点与失真

<span style="color:#c62828;font-weight:700">🔴【静态量】：`IBQ、ICQ、VCEQ`；设置目的是让交流信号在放大区上下摆动。</span>

<span style="color:#c62828;font-weight:700">🔴【截止失真】：工作点偏低或输入摆幅使管子截止。【饱和失真】：工作点偏高或摆幅使管子饱和。</span>

<span style="color:#c62828;font-weight:700">🔴【温度稳定】：分压偏置配合发射极电阻形成直流负反馈，抑制温漂。</span>

⚫【旁路电容】：交流下旁路RE可提高增益；直流下电容开路，RE稳定静态点的作用仍保留。

#### 5. 三种基本组态

<span style="color:#c62828;font-weight:700">🔴【共射】：电压、电流和功率均可放大；输出与输入反相；输入、输出电阻中等。</span>

<span style="color:#c62828;font-weight:700">🔴【共集/射极跟随器】：电压增益约1、同相；输入电阻高、输出电阻低，适合缓冲与阻抗变换。</span>

<span style="color:#c62828;font-weight:700">🔴【共基】：电流增益约小于1，电压增益较大；输入电阻低，高频特性好。</span>

<span style="color:#1565c0;font-weight:700">🔵【共射中频近似】：若发射极交流接地，`Au≈−β(RC∥RL)/rbe`；负号表示反相。</span>

⚫【负载效应】：接入负载后有效集电极负载为 `RC∥RL`，增益绝对值通常下降。

#### 6. 场效应管

<span style="color:#c62828;font-weight:700">🔴【特点】：FET是电压控制器件，栅极电流很小、输入电阻高；BJT是电流控制器件。</span>

<span style="color:#c62828;font-weight:700">🔴【MOS管区分】：增强型需要足够栅源电压建立沟道；判断N/P沟道和截止/恒流/可变电阻区必须结合器件类型与电压极性。</span>

### 三、频率响应与多级放大

#### 7. 频率响应

<span style="color:#c62828;font-weight:700">🔴【通频带】：下限频率 `fL` 与上限频率 `fH` 之间；截止点电压增益降至中频的 `1/√2`，即−3 dB。</span>

<span style="color:#c62828;font-weight:700">🔴【低频下降】：耦合、旁路电容容抗增大。【高频下降】：结电容、杂散电容和米勒效应影响。</span>

<span style="color:#1565c0;font-weight:700">🔵【分贝】：电压增益 `20log10|Au|`；功率增益 `10log10(Po/Pi)`。</span>

#### 8. 多级放大

<span style="color:#c62828;font-weight:700">🔴【总增益】：各级电压增益相乘；以dB表示时相加。</span>

⚫【级间影响】：后级输入电阻是前级负载，实际各级增益不能简单按空载值相乘。

### 四、集成运算放大器

#### 9. 理想运放与负反馈条件

<span style="color:#c62828;font-weight:700">🔴【理想参数】：开环增益无穷大、输入电阻无穷大、输出电阻为0、带宽无穷大。</span>

<span style="color:#c62828;font-weight:700">🔴【虚短】：线性负反馈状态下 `u+≈u−`。【虚断】：`i+≈i−≈0`。</span>

⚫【适用条件】：必须有负反馈且输出未饱和；比较器开环/正反馈时不能套“虚短”。

#### 10. 常见运算电路

<span style="color:#c62828;font-weight:700">🔴【反相比例】：`uo=−(Rf/Rin)ui`，反相端近似虚地。</span>

<span style="color:#c62828;font-weight:700">🔴【同相比例】：`uo=(1+Rf/R1)ui`，输入电阻高。</span>

<span style="color:#c62828;font-weight:700">🔴【加法器】：按KCL叠加各输入电流；等电阻反相加法 `uo=−RfΣ(ui/Ri)`。</span>

<span style="color:#c62828;font-weight:700">🔴【积分器】：`uo=−(1/RC)∫ui dt`。【微分器】：`uo=−RC·dui/dt`。</span>

<span style="color:#1565c0;font-weight:700">🔵【解题步骤】：先确认负反馈 → 写虚短、虚断 → 对反相节点列KCL → 解输出 → 检查是否超过电源允许摆幅。</span>

#### 11. 比较器与滞回比较器

<span style="color:#c62828;font-weight:700">🔴【单限比较器】：比较 `u+` 和 `u−`，输出进入正或负饱和。</span>

<span style="color:#c62828;font-weight:700">🔴【滞回比较器】：使用正反馈形成两个不同阈值，提高抗噪能力；输入上升和下降时翻转点不同。</span>

### 五、反馈与振荡

#### 12. 反馈类型与负反馈影响

<span style="color:#c62828;font-weight:700">🔴【反馈判别】：输出取样量是电压/电流；输入端叠加方式是串联/并联；瞬时极性判断正负反馈。</span>

<span style="color:#c62828;font-weight:700">🔴【闭环增益】：负反馈 `Af=A/(1+AF)`；深度负反馈 `|AF|≫1` 时 `Af≈1/F`。</span>

<span style="color:#c62828;font-weight:700">🔴【负反馈作用】：稳定增益、减小非线性失真、扩展通频带、改善输入输出电阻，但通常降低增益。</span>

<span style="color:#1565c0;font-weight:700">🔵【电阻规律】：串联反馈提高输入电阻，并联反馈降低输入电阻；电压反馈降低输出电阻，电流反馈提高输出电阻。</span>

#### 13. 正弦振荡

<span style="color:#c62828;font-weight:700">🔴【自激条件】：幅值条件 `|AF|=1`，相位条件总相移为 `2kπ`；起振时通常要求环路增益略大于1。</span>

<span style="color:#c62828;font-weight:700">🔴【组成】：放大、正反馈、选频、稳幅。RC振荡适合低频，LC振荡适合高频，晶体振荡频率稳定度高。</span>

### 六、功率放大与直流稳压电源

#### 14. 功率放大

<span style="color:#c62828;font-weight:700">🔴【甲类】：失真小、效率低。【乙类】：静态功耗小、效率高但有交越失真。【甲乙类】：小静态偏置减小交越失真。</span>

<span style="color:#c62828;font-weight:700">🔴【互补对称】：NPN、PNP管分别放大正负半周；必须关注输出功率、效率、管耗和安全工作区。</span>

⚫【最大管耗】：不一定发生在最大输出功率点，不能只按负载功率判断器件安全。

#### 15. 稳压电源

<span style="color:#c62828;font-weight:700">🔴【功能链】：变压 → 整流 → 滤波 → 稳压。</span>

<span style="color:#c62828;font-weight:700">🔴【稳压管电路】：稳压管反向击穿，串联电阻同时承担限流和分压；必须校验稳压管最小、最大电流。</span>

<span style="color:#c62828;font-weight:700">🔴【串联反馈稳压】：取样比较 → 误差放大 → 调整管改变压降，构成负反馈稳定输出。</span>

### 考前完整性清单

<span style="color:#c62828;font-weight:700">🔴能否快速判定二极管通断、BJT截止/放大/饱和、共射相位、运放线性/饱和、反馈极性？</span>

<span style="color:#1565c0;font-weight:700">🔵能否独立算出静态工作点、共射增益、反相/同相运放输出、反馈闭环增益、整流平均值和滤波故障？</span>

⚫若本校使用童诗白教材，差分放大、镜像电流源、复合管和集成功放也可能进入范围；若教师课堂重点讲过，应回讲义和课件补上对应推导。

## 数字电子技术

### 一、数制、编码与逻辑代数

#### 1. 数制与常用编码

<span style="color:#c62828;font-weight:700">🔴【位权展开】：二进制转十进制按 `Σ(数码×2^位权)`；十进制整数转二进制用除2取余逆序。</span>

<span style="color:#1565c0;font-weight:700">🔵【分组转换】：二进制↔八进制每3位一组；二进制↔十六进制每4位一组，不足在高位补0。</span>

<span style="color:#c62828;font-weight:700">🔴【BCD码】：8421 BCD每4位表示一个十进制数字；例如十进制59写成 `0101 1001`，不是59的纯二进制。</span>

<span style="color:#c62828;font-weight:700">🔴【格雷码】：相邻码仅一位变化，常用于减少转换冒险；二进制转格雷为最高位不变、其余相邻二进制位异或。</span>

#### 2. 基本逻辑与公式

<span style="color:#c62828;font-weight:700">🔴【基本门】：与 `AB`，或 `A+B`，非 `Ā`；异或输入不同时为1，同或输入相同时为1。</span>

<span style="color:#c62828;font-weight:700">🔴【常用律】：`A+A=A`，`AA=A`，`A+0=A`，`A·1=A`，`A+Ā=1`，`AĀ=0`。</span>

<span style="color:#c62828;font-weight:700">🔴【吸收律】：`A+AB=A`，`A(A+B)=A`。</span>

<span style="color:#c62828;font-weight:700">🔴【德摩根】：`overline(A+B)=ĀB̄`，`overline(AB)=Ā+B̄`。</span>

<span style="color:#1565c0;font-weight:700">🔵【代数化简】：展开/提取公因式 → 用吸收律 → 配反变量 → 检查是否还能减少项或变量。</span>

### 二、逻辑函数表示与卡诺图

#### 3. 最小项、最大项和标准式

<span style="color:#c62828;font-weight:700">🔴【最小项】：每个变量出现一次，取原变量或反变量；某输入组合下仅对应一个最小项为1。</span>

<span style="color:#c62828;font-weight:700">🔴【标准与或式】：函数为1的最小项之和 `F=Σm(...)`；标准或与式用函数为0的最大项表示 `F=ΠM(...)`。</span>

⚫【编号】：变量顺序必须按题目约定，通常高位在左；不要把卡诺图格序当自然二进制顺序。

#### 4. 卡诺图化简

<span style="color:#c62828;font-weight:700">🔴【相邻性】：行列采用格雷码；左右边界、上下边界也相邻，四角可组成一圈。</span>

<span style="color:#c62828;font-weight:700">🔴【圈1规则】：与或式圈1；圈大小必须为 `1、2、4、8...`；圈尽量大、数量尽量少，所有1必须覆盖。</span>

<span style="color:#1565c0;font-weight:700">🔵【写项】：圈内保持不变的变量留下，变化变量消去；变量恒1写原变量，恒0写反变量。</span>

⚫【重叠与无关项】：圈可以重叠；约束中的无关项X可按0或1使用，但只有能简化时才利用。

### 三、门电路与组合逻辑

#### 5. TTL/CMOS基本常识

<span style="color:#c62828;font-weight:700">🔴【CMOS】：静态功耗低、输入电阻高、噪声容限较好；输入端不能悬空。</span>

<span style="color:#c62828;font-weight:700">🔴【三态门】：除0、1外还有高阻态Z，可用于总线共享；高阻态不是逻辑0。</span>

⚫【TTL悬空】：传统TTL输入悬空常等效高电平，但工程上仍应明确连接；题目指定系列时按器件手册。

#### 6. 组合电路分析与设计

<span style="color:#1565c0;font-weight:700">🔵【分析步骤】：由电路逐级写表达式 → 化简 → 列真值表 → 说明逻辑功能。</span>

<span style="color:#1565c0;font-weight:700">🔵【设计步骤】：读题定义输入输出 → 列真值表 → 写并化简函数 → 选门/译码器/数据选择器实现 → 检查无效输入。</span>

<span style="color:#c62828;font-weight:700">🔴【半加器】：`S=A⊕B，C=AB`。【全加器】：`S=A⊕B⊕Cin`，进位为任意两个输入同时为1。</span>

<span style="color:#c62828;font-weight:700">🔴【译码器】：n位输入最多产生 `2^n` 路互斥输出；关注使能端和输出高/低有效小圆圈。</span>

<span style="color:#c62828;font-weight:700">🔴【数据选择器】：`2^n`选1需要n位选择端；可把变量接选择端、剩余变量接数据端实现逻辑函数。</span>

<span style="color:#c62828;font-weight:700">🔴【编码器】：把有效输入位置编码；优先编码器允许多个输入同时有效并按优先级输出。</span>

#### 7. 竞争与冒险

<span style="color:#c62828;font-weight:700">🔴【原因】：同一信号经不同路径传播延迟不同，输入变化时可能产生瞬态毛刺。</span>

<span style="color:#1565c0;font-weight:700">🔵【常用消除】：增加冗余项、输出端滤波、用选通脉冲避开毛刺；同步系统还应满足时序约束。</span>

### 四、触发器

#### 8. RS、JK、D、T特性

<span style="color:#c62828;font-weight:700">🔴【RS】：置位、复位、保持；禁用组合取决于门型。或非型高有效禁用 `S=R=1`，与非型低有效禁用 `S̄=R̄=0`。</span>

<span style="color:#c62828;font-weight:700">🔴【JK】：`Q+=JQ̄+K̄Q`；`J=K=1` 翻转，无RS禁用状态。</span>

<span style="color:#c62828;font-weight:700">🔴【D】：`Q+=D`，有效沿把D送入Q。</span>

<span style="color:#c62828;font-weight:700">🔴【T】：`Q+=T⊕Q`；`T=0` 保持，`T=1` 翻转。</span>

⚫【异步端】：异步置位/清零端有效时不等待时钟，通常优先级最高；有效电平看小圆圈或题目说明。

#### 9. 波形题

<span style="color:#1565c0;font-weight:700">🔵【五步法】：确认触发器类型 → 看上升沿/下降沿/电平触发 → 标出异步端有效区 → 从给定初态开始 → 只在有效时刻代特性方程更新。</span>

⚫【陷阱】：边沿触发器不是时钟高电平期间连续透明；必须区分输入建立/保持时间与传播延迟。

### 五、时序逻辑与计数器

#### 10. 时序电路分析

<span style="color:#1565c0;font-weight:700">🔵【标准步骤】：写各触发器激励方程 → 代入特性方程得状态方程 → 写输出方程 → 列状态转换表 → 画状态图/时序图 → 说明功能。</span>

<span style="color:#c62828;font-weight:700">🔴【Moore型】：输出只与状态有关。【Mealy型】：输出与状态和当前输入都有关。</span>

<span style="color:#c62828;font-weight:700">🔴【自启动】：从任何无效状态都能在有限拍进入有效状态循环；设计后必须检查未用状态。</span>

#### 11. 寄存器与计数器

<span style="color:#c62828;font-weight:700">🔴【寄存器】：存储多位数据；移位寄存器可实现串/并转换、延时和序列产生。</span>

<span style="color:#c62828;font-weight:700">🔴【计数容量】：n个触发器最多有 `2^n` 个状态；模M计数器最少n满足 `2^n≥M`。</span>

<span style="color:#c62828;font-weight:700">🔴【同步计数器】：各级共用时钟，速度较高；异步计数器逐级触发，传播延迟累积。</span>

<span style="color:#1565c0;font-weight:700">🔵【任意进制设计】：确定状态数 → 选触发器数 → 编码与状态图 → 求激励方程 → 化简实现 → 检查自启动。</span>

### 六、脉冲、存储器与数模转换

#### 12. 555定时器

<span style="color:#c62828;font-weight:700">🔴【核心阈值】：典型内部比较参考为 `1/3VCC` 与 `2/3VCC`。</span>

<span style="color:#c62828;font-weight:700">🔴【单稳态】：一个稳定状态，触发后输出一个定宽脉冲。【多谐振荡器】：无稳定状态，连续产生矩形波。【施密特】：双阈值整形与抗噪。</span>

#### 13. 半导体存储器

<span style="color:#c62828;font-weight:700">🔴【ROM】：掉电不丢失，主要读出固定/少改内容。【RAM】：可读写，掉电通常丢失。</span>

<span style="color:#c62828;font-weight:700">🔴【容量扩展】：字扩展增加地址数量；位扩展增加每字位数；芯片数=`字数倍数×位数倍数`。</span>

#### 14. ADC与DAC

<span style="color:#c62828;font-weight:700">🔴【ADC过程】：采样保持 → 量化 → 编码；n位ADC有 `2^n` 个量化级，分辨率约为满量程/`2^n`。</span>

<span style="color:#c62828;font-weight:700">🔴【奈奎斯特抽样】：带限信号最高频率fm，理想条件 `fs≥2fm`。</span>

<span style="color:#c62828;font-weight:700">🔴【DAC】：把数字量转换为模拟量；常考倒T形电阻网络、分辨率、满量程和转换时间。</span>

⚫【量化误差】：理想均匀量化误差绝对值通常不超过半个量化间隔。

### 考前完整性清单

<span style="color:#c62828;font-weight:700">🔴是否能完成：代数/卡诺图化简、组合电路分析设计、四类触发器转换、波形推演、状态表/状态图、计数器模值？</span>

<span style="color:#1565c0;font-weight:700">🔵是否记住：有效电平与小圆圈、异步端优先、模M触发器数、ADC级数、555两个阈值？</span>

⚫若本校期末包含PLD、ROM阵列实现、同步时序约束或复杂序列检测器，应按教师课堂例题补充；本速记覆盖普通本科数字电子技术的主干范围。

## 通信原理与高频电子线路

### 一、通信系统与信息量

#### 1. 系统模型与传输方式

<span style="color:#c62828;font-weight:700">🔴【基本链路】：信源 → 发送设备 → 信道+噪声 → 接收设备 → 信宿。</span>

<span style="color:#c62828;font-weight:700">🔴【发送设备】：编码、调制、放大，使消息变为适合信道传输的信号；接收端完成逆过程。</span>

<span style="color:#c62828;font-weight:700">🔴【单工】：只能单向；【半双工】：双向但不能同时，如对讲机；【全双工】：双向同时，如电话。</span>

<span style="color:#c62828;font-weight:700">🔴【模拟/数字通信】：按传输信号取值连续或离散划分，不等同于信源本身是否模拟。</span>

#### 2. 信息量、码元与速率

<span style="color:#c62828;font-weight:700">🔴【自信息量】：`I=−log2P(x)` bit；等概率M进制每码元携带 `log2M` bit。</span>

<span style="color:#c62828;font-weight:700">🔴【码元速率】：`RB=1/TB`，单位Bd；【比特率】：单位bit/s。</span>

<span style="color:#1565c0;font-weight:700">🔵【关系】：理想等概率M进制 `Rb=RBlog2M`；只有二进制时数值才常相等。</span>

⚫【陷阱】：波特率不是“每秒比特数”，也不宜简单写成“波形变化次数”。

#### 3. 信噪比与香农容量

<span style="color:#c62828;font-weight:700">🔴【分贝】：功率比 `SNRdB=10log10(S/N)`；电压比在同阻抗下可用 `20log10(U2/U1)`。</span>

<span style="color:#c62828;font-weight:700">🔴【香农公式】：`C=B log2(1+S/N)` bit/s。</span>

<span style="color:#1565c0;font-weight:700">🔵【计算步骤】：dB先转功率比 `S/N=10^(dB/10)` → 带宽换Hz → 代公式 → 结果写bit/s或kbit/s。</span>

⚫【物理意义】：C是给定带宽和信噪比下可靠通信的理论上限；提高S/N的收益呈对数增长。

### 二、信道、噪声与基带传输

#### 4. 信道与噪声

<span style="color:#c62828;font-weight:700">🔴【信道】：有线/无线、恒参/随参；常见影响为衰减、失真、时延和噪声。</span>

<span style="color:#c62828;font-weight:700">🔴【白噪声】：功率谱密度在研究频段内近似常数；高斯白噪声的幅度服从高斯分布。</span>

<span style="color:#c62828;font-weight:700">🔴【误码率】：错误比特数/传输总比特数；不能与误码个数混为单位。</span>

#### 5. 无码间串扰与眼图

<span style="color:#c62828;font-weight:700">🔴【奈奎斯特第一准则】：接收抽样时刻本码元响应不为0、其他码元响应为0，可无码间串扰判决。</span>

<span style="color:#c62828;font-weight:700">🔴【理想低通信道最高码元率】：`RB,max=2B` Bd；若M进制，理想比特率上限 `2Blog2M`。</span>

<span style="color:#c62828;font-weight:700">🔴【升余弦】：带宽 `B=(1+α)RB/2`，`0≤α≤1`；α越大，时域拖尾更小但占用带宽增大。</span>

<span style="color:#c62828;font-weight:700">🔴【眼图】：眼睛张开越大，码间串扰与噪声通常越小；最佳抽样时刻在眼睛最张开处。</span>

### 三、抽样、量化与PCM

#### 6. 抽样定理

<span style="color:#c62828;font-weight:700">🔴【低通信号】：最高频率fm，理想无混叠要求 `fs≥2fm`。</span>

⚫【工程条件】：实际抗混叠滤波器有过渡带，抽样率常留裕量；等号是理想下限。

#### 7. PCM

<span style="color:#c62828;font-weight:700">🔴【流程】：抽样 → 量化 → 编码；接收端译码、低通重建。</span>

<span style="color:#c62828;font-weight:700">🔴【码率】：每秒fs个样值、每样值n bit，则 `Rb=fsn`；多路复用还要乘路数并考虑帧开销。</span>

<span style="color:#c62828;font-weight:700">🔴【量化级】：n位均匀量化有 `L=2^n` 级；量化间隔约为满量程/L，误差绝对值通常≤半个量化间隔。</span>

<span style="color:#c62828;font-weight:700">🔴【非均匀量化】：A律/μ律对小信号分得更细，改善小信号量化信噪比。</span>

⚫【13折线A律】：8位码常由极性、段落码和段内码构成；具体码字规则以教材表格为准。

### 四、模拟调制

#### 8. AM、DSB-SC与SSB

<span style="color:#c62828;font-weight:700">🔴【AM频谱】：载波fc及上下边带 `fc±fm`；基带最高频率fm时带宽 `BAM=2fm`。</span>

<span style="color:#c62828;font-weight:700">🔴【调幅系数】：单音AM为避免过调幅通常 `0≤ma≤1`；`ma>1` 包络检波会失真。</span>

<span style="color:#c62828;font-weight:700">🔴【DSB-SC】：抑制载波、保留双边带，带宽2fm，需相干解调。</span>

<span style="color:#c62828;font-weight:700">🔴【SSB】：只传一个边带，带宽fm，功率和频带利用率高，但收发实现更复杂。</span>

<span style="color:#1565c0;font-weight:700">🔵【相干解调】：乘以同频同相本振 → 低通滤波；频率或相位不同步会衰减、失真。</span>

#### 9. FM与PM

<span style="color:#c62828;font-weight:700">🔴【FM】：幅度理想不变，瞬时频率随调制信号变化；抗幅度噪声能力较好。</span>

<span style="color:#c62828;font-weight:700">🔴【调频指数】：单音FM `β=Δf/fm`；卡森公式 `BFM≈2(Δf+fm)=2(β+1)fm`。</span>

⚫【窄带/宽带】：β≪1为窄带近似，β较大为宽带；FM带宽不能一律写成2fm。

### 五、数字调制与差错控制

#### 10. ASK、FSK、PSK与QAM

<span style="color:#c62828;font-weight:700">🔴【2ASK】：用幅度表示比特，易受幅度噪声影响。【2FSK】：用两种频率表示。【2PSK/BPSK】：用相差π的载波表示。</span>

<span style="color:#c62828;font-weight:700">🔴【QPSK】：4种相位，每码元2 bit；M进制每码元 `log2M` bit。</span>

<span style="color:#c62828;font-weight:700">🔴【QAM】：同时利用幅度和相位，提高频谱效率，但阶数越高对信噪比和线性度要求越高。</span>

<span style="color:#c62828;font-weight:700">🔴【相干BPSK误码率】：常见AWGN条件下与 `Eb/N0` 有关；比较性能时必须统一检测方式和每比特能量。</span>

⚫【差分编码】：可解决PSK相干载波的相位模糊，但性能和实现代价需按题目模型判断。

#### 11. 信道编码

<span style="color:#c62828;font-weight:700">🔴【目的】：加入结构化冗余用于检错/纠错，以带宽或码率换可靠性。</span>

<span style="color:#c62828;font-weight:700">🔴【奇偶校验】：能检出奇数个比特错误，不能定位并纠正。【循环码】：利用生成多项式，系统码常由信息位+校验位组成。</span>

<span style="color:#c62828;font-weight:700">🔴【汉明距离】：要检出e个错误需 `dmin≥e+1`；纠正t个错误需 `dmin≥2t+1`。</span>

### 六、复用、移动与光纤通信

#### 12. 复用与多址

<span style="color:#c62828;font-weight:700">🔴【FDM】：不同频带同时传输。【TDM】：不同时间隙。【WDM】：光波长复用。【CDM】：不同码序列。</span>

⚫【多址与复用】：技术思想相关，但多址强调多个用户共享资源，复用强调多路信号合并到一条链路。

#### 13. 移动通信代际

<span style="color:#c62828;font-weight:700">🔴【主线】：1G模拟语音；2G数字语音与短信；3G移动数据；4G全IP移动宽带；5G增强移动宽带、海量连接、低时延高可靠场景。</span>

⚫【考试边界】：代际标准众多，避免把某一地区制式说成全球唯一；以课程给定标准为准。

#### 14. 电磁波与光纤

<span style="color:#c62828;font-weight:700">🔴【频率由低到高】：无线电/微波 → 红外 → 可见光 → 紫外 → X射线 → γ射线；波长顺序相反。</span>

<span style="color:#c62828;font-weight:700">🔴【光纤窗口】：常用 `1.31 μm` 与 `1.55 μm`；1.55 μm附近损耗更低，1.31 μm附近色散特性具有传统优势。</span>

<span style="color:#c62828;font-weight:700">🔴【数值孔径】：反映光纤集光能力；单模光纤带宽高、色散小、适合长距离高速传输。</span>

### 七、高频电子线路

#### 15. 高频小信号放大器

<span style="color:#c62828;font-weight:700">🔴【调谐回路】：完成选频、阻抗变换和负载作用；品质因数Q越高，选择性通常越好、通频带越窄。</span>

<span style="color:#1565c0;font-weight:700">🔵【带宽近似】：单调谐回路 `BW≈f0/Q`；多级级联提高选择性但总带宽可能变窄。</span>

#### 16. 高频功率放大器

<span style="color:#c62828;font-weight:700">🔴【丙类谐振功放】：导通角小于180°，效率高；集电极电流脉冲经谐振回路选出基波近似正弦。</span>

<span style="color:#c62828;font-weight:700">🔴【工作状态】：欠压、临界、过压按负载线和电流脉冲形状判断；最佳状态常在临界附近，具体以教材图为准。</span>

#### 17. 混频与超外差接收

<span style="color:#c62828;font-weight:700">🔴【混频】：非线性器件产生和频、差频等组合频率，选出固定中频 `fIF=|fLO−fRF|`。</span>

<span style="color:#c62828;font-weight:700">🔴【超外差链路】：天线/高放 → 混频+本振 → 中频放大 → 检波/解调 → 低频放大。</span>

<span style="color:#c62828;font-weight:700">🔴【镜像频率】：另一射频也能与本振产生同一中频；提高前端选择性可抑制镜像干扰。</span>

#### 18. 反馈振荡器与锁相环

<span style="color:#c62828;font-weight:700">🔴【振荡条件】：`|AF|=1` 且总相移为 `2kπ`；起振时环路增益略大于1，稳幅后回到1。</span>

<span style="color:#c62828;font-weight:700">🔴【PLL】：鉴相器+环路滤波器+压控振荡器；锁定后输出频率/相位跟踪输入，可用于频率合成、解调和载波恢复。</span>

### 考前完整性清单

<span style="color:#c62828;font-weight:700">🔴是否能完成：信息量与速率、dB换算、香农容量、升余弦带宽、抽样PCM码率、AM/FM带宽、M进制每码元位数？</span>

<span style="color:#1565c0;font-weight:700">🔵是否能画出并解释：通信系统框图、AM/DSB频谱、PCM流程、相干解调、超外差接收链路？</span>

⚫若“通信原理”和“高频电子线路”分开考试，高频线路还可能重点考谐振功放负载特性、变频干扰和PLL定量分析；按本校教师划定范围取舍。

## 软件工程

> > 定位：大学期末简答、分析和设计题速记。🔴背定义与原则；🔵背需求、建模、测试和项目计算步骤；⚫看适用条件和常见扣分点。本文件不放习题。

### 1 软件工程与过程

<span style="color:#c62828;font-weight:700">🔴【软件工程】应用系统化、规范化、可量化的方法开发、运行和维护软件，并研究这些方法。目标是在成本和进度约束下获得可维护的高质量软件。</span>

<span style="color:#c62828;font-weight:700">🔴【软件危机】规模复杂度增长导致延期、超支、质量差、难维护；根因包含需求不清、估算与管理不足、方法工具落后和复杂性本身。</span>

<span style="color:#c62828;font-weight:700">🔴【生命周期】可行性/计划→需求分析→概要设计→详细设计→编码→测试→部署运行→维护。阶段可迭代，不等于所有项目必须顺序瀑布。</span>

<span style="color:#c62828;font-weight:700">🔴【瀑布】阶段清晰、文档和评审严格，适合需求稳定、合规强项目，变更代价大。🔴【原型】用快速模型澄清需求，分抛弃型与演化型；不可把低质量原型直接无控制变产品。</span>

<span style="color:#c62828;font-weight:700">🔴【增量/迭代】分批交付价值、逐步完善，便于反馈和降低早期风险。🔴【螺旋】每轮目标—风险分析—开发验证—计划，适合大型高风险项目，风险管理能力要求高。</span>

<span style="color:#c62828;font-weight:700">🔴【敏捷】个体互动、可工作软件、客户协作、响应变化更受重视，但不等于不要流程、文档、合同和计划。Scrum有产品待办、Sprint、增量，角色常含产品负责人、Scrum Master、开发团队。</span>

⚫【模型选择】题目要求“说明理由”时按需求稳定性、风险、规模、交付频率、合规文档和团队能力匹配，不能只背优缺点。

### 2 可行性与需求工程

<span style="color:#c62828;font-weight:700">🔴【可行性】技术、经济、操作/社会、法律、进度等；经济评价常比较成本与收益，沉没成本不应影响未来决策。</span>

<span style="color:#c62828;font-weight:700">🔴【需求层次】业务需求说明目标；用户需求说明用户任务；系统需求细化功能与非功能约束。功能需求描述做什么；非功能需求含性能、可靠性、安全、易用性、兼容性等可度量质量要求。</span>

<span style="color:#c62828;font-weight:700">🔴【需求过程】获取→分析协商→规格说明→验证→变更管理。SRS应正确、完整、一致、无歧义、可验证、可追踪、可修改。</span>

<span style="color:#1565c0;font-weight:700">🔵【需求获取】访谈适合深入但受表达影响；问卷适合多人；观察发现隐性流程；工作坊促进协商；原型澄清交互；文档/接口分析了解现状。</span>

<span style="color:#1565c0;font-weight:700">🔵【用例】明确参与者、目标、前置条件、主成功场景、备选/异常流、后置条件。`include`表示必用公共行为，`extend`表示在扩展点有条件附加，泛化表示继承。</span>

⚫【参与者】参与者是与系统交互的外部角色，不一定是具体人；数据库若属于系统内部通常不是参与者。用例写用户目标，不写按钮级操作堆砌。

<span style="color:#c62828;font-weight:700">🔴【需求验证】评审正确性、完整性、一致性、可行性和可测试性；建立需求—设计—代码—测试双向追踪矩阵，变更需做影响分析和基线控制。</span>

### 3 结构化分析与设计

<span style="color:#c62828;font-weight:700">🔴【数据流图DFD】元素：外部实体、加工、数据流、数据存储。上下文图只画整个系统一个加工和外部交互；逐层分解要保持父子图输入/输出平衡。</span>

⚫【DFD规则】数据不能无加工地从外部实体直达外部实体、存储直达存储、外部实体直达存储；加工至少有输入和输出，避免黑洞、奇迹和灰洞。

<span style="color:#c62828;font-weight:700">🔴【数据字典】定义数据流、数据项、数据结构、数据存储和加工逻辑，是DFD的文字补充。加工逻辑可用结构化语言、判定表、判定树。</span>

<span style="color:#c62828;font-weight:700">🔴【模块化】信息隐藏和抽象控制复杂性；高内聚、低耦合。内聚由低到高常为偶然、逻辑、时间、过程、通信、顺序、功能；耦合由强到弱常为内容、公共、控制、标记、数据（教材分类略有差异）。</span>

<span style="color:#c62828;font-weight:700">🔴【设计原则】单一职责、开闭、里氏替换、接口隔离、依赖倒置；SOLID用于面向对象设计，不能机械套用导致过度设计。</span>

<span style="color:#1565c0;font-weight:700">🔵【概要设计题】划分子系统/模块→定义职责与接口→确定数据和控制关系→选择体系结构→检查内聚耦合、异常和质量属性→形成模块图、接口和部署决策。</span>

<span style="color:#c62828;font-weight:700">🔴【体系结构】分层利于职责隔离；客户机/服务器适合分布服务；MVC分模型、视图、控制；仓库以共享数据中心集成；管道过滤器适合变换流；事件驱动解耦生产者消费者但追踪和一致性更复杂。</span>

### 4 面向对象与UML

<span style="color:#c62828;font-weight:700">🔴【面向对象】对象有状态、行为和标识；类是对象抽象。封装隐藏实现，继承复用和表达“is-a”，多态使同一接口有不同实现。</span>

<span style="color:#c62828;font-weight:700">🔴【类关系】依赖是临时使用；关联是稳定结构关系；聚合是弱整体—部分，部分可独立；组合是强整体—部分，生命周期依赖；泛化是继承；实现是类实现接口。</span>

⚫【关系强度】依赖<关联<聚合<组合是常用理解，但实际建模以生命周期、所有权和导航需求为准，不能只看代码中“是否有成员变量”。

<span style="color:#c62828;font-weight:700">🔴【UML图】用例图描述功能与参与者；类图描述静态结构；顺序图按时间展示对象消息；活动图描述流程与并发；状态图描述对象状态转换；部署图描述软件构件到硬件结点。</span>

<span style="color:#1565c0;font-weight:700">🔵【顺序图】先识别参与者、边界/控制/实体对象；消息自上而下，激活条反映执行；用alt/opt/loop表达分支、可选和循环；返回消息可省略但关键数据要标。</span>

<span style="color:#1565c0;font-weight:700">🔵【状态图】状态必须有明确含义；迁移标为`事件[守卫]/动作`；检查初态、终态、每个事件的条件和不可达/死循环状态。</span>

### 5 软件设计细化

<span style="color:#c62828;font-weight:700">🔴【接口设计】定义输入输出、数据类型、前后置条件、错误码、幂等性、超时和版本兼容。契约清晰比暴露内部结构重要。</span>

<span style="color:#c62828;font-weight:700">🔴【数据库/数据设计】从概念数据模型到逻辑模式，再考虑索引、事务、一致性和归档；避免模块各自复制权威数据源。</span>

<span style="color:#c62828;font-weight:700">🔴【用户界面】一致性、及时反馈、容错与可恢复、减少记忆负担、可访问性；关键操作确认但不能处处弹窗。</span>

<span style="color:#c62828;font-weight:700">🔴【设计模式】创建型解决对象创建（工厂、单例）；结构型组合类对象（适配器、装饰器）；行为型职责协作（观察者、策略、状态）。答题要写意图、结构、适用场景和代价。</span>

⚫【模式误区】单例不是全局变量美化，可能造成隐藏依赖和测试困难；观察者解耦发布订阅双方，但要处理通知顺序、异常和取消订阅。

<span style="color:#1565c0;font-weight:700">🔵【详细设计表达】流程图直观；N-S图结构化；PAD支持自顶向下；伪代码便于评审。任何表示都应只用顺序、选择、循环等可验证控制结构。</span>

### 6 软件测试（必考）

<span style="color:#c62828;font-weight:700">🔴【原则】测试目标是发现缺陷，不是证明程序绝对无错；尽早测试、追溯需求、重视缺陷聚集、避免杀虫剂悖论；开发者可单测，独立团队更适合系统/验收测试。</span>

<span style="color:#c62828;font-weight:700">🔴【层次】单元测试模块内部；集成测试接口协作；系统测试完整系统与非功能需求；验收测试由用户/客户确认业务满足。回归测试在修改后检查原功能未被破坏。</span>

<span style="color:#c62828;font-weight:700">🔴【静态/动态】评审、走查、静态分析不执行程序；动态测试执行程序。黑盒依据规格，白盒依据内部结构，两者互补。</span>

<span style="color:#1565c0;font-weight:700">🔵【等价类】把输入域划为有效/无效等价类；每个有效类至少覆盖一次，每个无效类通常单独设计用例，避免一个用例含多个无效输入而无法定位。</span>

<span style="color:#1565c0;font-weight:700">🔵【边界值】对区间[a,b]常选a、b及相邻a−1、a+1、b−1、b+1（结合类型和鲁棒/一般边界策略）；边界比随机中间值更易暴露错误。</span>

<span style="color:#1565c0;font-weight:700">🔵【判定表】列条件桩与动作桩→列条件组合规则→填对应动作→合并无关条件；适合多条件组合，2^n只是理论组合数，可用约束消除不可能规则。</span>

<span style="color:#c62828;font-weight:700">🔴【白盒覆盖强度】语句覆盖<判定/分支覆盖；条件覆盖与判定覆盖互不包含；判定—条件覆盖二者都满足；条件组合覆盖更强；路径覆盖受循环导致路径无限，通常限界。</span>

<span style="color:#1565c0;font-weight:700">🔵【McCabe环形复杂度】连通控制流图`V(G)=E−N+2`，也可为判定结点数+1，或区域数；它给基本独立路径条数。复合条件按图中实际判定结点口径。</span>

<span style="color:#1565c0;font-weight:700">🔵【白盒题步骤】画控制流图并编号→数E/N或判定点算V(G)→列V(G)条基本独立路径→为每条路径选输入→核对覆盖目标。不可只列路径不写输入和期望结果。</span>

<span style="color:#c62828;font-weight:700">🔴【集成策略】自顶向下需桩模块，早验证主控逻辑；自底向上需驱动模块，早验证基础服务；大爆炸集成定位故障困难；三明治混合两者。</span>

### 7 质量、维护与配置管理

<span style="color:#c62828;font-weight:700">🔴【质量属性】功能适合性、性能效率、兼容性、易用性、可靠性、安全性、可维护性、可移植性。质量需求应可量化，例如响应时间百分位、可用性和恢复目标。</span>

<span style="color:#c62828;font-weight:700">🔴【维护类型】纠错性修缺陷；适应性适应环境变化；完善性增加/改进功能性能；预防性重构以降低未来风险。维护成本通常占生命周期大头。</span>

<span style="color:#c62828;font-weight:700">🔴【可维护性】可分析、可修改、可测试；受模块化、文档、复杂度、测试、命名和架构影响。重构不改变外部可观察行为，需测试保护。</span>

<span style="color:#c62828;font-weight:700">🔴【配置管理】识别配置项、版本控制、变更控制、状态记录、配置审计、构建发布。基线是正式评审批准、只能经变更流程修改的版本。</span>

<span style="color:#c62828;font-weight:700">🔴【版本】分支支持并行，合并集成变化；标签标记发布点；语义化版本常为主版本.次版本.修订号，兼容规则以项目约定。</span>

<span style="color:#c62828;font-weight:700">🔴【可靠性】可靠性是在给定时间和环境无故障运行概率；可用性约`MTBF/(MTBF+MTTR)`。提高可用性既可延长无故障时间，也可缩短修复时间。</span>

### 8 项目估算、进度与风险

<span style="color:#c62828;font-weight:700">🔴【规模】LOC直观但语言相关且早期难估；功能点按用户可见功能及复杂度估算，更早期、相对语言无关。估算应给假设和区间并持续校准。</span>

<span style="color:#c62828;font-weight:700">🔴【COCOMO基本式】工作量`E=a×(KLOC)^b`人月，开发时间`D=c×E^d`月，系数依项目类型/模型。人员数不等于任意缩短工期，遵循Brooks定律的沟通与上手开销。</span>

<span style="color:#1565c0;font-weight:700">🔵【PERT】三点估计期望工期`t=(a+4m+b)/6`，方差`((b−a)/6)^2`；路径期望为活动期望和，独立假设下路径方差为方差和。</span>

<span style="color:#c62828;font-weight:700">🔴【关键路径】网络图中总工期最长路径，关键活动总时差0；延误关键活动会延误项目（无储备时），缩短非关键活动不一定缩短总工期。</span>

<span style="color:#1565c0;font-weight:700">🔵【CPM步骤】正推求最早开始ES/最早完成EF（汇合取max）→逆推最迟完成LF/最迟开始LS（分叉取min）→总时差`TF=LS−ES=LF−EF`→TF=0为关键活动→列关键路径和工期。</span>

<span style="color:#1565c0;font-weight:700">🔵【挣值管理】PV计划价值、EV挣值、AC实际成本；成本偏差CV=EV−AC，进度偏差SV=EV−PV；CPI=EV/AC，SPI=EV/PV。CV/SV正值有利，指数>1有利。</span>

<span style="color:#c62828;font-weight:700">🔴【风险】风险暴露≈发生概率×损失。流程为识别、分析排序、应对计划、监控。应对有规避、减轻、转移、接受；问题是已发生事件，不再只是风险。</span>

### 9 敏捷、DevOps与工程实践

<span style="color:#c62828;font-weight:700">🔴【Scrum】产品负责人管理产品待办和价值；Scrum Master促进过程、移除障碍；开发者交付增量。Sprint内包含计划、每日站会、评审和回顾；完成定义DoD统一质量门槛。</span>

<span style="color:#c62828;font-weight:700">🔴【XP】小步发布、测试驱动、结对编程、持续集成、重构、简单设计、集体代码所有权。TDD循环红—绿—重构，测试先表达预期行为。</span>

<span style="color:#c62828;font-weight:700">🔴【CI/CD】持续集成频繁合并并自动构建测试；持续交付保证随时可发布，持续部署自动发布到生产。部署流水线仍需安全、审批、回滚和可观测性。</span>

<span style="color:#c62828;font-weight:700">🔴【评审】代码评审发现缺陷、传播知识和维持标准；重点看正确性、边界、安全、可测试性与可维护性，不做人身评价。</span>

⚫【敏捷误区】变更友好不等于不控制范围；短迭代不等于缺少架构；自动化测试多不等于可以省略探索性和验收测试。

<span style="color:#1565c0;font-weight:700">🔵【需求/建模】系统边界→参与者/实体→主流程→异常与约束→图形→一致性检查。图必须有文字解释，文字需能在图中追踪。</span>

## 信息安全

> > 只用于考前速记：🔴核心必背；🔵公式、流程、辨析与案例踩分词；⚫适用条件、绝对化陷阱和版本差异。涉及法律法规、标准或算法参数时，以题干指定版本和任课教师教材为准。

### 一、信息安全目标与基本概念

- 🔴 CIA 三要素：机密性 Confidentiality—防未授权泄露；完整性 Integrity—防未授权篡改/破坏；可用性 Availability—授权主体需要时可获得服务。
- 🔴 扩展目标：真实性、可控性、可审计/可追溯性、不可否认性；不同教材分类层次可能不同。
- 🔴 资产有价值；威胁是可能造成损害的因素；脆弱性是可被利用的弱点；攻击是威胁主体利用脆弱性的行为；风险综合可能性与影响。
- 🔵 常用风险表达：`风险 ≈ 发生可能性 × 影响程度`；风险评估还要识别资产、威胁、脆弱性、现有控制和剩余风险。
- 🔴 风险处置：规避、降低/缓解、转移、接受；任何处置都应明确责任人、期限和剩余风险。
- 🔴 最小权限：主体只获得完成任务所需最少权限；纵深防御：人员、终端、网络、应用、数据和运营部署相互补充控制。
- ⚫ “有漏洞”不等于已经被攻击；安全不是一次性产品，而是持续的治理、技术和运营过程。

### 二、安全管理与控制类型

- 🔴 安全控制可分管理、技术、物理控制；按作用可分预防、检测、响应、恢复和威慑/纠正控制。
- 🔴 安全策略规定目标和原则；标准给出强制要求；流程规定执行步骤；指南提供建议。
- 🔵 安全治理闭环：识别资产/责任 → 评估风险 → 制定控制 → 实施运行 → 监测审计 → 响应改进。
- 🔴 职责分离避免关键业务由一人独立完成；双人控制、审批复核和岗位轮换可降低舞弊风险。
- 🔴 最小化攻击面：关闭无用端口/服务、删除默认账号、及时补丁、基线配置、应用白名单和权限收敛。
- ⚫ 合规只是安全底线，不代表没有风险；购买安全设备但不配置、不监控、不维护，不能形成有效控制。

### 三、恶意代码

- 🔴 病毒通常依附宿主并具有复制/传染能力；蠕虫可独立运行并利用网络或漏洞自行传播；木马靠伪装诱导执行，定义重点不是自我复制。
- 🔴 勒索软件通过加密、锁定或窃取数据实施勒索；可借木马、漏洞、钓鱼或蠕虫能力传播，不等于所有勒索软件都是蠕虫。
- 🔴 后门绕过正常控制提供隐蔽访问；逻辑炸弹在特定条件触发；Rootkit 隐藏恶意活动并维持高权限；僵尸网络由受控设备组成。
- 🔴 宏病毒利用 Office 等应用宏机制传播；现代含宏格式常见 `.docm/.xlsm`，旧格式也可能含宏。
- 🔵 恶意代码处置：隔离主机 → 保留证据/判断范围 → 阻断传播与控制通道 → 清除或重装 → 修补入口 → 恢复验证 → 持续监测复盘。
- ⚫ 杀毒软件可能误报、漏报且依赖更新；“装了杀毒就绝对安全”错误，未经验证直接解密/交赎金也不能保证恢复。

### 四、社会工程与常见攻击入口

- 🔴 网络钓鱼冒充可信主体诱骗凭据、付款或执行恶意程序；鱼叉式钓鱼针对特定个人/组织；捕鲸面向高层或关键人员。
- 🔴 社会工程常利用权威、紧迫、恐惧、好奇、贪利和从众心理；形式包括钓鱼、冒充客服、尾随、诱饵、垃圾搜寻和肩窥。
- 🔵 可疑消息核验：停下操作 → 用独立可信渠道联系对方 → 核验域名/账号/业务背景 → 不直接打开附件或提供口令/验证码 → 上报。
- 🔴 常见入口：弱口令、凭据复用、未修补漏洞、恶意附件/链接、盗版软件、移动介质、暴露远程服务和供应链。
- 🔵 防护踩分词：安全意识、MFA、邮件过滤、最小权限、补丁、应用控制、宏限制、备份、日志监测。
- ⚫ 来自熟人账号的消息也可能由账号被盗产生；HTTPS 只表示传输加密和证书验证，不保证网站业务内容善意。

### 五、身份认证、授权与审计

- 🔴 Authentication 认证回答“你是谁”；Authorization 授权回答“你能做什么”；Accounting/Auditing 审计记录“你做过什么”。
- 🔴 认证因素：所知（口令/PIN）、所有（令牌/手机/密钥）、所是（指纹/人脸）；多因素应来自不同因素类别。
- 🔴 口令应长、唯一、不复用，服务端应使用带盐的专用口令哈希算法存储；盐不必保密，作用是抵抗预计算和彩虹表。
- 🔴 DAC 自主访问控制由资源所有者授权；MAC 强制访问控制依据安全标签；RBAC 按角色授予权限；ABAC 按主体、资源、动作和环境属性决策。
- 🔵 账号生命周期：申请 → 身份核验 → 审批授权 → 发放 → 定期复核 → 变更 → 停用/注销 → 审计留痕。
- 🔵 权限设计踩分词：默认拒绝、最小权限、职责分离、按需/按时授权、定期复核、离职立即回收。
- ⚫ 两个口令仍属于同一因素，不构成真正 MFA；生物特征也可能被仿冒且泄露后难以更换。

### 六、密码学基础

- 🔴 明文经加密算法和密钥变为密文，密文经解密恢复；现代密码系统的安全性应主要依赖密钥而非算法保密。
- 🔴 对称密码使用同一或可相互推导的共享密钥，速度快，适合大量数据，难点是密钥安全分发；典型算法 AES。
- 🔴 非对称密码使用公钥/私钥对，便于密钥协商和数字签名但开销较大；典型体系 RSA、ECC。
- 🔴 混合加密通常用公钥机制协商/封装会话密钥，再用对称算法加密大量业务数据。
- 🔴 哈希把任意长度输入映射成定长摘要，目标包括单向性、抗原像、抗第二原像和抗碰撞；哈希不是可逆加密。
- 🔵 完整性鉴别：共享密钥环境常用 MAC/HMAC；公开验证、来源认证和不可否认需求常用数字签名。
- ⚫ Base64 是编码不是加密；MD5、SHA-1 已不适合抗碰撞安全用途；RSA 不宜直接加密大文件。

### 七、数字签名、证书与 PKI

- 🔴 数字签名：发送者用签名私钥对消息摘要相关数据生成签名，接收者用相应公钥验证。
- 🔴 签名主要提供来源认证、完整性和一定条件下的不可否认性，不自动提供机密性。
- 🔵 保密方向：通常用接收者公钥加密/封装，接收者私钥解密；签名方向：签名者私钥生成，签名者公钥验证。
- 🔴 数字证书把主体身份与公钥等信息绑定，由 CA 签发；PKI 还包含注册、证书库、策略以及撤销/状态服务。
- 🔵 证书验证：验证签名链 → 有效期 → 主体/域名 → 密钥用途 → 撤销状态 → 信任锚。
- 🔴 HTTPS/TLS 通过证书认证服务器，并协商会话密钥保护传输机密性和完整性；具体握手随 TLS 版本和密码套件变化。
- ⚫ 证书有效不代表网站内容可信；签名私钥泄露会破坏身份可信度，需及时吊销并更换。

### 八、网络边界与安全设备

- 🔴 防火墙依据策略控制区域/主机之间通信，可进行包过滤、状态检测、代理和应用识别；默认拒绝并按需放行是常见安全思路。
- 🔴 IDS 侧重检测告警；IPS 通常串联并可实时阻断；误报是把正常行为判成攻击，漏报是未发现真实攻击。
- 🔴 VPN 在不可信网络上建立受保护隧道，常分远程接入和站点到站点；VPN 不会自动消除终端、账号和应用漏洞。
- 🔴 DMZ 用于放置对外服务并与内网分区；网络分段和零信任思路限制横向移动，强调持续验证和最小权限。
- 🔵 防火墙规则检查：源地址 → 目的地址 → 协议/端口 → 方向/接口 → 动作 → 优先级 → 日志 → 业务验证。
- ⚫ 路由器可集成 ACL/防火墙功能，下一代防火墙可集成 IPS；设备名称不如实际功能、部署位置和策略重要。

### 九、常见网络攻击

- 🔴 DoS 消耗带宽、连接或计算资源使服务不可用；DDoS 使用多个分布式来源。
- 🔴 中间人攻击介入通信进行窃听或篡改；正确验证证书和使用 TLS 可降低风险。
- 🔴 嗅探截获网络流量；欺骗可伪造 IP、ARP、DNS 等身份/映射；会话劫持夺取已建立会话。
- 🔴 端口扫描用于发现开放服务，本身可用于授权运维或攻击侦察，性质取决于授权与目的。
- 🔵 网络攻击案例：资产/业务 → 攻击入口 → 利用的弱点 → CIA 影响 → 预防 → 检测 → 响应恢复。
- ⚫ 加密流量提高机密性也可能降低中间设备可见性；防火墙无法包治合法账号滥用、绕行路径和允许流量中的所有攻击。

### 十、Web 与应用安全

- 🔴 SQL 注入源于把不可信输入拼接进 SQL；核心防护是参数化查询/预编译，并配合输入校验、最小数据库权限和错误信息控制。
- 🔴 XSS 使恶意脚本在受害者浏览器执行，分存储型、反射型和 DOM 型；核心防护是按输出上下文编码，并配合 CSP、安全 Cookie 和输入处理。
- 🔴 CSRF 利用用户已登录状态诱使浏览器发起非预期请求；防护包括 CSRF Token、SameSite Cookie、重新认证和来源校验。
- 🔴 越权包括水平越权访问同级他人资源和垂直越权获得更高权限；每次请求都必须在服务端检查授权。
- 🔴 缓冲区溢出、命令注入、路径遍历、不安全反序列化和文件上传也是常见漏洞类型。
- 🔵 安全开发生命周期：需求威胁建模 → 安全设计 → 规范编码 → 代码审查/测试 → 安全部署 → 漏洞响应维护。
- ⚫ 只做前端校验不能防攻击；转义方式依 HTML、属性、JavaScript、URL 等输出上下文而不同；过滤关键词不能替代参数化。

### 十一、主机、数据与备份安全

- 🔴 主机加固：补丁更新、关闭无用服务、最小权限、强认证、端点防护、磁盘加密、安全基线、日志和配置审计。
- 🔴 数据全生命周期：收集、存储、使用/加工、传输、提供、公开、归档和删除，各阶段都需分类分级和访问控制。
- 🔴 备份 3-2-1：至少 3 份数据、2 种介质、1 份异地；现代勒索场景还应保留离线或不可变副本并测试恢复。
- 🔴 RPO 恢复点目标表示可容忍丢失多少时间的数据；RTO 恢复时间目标表示中断后多快恢复服务。
- 🔵 备份方案：识别关键数据 → 定 RPO/RTO → 选全量/增量/差异备份 → 加密与异地/离线保存 → 定期恢复演练。
- ⚫ RAID 提高可用性但不能替代备份；有备份文件不等于可恢复，必须验证完整性、密钥和恢复流程。

### 十二、安全事件响应与取证

- 🔴 事件响应常见阶段：准备 → 识别/分析 → 遏制 → 根除 → 恢复 → 复盘改进。
- 🔵 初期处置：确认告警 → 分级上报 → 隔离受影响资产 → 保留易失证据 → 阻断扩散 → 维护业务连续性。
- 🔴 数字取证强调合法授权、证据完整、过程可重复和保管链；采集时记录人员、时间、地点、工具、哈希和交接。
- 🔵 日志分析关联账号、源/目的地址、时间、动作和结果；应统一时钟并保护日志免遭篡改。
- 🔴 业务连续性关注关键业务持续运行；灾难恢复侧重 IT 系统和数据恢复，二者相关但不相同。
- ⚫ 发现入侵后立即关机可能丢失内存等易失证据；是否断电、隔离或在线取证应按预案、风险和专业指挥决定。

### 十三、个人信息、法律与职业伦理

- 🔴 个人信息是与已识别或可识别自然人有关的各种信息，匿名化处理后的信息不再属于个人信息；去标识化仍可能重新识别。
- 🔴 处理个人信息应遵循合法、正当、必要、诚信、目的明确、最小影响、公开透明、质量和安全保障等要求。
- 🔴 敏感个人信息处理通常要求特定目的、充分必要和严格保护，并依法履行告知、单独同意等义务。
- 🔴 高频法律框架：《网络安全法》《数据安全法》《个人信息保护法》《密码法》《著作权法》及计算机软件保护相关规定。
- 🔴 等级保护强调定级、备案、建设整改、等级测评和监督检查；具体对象、等级与要求按现行标准和题干版本作答。
- 🔵 法律案例答题：主体 → 行为/数据 → 权限或合法基础 → 安全义务 → 影响后果 → 整改措施；不要只写“违法/合法”。
- ⚫ 同意不是个人信息处理的唯一合法基础；法律和标准会修订，具体日期、条文、罚则和等级要求必须核对题干版本。

### 十四、综合案例题统一框架

- 🔵 第一步定资产：数据、账号、设备、服务、声誉和人员安全。
- 🔵 第二步定问题：威胁利用何种脆弱性，经什么入口发生，影响 CIA 哪些目标。
- 🔵 第三步分层控制：人员培训 + 身份权限 + 终端补丁 + 网络分区 + 应用安全 + 数据加密备份 + 日志运营。
- 🔵 第四步写闭环：预防、检测、响应、恢复，并给责任人、证据和验证方法。
- 🔵 高分措辞：最小权限、默认拒绝、纵深防御、多因素、参数化、分区隔离、离线备份、持续监测、复盘整改。
- ⚫ 只罗列产品名通常得分低；措施必须对应已识别的威胁和薄弱点，同时说明局限与剩余风险。
