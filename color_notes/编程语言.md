# 编程语言（C + Python 3）· 三色回忆卡

> 先答后核对。题目未说明时，Python 按 Python 3；C 的整数宽度、字符集和部分行为以题设编译环境为准。

## 一、程序、类型与控制结构

### 回忆卡 1｜算法与程序错误

先答：算法的基本性质和三种控制结构是什么？语法、运行时、逻辑错误怎样区分？

- 🔴 算法强调有穷、确定、可行及输入输出；结构是顺序、选择、循环。语法错误违背规则，运行时错误执行中发生，逻辑错误能运行但结果错误。
- 🔵 开发流程：分析 → 算法 → 编码 → 运行 → 测试调试 → 维护。
- ⚫ 编译与解释不是所有实现的绝对二分，Python 通常还涉及字节码和虚拟机。

### 回忆卡 2｜类型、转换与运算

先答：整数、浮点、字符、布尔应怎样选择？隐式转换可能带来什么问题？

- 🔴 计数用整数、近似连续量用浮点、单字符用字符、真假条件用布尔；窄化转换可能溢出或丢精度。
- 🔵 不确定优先级时加括号；短路逻辑先判断左操作数是否已能确定结果。
- ⚫ 浮点是有限近似，`0.1+0.2` 未必精确等于 `0.3`。

### 回忆卡 3｜选择与循环跟踪

先答：怎样闭卷跟踪一段含分支、循环、break 和 continue 的程序？

- 🔴 `break` 结束最近循环，`continue` 跳过本轮余下语句，`return` 结束函数。
- 🔵 列“轮次—条件—关键变量—输出”表，依次核对初值、更新、0次/1次/末次和越界状态。
- ⚫ `while` 可能一次不执行；嵌套循环次数不能在边界相关时机械相乘。

## 二、函数、算法与测试

### 回忆卡 1｜函数、作用域与递归

先答：形参、实参、作用域、生命周期和递归基例分别是什么？

- 🔴 函数封装任务；形参在定义中，实参在调用中；递归必须有基例且每次逼近基例。
- 🔵 递归跟踪先写调用链，再按返回顺序向上代值；检查每条路径是否返回。
- ⚫ C 是值传递；Python 是对象共享传参，重新绑定形参不等于改调用者变量。

### 回忆卡 2｜查找与基础排序

先答：顺序、二分查找和冒泡、选择、插入排序各依靠什么条件？

- 🔴 顺序查找通常 `O(n)`；二分要求有序且为 `O(log n)`；三种基础排序通常为 `O(n²)`。
- 🔵 二分步骤：定 low/high → 求 mid → 比较 → 缩区间 → 找到或区间空。
- ⚫ 未排序数据不能直接二分；排序题要先确认升降序、稳定性和边界。

### 回忆卡 3｜测试与调试

先答：测试和调试有何区别？改错题怎样写完整？

- 🔴 测试用于发现缺陷，调试用于定位并修复；测试不能证明程序绝对无错。
- 🔵 改错写“位置 → 原因/后果 → 修改 → 用正常和边界数据验证”。
- ⚫ 只测正常数据会漏掉空输入、零、负数、重复值和上下界。

## 三、C 基础与输入输出

### 回忆卡 1｜程序结构与类型

先答：C 程序、语句、预处理和常见基本类型各有什么形式？

- 🔴 C 程序由函数组成，`main` 常为入口；预处理如 `#include` 在编译前处理，语句多以分号结束；常见类型有 `char/int/float/double`。
- 🔵 标识符不能数字开头、不能用关键字且区分大小写。
- ⚫ 各整数类型确切字节数依实现，不能脱离题设绝对化。

### 回忆卡 2｜C 运算与控制

先答：C 的整数除法、求余、逻辑运算和自增题怎样避免心算失误？

- 🔴 整数 `/` 截断小数，`%` 求余；逻辑与或非为 `&& || !`，比较相等用 `==`。
- 🔵 按括号和优先级逐步写中间值；复杂自增表达式先检查是否存在未定义或未指定行为。
- ⚫ `=` 是赋值；不要依赖不同编译器可能给出不同结果的表达式。

### 回忆卡 3｜格式化输入输出

先答：`printf`、`scanf` 中 `%d %f %lf %c %s` 分别怎样使用？

- 🔴 `%d` 整数，输出浮点常用 `%f`；`scanf` 中 `%f` 读 `float`、`%lf` 读 `double`；`%c` 字符、`%s` 字符串。
- 🔵 普通变量输入传地址，如 `scanf("%d", &x)`；输出题原样核对空格、换行和宽度精度。
- ⚫ `%c` 前残留空白是高频陷阱；格式与实参类型不匹配会导致错误行为。

## 四、C 数组、指针与资源

### 回忆卡 1｜数组、指针与字符串

先答：数组和指针为何相关却不相同？C 字符串怎样结束？

- 🔴 指针保存地址，`&` 取址、`*` 间接访问；数组名多数表达式中转成首元素指针，但数组不是指针；C 字符串以 `\0` 结束。
- 🔵 `a[i]` 等价于 `*(a+i)`；容量必须给终止符留空间。
- ⚫ 下标越界和无效指针访问属于未定义行为；`strlen` 不含终止符。

### 回忆卡 2｜函数修改外部数据

先答：C 函数怎样交换调用者的两个变量？

- 🔴 形参接收指针，调用时传两个变量地址，函数内用解引用修改其存储单元。
- 🔵 骨架：`swap(int *a,int *b)` → 临时量保存 `*a` → 交换 `*a,*b`。
- ⚫ 只交换值形参不会修改调用者变量；指针使用前必须指向有效对象。

### 回忆卡 3｜结构体、动态内存与文件

先答：结构体成员、动态内存和文本文件的关键操作是什么？

- 🔴 对象用`.`、结构体指针用`->`；动态内存用 `malloc/calloc/realloc/free`；文件流程是打开、检查、读写、关闭。
- 🔵 资源题检查申请/打开是否成功、边界、释放/关闭及错误路径。
- ⚫ 避免内存泄漏、重复释放、悬空指针；文本和二进制模式不能机械混用。

## 五、Python 对象、容器与控制

### 回忆卡 1｜对象、赋值与比较

先答：Python 变量、可变对象、`==` 与 `is` 应怎样理解？

- 🔴 变量绑定对象；列表、字典、集合通常可变，数值、字符串、元组通常不可变；`==` 比较值，`is` 比较身份。
- 🔵 类型转换常用 `int/float/str/list`，输入 `input()` 返回字符串。
- ⚫ 不用 `is` 代替普通数值或字符串相等判断。

### 回忆卡 2｜容器与切片

先答：列表、元组、字典、集合分别适合什么任务？切片 `a[start:stop:step]` 是否包含 stop？

- 🔴 列表有序可变；元组有序通常不可变；字典按键映射；集合去重并支持集合运算；切片不含 stop。
- 🔵 遍历字典明确取键、值或键值对；复制嵌套容器时区分浅拷贝与深拷贝。
- ⚫ `append(x)` 添加一个对象，`extend(iterable)` 逐项扩展；负步长切片要重判边界。

### 回忆卡 3｜Python 分支与循环

先答：Python 代码块、整除、逻辑运算和循环有什么高频边界？

- 🔴 代码块靠缩进；整除用 `//`，逻辑用 `and/or/not`；`range` 右端不包含；`for/while` 可带 `else`。
- 🔵 跟踪时同时记录变量值、容器是否原地改变和循环输出。
- ⚫ `//` 是向下取整，负数结果不等同于C整数除法的向零截断。

## 六、Python 函数、异常与综合输出

### 回忆卡 1｜函数参数与作用域

先答：位置、关键字、默认、可变参数怎样传递？默认可变对象为什么危险？

- 🔴 `*args` 收集额外位置参数，`**kwargs` 收集额外关键字参数；默认参数在函数定义时求值。
- 🔵 需要每次新容器时用 `None` 作默认值，再在函数内创建。
- ⚫ 可变默认对象可能被多次调用共享；局部赋值默认创建局部绑定。

### 回忆卡 2｜异常与文件

先答：`try/except/else/finally` 各在何时执行？文件为何推荐 `with`？

- 🔴 `except` 处理异常，`else` 在无异常时执行，`finally` 通常总执行；`with` 能在离开代码块时关闭文件。
- 🔵 文件题写路径/模式/编码 → 打开 → 读写 → 异常处理 → 自动关闭。
- ⚫ 不要用裸 `except` 吞掉所有错误；读写模式和字符编码必须匹配题意。

### 回忆卡 3｜C 与 Python 程序题

先答：拿到阅读、填空、改错或编程题，第一步分别是什么？

- 🔴 阅读先跟踪变量；填空先判断所需类型/条件/更新；改错先定位后果；编程先写输入输出和算法。
- 🔵 统一检查：初始化、边界、终止、返回值、输出格式、资源释放，再用典型与极端数据手测。
- ⚫ 不要混写 C 的分号、大括号、`&&` 与 Python 的冒号、缩进、`and`；先确认指定语言。
