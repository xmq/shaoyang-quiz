# 编程语言·期末复习三色笔记

> 只用于考前速记：🔴核心必背；🔵语法、步骤、代码阅读与踩分词；⚫语言差异、边界和常见陷阱。课程主线统一为 C 语言与 Python 3：C 侧重底层数据表示、指针和文件，Python 侧重容器、函数与工程化基础。

## 一、程序、算法与翻译方式

- 🔴 程序是为解决问题而编写的指令序列；算法应具有有穷性、确定性、可行性、输入和输出。
- 🔴 算法基本结构：顺序、选择、循环；常用表示有自然语言、流程图、伪代码和程序设计语言。
- 🔴 编译通常把整个源程序翻译为目标程序后执行；解释通常边翻译边执行。现代语言实现可能混合编译、字节码和虚拟机。
- 🔴 语法错误违反语言规则；运行时错误在执行中发生；逻辑错误能运行但结果不符合要求。
- 🔵 程序开发：问题分析 → 建模/设计算法 → 编码 → 编译/运行 → 测试调试 → 文档维护。
- 🔵 算法题踩分：写输入、输出、初始化、循环/选择条件、更新语句、终止条件和边界处理。
- ⚫ “编译型/解释型”不是所有实现的绝对二分；Python 常先编译成字节码再由虚拟机执行。

## 二、标识符、变量、常量与数据类型

- 🔴 标识符一般由字母、数字、下划线等构成，不能以数字开头，不能使用保留字；是否区分大小写由语言决定。
- 🔴 变量具有名称、类型/值、作用域和生命周期；常量在规定范围内不可修改。
- 🔴 基本类型通常包括整数、实数/浮点、字符、布尔；复合类型包括数组、记录/结构体、容器和对象等。
- 🔵 类型选择：计数用整数，连续近似量用浮点，单字符用字符，真假条件用布尔；金融精确小数要关注十进制类型或整数化处理。
- 🔴 隐式转换由语言自动完成，显式转换由程序员指定；窄化转换可能溢出或损失精度。
- ⚫ 浮点数是有限二进制近似，`0.1+0.2` 未必精确等于 `0.3`；比较浮点数常使用容差。

## 三、运算符、表达式与输入输出

- 🔴 运算符常分算术、关系、逻辑、赋值、位运算和条件运算；表达式由操作数和运算符组成并产生值。
- 🔴 高优先级通常先算，优先级相同时再看结合性；考试不确定时用括号明确顺序。
- 🔵 逻辑短路：`AND/&&` 左侧为假时可能不算右侧；`OR/||` 左侧为真时可能不算右侧。
- 🔵 整除与余数：C 整数 `/` 截断小数，`%` 求余；Python `//` 向下取整除、`%` 满足整除关系。
- 🔴 输入要做类型转换和合法性检查；输出题要按实际执行顺序记录变量变化、换行和格式控制。
- ⚫ `=` 在 C/Python 中是赋值，不是比较；比较相等都用 `==`。Python 还能用 `is` 比较对象身份，但不能用它代替一般数值或字符串相等判断。

## 四、选择与循环结构

- 🔴 单分支、双分支、多分支根据互斥条件选择路径；多分支要注意条件覆盖、顺序和默认分支。
- 🔴 计数循环适合次数已知；条件循环适合次数未知；循环必须有初始化、条件判断、循环体和状态更新。
- 🔵 程序阅读表：每轮依次记录“循环变量—条件结果—关键变量—输出”，不要凭感觉心算。
- 🔵 循环设计：初始化 → 判断 → 处理 → 更新；验证 0 次、1 次、最后一次和越界后状态。
- 🔴 `break` 结束当前最近循环；`continue` 跳过本轮余下语句进入下一轮；`return` 结束函数并可返回值。
- ⚫ `while` 可能一次不执行；传统 `do...while` 至少执行一次；嵌套循环总次数通常是各层次数乘积，但内层边界可能依赖外层。

## 五、函数、参数、作用域与递归

- 🔴 函数把特定任务封装为可复用单元，具有名称、形参、函数体和返回值；过程可视为不强调返回值的子程序。
- 🔴 形参是定义时参数，实参是调用时传入的值/对象；参数传递语义要按具体语言判断。
- 🔴 局部变量只在所属块/函数可见；全局变量作用域更大但增加耦合，应谨慎使用。
- 🔵 函数题写法：明确输入参数 → 处理逻辑 → 返回结果；调用方接收返回值；检查所有执行路径是否返回。
- 🔴 递归必须有基本情况和递归推进，且每次调用应更接近基本情况；调用栈保存各层局部状态和返回位置。
- 🔵 递归跟踪：先写调用链到基本情况，再按返回顺序向上代值；区分“调用阶段”和“回归阶段”的输出。
- ⚫ C 默认值传递；要让函数修改调用者变量通常传地址。Python 是对象引用传递语义，可变对象可能原地改变，重新绑定形参不等于修改调用者变量。

## 六、数组、字符串与常用算法

- 🔴 C 数组和 Python 序列通常从 0 开始；C 数组元素类型固定，Python 列表可保存不同类型对象。
- 🔴 顺序查找逐个比较，时间复杂度通常 `O(n)`；二分查找要求数据有序，每次折半，时间复杂度 `O(log n)`。
- 🔵 二分查找：`low/high` 初始化 → 当区间有效时求 `mid` → 比较 → 保留左半或右半 → 找到或区间为空。
- 🔴 冒泡排序重复比较相邻元素并交换逆序对；选择排序每轮选出最值放到目标位置；插入排序把当前元素插入前方有序区。
- 🔵 冒泡高频边界：第 `i` 轮后端已有序，内层上界应缩小；交换需要临时变量或安全的并行赋值。
- 🔴 字符串是字符序列；C 字符串以 `\0` 结束，字符数组容量要给终止符留位置。
- 🔵 字符串常考：长度、拼接、比较、查找、截取、大小写转换；注意函数返回新对象还是原地修改。
- ⚫ 二分查找不能直接用于无序数据；数组下标越界在 C 中可能导致未定义行为，Python 通常抛出 `IndexError`。

## 七、测试、调试与程序质量

- 🔴 测试是发现缺陷，调试是定位并修复缺陷；测试不能证明程序绝对无错。
- 🔵 黑盒边界值：最小值、略大于最小、正常值、略小于最大、最大值，以及刚好越界值。
- 🔵 调试步骤：稳定复现 → 缩小范围 → 观察变量/调用栈 → 提出原因 → 最小修改 → 回归测试。
- 🔴 良好程序强调正确性、可读性、健壮性、效率、可维护性和可移植性。
- 🔵 代码改错踩分：指出错误位置 → 说明错误类型/后果 → 给出正确代码 → 用边界数据验证。
- ⚫ “程序有输出”不代表正确；只测正常数据会漏掉空输入、零、负数、重复值和上下界问题。

## 八、C 语言：程序结构与数据类型

- 🔴 C 程序由函数组成，`main` 是常见入口；预处理指令如 `#include` 在编译前处理，语句多以分号结束。
- 🔴 C 区分大小写；合法标识符不能以数字开头；关键字不能作标识符。
- 🔴 常见类型：`char`、`short`、`int`、`long`、`float`、`double`；确切字节数与实现有关，只有相对范围受标准约束。
- 🔵 格式化输入输出：`printf` 输出，`scanf` 输入；常见格式 `%d` 整数、`%f` 浮点输出、`%lf` 双精度输入、`%c` 字符、`%s` 字符串。
- 🔵 `scanf` 普通变量通常传地址，如 `scanf("%d", &x)`；字符数组名用于 `%s` 时本身可退化为地址，通常不再加 `&`。
- ⚫ 输入字符前残留换行是高频陷阱；`scanf(" %c", &ch)` 前导空格可跳过空白，但是否应跳过取决于题意。

## 九、C 语言：指针、数组、字符串与结构体

- 🔴 指针保存地址；`&x` 取变量地址，`*p` 间接访问指针所指对象；使用前必须指向有效存储区域。
- 🔴 数组名在多数表达式中转换为首元素指针，但数组与指针不是完全相同类型；`a[i]` 等价于 `*(a+i)`。
- 🔵 交换调用者变量：函数形参接收指针 → 调用时传 `&变量` → 函数内用 `*指针` 修改。
- 🔴 常见字符串函数：`strlen` 长度、`strcpy/strncpy` 复制、`strcat` 连接、`strcmp` 比较；必须保证目标空间足够并关注终止符。
- 🔴 结构体用 `struct` 把不同类型字段组合；普通对象用 `.` 访问成员，结构体指针用 `->`。
- 🔴 动态内存常用 `malloc/calloc/realloc/free`；申请后检查是否成功，使用后释放并避免悬空指针、重复释放和越界。
- ⚫ `sizeof(数组)` 在数组定义所在作用域可得整个数组字节数，但数组传入函数后形参通常退化为指针；`strlen` 不包含 `\0`。

## 十、C 语言：函数、文件与预处理

- 🔴 函数调用前应有声明/原型；返回类型、参数类型和调用方式应一致。
- 🔴 存储类别与作用域要区分：局部 `static` 变量生命周期贯穿程序但作用域仍在函数/块内；`extern` 用于引用外部定义。
- 🔵 文件流程：`fopen` → 检查返回指针 → `fscanf/fprintf` 或 `fread/fwrite` 读写 → 检查状态 → `fclose`。
- 🔴 文本模式按字符解释，二进制模式按字节读写；文件打开模式 `r/w/a` 分别常用于读、覆盖写、追加，`+` 表示可读写。
- 🔴 宏是预处理文本替换；带参数宏应给参数和整体加括号，如 `#define SQR(x) ((x)*(x))`。
- ⚫ `w` 模式会截断已有文件；宏参数可能被计算多次，不能简单等同于函数。

## 十一、Python：语法、对象与容器

- 🔴 Python 用缩进表示代码块，区分大小写，变量名绑定对象且类型动态确定。
- 🔴 常见不可变对象：数字、字符串、元组；常见可变对象：列表、字典、集合。
- 🔵 输入转换：`input()` 返回字符串；数值输入常用 `int(input())`、`float(input())`。输出用 `print`，格式化优先掌握 f-string。
- 🔴 切片常写 `序列[start:stop:step]`，包含 start、不包含 stop；省略值使用默认边界，负下标从末尾计数。
- 🔴 列表有序可重复；元组有序通常不可变；集合无重复并支持集合运算；字典存储键值映射，键须可哈希。
- 🔵 高频方法：列表 `append/extend/insert/pop/sort`；字符串 `split/join/strip/find/replace`；字典 `get/keys/values/items`；集合 `add/remove/discard`。
- ⚫ `list.sort()` 原地排序并返回 `None`，`sorted()` 返回新列表；`append` 加一个对象，`extend` 逐项扩展；集合无固定位置下标。

## 十二、Python：控制、函数、异常与文件

- 🔵 分支：`if/elif/else`；循环：`for 变量 in 可迭代对象`、`while 条件`；`range(start,stop,step)` 不包含 stop。
- 🔴 函数用 `def` 定义，`return` 返回结果；未显式返回时结果为 `None`。
- 🔴 形参形式可含位置参数、关键字参数、默认参数、`*args` 和 `**kwargs`；默认参数在定义时求值，可变默认参数会跨调用共享。
- 🔴 LEGB 名称查找：Local 局部、Enclosing 嵌套、Global 全局、Built-in 内置；修改全局绑定需谨慎使用 `global`。
- 🔵 异常：`try` 放可能失败代码，`except` 处理特定异常，`else` 在无异常时执行，`finally` 通常用于清理资源。
- 🔵 文件推荐 `with open(path, mode, encoding="utf-8") as f:`，离开块自动关闭；文本读写常用 `read/readline/readlines/write`。
- 🔴 类用 `class` 定义，实例方法首参通常写 `self`；`__init__` 初始化实例状态；继承可复用并扩展父类行为。
- ⚫ `except:` 捕获过宽会掩盖错误；读文件模式不存在文件会失败，写模式会覆盖；列表复制要防共享嵌套对象。

## 十三、C 与 Python 高频对照

| 概念 | C | Python 3 |
|---|---|---|
| 🔴 类型系统 | 静态类型，变量声明类型 | 动态类型，名称绑定对象 |
| 🔵 相等/不等 | `==` / `!=` | `==` / `!=` |
| 🔵 逻辑与/或/非 | `&&` / `||` / `!` | `and` / `or` / `not` |
| 🔵 整除 | 整数 `/` 向零截断 | `//` 向下取整 |
| 🔵 代码块 | `{}`，语句常以 `;` 结尾 | 冒号加一致缩进 |
| 🔴 序列 | 固定类型数组、字符数组 | 列表、元组、字符串等对象 |
| 🔴 字符串 | `char` 数组，以 `\0` 结束 | 不可变 `str` 对象 |
| 🔴 参数影响 | 值传递；传指针可改外部对象 | 对象共享传参；可变对象可原地改变 |

- ⚫ 表格给出常见高校教学口径；C 的整数宽度、字符集和标准库支持随实现变化，Python 未注明时按 Python 3。

## 十四、期末程序题统一策略

- 🔵 阅读题：标变量初值 → 标运算优先级 → 按控制流逐句执行 → 列循环跟踪表 → 原样写输出格式。
- 🔵 填空题：先看上下文所需“类型、条件、更新、返回值”，再补语法；填后从前到后模拟一次。
- 🔵 改错题：语法、类型、边界、初始化、条件、更新、输入输出、资源释放逐项检查。
- 🔵 编程题：写输入输出 → 选数据结构 → 写核心算法 → 处理边界 → 模块化 → 用典型与极端数据手工验证。
- ⚫ 不要把 C 的分号、大括号、`&&` 与 Python 的冒号、缩进、`and` 混写；先确认题目指定语言。
