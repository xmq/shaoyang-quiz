# 编程语言：三色笔记

三色按考频和出题稳定性划分：红色优先背熟，蓝色用于补齐得分点，黑色用于防漏和边界判断。

## 🔴 高频考点

- 🔴 **变量与数据类型**：变量保存数据，类型决定取值范围和可用操作。 常见考法：判断整型、浮点、字符、字符串。 易错边界：字符串和字符不是同一概念。
- 🔴 **表达式与运算符**：算术、关系、逻辑运算构成条件和计算。 常见考法：判断优先级、结果类型。 易错边界：赋值 `=` 与相等判断 `==` 不同。
- 🔴 **顺序、选择、循环**：程序基本控制结构是顺序、分支和循环。 常见考法：读程序写结果，补全条件。 易错边界：循环要关注初值、条件、更新。
- 🔴 **函数**：函数把可复用逻辑封装成有输入和输出的模块。 常见考法：判断参数、返回值、调用过程。 易错边界：形参和实参不是同一个变量。
- 🔴 **数组/列表**：连续或序列结构用于保存多个同类或相关数据。 常见考法：下标访问、遍历、求和计数。 易错边界：多数语言下标从 0 开始。
- 🔴 **程序调试**：语法错误、运行错误、逻辑错误要分开判断。 常见考法：根据报错或结果异常找原因。 易错边界：能运行不代表逻辑正确。

## 🔵 中频考点

- 🔵 **指针或引用**：C 常考指针地址，Python 常考对象引用。 常见考法：判断传参影响和地址/值。 易错边界：不同语言机制不同，不要混套。
- 🔵 **文件操作**：打开、读写、关闭是文件处理基本流程。 常见考法：判断读写模式。 易错边界：写文件可能覆盖原内容。
- 🔵 **异常处理**：异常用于处理运行时非正常情况。 常见考法：判断 try/except 或错误处理作用。 易错边界：异常处理不能替代输入校验。
- 🔵 **面向对象**：类是模板，对象是实例，封装、继承、多态是基本特征。 常见考法：判断类、对象、方法。 易错边界：面向对象不是必须用于所有小程序。
- 🔵 **算法实现**：常见题要求把排序、查找、统计转换成代码。 常见考法：读伪代码或补程序。 易错边界：先明确边界条件。

## ⚫ 低频了解

- ⚫ **编译与解释**：编译先整体转换，解释边解释边执行。 常见考法：判断 C、Python 执行方式差异。 易错边界：现代语言实现可能混合多种机制。
- ⚫ **库与模块**：模块复用已有功能。 常见考法：判断 import/include 的作用。 易错边界：导入库不等于自动执行所有功能。
- ⚫ **编码规范**：命名、缩进、注释提高可读性。 常见考法：判断程序风格问题。 易错边界：注释不能修复错误逻辑。

## 考前规则与简答

### 必会规则

- **循环执行次数**：先确定初值，再检查进入条件，最后跟踪每轮更新。形如 `for(i=0;i<n;i++)` 通常执行 `n` 次，对应下标 `0` 到 `n-1`。
- **整数与实数运算**：C语言中两个整数相除得到整数商；要保留小数，至少一个操作数应转为浮点数。Python的 `/` 为真除法，`//` 为向下取整除法，不能混套语言规则。
- **函数传参**：形参是函数内部接收值的变量，实参是调用处给出的值或对象。修改形参是否影响调用者，取决于语言传参机制和对象是否可变。

### 常见简答题

- **程序出现错误应怎样定位？** 先区分语法错误、运行时异常和逻辑错误，再构造最小输入，观察关键变量与分支，逐步缩小范围；程序能运行只说明语法和部分运行条件通过，不代表结果正确。
- **函数有什么作用？** 函数封装相对独立的功能，减少重复代码，提高可读性、可测试性和复用性，并通过参数和返回值与其他模块协作。
