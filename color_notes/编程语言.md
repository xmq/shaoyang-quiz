# 编程语言基础（C + Python 3）· 三色回忆卡

> 先答后核对。只要求基本概念、简单语法、三种结构、函数、数组或容器和代码输出；Python按Python 3。

## 一、程序、算法与语言

### 回忆卡 1｜程序与算法

先答：程序和算法有何区别？算法有哪些基本性质？


- ⚫ 算法是解决问题的明确步骤，程序是算法用具体语言实现的指令；算法强调有穷、确定、可行、输入和输出。
- 🔵 简答先分别定义，再列性质和二者关系。
- 🔴 程序不等于算法，算法也不能无限执行而无结果。
### 回忆卡 2｜三种基本结构

先答：顺序、选择、循环结构分别怎样控制程序执行？


- 🔵 顺序依次执行；选择按条件选分支；循环在条件控制下重复语句。
- 🔴 看代码先画执行顺序，再标条件和循环返回位置。
- ⚫ 复杂程序也可由三种基本结构组合，不需要把函数当作第四种控制结构。
### 回忆卡 3｜编译、解释与错误

先答：编译和解释有何基本区别？三类程序错误怎样辨认？


- ⚫ 编译通常先整体翻译再执行，解释通常运行中逐句翻译；错误分语法、运行时和逻辑错误。
- 🔵 按“何时发现—能否运行—结果是否正确”区分错误。
- 🔴 编译和解释不是所有实现的绝对二分，测试也不能证明程序绝对无错。
## 二、变量、类型与运算

### 回忆卡 1｜标识符、变量与类型

先答：合法标识符要满足什么？常见基本类型有哪些？


- ⚫ 标识符不能数字开头、不能用关键字且区分大小写；常见类型有整数、浮点、字符/字符串和布尔。
- 🔵 选择类型看数据是计数、小数、文本还是真假。
- 🔴 浮点是有限近似；`name`与`Name`是不同标识符。
### 回忆卡 2｜算术、关系与逻辑运算

先答：C和Python的整除、相等比较、逻辑与或非分别怎样写？


- 🔵 C整数`/`截去小数，Python普通除法`/`、整除`//`；相等都用`==`；C用`&&/||/!`，Python用`and/or/not`。
- 🔴 复杂表达式先算括号，不确定优先级时主动加括号。
- ⚫ `=`是赋值不是相等比较；短路时右侧可能不执行。
### 回忆卡 3｜输入输出

先答：C的scanf/printf和Python的input/print各做什么？


- 🔵 C用`scanf`输入、`printf`输出；Python用`input`读字符串、`print`输出，需要数值时用`int/float`转换。
- 🔴 读输出题严格核对变量当前值、空格和换行。
- ⚫ Python的`input()`默认不返回整数；C读取普通变量通常要传地址。
## 三、选择与循环

### 回忆卡 1｜选择结构

先答：C与Python的if语句在代码块表示上有什么区别？


- 🔵 C条件常写圆括号并用大括号组织块；Python条件后写冒号并靠缩进组织块。
- 🔴 多分支从上到下判断，命中后只执行对应分支。
- ⚫ 条件范围顺序不当可能让后续分支永远无法执行。
### 回忆卡 2｜while与for

先答：while为什么可能一次不执行？C for和Python range循环各怎么看次数？


- 🔵 while先判断条件；C for看初始化、条件、更新；Python `range(n)`通常产生0到n-1。
- 🔴 列“轮次—条件—变量—输出”表逐轮跟踪。
- ⚫ `range(5)`不包含5，循环次数也不能只凭最后一个数字判断。
### 回忆卡 3｜break与continue

先答：break和continue各改变什么？


- 🔴 break结束最近一层循环，continue跳过本轮剩余语句进入下一轮。
- 🔵 跟踪时画出跳转位置，并继续检查循环变量是否更新。
- ⚫ break不等于结束整个程序，也不会自动结束所有嵌套循环。
## 四、函数、数组与容器

### 回忆卡 1｜函数与参数

先答：函数有什么作用？形参、实参、返回值怎样区别？


- ⚫ 函数封装任务、减少重复并便于维护；形参在定义中，实参在调用处，返回值把结果交给调用者。
- 🔵 简答按“定义—组成—作用”写，阅读题先代入实参再执行函数体。
- 🔴 普通形参与调用时的实参不是同一个术语，函数也不一定都有显式返回值。
### 回忆卡 2｜C数组与字符串

先答：长度为n的C数组常用下标范围是什么？C字符串怎样结束？


- ⚫ 常用下标为0到n-1；C字符串是以`\0`结束的字符数组。
- 🔵 访问数组先核对长度、下标和循环边界。
- 🔴 长度为5的数组没有下标5；字符串空间要给终止符留位置。
### 回忆卡 3｜Python容器

先答：列表、元组、字典、集合分别有什么特点？切片是否包含stop？


- 🔵 列表有序可改，元组有序通常不可改，字典按键映射，集合元素不重复；切片不包含stop。
- 🔴 看输出时记录容器是否原地修改，以及访问使用下标还是键。
- ⚫ 字典不靠连续整数下标工作，列表和元组也不是都可修改。
## 五、指针、结构体、文件与异常

### 回忆卡 1｜C指针

先答：指针保存什么？`&x`、`*p`分别表示什么？


- 🔵 指针保存地址；`&x`取得x的地址，`*p`访问或修改p所指对象。
- 🔴 读码先写“p指向谁”，再把`*p=值`还原为“修改所指变量”。
- ⚫ 空指针和未初始化指针不能直接解引用；数组与指针相关但不是完全相同对象。
### 回忆卡 2｜结构体与C文本文件

先答：结构体解决什么问题？C文本文件的打开、读写、关闭和三种模式是什么？


- ⚫ 结构体组合不同类型成员，变量用`.`访问；C文件用`fopen`、读写函数、`fclose`，`r/w/a`分别为读、覆盖写、追加。
- 🔵 一眼读码先看打开模式和是否成功，再看写入对象，最后确认关闭。
- 🔴 `w`通常清空已有内容；结构体成员不要求类型相同，结构体指针常用`->`访问。
### 回忆卡 3｜Python异常与with open

先答：try、except、else、finally各何时执行？为什么推荐`with open`？


- 🔵 try放可能出错语句，except处理异常，else在无异常时执行，finally通常总执行；`with open`可在退出代码块时自动关闭文件。
- 🔴 文件模式`r/w/a`为读、覆盖写、追加；先确认模式，再执行read/write。
- ⚫ 异常处理不应隐藏所有错误；`w`可能清空原文件，with也不能替代备份。
## 六、C/Python比较与基础答题

### 回忆卡 1｜语法对照

先答：C和Python在代码块、语句结尾、逻辑运算、整除和常用输入输出上怎样对照？


- 🔵 C用大括号、常用分号、`&&/||/!`、整数`/`、scanf/printf；Python用缩进、通常无分号、and/or/not、`//`、input/print。
- 🔴 拿到代码先确认语言，再按该语言规则解释。
- ⚫ 不要混写C的大括号分号与Python的冒号缩进。
### 回忆卡 2｜程序输出题

先答：读简单顺序、分支或循环代码时，统一步骤是什么？


- 🔵 写变量初值 → 沿控制流逐句执行 → 分支算真假 → 循环逐轮记录 → 原样写输出。
- 🔴 循环至少记录轮次、条件、关键变量和输出。
- ⚫ 不要把变量初值当最终值，也不要忽略空格、换行和最后一次更新。
### 回忆卡 3｜简答与判断纠错

先答：定义区别题和程序判断题怎样作答？


- ⚫ 定义题写本质和作用；区别题写发生阶段、表现和示例；判断题指出错误点并改成正确说法。
- 🔵 高频检查：标识符、`=`/`==`、input类型、range右端、数组下标、break作用范围。
- 🔴 看到“永远、一定、完全”要核对条件，但不能只凭绝对词判错。
### 回忆卡 4｜读码与容器选型

先答：程序输出题先做哪四步？Python 列表、字典、集合分别适合什么需求？

- 🔵 读码按“初值 → 运算和类型 → 控制流逐句更新 → 输出格式”；列表重顺序和下标，字典按键查找，集合重去重和成员判断。
- 🔴 循环列变量跟踪表；选容器写清主要操作和是否需要顺序、重复值、键值映射。
- ⚫ C 整数 `/` 与 Python 3 `/` 规则不同；集合不能直接代替需要稳定业务顺序或重复记录的列表。
