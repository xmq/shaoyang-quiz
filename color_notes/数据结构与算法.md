# 数据结构与算法 · 三色回忆卡

> 先答后核对。🔴记结论；🔵记一步判断；⚫记条件与纠错。

## 一、概念与线性结构

### 回忆卡 1｜结构与复杂度

先答：逻辑结构、存储结构、数据运算是什么？常见复杂度怎样排序？

- 🔴 三者分别描述元素关系、机器表示和查找插删等操作；`O(1)<O(logn)<O(n)<O(nlogn)<O(n²)`。
- 🔵 直接访问常为 O(1)，范围减半为 O(logn)，单遍历 O(n)，双重 n 循环 O(n²)。
- ⚫ O 是渐进数量级，不是精确时间；同一逻辑结构可用不同存储方式。

### 回忆卡 2｜顺序表与链表

先答：顺序表和链表怎样比较？单、双、循环链表有何区别？

- 🔴 顺序表连续、随机访问 O(1)、中间插删 O(n)；链表离散、按位查 O(n)、已知前驱插删 O(1)。双链表可前后找，循环链表尾连头。
- 🔵 比较按“存储 → 访问 → 插删 → 扩展 → 指针开销”。
- ⚫ 头结点不等于头指针；普通链表不能按下标 O(1) 访问。

### 回忆卡 3｜栈、队列、串

先答：栈、队列的规则和用途是什么？KMP 相比朴素匹配改进什么？

- 🔴 栈后进先出，用于递归；队列先进先出，用于排队/BFS；KMP 利用模式前后缀，使主串位置不回退。
- 🔵 比较栈队列按“操作端 → 出入顺序 → 用途”。
- ⚫ 空串长度 0，空格串含字符；循环队列判空/满依实现约定。

## 二、树

### 回忆卡 4｜树与二叉树

先答：结点度、树的度、叶、二叉树、满和完全二叉树是什么？

- 🔴 结点度是孩子数，树的度是最大度，叶度为0；二叉树每结点最多两个有序孩子；满树各层满，完全树末层从左连续。
- 🔵 n 个非空树结点有 n-1 条边；二叉树有 `n0=n2+1`。
- ⚫ 二叉树不是每结点必须两个孩子，也不是普通“度为2的树”。

### 回忆卡 5｜遍历与 BST

先答：四种遍历顺序是什么？BST 有什么性质？

- 🔴 先序根左右，中序左根右，后序左右根，层序逐层；BST 左小右大，中序递增。
- 🔵 遍历命名看根在左右访问中的位置，层序使用队列。
- ⚫ BST 极端时退化为链，查找不总是 O(logn)。

### 回忆卡 6｜AVL、堆、哈夫曼、B+树

先答：四类树各有什么关键性质和用途？

- 🔴 AVL 左右高度差≤1；堆是完全二叉树且父子有序；哈夫曼树 WPL 最小、用于编码；B+树多路平衡、叶层相连、用于磁盘索引。
- 🔵 作用题按“结构特点 → 优化目标 → 场景”。
- ⚫ 堆只保证父子关系，不保证全局有序。

## 三、图与查找

### 回忆卡 7｜图与存储

先答：连通/强连通是什么？邻接矩阵和邻接表怎样比较？

- 🔴 连通指无向图两点有路径，强连通指有向图两点互达；矩阵 O(n²)、判边快，邻接表 O(n+e)、适合稀疏图。
- 🔵 无向图度数和＝2倍边数；有向图入度和＝出度和＝边数。
- ⚫ 无向边在邻接表通常存两次；有路径不等于有直接边。

### 回忆卡 8｜图算法用途

先答：DFS、BFS、Prim/Kruskal、Dijkstra、拓扑排序分别解决什么？

- 🔴 DFS 用栈深入，BFS 用队列逐层；Prim/Kruskal 求最小生成树，Dijkstra 求非负权单源最短路，拓扑排序处理 DAG 先后关系。
- 🔵 算法题写“问题＋思想＋数据结构＋限制条件”。
- ⚫ Dijkstra 不适合普通负权图；有环图不能完成拓扑排序。

### 回忆卡 9｜查找

先答：顺序、折半、散列查找的条件和特点是什么？

- 🔴 顺序查找适用有序/无序表、O(n)；折半要求有序顺序表、O(logn)；散列按函数定位，冲突用开放定址或拉链。
- 🔵 先看是否有序、能否随机访问，再选查找法。
- ⚫ 普通链表不适合折半；开放定址删除不能随意置空。

## 四、排序与答题

### 回忆卡 10｜稳定性

先答：排序稳定性是什么？哪些常见排序稳定？

- 🔴 相等键相对次序不变为稳定；插入、冒泡、归并稳定，选择、快速、堆通常不稳定。
- 🔵 选择算法结合稳定性、时间、空间和初始有序度。
- ⚫ 本次未交换相等项不能证明算法稳定；稳定不等于更快。

### 回忆卡 11｜排序复杂度

先答：插入、冒泡、选择、快速、堆、归并的主要时间特点是什么？

- 🔴 前三平均 O(n²)；快速平均 O(nlogn)、最坏 O(n²)；堆和归并最坏 O(nlogn)，归并需 O(n) 空间。
- 🔵 基本有序考虑插入，稳定考虑归并，原地且最坏有保证考虑堆。
- ⚫ 快速排序并非任何输入都最快。

### 回忆卡 12｜基础简答框架

先答：定义特点、结构比较和算法作用题怎样作答？

- 🔴 定义写“规则＋操作＋用途”；比较写“存储＋访问＋插删＋空间”；算法写“问题＋思想＋复杂度＋条件”。
- 🔵 先圈题目动词，列 3—5 个维度后逐点扩写。
- ⚫ 只写名称或复杂度、不写前提和用途，答案不完整。

### 回忆卡 13｜判断纠错

先答：“链表随机访问O(1)、栈先进先出、二叉树都有两孩子、快速排序总为O(nlogn)”怎样改？

- 🔴 均错误：链表按位查 O(n)；栈后进先出；二叉树每结点最多两孩子；快速排序最坏 O(n²)。
- 🔵 统一写“错误＋混淆点＋正确结论＋条件”。
- ⚫ 不要把逻辑结构与存储结构、稳定性与速度混为一谈。

## 五、场景选型

### 回忆卡 14｜从操作需求选结构

先答：尾部追加、下标访问、先进先出、编号等值查找分别优先选什么？

- 🔴 尾部追加兼顾下标访问选动态数组；先进先出选队列；已知位置频繁插删选链表；按唯一编号等值查找选散列表。
- 🔵 理由必须写“主要操作＋复杂度＋空间/顺序要求”，需求冲突时写主结构和辅助索引。
- ⚫ 链表插删 O(1) 以已知位置为前提；数组尾部追加是均摊 O(1)；散列平均 O(1) 不等于最坏 O(1)。

### 回忆卡 15｜优先级、磁盘索引与得分词

先答：有优先级的任务、磁盘范围索引各选什么？怎样把“为什么”答完整？

- 🔴 优先任务选优先队列，同级FIFO可加到达序号；磁盘范围索引常选B+树，因多路低树高且叶层有序相连。
- 🔵 理由按“主要操作 → 结构机制 → 复杂度或I/O收益 → 限制条件”组织。
- ⚫ 优先队列不保证全部元素有序；B+树查找仍可能发生磁盘I/O；结构名称本身不是完整原因。
