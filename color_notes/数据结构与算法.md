# 数据结构与算法：三色笔记

三色按考频和出题稳定性划分：红色优先背熟，蓝色用于补齐得分点，黑色用于防漏和边界判断。

## 🔴 高频考点

- 🔴 **线性表**：数组支持随机访问，链表便于插入删除。 常见考法：按订单尾部增加、随机查询等场景选结构。 易错边界：数组尾插快但中间插入移动成本高。
- 🔴 **栈与队列**：栈后进先出，队列先进先出。 常见考法：判断表达式、括号匹配、排队服务。 易错边界：循环队列要区分队空和队满。
- 🔴 **树与二叉树**：二叉树、遍历、二叉搜索树是基础。 常见考法：给树写前中后序，判断查找路径。 易错边界：二叉搜索树中序遍历有序。
- 🔴 **图**：图由顶点和边组成，DFS/BFS 是基本遍历。 常见考法：判断路径、连通、最短路思想。 易错边界：有向图和无向图度的概念不同。
- 🔴 **排序**：冒泡、选择、插入、快速、归并常考复杂度和稳定性。 常见考法：判断算法过程、时间复杂度。 易错边界：快速排序平均快但最坏可到 `O(n^2)`。
- 🔴 **查找**：顺序查找、折半查找、哈希查找各有条件。 常见考法：判断查找效率和适用场景。 易错边界：折半查找要求顺序存储且有序。

## 🔵 中频考点

- 🔵 **时间复杂度**：用数量级描述算法随规模增长的耗时。 常见考法：判断 `O(1)`、`O(n)`、`O(log n)`、`O(n^2)`。 易错边界：忽略常数不等于忽略主要循环层数。
- 🔵 **递归**：递归要有终止条件和规模缩小。 常见考法：判断递归出口、调用次数。 易错边界：没有正确出口会无限递归。
- 🔵 **哈希表**：用哈希函数把关键字映射到存储位置。 常见考法：判断冲突处理和查找效率。 易错边界：哈希查找平均快，但冲突会影响性能。
- 🔵 **堆**：堆用于优先队列，常见大根堆和小根堆。 常见考法：判断堆顶元素。 易错边界：堆不是内存中的堆区概念。
- 🔵 **广义表与串**：串是字符序列，常考模式匹配基础。 常见考法：判断子串、长度、匹配。 易错边界：下标从 0 还是 1 要看题目约定。

## ⚫ 低频了解

- ⚫ **外部排序**：数据量大到内存放不下时使用。 常见考法：概念判断。 易错边界：普通内部排序默认数据在内存中。
- ⚫ **平衡树**：通过旋转控制树高，提高查找效率。 常见考法：了解 AVL/红黑树用途。 易错边界：一般基础考试不要求手算复杂旋转。
- ⚫ **算法设计思想**：分治、贪心、动态规划是常见思想。 常见考法：判断场景适合哪类思想。 易错边界：贪心需要证明局部最优可导向全局最优。

## 考前公式与简答

### 必会公式

- **复杂度数量级**：常见增长速度为 `O(1)<O(log n)<O(n)<O(n log n)<O(n²)`。`n` 是输入规模，`O` 表示规模增大时基本操作次数的增长上界，比较时忽略低阶项和常数系数。
- **折半查找**：每次把范围减半，比较次数数量级约为 `log₂n`；前提是数据有序且能按下标快速访问，普通链表不适合直接折半。
- **树的基本关系**：一棵含 `n` 个结点的非空树有 `n-1` 条边；二叉树第 `k` 层最多有 `2^(k-1)` 个结点。层数从1还是0开始必须看题目约定。

### 常见简答题

- **数组与链表如何选择？** 需要按下标快速访问、数据规模相对稳定时选数组；插入删除频繁且不要求随机访问时链表更合适。还应考虑内存连续性和指针开销。
- **栈和队列有什么区别？** 栈按后进先出组织，常用于函数调用、括号匹配；队列按先进先出组织，常用于排队服务、广度优先搜索和任务调度。
