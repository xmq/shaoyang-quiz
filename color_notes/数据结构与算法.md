# 数据结构与算法 · 三色回忆卡

> 使用方法：先手算或口述“先答”，再核对。🔴只记最小结论；🔵只记算法过程；⚫只记条件、边界和反例。

## 一、复杂度与线性结构

### 回忆卡 1｜复杂度

先答：怎样判断顺序、嵌套、倍增循环和递归的时间/空间复杂度？

- 🔴 顺序取最大阶，嵌套相乘，变量按常数倍增长常为对数；递归先写递推式。增长次序为 `1<log n<n<nlog n<n²<2ⁿ<n!`。
- 🔵 选基本操作并求执行次数数量级；递归再计调用栈或辅助数组。
- ⚫ O 是渐进上界，不是精确时间；二分 O(log n) 以有序且随机访问为前提。

### 回忆卡 2｜顺序表与链表

先答：顺序表和链表在访问、插删、空间上怎样取舍？

- 🔴 顺序表随机访问 O(1)、插删平均 O(n)；链表按位访问 O(n)，已知前驱时插删 O(1)。
- 🔵 链表改指针先保存将丢失的后继，再接新链，最后释放被删结点。
- ⚫ “已知结点即可 O(1) 删除”对尾结点或要求保持结点身份时不成立。

### 回忆卡 3｜单链表操作

先答：怎样头插/尾插、在 p 后插入、删除 p 后继并原地逆置？

- 🔴 头插使输入逆序，尾插保持顺序；插入先令新结点指向后继再改 p；逆置用 pre、cur、next 三指针。
- 🔵 逆置每轮“保存 next → cur 指回 pre → pre/cur 前移”，结束后头指针指 pre。
- ⚫ 头结点不存有效数据；空表、首尾和释放内存必须单独检查。

### 回忆卡 4｜栈、队列与表达式

先答：栈和队列分别用于什么？中缀转后缀与后缀求值怎样做？

- 🔴 栈后进先出，队列先进先出；中缀转后缀用运算符栈，后缀求值用操作数栈。
- 🔵 运算符按括号和优先级弹栈；求值遇运算符先弹右数 b、再弹左数 a，算 `a op b`。
- ⚫ 相同优先级是否弹出还受结合性影响；减法和除法的弹栈次序不可反。

### 回忆卡 5｜循环队列

先答：数组长 m 且牺牲一个单元时，循环队列怎样判空、判满、求长？

- 🔴 空：`front=rear`；满：`(rear+1)%m=front`；长度：`(rear-front+m)%m`，实际容量 m-1。
- 🔵 入队先写 rear 所指位置再 rear 后移；出队取 front 所指位置再 front 后移，按题设定义调整。
- ⚫ 若另设 size/tag 可用满 m 个单元，不能混用判满公式。

### 回忆卡 6｜KMP

先答：KMP 为什么主串指针不回退？怎样手算前缀函数？

- 🔴 它利用模式串已匹配部分的最长相等真前后缀，使失配后模式跳转，复杂度 O(n+m)。
- 🔵 对每个模式前缀求最长相等真前后缀长度；失配时沿此前缀值回退，直到可匹配或为 0。
- ⚫ next 有 -1/0/1 下标版本，必须先声明；真前后缀不能等于串自身。

## 二、树与查找

### 回忆卡 7｜二叉树性质

先答：二叉树层结点上限、总结点上限和叶/度 2 结点关系是什么？

- 🔴 第 i 层最多 `2^(i-1)`，高度 h 最多 `2^h-1`；非空二叉树 `n0=n2+1`。
- 🔵 完全二叉树编号 i 的孩子为 2i、2i+1，双亲为 `floor(i/2)`，最后非叶结点为 `floor(n/2)`。
- ⚫ 层数从 0 还是 1、高度按边还是结点计，必须服从题设。

### 回忆卡 8｜遍历还原

先答：怎样由先序/后序与中序还原二叉树？为什么先序加后序通常不唯一？

- 🔴 先序首个或后序末个定根，在中序中按根划左右子树，再递归。
- 🔵 每轮写根、左子树长度、右子树长度，再同步切另一遍历序列。
- ⚫ 普通二叉树只给先序和后序，单孩子方向无法确定，通常不唯一。

### 回忆卡 9｜BST 与 AVL

先答：BST 删除双孩子结点怎样做？AVL 插入后怎样判 LL、RR、LR、RL？

- 🔴 BST 用中序前驱/后继替换后再删；AVL 从插入点向上找首个失衡结点，按向下两步方向判型：LL 右旋、RR 左旋、LR 先左后右、RL 先右后左。
- 🔵 手算每次插入后更新高度和平衡因子，只在首个失衡处旋转并重算。
- ⚫ 重复键策略按题设；BST 最坏退化为链，查找 O(n)。

### 回忆卡 10｜哈夫曼树

先答：怎样构造哈夫曼树并求 WPL、编码和总结点数？

- 🔴 每次合并最小两权，父权为和；`WPL=Σ叶权×深度`，也等于各次合并权之和；n 叶共有 2n-1 个结点。
- 🔵 维护有序权值集合，逐轮记录被合并两项和新权；由根到叶标 0/1 得前缀码。
- ⚫ 同权可有不同树形和编码，但最小 WPL 相同；左右 0/1 可互换。

### 回忆卡 11｜堆

先答：怎样建大根堆、插入、删除堆顶并用于升序排序？

- 🔴 建堆从最后非叶结点向前下滤；插入从末尾上滤；删顶用末元素补根再下滤；升序使用大根堆。
- 🔵 建堆 → 根与有效区末尾交换 → 堆长减 1 → 根下滤，重复。
- ⚫ 建堆 O(n)，单次调整 O(log n)，整个堆排序 O(nlog n) 且不稳定。

### 回忆卡 12｜折半查找

先答：折半查找的前提、过程和成功 ASL 怎样算？

- 🔴 要求有序顺序表；比较中点后缩到左/右半区，时间 O(log n)。成功 ASL 为各成功结点比较次数之和除元素数。
- 🔵 先声明 mid 取整规则，画判定树并按层数统计成功/失败比较次数。
- ⚫ 链表不适合折半；不同 mid 规则会改变具体树形和 ASL。

### 回忆卡 13｜散列表

先答：开放定址与拉链怎样处理冲突？怎样手算插入、查找和 ASL？

- 🔴 开放定址按探测序列找空位，拉链把同址元素链接；装填因子 `α=记录数/表长`。
- 🔵 按插入序计算初址，冲突逐次写探测地址；查找沿同序列到命中或从未占用位置，并累计比较次数。
- ⚫ 开放定址删除应设删除标记，直接置空会截断后续查找；拉链 α 可大于 1。

## 三、图算法

### 回忆卡 14｜图存储与遍历

先答：邻接矩阵/表的空间与判边代价是什么？DFS/BFS 用什么结构？

- 🔴 矩阵空间 O(n²)、判边 O(1)，适合稠密图；邻接表 O(n+e)，适合稀疏图。DFS 用递归/栈，BFS 用队列。
- 🔵 严格按题定邻接次序维护 visited 与栈/队列；非连通图从每个未访问点重启。
- ⚫ 未指定邻接次序时序列可能不唯一；无向边在邻接表存两次。

### 回忆卡 15｜Prim 与 Kruskal

先答：两种最小生成树算法每轮选什么，怎样防止成环？

- 🔴 Prim 每次选连接已选集与外部的最小边；Kruskal 按边权升序选不成环边，常用并查集。
- 🔵 逐轮写候选边、选中边和顶点集/连通分量，选满 n-1 边后核对覆盖且无环。
- ⚫ 只适用于连通无向加权图；有同权不必然产生多棵 MST。

### 回忆卡 16｜并查集

先答：并查集怎样支持 Kruskal 的判环？路径压缩和按秩合并做什么？

- 🔴 Find 找集合代表，Union 合并不同集合；一条边两端代表相同则加入会成环。
- 🔵 每选边先 Find 两端，不同才加入并 Union；路径压缩缩短查找链，按秩让矮树挂高树。
- ⚫ 并查集判断的是当前连通分量，不直接计算最短路径。

### 回忆卡 17｜Dijkstra

先答：Dijkstra 每轮“确定”和“松弛”什么？怎样还原路径？

- 🔴 每轮确定未确定且 dist 最小的 u；对出边检查 `dist[u]+w<dist[v]`，成立则更新 dist[v] 和 pre[v]。
- 🔵 初始化源点 0、其余无穷 → 选最小 → 松弛 → 列 dist/pre 表 → 从终点沿 pre 逆推路径。
- ⚫ 普通 Dijkstra 不适用于负权边；不可达点保持无穷。

### 回忆卡 18｜Floyd

先答：Floyd 的状态含义、更新式和循环顺序是什么？

- 🔴 以 1…k 为中间点时，比较原路径与经 k 路径：`D[i][j]=min(D[i][j],D[i][k]+D[k][j])`。
- 🔵 k 必须最外层；每轮复制矩阵并更新，需要路径时同步记录 next/pre。
- ⚫ 时间 O(n³)、空间 O(n²)；不可达的无穷参与加法时要防溢出。

### 回忆卡 19｜拓扑排序

先答：怎样手算拓扑序并用它判断有环？

- 🔴 反复输出入度 0 顶点并删除其出边；输出数少于 n 则有环。
- 🔵 建入度表和零入度栈/队列，每输出一项逐条减后继入度。
- ⚫ 只适用于 DAG；多个零入度点使拓扑序可能不唯一。

### 回忆卡 20｜关键路径

先答：AOE 网怎样求 ve、vl、活动余量和总工期？

- 🔴 正拓扑取 max 求 ve；逆拓扑取 min 求 vl；活动最早 `e=ve[起点]`，最迟 `l=vl[终点]-工期`，`l-e=0` 为关键活动。
- 🔵 拓扑序 → ve → 以终点 ve 初始化 vl → 逆序 vl → 逐边算 e/l → 写关键路径。
- ⚫ 多条关键路径时，只缩短其中一条上的活动未必缩短总工期。

## 四、排序与算法设计

### 回忆卡 21｜排序性质总表

先答：哪些常见排序稳定？哪些保证最坏 O(nlog n)？

- 🔴 稳定：插入、冒泡、归并、基数；通常不稳定：选择、希尔、快速、堆。归并和堆最坏 O(nlog n)，快速最坏 O(n²)。
- 🔵 选算法先看数据规模/有序度、稳定性、额外空间和最坏界。
- ⚫ “本次未交换相等项”不能证明算法稳定；具体实现可能改变稳定性。

### 回忆卡 22｜快速排序

先答：一次分区保证什么？怎样手算并分析退化？

- 🔴 一趟只保证枢轴最终到位、两侧满足大小关系，不保证两侧内部有序。
- 🔵 先声明枢轴和分区版本，逐次写指针移动/交换，再递归左右区间。
- ⚫ 近乎有序且固定取首/尾枢轴可能退化到 O(n²)；快速排序通常不稳定。

### 回忆卡 23｜归并、插入与冒泡

先答：归并、直接插入、冒泡每一趟分别形成什么不变量？

- 🔴 归并把两个有序段合成有序段；插入保持前缀有序；冒泡每趟把一个极值送到边界。
- 🔵 手算逐趟写有效区和比较/移动结果；归并相等时先取左段可保持稳定。
- ⚫ 插入和带提前终止的冒泡最好 O(n)；归并需 O(n) 辅助空间。

### 回忆卡 24｜算法设计题

先答：算法大题怎样写出输入输出、正确性和复杂度的完整得分点？

- 🔴 写输入/输出与边界、辅助结构含义、伪代码、终止条件、正确性要点、时间和额外空间。
- 🔵 用循环不变式写初始化—保持—终止，或递归写基例—规模缩小—组合结果。
- ⚫ 必查空表、单元素、重复键、首尾、图不连通、同权、下标和内存释放。
