# 计算机网络 · 三色回忆卡

> 先答后核对。🔴记结论；🔵记一步判断；⚫记条件与纠错。

## 一、概述与分层

### 回忆卡 1｜定义、功能、指标

先答：网络是什么？主要功能及带宽、吞吐量、时延各是什么？


- 🔴 网络是在协议控制下连接计算机以通信和共享资源的系统；带宽是标称能力，吞吐是实际速率，时延是传送耗时。
- 🔵 发送时延＝比特数/速率，先换算 `1B=8bit`。
- ⚫ 吞吐通常不高于带宽；分组交换共享链路但可能排队、丢包。
### 回忆卡 2｜层次与 PDU

先答：OSI 七层、五层模型和各层数据单位是什么？


- ⚫ OSI 为物理、链路、网络、运输、会话、表示、应用；五层合并后三层。PDU 主线是比特、帧、IP 数据报、段/用户数据报、报文。
- 🔵 发送逐层封装，接收逐层解封。
- 🔴 协议是对等层规则，服务是下层向上层提供的能力。
### 回忆卡 3｜地址与通信方向

先答：域名、IP、MAC、端口各标识什么？单工、半双工、全双工怎样区分？


- ⚫ 域名便于记忆，IP 跨网寻址，MAC 本链路交付，端口标识进程；三种方向依次为单向、可双向但不同时、可同时双向。
- 🔵 判断地址先问“给人、找接口、找本跳、找进程”。
- 🔴 对讲机是半双工；端口不标识一台计算机。
### 回忆卡 4｜网络拓扑

先答：总线、星型、环型、树型和网状拓扑各有什么主要特点？


- ⚫ 总线共享主干；星型便于管理但依赖中心；环型首尾相连；树型分层扩展；网状路径多、可靠但成本高。
- 🔵 读图先找中心、主干、闭环和冗余路径，再判断拓扑。
- 🔴 物理拓扑描述线路连接，逻辑拓扑描述数据流动，两者不一定相同。
## 二、介质与局域网

### 回忆卡 5｜介质、复用、线序

先答：双绞线、光纤、无线有何特点？T568A/B 和直通/交叉线怎样记？


- 🔵 双绞线经济，光纤高速抗电磁干扰，无线部署灵活；A 以白绿绿开头，B 以白橙橙开头，只交换橙绿对；同标直通、异标交叉。
- 🔴 复用辨认：FDM 分频带，TDM 分时隙，WDM 分光波长。
- ⚫ Auto-MDI/MDIX 可适配线型，但规范压线仍应遵守线序。
### 回忆卡 6｜交换机与设备

先答：交换机怎样学习转发？集线器、交换机、路由器分别在哪层？


- ⚫ 交换机按源 MAC 学习，已知单播定向转发，未知/广播泛洪；三设备依次在物理、链路、网络层。
- 🔵 比较按“层次 → 识别地址 → 冲突域 → 广播域”。
- 🔴 普通二层交换机默认不隔离广播域；全双工交换以太网不用 CSMA/CD。
### 回忆卡 7｜VLAN、STP、CRC

先答：VLAN、STP、CRC 分别解决什么问题？


- 🔵 VLAN 划广播域，STP 防二层环路，CRC 检测传输差错。
- 🔴 跨 VLAN 需三层转发；读拓扑先找广播域再找闭环。
- ⚫ CRC 检错不等于纠错；未检出错误不代表绝对无错。
### 回忆卡 8｜WLAN、AP与频段

先答：WLAN、AP、SSID是什么？2.4 GHz和5 GHz通常怎样比较？


- 🔵 WLAN常用802.11/Wi-Fi；AP提供无线接入，SSID是网络名称；2.4 GHz覆盖穿墙通常较好，5 GHz信道多、速度和抗干扰通常较好。
- 🔴 比较按“覆盖距离—穿墙—信道/干扰—速度”作答。
- ⚫ AP不等于完整无线路由器；隐藏SSID不等于加密；5 GHz不一定覆盖更远。
## 三、IP 与路由

### 回忆卡 9｜IPv4、子网划分与私有地址

先答：/24平均分4个子网怎样算？地址块、可用主机数怎样求？


- 🔵 借2位得/26，掩码255.255.255.192；块大小64，每块传统可用`2^6-2=62`台，网络边界为0、64、128、192。
- 🔴 四步：按子网数借位 → `h=32-前缀` → 算块大小 → 写网络/广播/主机范围；同网判断用IP与掩码AND。
- ⚫ “减2”是传统普通子网规则，特殊/31按题干；私有段为10/8、172.16/12、192.168/16，172只含第二段16～31。
### 回忆卡 10｜IPv6、ARP、网关

先答：IPv6 的关键特点是什么？同网/异网发送时 ARP 查谁？


- 🔵 IPv6 为 128 位、无广播、路由器不分片；同网 ARP 查目的 MAC，异网查默认网关 MAC。
- 🔴 先判同网，再定下一跳 IP，最后取得本跳 MAC。
- ⚫ ARP 不跨路由器；每跳 MAC 通常变化，端到端 IP 通常不变，NAT 除外。
### 回忆卡 11｜ICMP 与路由

先答：ping、traceroute 基于什么？RIP、OSPF、BGP 用在哪里？


- 🔵 ping 用 ICMP 回显，traceroute 利用 TTL 超时；RIP/OSPF 用于自治系统内，BGP 用于自治系统间。
- 🔴 路由器在多条匹配中选最长前缀；无具体路由才用默认路由。
- ⚫ ping 失败不等于主机必关机；BGP 不只追求物理最短。
## 四、运输与应用

### 回忆卡 12｜TCP 与 UDP

先答：TCP、UDP 怎样比较？握手、挥手和两类控制各有什么作用？


- 🔵 TCP连接、可靠有序、字节流，靠校验和、序号、确认、重传和窗口；UDP无连接、尽力、保留报文边界。
- 🔴 订单场景按“为何选TCP → 哪些机制保证传输 → 业务层还要什么”作答。
- ⚫ TCP不保证请求只执行一次或库存天然一致；仍需唯一订单号、幂等校验和数据库事务/并发控制。
### 回忆卡 13｜Web 与 DNS

先答：DNS、HTTP、HTTPS 各做什么，常见端口是什么？


- 🔵 DNS 解析域名、53；HTTP 传 Web 内容、80；HTTPS 经 TLS 保护、443。
- 🔴 访问过程概括为“DNS解析 → 建连 → 请求响应”；能ping公网IP却不能访问域名，优先用`nslookup`查解析。
- ⚫ DNS只负责名称解析；HTTPS保护传输和证书认证，不保证网站业务内容一定可信。
### 回忆卡 14｜邮件、远程、DHCP

先答：SMTP、POP3/IMAP、SSH、Telnet、DHCP 的作用是什么？


- 🔵 SMTP 发邮件，POP3/IMAP 收邮件，SSH 安全远程，Telnet 明文远程；DHCP 分配网络参数，DORA 为发现、提供、请求、确认。
- 🔴 获得169.254.x.x时先查链路、DHCP服务/中继和租约；端口最小集含SSH22、Telnet23、SMTP25、DHCP67/68。
- ⚫ DHCP不负责域名解析，DNS故障也不会直接让主机取得169.254地址；端口默认值可修改。
## 五、安全、排障与答题

### 回忆卡 15｜防护设施

先答：防火墙、IDS、IPS、VPN 分别做什么？


- 🔵 防火墙按规则控流量，IDS 检测告警，IPS 可主动阻断，VPN 建受保护隧道。
- 🔴 比较按“检测/阻断 → 部署位置 → 是否提供隧道”。
- ⚫ VPN 不自动保证终端、账号和应用安全。
### 回忆卡 16｜排障与简答

先答：基础排障顺序和网络简答题框架是什么？


- ⚫ 排障按链路 → IP配置 → 回环/本机 → 网关 → 公网IP → 域名 → 路径/端口；机制简答写“结论＋机制＋适用原因＋边界”。
- 🔵 对应命令为ipconfig、ping、tracert、nslookup、arp、netstat；每一步都写检查对象、预期现象和后续判断。
- 🔴 只列命令不写检查对象，或只说“TCP可靠”不写序号/确认/重传，答案不完整。
### 回忆卡 17｜判断纠错

先答：“交换机隔离广播域、ARP 跨路由、TCP 面向报文、ping 失败必关机”怎样改？

- 🔵 均错误：普通二层交换机默认不隔离广播域；ARP 限本地链路；TCP 面向字节流；ping 还受链路、路由和防火墙影响。
- 🔴 统一写“错误＋错误词＋正确结论＋条件”。
- ⚫ 不能只改是非，要指出混淆的层次、对象或适用范围。
