# 计算机网络 · 三色回忆卡

> 使用方法：遮住答案，先口述“先答”，再核对。🔴只记最小结论；🔵只记计算/报文步骤；⚫只记条件、边界和反例。

## 一、分层、性能与物理层

### 回忆卡 1｜分层与封装

先答：OSI 七层、教学五层和各层 PDU 怎样对应？四类地址各标识什么？

- 🔴 OSI 自下而上为物理、链路、网络、运输、会话、表示、应用；五层合并为物理、链路、网络、运输、应用。PDU 依次为比特、帧、IP 数据报、段/用户数据报、报文。
- 🔵 发送逐层加首部/尾部，接收逐层解封；按“域名 → IP → MAC → 端口”追踪解析与交付。
- ⚫ MAC 只负责本链路，IP 负责跨网，端口标识主机内进程；设备可跨层实现，不能只凭产品名判层。

### 回忆卡 2｜时延与吞吐

先答：总时延由哪四项组成？发送时延、传播时延和时延带宽积怎样算？

- 🔴 总时延＝发送＋传播＋处理＋排队；发送＝比特数/速率；传播＝距离/传播速度；时延带宽积＝带宽×传播时延。
- 🔵 先统一 bit、B、s 单位，再按每段链路分别算；存储转发多跳要逐跳计发送。
- ⚫ 吞吐受瓶颈和协议开销限制；RTT 不是单向传播时延，多分组流水不能把单包时延简单乘包数。

### 回忆卡 3｜奈奎斯特与香农

先答：无噪声和有噪声信道的极限速率分别怎样求？

- 🔴 奈奎斯特：`C=2W log2M`；香农：`C=W log2(1+S/N)`，dB 转线性为 `10^(dB/10)`。
- 🔵 判断题先看是否给码元状态数 M 或信噪比，再统一带宽单位代入。
- ⚫ 奈奎斯特的 `2W` 是码元率；香农只给理论上限，不指定调制和编码方式。

### 回忆卡 4｜传输方式、编码与复用

先答：单工/半双工/全双工，曼彻斯特与 FDM/TDM/WDM/CDM 的最小区别是什么？

- 🔴 半双工可双向但不同步，全双工可同时双向；曼彻斯特每比特中间必跳变；FDM 分频带、TDM 分时隙、WDM 分光波长、CDM 分码片。
- 🔵 波形题先定时钟边界和编码约定，再逐位画中间/起始跳变。
- ⚫ 曼彻斯特约定 0/1 的跳变方向可能相反；同步 TDM 的空闲用户仍可能占固定时隙。

### 回忆卡 5｜双绞线线序

先答：T568A、T568B 的 1—8 芯线序是什么？直通线和交叉线怎样接？

- 🔴 A：白绿、绿、白橙、蓝、白蓝、橙、白棕、棕；B：白橙、橙、白绿、蓝、白蓝、绿、白棕、棕。两端同标准为直通，一端 A 一端 B 为交叉。
- 🔵 记住只交换橙、绿两对；压线后按 1—8 连通性测试。
- ⚫ 千兆链路使用四对线；现代设备常有 Auto-MDI/MDIX，不能据“能通信”反推线序一定规范。

## 二、链路层与局域网

### 回忆卡 6｜透明传输与 CRC

先答：字节/比特填充怎样做？CRC 的 FCS 怎样手算？

- 🔴 字节填充在特殊字节前加转义；比特填充在连续 5 个 1 后插 0。CRC 用生成多项式作模 2 除，余数为 FCS。
- 🔵 G 的阶数为 r → 数据补 r 个 0 → 异或长除 → 取 r 位余数 → 接收端以同一 G 验余数。
- ⚫ 余数 0 只表示未检出错误；模 2 除无进位、借位。

### 回忆卡 7｜停止等待、GBN 与 SR

先答：三种 ARQ 的窗口、确认和重传范围有什么不同？

- 🔴 停止等待一次一帧；GBN 累计确认、接收窗口常为 1，超时重传该帧及以后；SR 逐帧确认、缓存失序，只重传缺失帧。
- 🔵 序列题逐时刻维护发送窗、接收窗、计时器和 ACK；窗口利用率近似 `min(1,W×T发/(T发+RTT))`。
- ⚫ 序号空间必须避免新旧帧混淆；ACK 时长、捎带确认是否计入按题设。

### 回忆卡 8｜交换机与以太网

先答：交换机怎样自学习和转发？何时使用 CSMA/CD？

- 🔴 按源 MAC 学习入端口；目的已知定向转发，未知单播和广播向除入端口外泛洪。CSMA/CD 只用于共享半双工以太网。
- 🔵 表项题每收到一帧先更新源表项，再按目的查表决定过滤/转发/泛洪。
- ⚫ 交换式全双工无冲突；经典最小帧 64 B 不含前导码，VLAN 标签会改变常规帧长度。

### 回忆卡 9｜冲突域、广播域、VLAN 与 STP

先答：集线器、交换机、路由器怎样划分冲突域和广播域？VLAN、STP 各解决什么？

- 🔴 集线器全端口一个冲突域；交换机每端口独立冲突域但默认同一广播域；路由器隔离广播域。VLAN 逻辑分广播域，STP 阻塞冗余链路形成无环拓扑。
- 🔵 拓扑题先按三层边界划广播域，再按交换端口划冲突域；跨 VLAN 必须经三层转发。
- ⚫ 二层环路会导致广播风暴、重复帧和 MAC 表震荡；STP 保留冗余物理链路但阻塞部分逻辑路径。

## 三、IPv4、子网与路由

### 回忆卡 10｜CIDR 地址范围

先答：给定 IPv4/prefix，怎样求网络、广播和可用主机范围？

- 🔴 主机位 `h=32-prefix`，地址数 `2^h`；网络地址主机位全 0，广播全 1，传统可用数 `2^h-2`。
- 🔵 找掩码首个非 255 字节，块大小＝256－该字节；IP 所在块起点为整除块大小后乘回，下一块起点减 1 为广播。
- ⚫ `/31` 点到点和 `/32` 主机路由不套减 2；合法掩码必须连续 1 后连续 0。

### 回忆卡 11｜FLSM、VLSM 与聚合

先答：定长、变长子网和路由聚合分别怎样计算？

- 🔴 FLSM 借 k 位满足子网数；VLSM 按主机需求从大到小选最小 2 的幂地址块；聚合取连续且边界对齐网络的最长公共前缀。
- 🔵 FLSM 先验子网数再验主机数；VLSM 逐块写网络/广播/可用范围并查不重叠；聚合把网络地址转二进制找首个不同位。
- ⚫ 聚合不能夹入不应覆盖的网段；现代题的子网数是否减 2 按题设。

### 回忆卡 12｜ARP、同网与异网转发

先答：主机向同网和异网目的发送时，ARP 查询的 IP 分别是谁？每跳哪些地址改变？

- 🔴 同网 ARP 目的 IP；异网 ARP 默认网关。每跳帧的源/目的 MAC 改变，端到端 IP 通常不变。
- 🔵 先用 IP 与掩码 AND 判同网，再定下一跳，ARP 得下一跳 MAC，封装帧。
- ⚫ ARP 请求广播、应答通常单播且不跨路由器；NAT 会改变 IP/端口。

### 回忆卡 13｜IPv4 分片

先答：给定 MTU 和数据报长度，怎样求每片数据长度、偏移和 MF？

- 🔴 每片总长不超 MTU；除末片外数据长度为 8 B 倍数；偏移单位 8 B；非末片 MF=1，标识相同。
- 🔵 先减首部得每片最大数据，再向下取 8 的倍数；按累计数据字节/8 写偏移，逐片补首部长度。
- ⚫ 目的主机重组；IPv6 路由器不分片，首部总长度和片内数据长度不可混用。

### 回忆卡 14｜最长前缀匹配

先答：路由器怎样在多条匹配路由中选表项？默认路由何时使用？

- 🔴 目的 IP 与每条掩码相与，匹配者中选前缀最长；无更具体匹配才用 `0.0.0.0/0`。
- 🔵 逐项写“目的 AND 掩码＝网络？”再比较前缀长度，最后确定下一跳/接口。
- ⚫ 转发表选择不是按表项先后，也不是在不同前缀间直接选度量最小。

### 回忆卡 15｜RIP、OSPF、BGP

先答：RIP、OSPF、BGP 分属什么思想、用在什么范围？

- 🔴 RIP 是自治系统内距离向量、度量跳数；OSPF 是域内链路状态、用 Dijkstra；BGP 是自治系统间路径向量，重策略和可达性。
- 🔵 Dijkstra：初始化距离 → 每轮确定最小未定点 → 松弛邻边 → 记录前驱/下一跳。
- ⚫ Dijkstra 要求非负权；BGP 不以物理最短为唯一目标，RIP 的 16 跳表示不可达。

### 回忆卡 16｜ICMP 与排障

先答：ping、traceroute 为什么能工作？排障的自下而上顺序是什么？

- 🔴 ping 用 ICMP 回显；traceroute 利用 TTL 到 0 的超时报文定位逐跳路径。
- 🔵 链路 → IP/掩码/网关/DNS → ping 回环/本机/网关/公网 IP/域名 → ARP → 路由跟踪 → DNS → 端口。
- ⚫ 禁 ICMP 时 ping 失败不等于主机离线；ICMP 差错报文通常不再引发 ICMP 差错。

## 四、运输层与应用层

### 回忆卡 17｜UDP 与 TCP

先答：UDP 与 TCP 在连接、可靠性、边界和首部上有什么最小区别？

- 🔴 UDP 无连接、尽最大努力、保留报文边界、首部 8 B；TCP 面向连接、可靠有序、全双工、面向字节流。
- 🔵 选协议时按时延容忍、可靠/有序需求、应用是否自建机制和报文边界判断。
- ⚫ UDP 校验和不等于可靠交付；TCP 不保留应用消息边界，也不保证应用已经处理。

### 回忆卡 18｜三次握手与四次挥手

先答：给定初始序号 x、y，握手的 SYN/ACK 值是什么？为什么挥手常为四次？

- 🔴 握手：`SYN x` → `SYN y, ACK x+1` → `ACK y+1`；全双工两方向独立关闭，所以 FIN/ACK 常分四段。
- 🔵 时序题逐段标 flags、SEQ、ACK、数据长度；主动关闭方最后进入 TIME_WAIT 2MSL。
- ⚫ SYN、FIN 各占 1 个序号；纯 ACK 无数据且无 SYN/FIN 时通常不消耗序号。

### 回忆卡 19｜SEQ、ACK 与窗口

先答：怎样由段长计算下一 SEQ 和累计 ACK？发送窗口由什么限制？

- 🔴 下一序号＝本段 SEQ＋数据字节数＋SYN/FIN 占用；ACK=N 表示 N 前字节已收。实际发送窗口＝`min(rwnd,cwnd)`。
- 🔵 逐段维护“已发未确认区、下个期望字节、接收/拥塞窗口”，单位统一为字节或 MSS。
- ⚫ rwnd 是流量控制，cwnd 是拥塞控制；零窗口用持续计时器探测，避免更新丢失死锁。

### 回忆卡 20｜拥塞控制

先答：慢开始、拥塞避免、超时和三次重复 ACK 时 cwnd 怎样变化？

- 🔴 慢开始按 RTT 近似指数增，拥塞避免线性增；超时通常阈值减半并把 cwnd 降到较小值；三次重复 ACK 触发快速重传/恢复。
- 🔵 按题定初值、阈值和 TCP 版本逐 RTT 列 cwnd；带宽跑满所需窗口约为带宽×RTT。
- ⚫ Reno、Tahoe 等处理细节不同，不能混套；流量控制与拥塞控制目的不同。

### 回忆卡 21｜DNS、HTTP 与 HTTPS

先答：递归/迭代 DNS 如何区别？HTTP 无状态、持久连接与 HTTPS 分别意味着什么？

- 🔴 递归要求受托服务器继续查，迭代返回下一步线索；HTTP 请求—响应且无状态，持久连接复用连接；HTTPS 是 HTTP 经 TLS，提供加密、完整性和身份认证。
- 🔵 Web 时延题按 DNS 查询、建连、TLS、请求响应次数逐项计 RTT，并按是否并行/持久修正。
- ⚫ DNS 常用 UDP 53，但区传/大型响应可用 TCP；HTTPS 不保证网站业务内容可信。

### 回忆卡 22｜邮件、DHCP 与端口

先答：SMTP、POP3、IMAP、DHCP 各做什么？DORA 顺序是什么？

- 🔴 SMTP 发送/中继，POP3/IMAP 收取；DHCP 的 DORA 是 Discover、Offer、Request、ACK，使用 UDP 67/68。
- 🔵 端口最小集：SSH 22、SMTP 25、DNS 53、HTTP 80、POP3 110、IMAP 143、HTTPS 443。
- ⚫ 端口是默认约定可修改；客户端初始无 IP 时 DHCP 报文会使用广播。

### 回忆卡 23｜网络安全最小集

先答：防火墙、IDS、IPS、VPN 和数字签名分别提供什么，不提供什么？

- 🔴 防火墙按策略控流量；IDS 检测告警，IPS 可阻断；VPN 建受保护隧道；签名用私钥生成、公钥验证，保障来源与完整性。
- 🔵 案例按机密性、完整性、可用性定位目标，再选择边界、检测、加密、认证措施。
- ⚫ VPN 不自动保证终端安全；数字签名不提供内容保密。
