# 计算机网络：三色笔记

三色按考频和出题稳定性划分：红色优先背熟，蓝色用于补齐得分点，黑色用于防漏和边界判断。

## 🔴 高频考点

- 🔴 **网络分层**：OSI 七层和 TCP/IP 分层用于把复杂通信拆成层次职责。 常见考法：判断设备、协议属于哪一层。 易错边界：TCP 属传输层，IP 属网络层。
- 🔴 **IP 地址与子网划分**：子网掩码决定网络位和主机位。 常见考法：求网络地址、广播地址、可用主机数。 易错边界：可用主机数通常要减去网络地址和广播地址。
- 🔴 **TCP 与 UDP**：TCP 面向连接、可靠、有序；UDP 无连接、开销小。 常见考法：解释订单/收银同步为什么常选 TCP。 易错边界：TCP 可靠靠序号、确认、重传、滑动窗口等机制共同实现。
- 🔴 **常见网络命令**：`ping` 测连通性，`tracert/traceroute` 看路径，`ipconfig/ifconfig` 看地址。 常见考法：根据故障现象选排查命令。 易错边界：ping 不通不一定是网络断，可能被防火墙禁 ICMP。
- 🔴 **DNS 与 DHCP**：DNS 解析域名，DHCP 自动分配地址。 常见考法：判断上不了网、能 ping IP 不能访问域名的原因。 易错边界：DNS 失败不等于物理链路断。
- 🔴 **网络设备**：交换机主要工作在数据链路层，路由器连接不同网络。 常见考法：判断组网设备用途。 易错边界：家用路由器常集成交换、NAT、无线等功能。

## 🔵 中频考点

- 🔵 **HTTP/HTTPS**：HTTP 是应用层协议，HTTPS 在 HTTP 上加入 TLS 加密。 常见考法：判断网页访问、证书、加密传输。 易错边界：HTTPS 不保证网站内容一定真实可靠。
- 🔵 **NAT**：NAT 让内网地址复用公网地址访问外网。 常见考法：判断私有地址上网原理。 易错边界：NAT 不是安全防护的全部。
- 🔵 **VPN**：VPN 在公共网络上建立加密隧道。 常见考法：简答远程办公为什么用 VPN。 易错边界：VPN 不是加速器，本质是安全接入。
- 🔵 **无线网络**：Wi-Fi 受距离、遮挡、干扰和信道影响。 常见考法：判断无线不稳定原因。 易错边界：信号满格不代表吞吐一定高。
- 🔵 **网线标准**：双绞线按线序压接，常见 T568A/T568B。 常见考法：判断直通线、交叉线和水晶头线序。 易错边界：现代设备常支持自动翻转，但考试仍考标准线序。

## ⚫ 低频了解

- ⚫ **IPv6**：IPv6 地址空间更大，改进自动配置等能力。 常见考法：比较 IPv4 与 IPv6。 易错边界：IPv6 地址不是简单比 IPv4 多几个数字。
- ⚫ **等保 2.0**：强调安全技术和管理体系，覆盖云、大数据、物联网等场景。 常见考法：判断安全建设属于哪类要求。 易错边界：等保不是只买安全设备。
- ⚫ **网络拓扑**：星型、总线型、环型等描述节点连接方式。 常见考法：识别拓扑优缺点。 易错边界：现在局域网常见物理星型。

## 考前公式与简答

### 必会公式

- **IPv4主机数**：前缀长度为 `/p` 时，主机位数 `h=32-p`，传统子网可用主机数为 `2^h-2`。减去的两个地址通常是网络地址和广播地址；点到点等特殊场景应按题目约定。
- **传输时延**：`发送时延=数据量(bit)/链路速率(bit/s)`。数据量表示把全部比特推入链路所需规模；传播时延则是 `距离/传播速度`，两者物理含义不同。
- **端口与层次**：IP地址定位网络中的主机或接口，端口号定位主机上的应用进程，MAC地址用于本地链路交付。排障时不要把三者混为一种地址。

### 常见简答题

- **TCP为什么可靠？** TCP通过序号与确认、超时重传、校验、滑动窗口和流量/拥塞控制，使字节流尽量按序、无重复地交付；可靠不代表绝不失败，应用仍需处理断线和超时。
- **能访问IP却不能访问域名应怎样判断？** 物理链路和IP路由可能基本正常，应优先检查DNS服务器配置、域名解析结果和本地缓存；不能直接判定为网线故障。
