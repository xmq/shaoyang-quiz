# 信息安全：三色笔记

三色按考频和出题稳定性划分：红色优先背熟，蓝色用于补齐得分点，黑色用于防漏和边界判断。

## 🔴 高频考点

- 🔴 **CIA 三要素**：机密性、完整性、可用性是信息安全核心目标。 常见考法：按攻击后果判断损害哪一项。 易错边界：三者可能同时受影响。
- 🔴 **身份认证与访问控制**：认证确认是谁，授权决定能做什么。 常见考法：判断密码、验证码、权限控制。 易错边界：认证通过不代表拥有所有权限。
- 🔴 **加密与数字签名**：加密保护机密性，数字签名保护身份和完整性。 常见考法：区分对称加密、非对称加密、签名。 易错边界：公钥加密和私钥签名用途不同。
- 🔴 **恶意代码**：病毒、蠕虫、木马、勒索软件危害不同。 常见考法：按传播和行为分类。 易错边界：杀毒软件不是唯一防护手段。
- 🔴 **防火墙与入侵检测**：防火墙控制流量，IDS/IPS 发现或阻断攻击。 常见考法：按安全需求选设备。 易错边界：防火墙不能替代主机加固。
- 🔴 **备份与恢复**：备份降低数据丢失风险。 常见考法：判断勒索、误删后的应对。 易错边界：备份要测试可恢复性。

## 🔵 中频考点

- 🔵 **等保 2.0**：强调安全技术和安全管理并重。 常见考法：判断安全建设内容。 易错边界：等保不是一次测评后永久有效。
- 🔵 **网络攻击**：口令破解、钓鱼、DDoS、SQL 注入是常见攻击。 常见考法：按现象判断攻击类型。 易错边界：DDoS 主要损害可用性。
- 🔵 **安全策略**：最小权限、纵深防御、分区分域是常用原则。 常见考法：简答安全设计原则。 易错边界：方便性和安全性常需平衡。
- 🔵 **日志审计**：日志用于追踪行为和定位事件。 常见考法：判断日志用途。 易错边界：有日志不代表没人攻击。
- 🔵 **数据脱敏**：脱敏减少敏感数据暴露。 常见考法：判断身份证、手机号处理。 易错边界：脱敏后仍需控制访问。

## ⚫ 低频了解

- ⚫ **物理安全**：机房门禁、电源、消防属于基础安全。 常见考法：概念判断。 易错边界：信息安全不只发生在网络层。
- ⚫ **安全意识**：弱口令、随意点链接、外接 U 盘是常见风险。 常见考法：判断个人安全行为。 易错边界：技术措施无法替代所有管理要求。
- ⚫ **隐私保护**：个人信息处理应遵循最小必要等原则。 常见考法：判断是否过度收集。 易错边界：授权同意不等于可以无限使用。
