# 信息安全 · 三色回忆卡

> 先答后核对；答不出回讲义，答得犹豫进入复习。法规、标准和算法参数以题干版本为准。

## 一、安全目标、风险与权限

### 回忆卡 1｜CIA 与风险链

先答：CIA 保护什么？资产、威胁、脆弱性、攻击和风险怎样关联？

- 🔴 CIA 是机密性、完整性、可用性；威胁利用脆弱性危害资产，风险综合可能性与影响。
- 🔵 案例先写“资产 → 威胁 → 脆弱性 → CIA 影响”。
- ⚫ 有漏洞不等于已遭攻击，风险也通常不能绝对消除。

### 回忆卡 2｜处置与控制

先答：四种风险处置是什么？控制可怎样分类？

- 🔴 处置是规避、降低、转移、接受；控制可分管理/技术/物理，也可分预防/检测/响应/恢复。
- 🔵 每项措施对应一个风险，并写责任、验证和剩余风险。
- ⚫ 合规只是底线，设备若不配置和监测也不是有效控制。

### 回忆卡 3｜认证、授权与通用原则

先答：认证、授权、审计分别回答什么？MFA 和最小权限有何要求？

- 🔴 认证确认是谁，授权决定能做什么，审计记录做过什么；MFA 要跨所知/所有/所是类别。
- 🔵 权限按“默认拒绝 → 最小授权 → 职责分离 → 定期复核 → 离职回收”。
- ⚫ 两个口令仍是单因素；纵深防御要求多层控制而非多买产品。

## 二、恶意行为与密码技术

### 回忆卡 1｜恶意代码与钓鱼

先答：病毒、蠕虫、木马、勒索怎样区分？可疑消息如何核验？

- 🔴 病毒通常依附宿主；蠕虫可独立传播；木马靠伪装；勒索以锁定、加密或窃取数据勒索为目的。
- 🔵 停止操作 → 独立渠道核实主体和业务 → 不交口令/验证码 → 上报。
- ⚫ 熟人账号也可能被盗；HTTPS 不保证网站内容善意。

### 回忆卡 2｜加密、哈希与 MAC

先答：对称、非对称、哈希、MAC 分别做什么？

- 🔴 对称加密快但分发密钥难；非对称便于协商和签名但较慢；哈希生成不可逆摘要；MAC 用共享密钥鉴别完整性和来源。
- 🔵 混合加密先协商/封装会话密钥，再用对称算法加密数据。
- ⚫ Base64 是编码；哈希不能解密，RSA 不宜直接加密大文件。

### 回忆卡 3｜签名、证书与 TLS

先答：保密和签名各用谁的密钥？证书验证查什么？

- 🔴 保密通常接收者公钥加密、私钥解密；签名者私钥签、公钥验；证书绑定身份与公钥。
- 🔵 验证证书链、有效期、主体/域名、用途、撤销状态和信任锚。
- ⚫ 签名不自动保密，证书有效也不保证网站业务可信。

## 三、网络、应用与数据防护

### 回忆卡 1｜边界设备

先答：防火墙、IDS、IPS、VPN、DMZ 各解决什么问题？

- 🔴 防火墙控流；IDS 检测告警；IPS 可阻断；VPN 建隧道；DMZ 隔离对外服务与内网。
- 🔵 规则核对“源—目的—协议/端口—方向—动作—优先级—日志”。
- ⚫ VPN 不会修复终端、账号和应用漏洞；判断设备要看功能和部署。

### 回忆卡 2｜攻击与应用漏洞

先答：DoS、欺骗、中间人、SQL 注入、XSS、CSRF、越权各攻击哪里？

- 🔴 依次针对资源、身份/映射、通信、查询、浏览器脚本、登录态请求和服务端授权。
- 🔵 防护抓参数化查询、按上下文编码、Token/SameSite、逐请求鉴权、TLS 和日志。
- ⚫ 前端校验不能代替服务端控制，关键词过滤不能代替参数化。

### 回忆卡 3｜备份与恢复

先答：3-2-1、RPO、RTO 是什么？RAID 为什么不是备份？

- 🔴 3-2-1 是三份数据、两种介质、一份异地；RPO 是容忍丢失的数据时间，RTO 是容忍的恢复时间。
- 🔵 定目标 → 选备份方式 → 加密并离线/异地保存 → 定期恢复演练。
- ⚫ RAID 主要提高可用性，不能防误删、勒索和站点灾害。

## 四、事件响应、取证与合规

### 回忆卡 1｜响应闭环

先答：安全事件处置顺序是什么？初期最重要的动作有哪些？

- 🔴 准备 → 识别分析 → 遏制 → 根除 → 恢复 → 复盘。
- 🔵 分级上报、隔离资产、保留易失证据、阻断扩散并维持关键业务。
- ⚫ 未评估就关机可能丢失内存证据，应按预案决定断电或在线取证。

### 回忆卡 2｜取证与业务连续性

先答：如何证明数字证据未被篡改？业务连续性与灾难恢复有何区别？

- 🔴 取证强调授权、完整、可重复和保管链；前者关注业务持续，后者侧重 IT 系统与数据恢复。
- 🔵 记录人员、时间、地点、工具、哈希、原始介质和交接。
- ⚫ 截图没有来源和过程记录时证据能力有限；日志需统一时钟并防篡改。

### 回忆卡 3｜个人信息与案例

先答：个人信息案例怎样分析？取得同意是否一定合法？

- 🔴 处理须有合法基础，并遵循目的明确、最小必要、透明和安全保障；敏感信息条件更严。
- 🔵 主体与数据 → 行为目的 → 合法基础/权限 → 义务 → 影响 → 整改验证。
- ⚫ 同意不是唯一合法基础，也不是无限授权；法律和等保题按题干版本作答。
