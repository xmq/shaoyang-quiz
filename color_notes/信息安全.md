# 信息安全·期末复习三色笔记

> 只用于考前速记：🔴核心必背；🔵公式、流程、辨析与案例踩分词；⚫适用条件、绝对化陷阱和版本差异。涉及法律法规、标准或算法参数时，以题干指定版本和任课教师教材为准。

## 一、信息安全目标与基本概念

- 🔴 CIA 三要素：机密性 Confidentiality—防未授权泄露；完整性 Integrity—防未授权篡改/破坏；可用性 Availability—授权主体需要时可获得服务。
- 🔴 扩展目标：真实性、可控性、可审计/可追溯性、不可否认性；不同教材分类层次可能不同。
- 🔴 资产有价值；威胁是可能造成损害的因素；脆弱性是可被利用的弱点；攻击是威胁主体利用脆弱性的行为；风险综合可能性与影响。
- 🔵 常用风险表达：`风险 ≈ 发生可能性 × 影响程度`；风险评估还要识别资产、威胁、脆弱性、现有控制和剩余风险。
- 🔴 风险处置：规避、降低/缓解、转移、接受；任何处置都应明确责任人、期限和剩余风险。
- 🔴 最小权限：主体只获得完成任务所需最少权限；纵深防御：人员、终端、网络、应用、数据和运营部署相互补充控制。
- ⚫ “有漏洞”不等于已经被攻击；安全不是一次性产品，而是持续的治理、技术和运营过程。

## 二、安全管理与控制类型

- 🔴 安全控制可分管理、技术、物理控制；按作用可分预防、检测、响应、恢复和威慑/纠正控制。
- 🔴 安全策略规定目标和原则；标准给出强制要求；流程规定执行步骤；指南提供建议。
- 🔵 安全治理闭环：识别资产/责任 → 评估风险 → 制定控制 → 实施运行 → 监测审计 → 响应改进。
- 🔴 职责分离避免关键业务由一人独立完成；双人控制、审批复核和岗位轮换可降低舞弊风险。
- 🔴 最小化攻击面：关闭无用端口/服务、删除默认账号、及时补丁、基线配置、应用白名单和权限收敛。
- ⚫ 合规只是安全底线，不代表没有风险；购买安全设备但不配置、不监控、不维护，不能形成有效控制。

## 三、恶意代码

- 🔴 病毒通常依附宿主并具有复制/传染能力；蠕虫可独立运行并利用网络或漏洞自行传播；木马靠伪装诱导执行，定义重点不是自我复制。
- 🔴 勒索软件通过加密、锁定或窃取数据实施勒索；可借木马、漏洞、钓鱼或蠕虫能力传播，不等于所有勒索软件都是蠕虫。
- 🔴 后门绕过正常控制提供隐蔽访问；逻辑炸弹在特定条件触发；Rootkit 隐藏恶意活动并维持高权限；僵尸网络由受控设备组成。
- 🔴 宏病毒利用 Office 等应用宏机制传播；现代含宏格式常见 `.docm/.xlsm`，旧格式也可能含宏。
- 🔵 恶意代码处置：隔离主机 → 保留证据/判断范围 → 阻断传播与控制通道 → 清除或重装 → 修补入口 → 恢复验证 → 持续监测复盘。
- ⚫ 杀毒软件可能误报、漏报且依赖更新；“装了杀毒就绝对安全”错误，未经验证直接解密/交赎金也不能保证恢复。

## 四、社会工程与常见攻击入口

- 🔴 网络钓鱼冒充可信主体诱骗凭据、付款或执行恶意程序；鱼叉式钓鱼针对特定个人/组织；捕鲸面向高层或关键人员。
- 🔴 社会工程常利用权威、紧迫、恐惧、好奇、贪利和从众心理；形式包括钓鱼、冒充客服、尾随、诱饵、垃圾搜寻和肩窥。
- 🔵 可疑消息核验：停下操作 → 用独立可信渠道联系对方 → 核验域名/账号/业务背景 → 不直接打开附件或提供口令/验证码 → 上报。
- 🔴 常见入口：弱口令、凭据复用、未修补漏洞、恶意附件/链接、盗版软件、移动介质、暴露远程服务和供应链。
- 🔵 防护踩分词：安全意识、MFA、邮件过滤、最小权限、补丁、应用控制、宏限制、备份、日志监测。
- ⚫ 来自熟人账号的消息也可能由账号被盗产生；HTTPS 只表示传输加密和证书验证，不保证网站业务内容善意。

## 五、身份认证、授权与审计

- 🔴 Authentication 认证回答“你是谁”；Authorization 授权回答“你能做什么”；Accounting/Auditing 审计记录“你做过什么”。
- 🔴 认证因素：所知（口令/PIN）、所有（令牌/手机/密钥）、所是（指纹/人脸）；多因素应来自不同因素类别。
- 🔴 口令应长、唯一、不复用，服务端应使用带盐的专用口令哈希算法存储；盐不必保密，作用是抵抗预计算和彩虹表。
- 🔴 DAC 自主访问控制由资源所有者授权；MAC 强制访问控制依据安全标签；RBAC 按角色授予权限；ABAC 按主体、资源、动作和环境属性决策。
- 🔵 账号生命周期：申请 → 身份核验 → 审批授权 → 发放 → 定期复核 → 变更 → 停用/注销 → 审计留痕。
- 🔵 权限设计踩分词：默认拒绝、最小权限、职责分离、按需/按时授权、定期复核、离职立即回收。
- ⚫ 两个口令仍属于同一因素，不构成真正 MFA；生物特征也可能被仿冒且泄露后难以更换。

## 六、密码学基础

- 🔴 明文经加密算法和密钥变为密文，密文经解密恢复；现代密码系统的安全性应主要依赖密钥而非算法保密。
- 🔴 对称密码使用同一或可相互推导的共享密钥，速度快，适合大量数据，难点是密钥安全分发；典型算法 AES。
- 🔴 非对称密码使用公钥/私钥对，便于密钥协商和数字签名但开销较大；典型体系 RSA、ECC。
- 🔴 混合加密通常用公钥机制协商/封装会话密钥，再用对称算法加密大量业务数据。
- 🔴 哈希把任意长度输入映射成定长摘要，目标包括单向性、抗原像、抗第二原像和抗碰撞；哈希不是可逆加密。
- 🔵 完整性鉴别：共享密钥环境常用 MAC/HMAC；公开验证、来源认证和不可否认需求常用数字签名。
- ⚫ Base64 是编码不是加密；MD5、SHA-1 已不适合抗碰撞安全用途；RSA 不宜直接加密大文件。

## 七、数字签名、证书与 PKI

- 🔴 数字签名：发送者用签名私钥对消息摘要相关数据生成签名，接收者用相应公钥验证。
- 🔴 签名主要提供来源认证、完整性和一定条件下的不可否认性，不自动提供机密性。
- 🔵 保密方向：通常用接收者公钥加密/封装，接收者私钥解密；签名方向：签名者私钥生成，签名者公钥验证。
- 🔴 数字证书把主体身份与公钥等信息绑定，由 CA 签发；PKI 还包含注册、证书库、策略以及撤销/状态服务。
- 🔵 证书验证：验证签名链 → 有效期 → 主体/域名 → 密钥用途 → 撤销状态 → 信任锚。
- 🔴 HTTPS/TLS 通过证书认证服务器，并协商会话密钥保护传输机密性和完整性；具体握手随 TLS 版本和密码套件变化。
- ⚫ 证书有效不代表网站内容可信；签名私钥泄露会破坏身份可信度，需及时吊销并更换。

## 八、网络边界与安全设备

- 🔴 防火墙依据策略控制区域/主机之间通信，可进行包过滤、状态检测、代理和应用识别；默认拒绝并按需放行是常见安全思路。
- 🔴 IDS 侧重检测告警；IPS 通常串联并可实时阻断；误报是把正常行为判成攻击，漏报是未发现真实攻击。
- 🔴 VPN 在不可信网络上建立受保护隧道，常分远程接入和站点到站点；VPN 不会自动消除终端、账号和应用漏洞。
- 🔴 DMZ 用于放置对外服务并与内网分区；网络分段和零信任思路限制横向移动，强调持续验证和最小权限。
- 🔵 防火墙规则检查：源地址 → 目的地址 → 协议/端口 → 方向/接口 → 动作 → 优先级 → 日志 → 业务验证。
- ⚫ 路由器可集成 ACL/防火墙功能，下一代防火墙可集成 IPS；设备名称不如实际功能、部署位置和策略重要。

## 九、常见网络攻击

- 🔴 DoS 消耗带宽、连接或计算资源使服务不可用；DDoS 使用多个分布式来源。
- 🔴 中间人攻击介入通信进行窃听或篡改；正确验证证书和使用 TLS 可降低风险。
- 🔴 嗅探截获网络流量；欺骗可伪造 IP、ARP、DNS 等身份/映射；会话劫持夺取已建立会话。
- 🔴 端口扫描用于发现开放服务，本身可用于授权运维或攻击侦察，性质取决于授权与目的。
- 🔵 网络攻击案例：资产/业务 → 攻击入口 → 利用的弱点 → CIA 影响 → 预防 → 检测 → 响应恢复。
- ⚫ 加密流量提高机密性也可能降低中间设备可见性；防火墙无法包治合法账号滥用、绕行路径和允许流量中的所有攻击。

## 十、Web 与应用安全

- 🔴 SQL 注入源于把不可信输入拼接进 SQL；核心防护是参数化查询/预编译，并配合输入校验、最小数据库权限和错误信息控制。
- 🔴 XSS 使恶意脚本在受害者浏览器执行，分存储型、反射型和 DOM 型；核心防护是按输出上下文编码，并配合 CSP、安全 Cookie 和输入处理。
- 🔴 CSRF 利用用户已登录状态诱使浏览器发起非预期请求；防护包括 CSRF Token、SameSite Cookie、重新认证和来源校验。
- 🔴 越权包括水平越权访问同级他人资源和垂直越权获得更高权限；每次请求都必须在服务端检查授权。
- 🔴 缓冲区溢出、命令注入、路径遍历、不安全反序列化和文件上传也是常见漏洞类型。
- 🔵 安全开发生命周期：需求威胁建模 → 安全设计 → 规范编码 → 代码审查/测试 → 安全部署 → 漏洞响应维护。
- ⚫ 只做前端校验不能防攻击；转义方式依 HTML、属性、JavaScript、URL 等输出上下文而不同；过滤关键词不能替代参数化。

## 十一、主机、数据与备份安全

- 🔴 主机加固：补丁更新、关闭无用服务、最小权限、强认证、端点防护、磁盘加密、安全基线、日志和配置审计。
- 🔴 数据全生命周期：收集、存储、使用/加工、传输、提供、公开、归档和删除，各阶段都需分类分级和访问控制。
- 🔴 备份 3-2-1：至少 3 份数据、2 种介质、1 份异地；现代勒索场景还应保留离线或不可变副本并测试恢复。
- 🔴 RPO 恢复点目标表示可容忍丢失多少时间的数据；RTO 恢复时间目标表示中断后多快恢复服务。
- 🔵 备份方案：识别关键数据 → 定 RPO/RTO → 选全量/增量/差异备份 → 加密与异地/离线保存 → 定期恢复演练。
- ⚫ RAID 提高可用性但不能替代备份；有备份文件不等于可恢复，必须验证完整性、密钥和恢复流程。

## 十二、安全事件响应与取证

- 🔴 事件响应常见阶段：准备 → 识别/分析 → 遏制 → 根除 → 恢复 → 复盘改进。
- 🔵 初期处置：确认告警 → 分级上报 → 隔离受影响资产 → 保留易失证据 → 阻断扩散 → 维护业务连续性。
- 🔴 数字取证强调合法授权、证据完整、过程可重复和保管链；采集时记录人员、时间、地点、工具、哈希和交接。
- 🔵 日志分析关联账号、源/目的地址、时间、动作和结果；应统一时钟并保护日志免遭篡改。
- 🔴 业务连续性关注关键业务持续运行；灾难恢复侧重 IT 系统和数据恢复，二者相关但不相同。
- ⚫ 发现入侵后立即关机可能丢失内存等易失证据；是否断电、隔离或在线取证应按预案、风险和专业指挥决定。

## 十三、个人信息、法律与职业伦理

- 🔴 个人信息是与已识别或可识别自然人有关的各种信息，匿名化处理后的信息不再属于个人信息；去标识化仍可能重新识别。
- 🔴 处理个人信息应遵循合法、正当、必要、诚信、目的明确、最小影响、公开透明、质量和安全保障等要求。
- 🔴 敏感个人信息处理通常要求特定目的、充分必要和严格保护，并依法履行告知、单独同意等义务。
- 🔴 高频法律框架：《网络安全法》《数据安全法》《个人信息保护法》《密码法》《著作权法》及计算机软件保护相关规定。
- 🔴 等级保护强调定级、备案、建设整改、等级测评和监督检查；具体对象、等级与要求按现行标准和题干版本作答。
- 🔵 法律案例答题：主体 → 行为/数据 → 权限或合法基础 → 安全义务 → 影响后果 → 整改措施；不要只写“违法/合法”。
- ⚫ 同意不是个人信息处理的唯一合法基础；法律和标准会修订，具体日期、条文、罚则和等级要求必须核对题干版本。

## 十四、综合案例题统一框架

- 🔵 第一步定资产：数据、账号、设备、服务、声誉和人员安全。
- 🔵 第二步定问题：威胁利用何种脆弱性，经什么入口发生，影响 CIA 哪些目标。
- 🔵 第三步分层控制：人员培训 + 身份权限 + 终端补丁 + 网络分区 + 应用安全 + 数据加密备份 + 日志运营。
- 🔵 第四步写闭环：预防、检测、响应、恢复，并给责任人、证据和验证方法。
- 🔵 高分措辞：最小权限、默认拒绝、纵深防御、多因素、参数化、分区隔离、离线备份、持续监测、复盘整改。
- ⚫ 只罗列产品名通常得分低；措施必须对应已识别的威胁和薄弱点，同时说明局限与剩余风险。
