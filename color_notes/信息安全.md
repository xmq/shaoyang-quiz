# 信息安全基础 · 三色回忆卡

> 先答后核对，重点提取定义、分类、作用、区别和判断条件；法律与标准以题干版本为准。

## 一、安全目标、风险与控制

### 回忆卡 1｜CIA三要素

先答：机密性、完整性、可用性分别防什么问题？

- 🔴 机密性防未授权泄露，完整性防未授权篡改破坏，可用性保证授权主体需要时能使用信息和服务。
- 🔵 案例先判断是“看到了、改坏了、用不了”中的哪一类影响。
- ⚫ 信息安全不只保护机密性，三项目标也可能同时受损。

### 回忆卡 2｜资产、威胁、脆弱性与风险

先答：五个概念怎样组成基本关系？

- 🔴 资产有价值，威胁可能造成损害，脆弱性是弱点，攻击利用弱点，风险综合发生可能性和影响。
- 🔵 案例按“资产 → 威胁 → 脆弱性 → CIA影响”作答。
- ⚫ 有漏洞不等于已遭攻击，没有发现攻击也不等于没有风险。

### 回忆卡 3｜控制与原则

先答：安全控制可怎样分类？最小权限、默认拒绝和纵深防御是什么意思？

- 🔴 控制可分管理/技术/物理，也可分预防/检测/响应/恢复；三项原则分别限制权限、只放行明确允许项、设置多层措施。
- 🔵 防护题按人员、制度、终端、网络、数据分层写。
- ⚫ 合规只是底线，安全也不是买一台设备就完成。

### 回忆卡 4｜等保2.0

先答：等保五级怎样理解？基础工作流程是什么？为什么不是一次认证？

- 🔴 保护对象按受破坏后的危害程度分一至五级，等级越高要求越严；流程为定级、备案、建设整改、等级测评、监督检查与持续改进。
- 🔵 简答先写“按危害定级”，再按流程逐项说明目的。
- ⚫ 测评通过不代表永久合规；业务、系统或风险变化后仍需复查、整改并按要求测评。

## 二、恶意代码、社会工程与身份

### 回忆卡 1｜恶意代码辨析

先答：病毒、蠕虫、木马、勒索软件最关键的区别是什么？

- 🔴 病毒通常依附宿主并复制，蠕虫可独立传播，木马靠伪装诱导运行，勒索以锁定、加密或窃取数据为目的。
- 🔵 按“是否依附—是否自传播—进入方式—主要目的”比较。
- ⚫ 勒索软件可借木马或蠕虫方式传播，攻击目的与传播方式不能混同。

### 回忆卡 2｜社会工程与基本防护

先答：可疑邮件或转账消息怎样核验？日常防护有哪些？

- 🔴 停止操作，用独立渠道核实身份业务，不交密码验证码并上报；同时做好补丁、可信安装、终端防护、MFA和备份。
- 🔵 简答从人员意识、账号、软件、设备和数据五方面写。
- ⚫ 熟人账号也可能被盗，杀毒软件也不能保证发现所有威胁。

### 回忆卡 3｜认证、授权与审计

先答：三者分别回答什么？两个口令是否构成多因素？

- 🔴 认证确认是谁，授权决定能做什么，审计记录做过什么；MFA要跨所知、所有、所是类别。
- 🔵 权限管理写申请、审批、最小授权、复核、变更和注销。
- ⚫ 通过认证不等于能访问全部资源，两个口令仍是同一因素。

## 三、密码技术基础

### 回忆卡 1｜对称与非对称加密

先答：两者在密钥、速度和用途上有何区别？

- 🔴 对称加密共享密钥、速度快、适合大量数据但分发难；非对称使用公私钥，便于协商和签名但较慢。
- 🔵 区别题按“密钥关系—速度—用途—管理难点”写。
- ⚫ 非对称加密并非在任何场景都优于对称加密。

### 回忆卡 2｜哈希与编码

先答：哈希用于什么？它与加密、Base64有何不同？

- 🔴 哈希把数据映射成固定长度摘要，常用于完整性和口令保护；加密可配合密钥恢复明文，Base64只是编码。
- 🔵 判断时问“是否可逆、是否使用密钥、目的是什么”。
- ⚫ 不能解密哈希摘要恢复原文，也不能把Base64当安全保护。

### 回忆卡 3｜签名与证书

先答：数字签名通常怎样生成和验证？证书绑定什么？

- 🔴 签名者用私钥生成签名，接收者用对应公钥验证；证书把主体身份与公钥等信息绑定。
- 🔵 签名作用写来源认证、完整性和防抵赖。
- ⚫ 数字签名本身不提供内容保密，证书有效也不代表网站内容绝对可信。

## 四、网络、主机与数据安全

### 回忆卡 1｜防火墙、IDS、IPS、VPN

先答：四者各自的基本作用是什么？

- 🔴 防火墙按规则控流，IDS检测告警，IPS可检测并阻断，VPN建立受保护通信通道。
- 🔵 区别设备时写“部署位置—主要动作—是否主动阻断”。
- ⚫ VPN不能修复终端病毒和弱口令，IDS与IPS也不是完全相同。

### 回忆卡 2｜常见网络攻击

先答：DoS/DDoS、中间人、欺骗和口令攻击分别做什么？

- 🔴 DoS消耗资源使服务不可用；中间人介入通信；欺骗伪造身份或映射；口令攻击猜测、窃取或复用凭据。
- 🔵 案例写攻击对象、利用弱点、CIA影响和对应防护。
- ⚫ 扫描等工具既可合法管理也可用于攻击，关键看授权和目的。

### 回忆卡 3｜备份与数据保护

先答：3-2-1备份是什么？全量、增量、差异备份怎样区别？

- 🔴 3-2-1是三份数据、两种介质、一份异地；全量备全部，增量备上次备份后变化，差异备上次全量后变化。
- 🔵 数据防护写分类、权限、加密、备份、审计和恢复验证。
- ⚫ RAID主要提高可用性，不能代替备份；有备份也不等于一定能恢复。

### 回忆卡 4｜事件响应与灾备

先答：安全事件响应的闭环是什么？它与灾难恢复、业务连续性有何区别？

- 🔴 闭环是准备、发现报告、分析确认、遏制、根除、恢复、复盘；灾难恢复重系统数据恢复，业务连续性重关键业务维持。
- 🔵 处置题先控影响，再清原因、验证恢复，最后留记录并改进。
- ⚫ 恢复备份只是一个环节，不等于完成事件响应或完整灾备。

## 五、基础答题与判断纠错

### 回忆卡 1｜定义分类题

先答：安全概念定义和恶意代码分类题怎样写？

- 🔴 定义题写本质、对象和作用；分类题先说明依据，再写名称与特点。
- 🔵 恶意代码可按依附、传播和目的比较，控制可按性质或作用分类。
- ⚫ 只罗列名词而不解释分类依据和特点，得分点不完整。

### 回忆卡 2｜区别与防护题

先答：比较密码技术或安全设备、防护措施题怎样组织？

- 🔴 区别按定义、作用、对象和关键差异；防护按人员、制度、终端、网络和数据分层。
- 🔵 每项措施对应一个具体威胁或弱点，避免空写“加强管理”。
- ⚫ 不能把任何单一产品写成“绝对安全、彻底防止”。

### 回忆卡 3｜高频判断陷阱

先答：为什么“漏洞=已攻击、两个口令=MFA、签名=保密、RAID=备份”都错？

- 🔴 漏洞只是弱点；两个口令同属所知；签名主要验来源完整性；RAID主要提高可用性。
- 🔵 判断题指出错在何处，再写一条正确表述。
- ⚫ 看到“所有、绝对、完全”要核对条件，但不能只凭措辞机械判错。
