# 多媒体技术 · 三色回忆卡

> 先答后核对。容量题先声明 `1 B=8 bit`，KB/KiB 按题干采用 1000 或 1024；软件名称和菜单以指定版本为准。

## 一、多媒体与数字化

### 回忆卡 1｜多媒体与超媒体

先答：多媒体的核心特征是什么？超文本与超媒体差在哪里？

- 🔴 多媒体集成文本、图形图像、音频、动画和视频，强调数字化、集成与交互；超媒体把超文本链接对象扩展到多种媒体。
- 🔵 简答按“媒体种类—集成方式—交互—应用”输出。
- ⚫ 同时播放声音和图片不必然构成有意义的交互式多媒体系统。

### 回忆卡 2｜采样、量化、编码

先答：模拟信号数字化的三个环节分别改变什么？

- 🔴 采样离散时间，量化离散幅值，编码把量化结果表示成数字码。
- 🔵 n 位量化有 `2^n` 个级；带限信号理想抽样满足 `fs≥2fmax`。
- ⚫ 采样率和位深提高会增加数据量，也不能恢复源信号中已经丢失的信息。

### 回忆卡 3｜容量计算总入口

先答：图像、PCM 音频、未压缩视频和码率视频分别看哪些量？

- 🔴 图像看像素×位深；音频看采样率×位深×通道×时间；未压缩视频再乘帧率；压缩视频直接用总码率×时间。
- 🔵 最后统一单位并除以 8，从 bit 转 B。
- ⚫ 已给总码率时通常不再乘像素、位深和帧率。

## 二、数字图像与颜色

### 回忆卡 1｜位图、矢量与分辨率

先答：位图和矢量图缩放时为什么表现不同？

- 🔴 位图由像素组成，放大会显像素；矢量图由几何描述组成，可重新计算绘制。
- 🔵 位图原始容量约为 `宽×高×位深÷8`；n 位色深最多表示 `2^n` 种颜色。
- ⚫ PPI 描述像素密度，DPI 常描述输出设备点密度，不能机械等同。

### 回忆卡 2｜颜色模型

先答：RGB、CMYK、HSV/HSB、YCbCr 各适合什么场景？

- 🔴 RGB 用于发光显示；CMYK 用于印刷；HSV/HSB 便于按色相饱和度亮度调色；YCbCr 分离亮度与色度，利于视频压缩。
- 🔵 选模型时先说输出介质，再说模型优势。
- ⚫ 屏幕 RGB 色彩不保证印刷后完全一致，二者色域不同。

### 回忆卡 3｜图像格式与非破坏编辑

先答：JPEG、PNG、GIF、SVG、PSD 的关键区别是什么？

- 🔴 JPEG 有损且适合照片；PNG 无损并支持透明；GIF 颜色有限并可简单动画；SVG 是矢量；PSD 保留图层编辑结构。
- 🔵 非破坏流程：保留原图 → 智能对象/调整图层 → 蒙版 → 分层保存 → 按用途导出。
- ⚫ JPEG 通常不保留透明和图层；导出格式不能代替可编辑源文件。

## 三、音频、动画与视频

### 回忆卡 1｜数字音频

先答：音频质量和未压缩容量由哪四项决定？WAV、MP3、FLAC、MIDI 有何区别？

- 🔴 关键量是采样率、位深、通道和时间；MP3 常有损，FLAC 无损，WAV 是容器且常装 PCM，MIDI 记录演奏事件而非声波。
- 🔵 PCM 容量=`采样率×位深×通道×时间÷8`。
- ⚫ WAV 不必然无压缩，MIDI 也不是录音。

### 回忆卡 2｜动画

先答：逐帧、运动补间和形状补间分别适合什么变化？

- 🔴 逐帧逐个制作画面；运动补间改变对象位置、缩放等；形状补间用于矢量形状变化。
- 🔵 时长=`总帧数÷帧率`，总帧数=`时长×帧率`。
- ⚫ 关键帧不等于每一帧都有新图像；帧率越高也会增加计算和容量。

### 回忆卡 3｜数字视频

先答：编码器与容器为什么不能混为一谈？I、P、B 帧有何核心关系？

- 🔴 H.264/H.265/AV1 等编码器决定压缩方法；MP4/MKV/MOV 等容器封装媒体；I 帧可独立解码，P/B 帧利用其他帧预测。
- 🔵 压缩视频容量=`(视频码率+音频码率)×时间÷8`。
- ⚫ `.mp4` 不唯一决定编码方式；分辨率相同也不代表质量和容量相同。

## 四、压缩、编码与传输

### 回忆卡 1｜有损与无损

先答：什么内容必须无损？什么内容通常可以有损？

- 🔴 无损可还原原数据，适合文本、程序和中间素材；有损利用感知冗余换更高压缩比，常用于照片、音频和视频分发。
- 🔵 压缩比常写 `原始量/压缩后量`，代入前先确认题干定义。
- ⚫ 有损文件反复转码会继续损失，文件变大也不会恢复真实细节。

### 回忆卡 2｜JPEG 与视频压缩

先答：JPEG 的主要流程是什么？视频为何还要做帧间预测？

- 🔴 JPEG 常经颜色变换/抽样、分块、DCT、量化、熵编码；视频再利用相邻帧的时间冗余。
- 🔵 量化是 JPEG 主要有损环节；熵编码减少统计冗余。
- ⚫ DCT 本身不是主要丢失来源，不能把所有压缩都说成有损。

### 回忆卡 3｜流媒体质量

先答：流媒体卡顿应检查哪些指标？自适应码率做了什么权衡？

- 🔴 关键指标有可用吞吐、启动时延、缓冲、抖动和丢包；自适应码率按网络状况切换清晰度以维持播放。
- 🔵 排查顺序：源文件/编码 → 服务器 → 网络吞吐与时延 → 缓冲策略 → 终端解码。
- ⚫ 标称带宽不等于持续可用吞吐，码率越高也不保证主观质量一定更好。

## 五、作品设计、交互与发布

### 回忆卡 1｜制作流程

先答：从需求到发布，多媒体作品要经过哪些检查点？

- 🔴 需求与受众 → 脚本/结构 → 素材 → 集成制作 → 测试修改 → 编码发布 → 维护。
- 🔵 每阶段保存可编辑源文件、版本、素材来源和输出参数。
- ⚫ 先堆素材再决定目标，容易造成结构混乱和认知负荷。

### 回忆卡 2｜交互与界面

先答：怎样判断一个交互设计是否可用，而不只看“好看”？

- 🔴 看导航一致、状态可见、反馈及时、错误可恢复、文字可读和无障碍支持。
- 🔵 测试用典型任务记录完成率、错误点、时间和用户反馈，再迭代。
- ⚫ 动画越多不等于交互越好；装饰信息可能妨碍理解。

### 回忆卡 3｜版权、隐私与发布

先答：发布前为什么必须检查授权、隐私、兼容和可访问性？

- 🔴 素材要有合法来源与授权，个人信息遵循最小必要；同时检查字体、链接、字幕、编码和多设备兼容。
- 🔵 发布清单：版权/隐私 → 文件打包 → 链接 → 清晰度/码率 → 字幕/替代文本 → 多端测试。
- ⚫ 网上可下载不等于可商用；压缩、裁剪或改色不会自动消除著作权义务。
