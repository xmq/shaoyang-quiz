# 操作系统原理：三色笔记

三色按考频和出题稳定性划分：红色优先背熟，蓝色用于补齐得分点，黑色用于防漏和边界判断。

## 🔴 高频考点

- 🔴 **进程与程序**：程序是静态代码，进程是程序一次运行的动态实体。 常见考法：简答两者区别。 易错边界：同一程序可产生多个进程。
- 🔴 **进程状态与调度**：就绪、运行、阻塞是核心状态，调度决定谁使用 CPU。 常见考法：判断状态转换、FCFS/SJF/时间片轮转。 易错边界：阻塞是等待事件，不是等待 CPU。
- 🔴 **同步与互斥**：互斥保护临界资源，同步协调执行顺序。 常见考法：判断信号量 P/V、生产者消费者。 易错边界：互斥是同步的一种特殊关系。
- 🔴 **死锁**：互斥、占有并等待、不可剥夺、循环等待四条件同时满足可能死锁。 常见考法：判断死锁条件和避免办法。 易错边界：资源不足不一定就是死锁。
- 🔴 **内存管理**：分页、分段、虚拟存储解决程序装入和地址转换。 常见考法：求页号、页内偏移、缺页次数。 易错边界：页号通常从 0 开始。
- 🔴 **文件系统**：文件系统负责文件命名、组织、权限和存取。 常见考法：判断目录、路径、权限。 易错边界：文件扩展名不等于文件真实格式。

## 🔵 中频考点

- 🔵 **线程**：线程是进程内的执行流，共享进程资源。 常见考法：比较进程与线程。 易错边界：线程切换通常比进程轻量，但同步更复杂。
- 🔵 **中断**：中断让 CPU 响应外部或内部事件。 常见考法：解释 I/O 为什么需要中断。 易错边界：中断不是程序错误的同义词。
- 🔵 **I/O 管理**：缓冲、缓存、设备驱动提高 I/O 效率。 常见考法：判断缓冲作用。 易错边界：缓存和缓冲概念相近但侧重点不同。
- 🔵 **页面置换**：FIFO、LRU 等算法处理内存不足。 常见考法：按访问序列求缺页。 易错边界：物理块数增加不一定所有算法缺页都减少。
- 🔵 **用户态与内核态**：内核态可执行特权操作，用户态受限制。 常见考法：判断系统调用。 易错边界：应用不能直接操作硬件核心资源。

## ⚫ 低频了解

- ⚫ **作业管理**：早期批处理系统常考作业周转时间。 常见考法：计算平均周转时间。 易错边界：周转时间从提交到完成，不只运行时间。
- ⚫ **磁盘调度**：FCFS、SSTF、SCAN 影响寻道效率。 常见考法：判断磁头移动顺序。 易错边界：离当前最近不一定整体最优。
- ⚫ **操作系统类型**：批处理、分时、实时、嵌入式各有目标。 常见考法：按场景判断系统类型。 易错边界：实时系统强调响应时间确定性。
