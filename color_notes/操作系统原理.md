# 操作系统原理 · 三色回忆卡

> 先答后核对。🔴记结论；🔵记简答顺序；⚫记判断纠错。

## 一、系统与接口

### 回忆卡 1｜定义与作用

先答：操作系统是什么？五项主要作用是什么？


- 🔵 操作系统是管理软硬件资源、控制程序运行并提供服务的系统软件；作用是处理机、内存、文件、设备管理和提供接口。
- 🔴 简答按“定义 → 管理对象 → 五项作用”分点写。
- ⚫ 操作系统不是普通应用软件，也不只负责管理硬件。
### 回忆卡 2｜四个特征

先答：并发、共享、虚拟、异步分别表示什么？


- ⚫ 并发是多个程序在一段时间共同推进；共享是共同使用资源；虚拟是把物理资源抽象扩展；异步是以不可预知但受控速度推进。
- 🔵 辨析并发/并行时先看“一段时间”还是“同一时刻”。
- 🔴 单核也能并发，但同一时刻通常只能执行一个指令流。
### 回忆卡 3｜系统类型与接口

先答：批处理、分时、实时系统分别强调什么？系统调用与中断/异常怎样区分？


- ⚫ 批处理重吞吐，分时重交互，实时重规定时间响应；系统调用是程序主动请求服务，中断多来自外部，异常由当前指令引起。
- 🔵 类型题抓“成批、多人交互、截止时间”三个关键词。
- 🔴 库函数不一定触发系统调用；实时不等于速度绝对最快。
## 二、进程、线程与调度

### 回忆卡 4｜程序、进程、线程

先答：程序、进程、线程的最小区别是什么？


- 🔵 程序是静态指令集合；进程是程序执行过程和资源分配单位；线程是进程内执行流和 CPU 调度单位。
- 🔴 比较按“动静态 → 资源单位 → 调度单位 → 共享内容 → 切换开销”回答。
- ⚫ 同进程线程共享地址空间和文件，但各有 PC、寄存器和栈。
### 回忆卡 5｜三种基本状态

先答：就绪、运行、阻塞分别缺什么条件？怎样转换？


- 🔵 就绪只缺 CPU，运行正占用 CPU，阻塞在等待事件；阻塞事件完成后先到就绪，调度后才到运行。
- 🔴 看状态题先问“等 CPU 还是等事件”。
- ⚫ 阻塞不能直接变运行；时间片到通常使运行转就绪。
### 回忆卡 6｜调度方法

先答：FCFS、SJF、优先级和 RR 各按什么规则选进程？


- 🔵 FCFS 按到达，SJF 选短作业，优先级法选高优先级，RR 按时间片轮转。
- 🔴 简答写“选择规则＋主要优点＋主要不足＋适用系统”。
- ⚫ SJF/优先级可能饥饿；RR 时间片过大接近 FCFS，过小会增加切换开销。
## 三、同步与死锁

### 回忆卡 7｜同步、互斥与信号量

先答：同步、互斥、临界资源和 P/V 操作分别是什么？


- 🔵 同步约束先后，互斥禁止同时访问临界资源；P 用于申请/等待，V 用于释放/通知。
- 🔴 判断题先分清“次序不对”还是“同时进入”。
- ⚫ 互斥信号量常为 1，资源信号量初值常等于资源数，但应服从题设。
### 回忆卡 8｜死锁

先答：死锁定义、四个必要条件和四类处理方法是什么？


- ⚫ 死锁是进程因资源竞争而永久等待；条件为互斥、请求并保持、不剥夺、循环等待；处理有预防、避免、检测、解除。
- 🔵 作答按“定义 → 四条件 → 破坏条件或处理方法”展开。
- 🔴 安全状态一定无死锁；不安全状态不等于已经死锁。
## 四、存储管理

### 回忆卡 9｜碎片与连续分配

先答：首次、最佳、最坏适应怎样选择空闲区？内部与外部碎片怎样区分？


- ⚫ 首次取首个足够区，最佳取最小足够区，最坏取最大区；内部碎片在已分配区内，外部碎片是空闲区分散。
- 🔵 选择题抓“第一个、最小、最大”三个动作词。
- 🔴 最佳适应不保证长期效果最佳；分页也可能产生页内碎片。
### 回忆卡 10｜分页与分段

先答：分页和分段在单位大小、逻辑意义、碎片和共享保护上怎样比较？


- 🔴 页定长、对用户透明、主要有内部碎片；段变长、反映程序逻辑、便于共享保护但可能有外部碎片。
- 🔵 比较题按“划分依据 → 长度 → 地址结构 → 碎片 → 用户可见性”写。
- ⚫ 页和页框等大；段内偏移达到段长即越界。
### 回忆卡 11｜虚拟存储、TLB 与缺页

先答：虚拟存储依靠什么原理？TLB 未命中与缺页有什么不同？


- ⚫ 虚拟存储依靠时间、空间局部性按需装入；TLB 未命中要查页表，缺页表示页面不在内存、需从外存调入。
- 🔵 事件顺序写“查 TLB → 查页表 → 必要时缺页调入”。
- 🔴 TLB 未命中、Cache 未命中和缺页不是同一事件。
### 回忆卡 12｜页面置换与抖动

先答：OPT、FIFO、LRU、CLOCK 各淘汰什么页面？抖动是什么？


- ⚫ OPT 淘汰未来最久不用，FIFO 淘汰最早装入，LRU 淘汰过去最久未用，CLOCK 循环寻找访问位 0；抖动是频繁缺页换页。
- 🔵 算法辨认只找“未来、进入时间、过去访问、访问位”关键词。
- 🔴 OPT 不能实际预知未来；FIFO 可能出现增加页框反而缺页更多。
## 五、文件与设备

### 回忆卡 13｜文件分配

先答：连续、链接、索引分配各有什么主要优缺点？


- 🔴 连续分配访问快但扩展难；链接分配易扩展但随机访问差；索引分配支持随机访问但需索引空间。
- 🔵 简答按“存放方式 → 访问性能 → 扩展性 → 空间代价”比较。
- ⚫ inode 通常不保存文件名；目录项负责名称到文件标识的联系。
### 回忆卡 14｜目录与链接

先答：绝对/相对路径、硬链接/符号链接怎样区分？


- ⚫ 绝对路径从根开始，相对路径从当前目录开始；硬链接指向同一 inode，符号链接保存目标路径。
- 🔵 判断链接先看“同 inode”还是“路径文件”。
- 🔴 硬链接通常不能跨文件系统；符号链接目标删除后可能悬空。
### 回忆卡 15｜I/O 控制与缓冲

先答：查询、中断、DMA 中 CPU 的参与程度怎样变化？缓冲有什么作用？


- ⚫ 查询由 CPU 轮询；中断由设备完成后通知；DMA 成块直传设备与主存；缓冲用于速度匹配、减少中断和提高并行性。
- 🔵 比较按“是否忙等 → 数据是否逐次经 CPU → 硬件复杂度”写。
- 🔴 DMA 仍需 CPU 初始化和结束处理，也会争用总线。
### 回忆卡 16｜磁盘调度

先答：FCFS、SSTF、SCAN、LOOK、C-SCAN 的服务规则是什么？


- 🔵 FCFS 按到达，SSTF 选最近，SCAN 到端点再反向，LOOK 到最远请求再反向，C-SCAN 只单向服务。
- 🔴 辨认题先看是否“最近、往返、只单向、是否到物理端点”。
- ⚫ SSTF 可能使远端请求饥饿；固态硬盘没有机械寻道过程。
## 六、答题与纠错

### 回忆卡 17｜基础简答框架

先答：定义、比较、组成作用类简答题分别怎样组织？


- 🔵 定义类写“是什么＋对象＋作用”；比较类写“定义＋共同点＋差异维度＋场景”；组成类写“总述＋名称＋作用”。
- 🔴 答题先圈题目动词，再列 3—5 个关键词，最后逐点扩一句。
- ⚫ 只列名词、只写优缺点或使用“越多越好”等绝对表述，通常不完整。
### 回忆卡 18｜判断纠错

先答：遇到“阻塞直接运行、TLB 未命中即缺页、DMA 完全不需 CPU”应怎样改？


- 🔴 三句都错误：阻塞须先到就绪；TLB 未命中先查页表；DMA 仍由 CPU 初始化和处理结束。
- 🔵 统一写“错误＋指出错误词＋正确表述＋必要条件”。
- ⚫ 看到“任何、一定、只能、完全”先检查是否忽略条件，但不能仅凭绝对词机械判错。
## 七、应用决策

### 回忆卡 19｜程序、进程与线程为什么不同

先答：为什么程序不能代替进程概念？为什么多线程仍要同步？

- 🔴 程序是静态指令，进程描述一次执行及其资源和状态；线程共享进程资源，能降低并发开销，但共享写入可能竞争。
- 🔵 场景题按“执行主体 → 共享什么 → 是否争用 → 用互斥还是同步 → 业务层边界”回答。
- ⚫ 同一程序可产生多个进程；线程锁只能保护进程内临界区，不能独自保证跨系统订单恰好处理一次。
