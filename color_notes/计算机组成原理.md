# 计算机组成原理：三色笔记

三色按考频和出题稳定性划分：红色优先背熟，蓝色用于补齐得分点，黑色用于防漏和边界判断。

## 🔴 高频考点

- 🔴 **计算机系统组成**：硬件包括运算器、控制器、存储器、输入和输出设备。 常见考法：写冯·诺依曼结构组成。 易错边界：CPU 主要由运算器和控制器等组成。
- 🔴 **数制与数据表示**：二进制、原码、反码、补码是机器数基础。 常见考法：进制转换、补码表示、溢出判断。 易错边界：补码符号扩展要复制符号位。
- 🔴 **指令系统**：指令由操作码和地址码等字段组成。 常见考法：判断指令功能、寻址方式。 易错边界：地址码不一定直接就是操作数。
- 🔴 **CPU 工作过程**：取指、译码、执行是指令周期核心。 常见考法：排序 CPU 执行步骤。 易错边界：机器周期、时钟周期、指令周期不要混淆。
- 🔴 **存储系统**：寄存器、Cache、主存、外存形成层次结构。 常见考法：比较速度、容量、成本。 易错边界：Cache 对程序员通常透明。
- 🔴 **总线**：数据总线、地址总线、控制总线分别传数据、地址和控制信号。 常见考法：判断总线宽度影响。 易错边界：地址总线宽度决定可寻址空间。

## 🔵 中频考点

- 🔵 **运算器**：ALU 完成算术和逻辑运算。 常见考法：判断加法器、移位、逻辑运算。 易错边界：溢出主要针对有符号数运算。
- 🔵 **控制器**：控制器产生控制信号协调各部件。 常见考法：判断微程序控制与硬布线控制特点。 易错边界：控制器不直接长期存放用户数据。
- 🔵 **I/O 方式**：程序查询、中断、DMA 是常见输入输出方式。 常见考法：判断大量数据传输适合 DMA。 易错边界：DMA 仍需总线和控制协调。
- 🔵 **性能指标**：字长、主频、CPI、MIPS 等描述不同性能侧面。 常见考法：判断指标含义。 易错边界：主频高不必然整体性能高。
- 🔵 **浮点数**：浮点数由阶码和尾数表示较大范围数。 常见考法：概念判断。 易错边界：浮点数有精度限制。

## ⚫ 低频了解

- ⚫ **流水线**：把指令执行分阶段重叠，提高吞吐率。 常见考法：判断流水线加速思想。 易错边界：流水线提高吞吐，不一定缩短单条指令延迟。
- ⚫ **RISC 与 CISC**：RISC 指令简单规整，CISC 指令复杂多样。 常见考法：比较体系结构特点。 易错边界：现代处理器常有混合实现。
- ⚫ **多核处理器**：多个核心可并行执行任务。 常见考法：概念判断。 易错边界：多核性能提升受程序并行度限制。

## 考前公式与简答

### 必会公式

- **存储容量**：`容量(bit)=存储单元数×字长`；若有 `n` 根地址线，最多选择 `2^n` 个地址。字长表示每个地址保存的位数，换算Byte时再除以8。
- **CPU执行时间**：`CPU时间=指令数×平均CPI/时钟频率`。指令数是程序执行的指令总量，CPI是每条指令平均需要的时钟周期，频率越高单周期越短；三个因素共同决定性能。
- **平均访问时间**：`T=命中率×Cache时间+(1-命中率)×未命中代价`。命中率表示请求直接在Cache找到的比例，未命中代价包含访问下一层存储器的额外时间。

### 常见简答题

- **为什么要设置Cache？** CPU速度远高于主存，Cache利用程序的时间局部性和空间局部性保存近期常用数据，降低平均访存时间，但不能替代容量更大的主存。
- **流水线为什么提高吞吐率却不一定缩短单条指令时间？** 流水线让多条指令的不同阶段重叠执行，提高单位时间完成数；单条指令仍需经过各阶段，还可能受到流水级开销和冲突影响。
