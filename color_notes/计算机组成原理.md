# 计算机组成原理 · 三色回忆卡

> 使用方法：先口述“先答”，再核对答案。🔴只保留最小结论；🔵只保留计算或设计动作；⚫只保留条件与反例。

## 一、系统与数据表示

### 回忆卡 1｜冯·诺依曼与 CPU

先答：冯·诺依曼机的核心思想和五大部件是什么？PC、IR、MAR、MDR 各存什么？

- 🔴 存储程序、程序控制，指令和数据均二进制存储；五部件是运算器、控制器、存储器、输入、输出。PC 存下条指令地址，IR 存当前指令，MAR/MDR 存访存地址/数据。
- 🔵 画通路时按“地址由谁给 MAR、数据经 MDR 去哪、PC 何时更新”追踪。
- ⚫ ISA 说明软件可见的“能做什么”，组成说明硬件“怎样实现”。

### 回忆卡 2｜性能计算

先答：怎样由 IC、CPI、主频求 CPU 时间？Amdahl 定律怎样算总加速比？

- 🔴 `CPU时间=IC×CPI/主频`；可改进比例 f、局部加速 s 时，`S=1/[(1-f)+f/s]`。
- 🔵 统一单位后分别算改进前后时间；Amdahl 先找“原时间占比”，不是指令占比。
- ⚫ 主频高不必然更快；局部无限加速的总上限是 `1/(1-f)`。

### 回忆卡 3｜补码与溢出

先答：n 位无符号数和补码范围是什么？补码加减怎样判溢出？

- 🔴 无符号范围 `0～2^n-1`，补码范围 `-2^(n-1)～2^(n-1)-1`；同号相加结果异号即溢出。
- 🔵 减法把减数取负后相加，符号位参与、最高进位丢弃；也可用符号位与最高数值位进位异或判溢出。
- ⚫ 进位反映无符号超界，溢出反映有符号超界，二者不等价。

### 回忆卡 4｜乘除法

先答：原码乘法、Booth 乘法和恢复余数除法每轮做什么？

- 🔴 原码符号异或、绝对值移位加；Booth 看相邻位：01 加、10 减、00/11 不动后算术右移；恢复余数法试减，负则恢复并商 0，非负商 1。
- 🔵 手算逐轮列 A/Q/Q-1 或题设寄存器，写“判位→加减→移位”，最后解释符号和位宽。
- ⚫ Booth 位序、除法初值和先移位/先试减有教材差异，必须服从题设微操作。

### 回忆卡 5｜IEEE 754

先答：单、双精度字段和规格化真值是什么？怎样完成十进制编码？

- 🔴 单精度为 `1|8|23`、偏置 127；双精度 `1|11|52`、偏置 1023；规格化值 `(-1)^S×1.F×2^(E-Bias)`。
- 🔵 转二进制并规格化 → 定 S → 阶码加偏置 → 填尾数并舍入 → 拼字段/转十六进制。
- ⚫ 阶码全 0 表示零或非规格化数；全 1 表示无穷或 NaN；阶码管范围、尾数管精度。

### 回忆卡 6｜浮点加减

先答：浮点加减的五步是什么，主要误差在哪里产生？

- 🔴 对阶 → 尾数加减 → 规格化 → 舍入 → 判断阶码上溢/下溢。
- 🔵 小阶向大阶对齐并右移尾数，保留保护/舍入/粘滞位后按题定舍入。
- ⚫ 对阶右移会丢低位；许多十进制小数不能被二进制浮点精确表示。

## 二、主存与 Cache

### 回忆卡 7｜存储层次与器件

先答：寄存器、Cache、主存、辅存的速度/容量关系是什么？SRAM 与 DRAM 怎样选？

- 🔴 层次向下速度降、容量增、每位价格降；SRAM 快且无需刷新，常作 Cache；DRAM 密度高且需刷新，常作主存。
- 🔵 解释局部性时分别举近期重复访问和邻近地址访问。
- ⚫ Cache—主存弥补速度差，主存—辅存构成虚拟存储，作用不可互换。

### 回忆卡 8｜芯片扩展与地址范围

先答：用 aK×b 位芯片组成 cK×d 位存储器，怎样求片数、接线和地址范围？

- 🔴 字扩展倍数 `c/a`，位扩展倍数 `d/b`，片数为二者乘积；低位作片内地址，高位译码片选。
- 🔵 先算字/位扩展 → 同字组数据位并联 → 高位译码 → 写每组首末地址并查重叠。
- ⚫ 字扩展增加地址空间，位扩展增加字长；不完全译码会出现地址镜像。

### 回忆卡 9｜Cache 地址字段

先答：m 位地址、块 B=2^b B、容量 C、E 路组相联时，Offset、Index、Tag 各几位？

- 🔴 Offset 为 b 位；组数 `S=C/(B×E)`，Index 为 `log2S` 位；Tag 为 `m-Index-Offset` 位。
- 🔵 先确认 C 是否仅指数据区、地址是否按字节；再由右向左切“Tag｜组号｜块内偏移”。
- ⚫ 全相联无 Index；若容量含标记/有效位，不能直接套数据容量公式。

### 回忆卡 10｜映射与访问序列

先答：直接、全相联、组相联分别把主存块放到哪里？怎样手算命中率？

- 🔴 直接映射到 `块号 mod 行数`；组相联到 `块号 mod 组数` 后组内任选路；全相联可到任意行。
- 🔵 地址除块大小得块号/偏移 → 定位行/组 → 比有效位和 Tag → 未命中装入或按策略替换 → 逐次计命中。
- ⚫ 增加相联度主要减少冲突失效，不消除首次访问的强制失效。

### 回忆卡 11｜写策略与 AMAT

先答：写直达/写回、写分配/非写分配怎样配合？AMAT 怎样列？

- 🔴 写直达同时写主存；写回只改 Cache 并置脏，淘汰时回写；`AMAT=命中时间+失效率×失效代价`。
- 🔵 序列题对每次写标命中、是否装块、是否置脏、淘汰时是否多一次写回。
- ⚫ 常见配对不是绝对规则；失效代价若已含某段时间，不能重复相加。

## 三、指令、寻址与控制

### 回忆卡 12｜指令格式与操作码

先答：操作码和地址码各做什么？定长与扩展操作码怎样计算？

- 🔴 操作码规定操作，地址字段给操作数位置或寻址信息；N 种定长操作至少需 `ceil(log2N)` 位。
- 🔵 扩展操作码按地址数由短到长逐级分配，并保留可继续扩展的前缀数。
- ⚫ 已分配给短指令的编码不能再作为长指令前缀；地址少常使指令短但程序指令数增多。

### 回忆卡 13｜立即、直接与间接寻址

先答：立即、直接、间接寻址的操作数在哪里？EA 怎样求？

- 🔴 立即数在指令内；直接寻址 `EA=A`；间接寻址 `EA=M[A]`，再由 `M[EA]` 取操作数。
- 🔵 先求 EA，最后才取操作数；每多一层方括号就多一次按地址读存储器。
- ⚫ 立即数没有可修改的存储目的地址；EA 与操作数值不是同一个量。

### 回忆卡 14｜寄存器、基址、变址与相对寻址

先答：寄存器、寄存器间接、基址、变址、相对寻址的 EA 各是什么，典型用途是什么？

- 🔴 寄存器数在 R；寄存器间接 `EA=(R)`；基址 `EA=(BR)+A` 利于重定位；变址 `EA=(IX)+A` 利于数组；相对 `EA=(PC)+A` 利于转移。
- 🔵 题中先圈定“寄存器内容、形式地址、存储内容”，按括号层次代入，再验 EA 范围。
- ⚫ PC 采用加前还是加后的值、位移是否符号扩展均以题设为准。

### 回忆卡 15｜指令周期与取指微操作

先答：取指阶段至少有哪些微操作？单总线数据通路要检查什么？

- 🔴 典型过程：PC 送 MAR 并读；存储字送 MDR；MDR 送 IR；PC 加指令长度。
- 🔵 按每个时钟写“源 → 总线/ALU → 目的”，同步标读写、ALU 功能和装入信号。
- ⚫ 单总线同一时刻不能有多个源驱动；不同指令不一定都有间址阶段。

### 回忆卡 16｜硬布线与微程序控制

先答：硬布线、微程序控制和水平/垂直微指令怎样比较？

- 🔴 硬布线快但修改难；微程序把控制信号编码为微指令，易扩展但通常较慢；水平型字长长并行度高，垂直型编码高而并行度低。
- 🔵 微程序题追踪“现微地址 → 控制字段发信号 → 后继地址形成 → 下条微指令”。
- ⚫ 微程序不是机器指令程序；控制存储器存微指令，不等同主存。

## 四、流水线、总线与 I/O

### 回忆卡 17｜理想流水线

先答：k 段流水完成 n 条指令的时间、吞吐率、加速比和效率怎样求？

- 🔴 等长段周期 Δt 时，`T=(k+n-1)Δt`，`TP=n/T`，`S=nk/(k+n-1)`，`E=n/(k+n-1)`。
- 🔵 先画逐周期流水图，数最后完成周期；不等长时周期取最长阶段加寄存器开销。
- ⚫ 流水提高吞吐率，不必缩短单条延迟；实际加速比通常小于段数。

### 回忆卡 18｜流水冒险

先答：结构、数据、控制冒险各因什么产生，怎样处理？

- 🔴 结构冒险争硬件，数据冒险源于相关，控制冒险源于分支；可用资源复制、转发/停顿/调度、预测/清空处理。
- 🔵 排图时先找 RAW，再看结果在哪阶段可转发；按分支决定阶段插气泡或清空，最后算周期/CPI。
- ⚫ 五段流水的 load-use 即使有转发常仍停 1 周期，具体服从阶段时序。

### 回忆卡 19｜总线带宽与仲裁

先答：总线带宽怎样计算？链式、计数器、独立请求仲裁的主要差异是什么？

- 🔴 `带宽=(总线位宽/8)×频率×每周期传输次数`；链式线少但固定优先，独立请求快但线多，计数器折中。
- 🔵 先统一 B/s 与 bit/s、十进制与二进制单位，再按有效传输率修正。
- ⚫ 地址总线宽度决定可寻址单元数，不直接等于数据总线一次传输量。

### 回忆卡 20｜查询、中断与 DMA

先答：三种 I/O 控制怎样占用 CPU？DMA 有哪些总线使用方式？

- 🔴 查询让 CPU 忙等；中断完成后通知 CPU；DMA 控制器在设备与主存间成块传输。DMA 可停止 CPU 访存、周期窃取或交替访问。
- 🔵 中断题按“请求 → 响应判优 → 保存现场 → 服务 → 恢复”画时序；DMA 题标初始化、传输、结束中断。
- ⚫ DMA 仍与 CPU 争主存/总线；响应优先级与处理优先级可由屏蔽字不同地安排。

### 回忆卡 21｜虚拟地址字段

先答：多级页表下怎样拆虚拟地址？Cache 未命中、TLB 未命中、缺页各去哪里？

- 🔴 页大小决定页内偏移位数，剩余虚页号按各级页表项数拆分；Cache 未命中去主存，TLB 未命中查页表，缺页去辅存。
- 🔵 先由页大小取 offset，再由“每级表大小/表项大小”求各级索引位，最后按“一级｜二级｜…｜偏移”切地址。
- ⚫ 多级页表节省的是未用下级表空间，不缩短一次未命中的查表次数。
