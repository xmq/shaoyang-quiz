# 计算机组成原理 · 三色回忆卡

> 先答后核对。🔴记结论；🔵记一步判断；⚫记条件与纠错。

## 一、系统与数据

### 回忆卡 1｜冯·诺依曼结构

先答：冯·诺依曼机的核心思想和五大部件是什么？


- 🔵 核心是存储程序、程序控制，指令和数据均二进制存储；部件为运算器、控制器、存储器、输入和输出。
- 🔴 简答按“思想 → 五部件 → 各一句作用”写。
- ⚫ 指令与数据形式相同，但 CPU 会按所处阶段和指令规则解释。
### 回忆卡 2｜性能指标

先答：主频、时钟周期、字长、吞吐量和响应时间分别表示什么？


- 🔵 主频表示时钟快慢，周期是其倒数；字长是一次处理位数；吞吐量看单位时间工作量，响应时间看单项任务耗时。
- 🔴 比较性能先说明使用同一任务和指标，再综合 CPU、存储与程序。
- ⚫ 主频高不等于整机性能一定高；字长也不等于地址总线宽度。
### 回忆卡 3｜数制与编码

先答：二进制、八进制、十六进制怎样对应？ASCII、Unicode、BCD 各做什么？


- 🔵 一位八进制对应 3 位二进制，一位十六进制对应 4 位；ASCII 编常用英文字符，Unicode 统一多语言字符，BCD 用 4 位表示一位十进制数。
- 🔴 简单转换从小数点向两边分组，不足位补 0。
- ⚫ 编码是表示规则，不等于加密；标准 ASCII 使用 7 位。
### 回忆卡 4｜补码与范围

先答：正负数补码怎样得到？n 位无符号数和补码范围是什么？


- ⚫ 正数补码同二进制真值；负数由正数按位取反加 1；无符号范围 `0～2^n-1`，补码范围 `-2^(n-1)～2^(n-1)-1`。
- 🔵 一步题先确认位数和“有符号/无符号”，再代范围。
- 🔴 进位不等于有符号溢出；8 位补码是 -128～127，不是 -127～127。
### 回忆卡 5｜定点与浮点

先答：定点数和浮点数的主要区别是什么？浮点符号、阶码、尾数各影响什么？


- ⚫ 定点小数点固定、结构简单；浮点小数点可浮动、范围大。符号定正负，阶码主控范围，尾数主控精度。
- 🔵 比较按“表示形式 → 范围 → 精度 → 硬件复杂度”写。
- 🔴 浮点数存在舍入误差，不能精确表示所有十进制小数。
## 二、CPU 与指令

### 回忆卡 6｜CPU 与寄存器

先答：ALU、控制器、PC、IR、MAR、MDR 各有什么作用？


- 🔵 ALU 运算，控制器发控制信号；PC 存下一指令地址，IR 存当前指令，MAR/MDR 存访存地址/数据。
- 🔴 取指顺序概括为“PC 给地址 → 存储器读出 → IR 保存 → PC 更新”。
- ⚫ PC 不保存当前指令内容；CPU 不只由运算器组成。
### 回忆卡 7｜基本寻址

先答：立即、直接、间接、寄存器和寄存器间接寻址的操作数或地址在哪里？


- 🔵 立即数在指令中；直接字段给地址；间接字段指向保存地址的单元；寄存器寻址的数在寄存器；寄存器间接的地址在寄存器。
- 🔴 辨认只问两步：“字段/寄存器里是数还是地址？还要不要再访存？”
- ⚫ 立即数通常不能作为可修改存储目的地；间接寻址访问范围大但访存次数多。
### 回忆卡 8｜基址、变址与相对寻址

先答：基址、变址、相对寻址的典型用途是什么？


- 🔵 基址加位移利于重定位，变址加位移利于数组，相对寻址用 PC 加位移利于转移和位置无关代码。
- 🔴 场景题抓“程序搬移、数组下标、转移距离”对应三类。
- ⚫ 具体使用 PC 更新前还是更新后以及位移符号，以题设处理器规则为准。
### 回忆卡 9｜RISC 与 CISC

先答：RISC 和 CISC 在指令、寻址、寄存器和流水方面怎样比较？


- 🔵 RISC 指令规整、寻址少、常用 Load/Store 和较多寄存器，利于流水；CISC 指令和寻址丰富、长度可能变化。
- 🔴 比较按“指令复杂度 → 寻址 → 访存 → 控制实现 → 流水”写。
- ⚫ 现代处理器会融合两者思想，不能绝对说 RISC 一定更快。
## 三、存储系统

### 回忆卡 10｜存储层次

先答：寄存器、Cache、主存、辅存从速度、容量和价格怎样排列？


- 🔵 向下速度降低、容量增大、每位价格降低；Cache 缓解 CPU 与主存速度差，虚拟存储结合主存与辅存。
- 🔴 简答先画层次，再逐项写“快/小/贵”到“慢/大/便宜”。
- ⚫ Cache 和主存都不是永久保存数据的可靠外存。
### 回忆卡 11｜SRAM、DRAM、ROM、Flash

先答：四类存储器是否刷新、是否易失、常用位置是什么？


- 🔵 SRAM 快且无需刷新，常作 Cache；DRAM 需刷新，常作主存；ROM、Flash 非易失，Flash 可电擦写。
- 🔴 类型题按“易失性 → 刷新 → 可写性 → 典型用途”判断。
- ⚫ 需要刷新的不是 SRAM；RAM 通常易失，但“可随机访问”不是 DRAM 独有。
### 回忆卡 12｜容量与地址线

先答：芯片“1K×8 位”表示什么？n 根地址线可选择多少编址单元？


- 🔵 1K×8 表示 1024 个单元、每单元 8 位；n 根地址线可选择 `2^n` 个单元。
- 🔴 一步题先确认按字节还是按字编址，再把单元数乘每单元字节数。
- ⚫ 位扩展增加每字位数，字扩展增加单元数量，两者不能写反。
### 回忆卡 13｜Cache 基础

先答：Cache 为什么有效？直接、全相联、组相联映射各有什么特点？


- 🔵 Cache 依靠时间和空间局部性；直接映射固定行、简单但冲突多，全相联任意行、复杂但冲突少，组相联折中。
- 🔴 简答按“放置范围 → 硬件复杂度 → 冲突概率”比较。
- ⚫ 提高相联度主要减少冲突失效，不能消除首次访问失效。
### 回忆卡 14｜Cache 写策略

先答：写直达和写回怎样更新主存？Cache 未命中与缺页有何区别？


- 🔵 写直达同时更新主存；写回先改 Cache，淘汰脏块时回写。Cache 未命中通常去主存，缺页才去辅存。
- 🔴 判断事件先看缺的是“数据块、地址转换项还是内存页面”。
- ⚫ 写回性能较好但控制更复杂；Cache 未命中不等于程序错误。
## 四、控制、总线与 I/O

### 回忆卡 15｜控制器与流水线

先答：硬布线/微程序控制器怎样比较？流水线主要提高什么？


- ⚫ 硬布线快但修改难，微程序规整易扩展但通常较慢；流水线让指令阶段重叠，主要提高吞吐率。
- 🔵 冲突辨认：争硬件是结构冲突，依赖结果是数据冲突，分支改地址是控制冲突。
- 🔴 流水线不保证缩短单条指令延迟，也不消除所有冲突。
### 回忆卡 16｜三类总线

先答：数据、地址、控制总线各传什么，宽度分别影响什么？


- 🔵 数据总线传数据、宽度影响一次传输量；地址总线传地址、宽度影响寻址范围；控制总线传读写和中断等信号。
- 🔴 组成作用题写“名称＋传输内容＋宽度/方向影响”。
- ⚫ 控制总线各信号方向不一；地址总线宽不等于一次数据传得多。
### 回忆卡 17｜I/O 控制

先答：查询、中断、DMA 中 CPU 怎样参与？统一编址与独立编址有何不同？


- 🔵 查询由 CPU 轮询；中断由设备完成后通知；DMA 在设备与主存间成块传输。统一编址占用内存地址空间，独立编址使用专用空间和指令。
- 🔴 比较按“是否忙等 → 数据是否经 CPU → 并行性 → 硬件代价”写。
- ⚫ DMA 仍需 CPU 初始化和结束处理，也会争用总线。
## 五、答题与纠错

### 回忆卡 18｜简答框架

先答：组成作用、分类比较和工作原理题怎样组织答案？


- ⚫ 组成题写“总述＋部件＋作用”；比较题写“定义＋差异维度＋场景”；原理题写“起点＋经过部件＋结果”。
- 🔵 先圈“组成/比较/说明原理”，列 3—5 个关键词后逐点扩写。
- 🔴 只写缩写、只列部件或只写“更快更好”均不完整。
### 回忆卡 19｜判断纠错

先答：“主频高一定快、PC 存当前指令、DRAM 无需刷新、流水线缩短单条延迟”怎样纠错？


- ⚫ 均错误：性能需综合判断；当前指令在 IR；DRAM 需刷新；流水线主要提高吞吐率。
- 🔵 统一写“错误＋错误词＋正确结论＋适用条件”。
- 🔴 不能只因句中有“一定”就判错，要能写出被忽略的条件。
## 六、信息流应用

### 回忆卡 20｜数据通路与中断过程

先答：取指时信息怎样流动？中断为什么比轮询更有效，边界是什么？


- 🔵 取指可概括为 `PC→MAR→主存→MDR→IR`；中断让设备就绪后通知 CPU，避免 CPU 持续忙等。
- 🔴 机制题按“请求 → 保存现场 → 找入口 → 服务 → 恢复返回”复述中断过程。
- ⚫ MAR 保存地址、MDR 暂存数据；中断不等于成块数据自动传完，DMA 也仍需 CPU 初始化和收尾。
### 回忆卡 21｜CPU时间与流水边界

先答：CPU执行时间由哪三个量决定？为什么主频或流水级数不能单独代表性能？

- 🔵 `CPU时间＝指令数×平均CPI÷主频`；流水吞吐还受最慢阶段、相关、分支和级间开销影响。
- 🔴 简答按“公式 → 每个量的含义 → 同一任务前提 → 存储/流水限制”作答。
- ⚫ 主频更高仍可能因指令更多或CPI更大而较慢；流水级数更多也不保证收益更高。
