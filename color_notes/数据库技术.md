# 数据库技术 · 三色回忆卡

> 先答后核对。🔴记结论；🔵记一步 SQL/简答；⚫记条件与纠错。

## 一、系统与建模

### 回忆卡 1｜DB、DBMS、DBS 与模式

先答：DB、DBMS、DBS 怎样区分？三级模式和两种独立性是什么？


- ⚫ DB 是数据集合，DBMS 是管理软件，DBS 还含应用、人员和环境；外模式面向用户，模式管全局逻辑，内模式管物理存储。
- 🔵 外/模映像支持逻辑独立性，模/内映像支持物理独立性。
- 🔴 一个库通常一个模式和内模式、多个外模式。
### 回忆卡 2｜E-R 与转表

先答：实体、属性、码、联系是什么？1:n 和 m:n 怎样转表？


- 🔴 实体是对象，属性描述特征，码唯一标识，联系描述关系；1:n 把 1 端码放多端作外码，m:n 单独建联系表。
- 🔵 圈实体/码/业务动词，双向判断基数，再写表名、主码和外码。
- ⚫ 外码应放多端；m:n 不能只在一方加单个外码。
### 回忆卡 3｜关系与键

先答：关系、元组、属性、域及超码、候选码、主码、外码是什么？


- 🔴 关系是表，元组是行，属性是列，域是取值范围；候选码是最小超码，主码是选定候选码，外码引用其他键。
- 🔵 键辨析按“唯一 → 最小 → 选定 → 引用”。
- ⚫ 候选码可有多个，主码通常只选一个。
## 二、完整性与 SQL

### 回忆卡 4｜完整性与 NULL

先答：三类完整性分别约束什么？NULL 怎样判断？


- ⚫ 实体完整性要求主码唯一非空；参照完整性要求外码为空或引用已有键；用户定义完整性管类型、范围和业务规则。
- 🔵 NULL 用 `IS NULL`；`COUNT(列)` 忽略 NULL，`COUNT(*)` 统计行。
- 🔴 NULL 不等于 0 或空串；外码是否可空还看列约束。
### 回忆卡 5｜关系运算与 SQL 分类

先答：选择、投影、连接分别做什么？CREATE、SELECT、INSERT/UPDATE/DELETE、GRANT 各属什么？


- 🔴 选择筛行，投影取列，连接组合相关表；四组命令依次为定义、查询、操纵、权限控制。
- 🔵 读题先定结果列，再定表、连接和行条件。
- ⚫ 漏连接条件会产生笛卡尔积；DROP 删对象，DELETE 删行。
### 回忆卡 6｜约束与查询结构

先答：五类常用约束是什么？SELECT 查询各子句做什么？


- ⚫ PRIMARY KEY、FOREIGN KEY、UNIQUE、NOT NULL、CHECK 分别管主标识、引用、唯一、非空、条件；SELECT/FROM/WHERE/GROUP BY/HAVING/ORDER BY 依次定列、表、行、组、组条件、顺序。
- 🔵 按“查什么 → 从哪查 → 行条件 → 分组筛组 → 排序”写。
- 🔴 WHERE 筛行，HAVING 筛组；主码同时唯一且非空。
### 回忆卡 7｜聚合与连接

先答：聚合函数有哪些？INNER、LEFT、RIGHT、FULL JOIN 保留哪些行？


- ⚫ 聚合有 COUNT、SUM、AVG、MAX、MIN；INNER 留匹配，LEFT/RIGHT 留指定侧，FULL 留两侧并以 NULL 补缺。
- 🔵 先写主外键连接，再判断是否保留无匹配记录。
- 🔴 LEFT JOIN 后在 WHERE 筛右表列可能滤掉补出的 NULL。
### 回忆卡 8｜子查询与集合

先答：IN、EXISTS、UNION、UNION ALL 分别做什么？


- 🔴 IN 判断集合成员，EXISTS 判断存在行；UNION 合并去重，UNION ALL 保留重复。
- 🔵 先判断子查询返回单值、集合还是存在性，再选操作符。
- ⚫ `NOT IN` 遇 NULL 可能产生未知；集合运算要求列数和对应类型兼容。
## 三、设计、事务与安全

### 回忆卡 9｜视图、索引与阶段

先答：视图和索引各有什么作用？数据库设计经过哪些阶段？


- 🔵 视图简化查询并限制可见数据；索引用空间和写维护换查询速度；设计为需求、概念、逻辑、物理、实施、运行维护。
- 🔴 比较写“是否保存结果 → 主要作用 → 更新代价”。
- ⚫ 普通视图通常不存结果；索引不是越多越好。
### 回忆卡 10｜规范化

先答：1NF、2NF、3NF、BCNF 的最小结论和目的是什么？


- 🔵 1NF 属性原子，2NF 消部分依赖，3NF 消传递依赖，BCNF 要求决定因素为超码；目的是减少冗余和更新异常。
- 🔴 基础题先找重复、插入/删除/更新异常，再说明拆分实体或联系。
- ⚫ 范式越高不代表查询一定更快。
### 回忆卡 11｜ACID 与异常

先答：ACID 和丢失修改、脏读、不可重复读、幻读分别是什么？


- ⚫ ACID 为原子、一致、隔离、持久；四异常依次为写覆盖、读未提交、同一行前后变化、条件结果行集合变化。
- 🔵 区分不可重复读与幻读时看“同一行”还是“行集合”。
- 🔴 一致性是业务约束成立，不等于任何副本时刻都完全相同。
### 回忆卡 12｜锁、日志、备份

先答：S 锁、X 锁、WAL、备份分别做什么？


- ⚫ S 锁允许共享读，X 锁排斥其他读写；WAL 强调数据页前先写日志；备份保存独立副本。
- 🔵 简答按“并发隔离 → 故障恢复 → 独立副本”写。
- 🔴 封锁可能死锁；RAID 和日志都不能代替备份。
## 四、安全与答题

### 回忆卡 13｜安全与权限

先答：身份鉴别、授权、审计分别回答什么？GRANT、REVOKE 做什么？


- ⚫ 鉴别回答是谁，授权回答能做什么，审计记录做过什么；GRANT 授权，REVOKE 收权。
- 🔵 权限题按“鉴别 → 最小授权 → 操作 → 审计 → 回收”。
- 🔴 完整性保证数据规则，权限控制访问主体，二者不能替代。
### 回忆卡 14｜基础简答框架

先答：定义作用、键比较和 SQL 读写题怎样组织？


- 🔴 定义写“是什么＋对象＋作用”；键写“唯一＋最小＋选定＋引用”；SQL 写“列＋表＋连接＋条件＋分组排序”。
- 🔵 写完检查 NULL、重复、连接遗漏和聚合条件。
- ⚫ 只写命令不解释语义，或只列主码不写外码关系，答案不完整。
### 回忆卡 15｜判断纠错

先答：“候选码只有一个、NULL 等于 0、WHERE 筛组、索引越多越好”怎样改？


- ⚫ 均错误：候选码可多个；NULL 是未知/缺失；HAVING 筛组；索引增加空间和写维护。
- 🔵 统一写“错误＋错误词＋正确结论＋条件或代价”。
- 🔴 不要把主码与候选码、数据完整性与访问权限混为一谈。
## 五、业务一致性

### 回忆卡 16｜事务、索引与业务边界

先答：收银订单为什么用事务？按订单号查询怎样建索引？哪些问题不能只靠事务或 TCP？

- 🔵 写订单、扣库存等整体用事务保证全做或全不做；订单号等值查询适合唯一索引；重复请求还需唯一键和幂等处理。
- 🔴 场景题按“业务规则 → ACID/并发异常 → 锁或原子更新 → 索引 → 重试边界”说明。
- ⚫ TCP 只保证连接内字节流可靠有序，不保证订单只执行一次；隔离级别和索引都不是越多越强越好。
