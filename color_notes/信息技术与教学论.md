# 信息技术与教学论·期末复习三色笔记

> 只用于考前速记：🔴核心必背；🔵模型、步骤与简答踩分词；⚫适用条件、辨析和教材口径。义务教育以《义务教育信息科技课程标准（2022年版）》为主，高中以《普通高中信息技术课程标准（2017年版2020年修订）》为主。

## 一、课程性质、理念与核心素养

- 🔴 义务教育课程名称为“信息科技”，具有基础性、实践性和综合性；高中课程名称为“信息技术”。
- 🔴 信息科技不是单纯的软件操作课，既学习数据、算法、网络和系统等原理，也发展利用技术解决问题的能力和社会责任。
- 🔴 四项核心素养：信息意识、计算思维、数字化学习与创新、信息社会责任。
- 🔵 信息意识踩分词：信息敏感度、价值判断、来源可靠性、时效性、主动获取与利用。
- 🔵 计算思维踩分词：分解、抽象、建模、算法设计、自动化、评价与优化。
- 🔵 数字化学习与创新踩分词：工具选择、资源管理、协同学习、作品创作、迁移创新。
- 🔵 信息社会责任踩分词：法律伦理、隐私安全、知识产权、文明交流、负责任发布。
- ⚫ 计算思维不等于只会编程；用了电脑不等于实现数字化学习；核心素养四项相互支持，不能割裂。

## 二、主要学习理论

- 🔴 行为主义关注刺激—反应、练习、强化和可观察行为，适合基础操作训练与即时反馈。
- 🔴 认知主义关注信息加工、认知结构、先备知识和策略，可用先行组织者、分块、概念图和精加工。
- 🔴 建构主义强调学习者在情境、协作、会话和资源支持下主动建构意义，教师提供支架与反馈。
- 🔴 人本主义重视完整人格、情感、需要、自我实现和有意义学习，强调尊重与选择。
- 🔴 社会学习理论强调观察学习、榜样、替代性强化和自我效能；情境学习强调知识与真实活动、共同体和文化情境相联系。
- 🔴 最近发展区：学生独立水平与在帮助下可达到水平之间的区域；支架应随能力提高逐步撤除。
- 🔵 理论匹配：操作熟练—行为主义；知识结构—认知主义；真实项目协作—建构主义；情感需要—人本主义。
- ⚫ “学生主体”不等于教师退出；发现学习也不是完全不给指导。

## 三、教学设计的系统观与模型

- 🔴 教学设计是以学习目标达成为中心，对学习者、内容、活动、资源、环境和评价进行系统规划并持续改进。
- 🔵 通用流程：需求/课标分析 → 学习者与内容分析 → 制定目标 → 设计活动、策略、资源和评价 → 开发资源 → 实施 → 反馈修订。
- 🔴 ADDIE：Analysis 分析 → Design 设计 → Development 开发 → Implementation 实施 → Evaluation 评价；评价可贯穿全过程。
- 🔴 教—学—评一致性：目标规定学什么，活动提供学习经历，评价收集目标达成证据，三者必须对应。
- 🔵 逆向设计：确定预期结果 → 确定可接受证据 → 设计学习经历与教学活动。
- 🔵 ASSURE 常见骨架：分析学习者 → 陈述目标 → 选择方法、媒体和材料 → 使用媒体材料 → 要求学习者参与 → 评价与修订。
- ⚫ 不同教材可能采用 ADDIE、迪克—凯瑞、肯普或本土流程，名称不同但都强调分析、目标、策略、实施、评价和修订。

## 四、学习者、内容与教学目标分析

- 🔴 学习者分析：先备知识技能、认知发展、动机兴趣、学习经验、个体差异、设备条件、特殊学习需要。
- 🔵 获取证据：前测、访谈、问卷、课堂观察、作品分析和学习数据；不能只凭教师印象。
- 🔴 内容分析要确定知识结构、重点难点、先后关系、任务类型及其与课标和生活情境的联系。
- 🔴 教学重点由课标要求、内容结构和育人价值决定；教学难点由内容特性与具体学习者基础共同决定。
- 🔴 目标应面向核心素养，并转化为可观察、可评价的学习表现。
- 🔵 ABCD：A 学习者，B 可观察行为，C 行为条件，D 达成标准；常用动词为识别、解释、比较、设计、调试、评价、创作。
- 🔵 目标表述示例骨架：“在给定……条件下，学生能……，达到……标准”。
- ⚫ “了解、掌握、提高、培养”若不说明行为证据，难直接评价；重点不必然是难点。

## 五、教学策略与常用方法

- 🔴 讲授法适合系统解释概念和原理，效率高；应结合提问、案例、可视化和练习。
- 🔴 演示练习法：教师分段示范并口述关键思路，学生模仿、独立练习，再迁移到新任务。
- 🔴 任务驱动：把知识技能融入有意义任务，学生在分析、实施和评价任务中学习。
- 🔵 任务驱动流程：创设情境 → 明确任务 → 分析并形成方案 → 自主/协作实施 → 展示交流 → 评价反思与迁移。
- 🔵 好任务踩分词：真实、目标一致、难度适切、层次递进、成果可评、安全可行、允许选择。
- 🔴 项目式学习围绕真实复杂问题持续开展规划、调研、设计、制作、测试、迭代和公开展示，周期通常长于单一任务。
- 🔵 项目流程：驱动性问题 → 规划分工 → 调研设计 → 制作测试 → 迭代改进 → 展示答辩 → 反思评价。
- 🔴 探究学习常经历提出问题、作出假设、收集分析证据、形成解释、交流评价。
- 🔴 合作学习要有共同目标、积极相互依赖、个体责任、促进性互动、合作技能和小组反思。
- 🔵 防“搭便车”：角色明确、个人任务可追踪、随机汇报、同伴互评、个人反思、过程记录。
- ⚫ 围坐在一起不等于合作；任务驱动不等于照着教师步骤操作；项目式学习不等于制作一个作品。

## 六、信息科技课堂教学组织

- 🔵 上机课流程：设备/账号检查 → 明确目标与安全规则 → 发布任务/分段示范 → 学生实践和教师巡视 → 保存提交 → 展示评价 → 退出账号并整理设备。
- 🔴 课堂管理同时关注人员、设备、账号、文件、网络、时间和数据安全。
- 🔵 故障处理：先保人身和数据安全 → 判断是否为个别/普遍故障 → 查电源、连接、账号、网络、软件和路径 → 启用离线资源/备用方案 → 记录复盘。
- 🔴 差异化教学可设置基础、进阶、挑战任务；为困难学生提供步骤卡、示例、半成品或同伴支架，为高水平学生提供开放拓展。
- 🔵 教师巡视看“过程证据”：任务理解、操作策略、调试记录、协作质量和安全规范，而不只看最后作品。
- ⚫ 技术故障不能简单转成无目标自由活动；教师演示过长会挤压学生实践时间。

## 七、信息化教学形态

- 🔴 WebQuest 是以网络资源为主要信息来源的探究活动，经典结构包括导言、任务、过程、资源、评价和结论。
- 🔴 微课围绕相对聚焦的目标，以短小数字资源为核心并配套学习活动；视频是常见载体，但微课不等于一段视频。
- 🔴 翻转课堂把部分知识获取前移，课堂更多用于问题解决、协作、反馈和个别指导。
- 🔴 混合式学习把线上与线下、同步与异步活动按目标进行有机整合，不是简单叠加平台和课堂。
- 🔴 移动学习突出便携、泛在、情境和碎片化支持；仍要防注意分散、数字鸿沟和隐私风险。
- 🔵 翻转设计：课前资源+检测 → 教师分析学情 → 课中问题解决/协作 → 即时反馈 → 课后巩固反思。
- ⚫ 微课“5～10 分钟”是常见经验而非绝对法规；翻转不等于课前看视频、课上只做题；线上资源多不等于混合质量高。

## 八、教学媒体与数字资源

- 🔴 媒体选择首先服从目标、内容、学习者、策略和环境，再考虑可获得性、成本、易用性和安全版权。
- 🔵 媒体选择踩分词：目标匹配、表征优势、认知负荷、交互反馈、无障碍、技术条件、成本效益。
- 🔴 多媒体学习应减少无关信息，突出关键线索，把相互关联的文字和图形邻近呈现，并允许适当分段和学习者控制。
- 🔴 资源评价：科学性、教育性、技术性、艺术性、交互性、可用性、安全性和版权合规。
- 🔵 资源开发流程：需求分析 → 教学设计 → 脚本/原型 → 素材采集制作 → 集成 → 测试评价 → 发布维护。
- ⚫ 动画、声音越多不一定越好；装饰性信息可能增加外在认知负荷；网络资源可以引用但必须核验来源和授权。

## 九、教学评价基本分类

- 🔴 诊断性评价用于教学前了解起点与困难；形成性评价用于教学中反馈改进；总结性评价用于阶段结束判定达成。
- 🔴 标准参照评价把表现与既定标准比较；常模参照评价与群体比较；个体内差异评价与本人过去比较。
- 🔴 定量评价用分数、频数等数据；定性评价用描述、观察和解释；应根据目标结合使用。
- 🔴 自评、互评、教师评价和社会/专家评价属于评价主体多元化。
- 🔵 信息科技证据：纸笔测验、上机任务、程序运行、作品、设计文档、过程日志、答辩和档案袋。
- ⚫ 同一测验因使用目的不同，可成为诊断、形成或总结性评价；不能只看形式下结论。

## 十、评价工具与测验质量

- 🔴 量规 Rubric 把评价维度及各水平表现描述清楚；维度必须对应目标，水平描述应可观察、可区分。
- 🔵 作品量规常用维度：功能正确、技术/算法合理、创意表达、规范安全、协作过程、说明反思。
- 🔴 信度是结果的一致性和稳定性；效度是是否测到预期目标；信度高是效度高的必要非充分条件。
- 🔴 难度指数若采用通过率 `P=答对人数/总人数`，P 越大题越容易；区分度反映题目区分不同水平学习者的能力。
- 🔵 客观题命制：题干独立清楚、一个最佳答案、干扰项同质且有迷惑性、避免暗示和无意线索。
- 🔵 形成性反馈应及时、具体、面向任务和改进策略，回答“哪里好、哪里需改、下一步怎么做”。
- ⚫ 不同教材对“难度值”可能采用相反定义，计算前先看公式；判断题中的“永远、完全、只能”常是陷阱但不能机械判错。

## 十一、案例分析与教学设计题

- 🔵 案例分析四步：判断做法 → 引用课标/理论依据 → 分析对学习的影响 → 提出可执行改进并说明评价证据。
- 🔵 教学设计答题骨架：课标与学情 → 素养目标 → 重点难点 → 情境任务 → 活动流程 → 资源支架 → 评价量规 → 安全伦理 → 反思改进。
- 🔵 高频得分词：学生主体、真实情境、问题解决、支架、合作与个体责任、分层任务、教—学—评一致、过程性证据。
- 🔴 技术伦理必须写：个人信息最小必要、账号权限、内容审核、知识产权、网络文明、数据备份与安全退出。
- ⚫ 只写“激发兴趣、活跃气氛、培养能力”而无具体行为和证据，通常难得高分；“教师包办操作、只评价成品外观、目标不可测”是高频反例。

## 十二、说课、听评课与教学研究

- 🔴 说课通常说明教什么、为什么教、怎么教、如何评价，内容包括教材/课标、学情、目标、重点难点、教法学法、过程、评价和反思。
- 🔴 听评课关注目标达成、学习活动、师生互动、资源使用、课堂生成、评价反馈和学生真实学习证据。
- 🔵 评课表述：先呈现观察证据 → 对照目标/理论解释 → 指出影响 → 给出具体建议；避免只给笼统好坏评价。
- 🔴 行动研究常见循环：计划 → 行动 → 观察 → 反思 → 再计划；用于教师在真实课堂中持续改进。
- ⚫ 教案是教学实施方案，说课是对方案依据和思路的解释，二者不能等同。
