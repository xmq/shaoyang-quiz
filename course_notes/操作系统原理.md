# 📚 操作系统原理

使用说明：🔴红色= 一级考点 / 核心术语 / 判断改错点（背下来）🔵蓝色= 简答题踩分关键词 / 调度算法特征 / 概念区分（考试时写上去就是分）⚫️黑色= 具体定义解释、例子、选择题常见正确/错误描述（理解即可）
第一章：操作系统概述
考点1：操作系统的定义与地位
🔴【定义】：操作系统（OS）是管理计算机软硬件资源、控制程序执行并为用户和应用提供服务的系统软件。传统教材常说它运行在裸机之上；现代系统还可能位于固件、虚拟机监控器之上。
🔴【地位】：
操作系统是用户与计算机硬件之间的接口。
操作系统是对硬件的第一次扩充。
操作系统是其他所有软件运行的基础平台。
⚫️【坑点】：操作系统是核心系统软件，不是一般应用软件。操作系统的功能不包括“高级程序设计语言的编译器”（编译器属于语言处理程序）。
考点2：操作系统的功能（五大管理）
🔴【五大功能】：
处理器（处理机）管理：CPU的分配和调度。
存储器管理：内存的分配、回收和保护。
设备管理：I/O设备的分配和管理。
文件管理（信息管理）：文件的存储、检索、共享和保护。
作业/用户接口管理：组织和控制作业，并通过命令接口、程序接口（系统调用）等提供服务。不同教材可能概括为四大管理、五大管理或另列用户接口。
⚫️【坑点】：存储器管理主要负责内存的分配与回收，不是外存（外存属于设备管理或文件管理）。
考点3：操作系统的分类（必考判断）
🔴【批处理操作系统】：
单道批处理：内存中只有一道程序。特点：单道性、顺序性、自动性。
多道批处理：内存中同时存放多道程序，共享CPU。特点：资源利用率高、系统吞吐量大，但无交互能力（用户无法干预）。设计时首先考虑资源利用率和吞吐量。
🔴【分时操作系统】：
将CPU时间划分为时间片，轮流分配给各终端用户。
四大特点：多路性、独立性、及时性、交互性。（口诀：多独及时交）
坑点：“自治性”不是分时操作系统的特点！
单核计算机实现多任务“同时”执行假象的机制是时间片轮转。
🔴【实时操作系统】：
对外部事件在规定时间内做出响应。
特点：正确性同时取决于逻辑结果与规定时限，强调可预测性和可靠性。实时系统可以具有很强的人机交互，“交互性弱”不是定义性特征。
应用：飞机控制、工业控制等硬实时场景，以及部分通信、媒体等软实时场景。订票系统更常归为联机事务处理，是否算实时系统取决于教材语境。
输出结果的正确性不仅依赖于结果的正确性，还依赖于产生结果的时间（实时性要求）。
⚫️【坑点】：
分时操作系统的时间片不是越小越好（过小会导致频繁切换，系统开销增大）。
批处理系统没有交互能力。
实时系统与分时系统的区别：实时系统交互性弱，响应时间敏感度更高。
考点4：操作系统的特征
🔴【四大特征】：并发、共享、虚拟、异步。（口诀：并共虚异）
🔴【并发】：两个或多个事件在同一时间间隔内发生（宏观同时，微观交替）。并发性是指若干事件在同一时间间隔发生。
🔴【共享】：系统资源供多个并发执行的进程共同使用。
🔴【虚拟】：把一个物理实体变成若干个逻辑上的对应物（如虚拟内存）。
🔴【异步】：进程以不可预知的速度向前推进（走走停停）。
⚫️【坑点】：
“并发”与“并行”的区别：并发是同一时间间隔（单核），并行是同一时刻（多核）。
多个进程的实体存在于同一内存中，在一段时间内都得到运行 → 这种性质称作并发性。
第二章：进程管理
考点5：进程与程序的区别（绝对高频）
🔴【程序】：静态的，是指令的有序集合，存储在外存中。
🔴【进程】：动态的，是程序的一次执行活动；其地址空间可部分驻留内存、部分位于外存，不能简单定义为“进程存储在内存”。
🔵【本质区别】：
进程是动态的，程序是静态的。
进程是资源分配的基本单位（有生命周期）。
进程 =程序 + PCB + 数据集合。
⚫️【坑点】：
进程是动态的，程序是静态的。这是它们的本质区别。
可执行程序通常以文件形式保存在外存，执行时创建进程；进程的运行状态由PCB等维护，其地址空间不要求全部常驻内存。
不同进程所执行的代码可以相同（如多个用户同时运行同一个程序）。
考点6：进程控制块（PCB）
🔴【PCB定义】：进程控制块，是进程存在的唯一标志。
🔴【PCB内容】：进程标识符、进程状态、程序计数器、寄存器、内存信息、资源清单等。
🔴【PCB作用】：系统根据PCB感知进程的存在，通过PCB控制和管理进程。
⚫️【坑点】：创建/撤销进程的核心操作包括建立/撤销PCB。PCB由内核维护，用户进程通常不能直接读写；应用可通过系统调用查询操作系统允许公开的部分进程信息。
考点7：进程的三种基本状态及其转换（必考）
🔴【三种状态】：
就绪态（Ready）：除CPU外，所有资源都已获得，等待CPU调度。
运行态（Running）：已获得CPU，正在执行。
阻塞态（Blocked/Waiting）：因等待某事件（如I/O完成）而暂停执行。
🔵【状态转换条件】：
⚫️【坑点】：
时间片用完 → 进程从运行态变为就绪态。
等待某一事件 → 进程从运行态变为阻塞态。
进程申请CPU得不到满足 → 处于就绪态，不是阻塞态。
单处理器系统中，同时存在12个进程，处于就绪队列的进程数量最多为11个（1个在运行）。
考点7A：进程与线程（高频辨析）
🔴【进程】：传统教材中是资源分配和保护的基本单位，拥有独立地址空间及资源。
🔴【线程】：是处理器调度的基本单位；同一进程内线程共享代码、数据和打开文件等进程资源，但拥有各自的程序计数器、寄存器和栈。
⚫️【坑点】：线程既有共享的进程资源，也有私有运行现场；“线程之间完全不共享资源”和“线程没有私有状态”都错误。
考点8：进程调度算法
🔴【先来先服务（FCFS）】：按就绪/到达先后服务，通常为非抢占，实现简单；长作业在前会使短作业久等，产生护航效应，但平均周转时间并非在任何任务集合中都必然最长。
🔴【短作业优先（SJF）】：选择运行时间最短的进程先执行。平均周转时间最短（所有作业同时到达时）。需要预知运行时间。
🔴【高响应比优先（HRRN）】：响应比 = (等待时间 + 要求服务时间) / 要求服务时间。兼顾等待时间和执行时间，既照顾短作业，又不使长作业长期等待。
🔴【时间片轮转（RR）】：分时系统使用。每个进程轮流获得固定时间片的CPU。
🔴【优先级调度】：选择优先级最高的进程执行。若使当前运行进程总是优先级最高的进程，应选择高优先级优先调度算法。
⚫️【坑点】：
所有作业同时到达时，平均周转时间最短的是短作业优先。
响应比高优先算法中：等待时间相同时，运行时间短的优先；运行时间相同时，等待时间长的优先。
某些调度策略会提高I/O密集型或交互型任务优先级，以改善响应并让I/O与CPU并行；优先级仍取决于系统目标，不能把“计算型必低于I/O型”当作通则。
考点9：进程同步与互斥
🔴【同步】：多个进程因合作而产生的直接制约关系（如流水线上下工序）。
🔴【互斥】：多个进程因竞争同一资源而产生的间接制约关系（如多个进程争抢打印机）。
🔴【临界资源】：一次只允许一个进程访问的资源（如打印机）。
🔴【临界区】：访问临界资源的那段程序代码。
🔴【信号量】：用于进程同步与互斥的机制。P操作（申请资源）和V操作（释放资源）是原子操作。
在部分教材的计数信号量定义中，值大于0表示可用资源数，值小于0时绝对值表示等待进程数；有些实现不允许用户直接观察负值，因此该解释依赖具体P/V定义。
⚫️【坑点】：
临界区是一段程序，不是缓冲区或数据区！
P、V操作既能实现进程互斥，也能实现进程同步。
按“P操作先减1、负值入队”的教材定义，有3个进程且最多允许2个进入时，信号量可出现2、1、0、-1；题目若采用不同信号量语义，不能机械套用。
信号量初值为3，当前为1 → 可用资源数M=1，等待进程数N=0。
考点10：死锁（极高频）
🔴【定义】：两个或多个进程因互相等待对方占有的资源而无限期阻塞的现象。
🔴【死锁产生的四个必要条件】（全部满足才可能死锁）：
互斥：资源一次只能分配给一个进程。
请求与保持：进程占有一些资源的同时，请求新的资源。
不可剥夺：进程已获得的资源在使用完毕前不能被强行剥夺。
环路等待：若干进程形成头尾相接的循环等待资源链。
🔴【死锁的预防】：在资源分配规则上破坏四个必要条件之一。
破坏“请求与保持”：一次性申请所有资源。
破坏“环路等待”：资源有序分配（给资源类型编号，按规定顺序申请）。它是常见预防方法之一，但可能降低资源利用率或增加编程约束。
🔴【死锁的避免】：银行家算法。系统分配资源前先计算是否会导致不安全状态。
🔴【死锁与安全状态的关系】：
死锁状态一定是不安全状态。
不安全状态不一定会死锁（但可能导致死锁）。
安全状态一定不会死锁。
⚫️【坑点】：
死锁的预防方法中，不太可能的一种是摒弃互斥条件（很多资源本身就是互斥的，无法摒弃）。
可共享资源通常能减少互斥竞争，但硬盘等设备还涉及控制块、缓冲区和锁，不能绝对说“不会引起死锁”。
死锁发生后可通过终止进程、资源剥夺、回滚等恢复；“不能被中断或唤醒”不是死锁定义，不可剥夺条件只表示资源在正常规则下不能被强制收回。
系统中出现环路不一定死锁（还需要其他条件）。
第三章：存储管理
考点11：存储管理的目的
🔴【主要功能】：内存的分配、回收、地址转换、存储保护和内存扩充（虚拟内存）。
🔴【地址转换（重定位）】：
静态重定位：程序装入时一次性完成地址转换。
动态重定位：程序执行过程中借助地址转换硬件完成逻辑地址到物理地址的转换。现代通用操作系统通常采用动态地址转换，但不能写成“所有现代操作系统只采用这一方式”。
⚫️【坑点】：静态重定位的时机是程序装入时，动态重定位的时机是程序运行时。
考点12：连续分配方式
🔴【单一连续分配】：内存分为系统区和用户区，只能用于单用户、单任务操作系统。最简单。
🔴【固定分区分配】：内存划分为多个固定大小的分区，每个分区可装入一道程序。分区大小可以不同，但预先固定。有内部碎片。
🔴【动态分区分配】：根据进程需要动态分配内存空间。有外部碎片。可通过“紧凑”解决。
🔴【首次适应算法】：按地址递增的顺序分配空闲分区。
⚫️【坑点】：固定分区中，分区大小可以不同但预先固定。首次适应算法要求空闲区按地址递增登记。
考点13：分页存储管理（离散分配）
🔴【核心思想】：将进程的逻辑地址空间分成页，将内存的物理地址空间分成块。页和块的大小相等。
🔴【页表】：记录页号到物理块号的映射关系。每个进程拥有一张页表。
🔴【地址结构】：逻辑地址 =页号 + 页内地址。物理地址 =块号 + 块内地址（块内地址 = 页内地址）。
🔴【访问内存次数】：未引入快表时，每次存取指令或数据需2次访问内存（第一次查页表，第二次存取数据）。
⚫️【坑点】：
块内位移量等于页内位移量，是因为页和块的大小相等。
每个进程拥有自己的页表，不是所有进程共享一张页表。
8个页面，每页1024字节 → 逻辑地址需要3（页号）+ 10（页内地址）= 13位二进制表示。
考点14：分段存储管理
🔴【核心思想】：将进程的地址空间按逻辑信息（代码段、数据段等）分成若干段。每段从0开始编址，大小不固定。
🔴【地址结构】：逻辑地址 =段号 + 段内地址。
🔴【与分页的区别】：
页是进程逻辑地址空间的固定大小划分，页框/块是物理内存的固定大小划分；分页主要出于系统管理需要，对用户通常不可见。段按程序逻辑信息划分，对用户可见。
页大小固定，段大小不固定。
分页是一维地址空间，分段是二维地址空间（段名+段内地址）。
考点15：段页式存储管理
🔴【核心思想】：分段 + 分页的结合。用分段方法管理用户地址空间，用分页方法管理物理存储空间。
⚫️【坑点】：段页式管理的实现原理是：用分段方法来分配和管理用户地址空间，用分页方法来管理物理存储空间。
考点16：虚拟存储管理（核心考点）
🔴【虚拟存储的定义】：基于局部性原理，将进程的部分程序和数据装入内存，其余放在外存，逻辑上扩充内存容量。
🔴【局部性原理】：程序执行时对内存访问的不均匀性（时间局部性：某条指令被执行后不久可能再次被执行；空间局部性：某存储单元被访问后不久其附近的存储单元也可能被访问）。
🔴【虚拟存储的实现】：常见为请求分页，也可采用请求分段或段页式；请求分页需要页表、缺页异常处理、地址变换和页面置换等支持。
⚫️【坑点】：虚拟存储的目的是逻辑上扩充主存容量（让用户感觉内存更大），不是物理上扩大主存。
考点17：页面置换算法（计算高频）
🔴【最佳置换算法（OPT）】：选择以后永不使用或最长时间内不再使用的页面淘汰。理论最优，但无法实现（无法预知未来）。
🔴【先进先出置换算法（FIFO）】：淘汰最早进入内存的页面。可能出现Belady异常（分配的物理块数增加，缺页次数反而增加）。
🔴【最近最久未使用置换算法（LRU）】：淘汰过去最长时间未被访问的页面，通常表现较好，但不是理论最优；精确实现开销较大，实际系统常采用近似LRU策略。
🔴【Clock/Second-Chance】：利用访问位循环检查页面，是近似LRU算法；基础Clock主要看访问位。增强Clock还结合修改位，通常优先选择访问位=0、修改位=0的页。
⚫️【坑点】：
精确LRU需要维护访问次序，通常需硬件访问位、时间戳或栈等支持，成本较高；实际常用Clock等近似实现。
系统发生抖动（频繁的页面换入换出）会使系统效率降低。
页面置换计算必须明确页框初始是否为空、页面序列和算法细节；应逐次记录页框状态，不能脱离前提死记某个缺页次数。
第四章：设备管理
考点18：I/O设备分类
🔴【块设备】：以数据块为单位传输，可随机访问。如：磁盘。I/O控制主要采取DMA方式。
🔴【字符设备】：以字符为单位传输，顺序访问。如：键盘、鼠标、打印机。
⚫️【坑点】：磁盘属于块设备，键盘属于字符设备。
考点19：缓冲技术
🔴【缓冲的作用】：
缓和CPU与I/O设备之间速度不匹配的矛盾。
减少对CPU的中断频率。
协调数据处理单位与传输单位不匹配的问题。
🔴【缓冲的实现】：软件缓冲通常是在内存中分配一段空间来实现的（不是在磁盘上）。
⚫️【坑点】：判断题说“软件缓冲通常是在磁盘上分配一段空间” →错！是在内存中。
考点20：SPOOLing技术（假脱机技术）
🔴【定义】：将独占设备改造为共享设备的技术（如打印机共享）。
🔴【核心】：利用磁盘作为缓冲区（输入井/输出井），在磁盘上形成队列。
🔴【作用】：实现虚拟设备功能。
⚫️【坑点】：SPOOLing技术的核心是把独占设备变为共享设备。
考点21：磁盘调度算法
🔴【先来先服务（FCFS）】：按请求顺序服务。
🔴【最短寻道时间优先（SSTF）】：选择距离当前磁头位置最近的磁道访问。每次移动距离最短。
🔴【扫描算法（SCAN，电梯算法）】：磁头按当前方向移动并服务沿途请求，到磁盘端点后反向；LOOK在当前方向无更远请求时即反向，不必走到物理端点。
🔴【循环扫描（C-SCAN）】：只在一个方向上服务请求，到端点后快速返回另一端再继续；C-LOOK则在最远请求处回绕。
⚫️【坑点】：
磁盘访问时间 =寻道时间 + 旋转延迟时间 + 读写时间。不包括CPU调度时间。
最短寻道时间优先算法会导致远处的请求饥饿。
第五章：文件管理
考点22：文件与文件系统
🔴【文件】：具有文件名的信息集合。
🔴【文件控制块（FCB）】：文件存在的标志，记录文件的属性信息（文件名、大小、地址、权限等）。
🔴【目录结构】：
单级目录：所有文件在同一目录下（有重名问题）。
二级目录：主目录 + 用户目录。
多级目录（树型目录）：最常用，允许不同用户的文件同名。形式为树型结构。
⚫️【坑点】：为了允许不同用户的文件具有相同的文件名，文件系统中采用多级目录。
考点23：文件的逻辑结构
🔴【流式文件（无结构文件）】：文件中的信息是字节流，无结构。如：文本文件（.txt）。
🔴【记录式文件（有结构文件）】：文件由一组记录组成。如：数据库文件。
⚫️【坑点】：数据库文件的逻辑结构形式是记录式文件。
考点24：文件的物理结构（分配方式）
🔴【连续分配】：文件占用连续磁盘块，支持顺序和随机访问，访问快；难以动态增长并会产生外部碎片。
🔴【链接分配】：文件占用不连续的磁盘块，通过指针链接。存取时必须按指针顺序进行，效率较低。支持顺序存取，不支持随机存取。
🔴【索引分配】：文件可占用不连续磁盘块，索引块保存各数据块地址，支持直接访问。这里的“索引分配”是文件物理块索引，不等同于数据库中“关键字→记录地址”的内容索引。
⚫️【坑点】：
对文件的存取时必须按指针进行，效率较低 →链接文件。
文件的逻辑结构、存取方法和物理分配方式是不同层次：顺序存取不要求文件一定连续分配，链接分配也可以顺序访问。
同一文件的不同副本可以采用相同或不同物理结构，取决于文件系统和介质；不存在“必须不同”的通则。
考点25：文件的存取方法
🔴【顺序存取】：按文件中的记录顺序依次访问（磁带、链接文件）。
🔴【随机存取（直接存取）】：可访问任意位置的记录（磁盘、索引文件）。

🔥 本章高频简答题汇总
1. 简述进程和程序的区别。
🔵【踩分点】：① 进程是动态的，程序是静态的。② 进程有生命周期（创建到撤销），程序是永久的。③ 进程 =程序 + PCB + 数据集合。④ 一个程序可对应多个进程。
2. 简述进程的三种基本状态及其转换。
🔵【踩分点】：①就绪态：等待CPU。②运行态：占有CPU执行。③阻塞态：等待某事件（如I/O）。转换：就绪→运行（调度选中），运行→就绪（时间片完），运行→阻塞（等待事件），阻塞→就绪（事件完成）。
3. 简述死锁产生的必要条件及预防方法。
🔵【踩分点】：必要条件：互斥、请求与保持、不可剥夺、环路等待。预防通过破坏至少一个必要条件；资源有序分配法破坏环路等待，是常见方法。
4. 简述分页和分段的区别。
🔵【踩分点】：① 页是物理单位（为管理内存），段是逻辑单位（为用户需求）。② 页大小固定，段大小不固定。③ 分页是一维地址空间，分段是二维地址空间。
5. 简述虚拟存储的原理。
🔵【踩分点】：基于局部性原理（程序执行时对内存访问不均匀），将程序的部分装入内存，部分放在外存，在逻辑上扩充内存容量。当访问的页面不在内存时，产生缺页中断，将其调入内存。
6. 简述SPOOLing技术的作用。
🔵【踩分点】：将独占设备（如打印机）改造为共享设备，实现虚拟设备功能，提高了设备利用率。
