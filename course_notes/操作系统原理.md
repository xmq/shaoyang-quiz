# 操作系统原理讲义

## 一、操作系统概述
<!-- exam: B | 单选、判断 -->

### 1. 定义与作用

操作系统是管理计算机硬件与软件资源、控制程序运行，并向用户和应用程序提供服务的系统软件。它位于硬件与应用软件之间。

主要作用有：

1. 管理处理机，决定哪个程序使用 CPU。
2. 管理内存，完成分配、回收和保护。
3. 管理文件，组织数据并控制访问。
4. 管理设备，协调主机与外设工作。
5. 提供接口，包括命令接口、图形界面和程序接口。

### 2. 基本特征

- 并发：多个程序在一段时间内交替推进。
- 共享：多个程序共同使用系统资源，可分互斥共享和同时共享。
- 虚拟：把一个物理资源抽象成多个逻辑资源，或扩大可见容量。
- 异步：程序以不可预知但受控制的速度向前推进。

并发是“时间段内共同推进”，并行是“同一时刻同时执行”。单核系统可以并发，多核系统才可能真正并行。

### 3. 常见分类

- 批处理系统：成批提交作业，吞吐量较高，交互性弱。
- 分时系统：多个用户按时间片共享主机，强调及时交互。
- 实时系统：在规定时间内响应，分硬实时和软实时。
- 网络操作系统：为网络资源共享和通信提供服务。
- 分布式操作系统：把多台计算机组织成统一系统。
- 嵌入式操作系统：面向专用设备，资源受限、可靠性要求高。

### 4. 操作系统怎样管理资源

处理机管理的核心是进程控制和调度：操作系统创建、撤销进程，保存运行现场，并在多个就绪进程之间分配 CPU。存储管理负责内存分配、地址转换、保护和扩充。文件管理把外存数据组织成文件和目录，设备管理则通过驱动程序屏蔽不同硬件细节。

操作系统还承担保护职责。例如用户程序不能随意修改内核数据，一个进程也不能未经许可访问另一个进程的内存。资源共享与资源保护并不矛盾：共享必须在权限和同步规则下进行。

常见用户接口有命令行界面和图形用户界面；程序接口主要表现为系统调用。命令解释程序和桌面环境是用户与系统交互的工具，但不等于整个操作系统内核。

### 5. 常见类型判断

银行夜间集中处理大量账单，突出吞吐量，具有批处理特点；多人同时登录服务器交互，强调响应，具有分时特点；汽车制动、工业控制必须在截止时间内完成，属于实时应用。实时系统的关键是可预测、守时，而不是平均速度最高。

网络操作系统强调各主机自主并提供共享服务；分布式操作系统更强调把多台机器组织成统一整体。二者都使用网络，但透明程度和管理方式不同。

## 二、用户态、内核态和系统接口
<!-- exam: A | 单选、判断、场景题 -->

CPU 通常设置用户态和内核态。内核态可以执行特权指令并访问关键资源；普通应用运行在用户态，不能直接操作硬件。

系统调用是应用程序主动请求操作系统服务的接口，例如创建进程、读写文件。中断多由外部设备发出，例如键盘输入或 I/O 完成；异常由当前指令执行引起，例如除零、越界或缺页。

判断要点：系统调用是“主动请求服务”，中断和异常是“触发控制转移的机制”；库函数不一定产生系统调用。

## 三、进程与线程
<!-- exam: A | 单选、判断、计算、场景题、简答 -->

### 1. 程序、进程和线程

程序是静态的指令集合；进程是程序的一次执行过程，是资源分配和保护的基本单位。进程控制块 PCB 保存进程标识、状态、现场和资源等信息，是进程存在的重要标志。

线程是进程中的执行单元，是 CPU 调度的基本单位。同一进程中的线程共享地址空间和打开的文件，但各自拥有程序计数器、寄存器和栈。

为什么要从“程序”引入“进程”？程序文件本身不会主动占用 CPU，也没有运行状态；操作系统需要用进程描述一次具体执行，为它分配内存和文件等资源，并在切换时保存现场。同一个程序可以同时形成多个进程，例如打开两个相互隔离的浏览器实例；一个进程也可能在运行中装入多个程序模块。因此“程序与进程一一对应”是错误的。

为什么还要引入线程？一个进程内若有界面响应、网络收发和数据处理等任务，多个线程可以共享进程资源并发推进，创建和切换通常比多个独立进程轻量。但共享也带来边界：线程间不是天然安全的；多个线程修改同一数据时仍要用互斥锁等机制，某线程破坏共享内存也可能影响整个进程。

### 2. 基本状态

```text
就绪 --调度--> 运行 --等待事件--> 阻塞
  ^              |                 |
  |--时间片到----|------事件完成----|
```

- 就绪：除 CPU 外，运行条件已经具备。
- 运行：正在使用 CPU。
- 阻塞：正在等待某个事件或资源。

阻塞进程必须先因事件完成变为就绪，不能直接变为运行。进程切换需要保存和恢复现场，通常比同一进程内的线程切换开销大。

### 3. 调度方法的基本比较

- 先来先服务 FCFS：按到达顺序，简单公平，但短作业可能被长作业拖延。
- 短作业优先 SJF：优先服务时间短的作业，平均等待时间较短，但长作业可能饥饿。
- 优先级调度：优先运行高优先级任务，可用“老化”缓解低优先级饥饿。
- 时间片轮转 RR：每个就绪进程轮流使用一个时间片，适合分时系统。

基础题只需识别规则：时间片过大时 RR 接近 FCFS；时间片过小时进程切换开销增加。

基础计算常给出到达时间和服务时间，要求按算法排出执行顺序。周转时间＝完成时间−到达时间，等待时间＝周转时间−实际运行时间。例如进程在 0 时刻到达、运行 3 个时间单位、5 时刻完成，则周转时间为 5，等待时间为 2。计算前必须先判断是否允许抢占，不要仅凭算法名称猜顺序。

### 4. 场景中的调度、同步和通信

- 收银前台要让多个任务轮流获得 CPU、保持交互响应，体现调度；时间片轮转适合交互任务，但不能保证某笔业务一定最先完成。
- 多个收银线程同时修改同一库存数量，需要互斥，避免更新互相覆盖；“先扣库存再生成出库记录”等先后关系还涉及同步。
- 线程间共享地址空间，交换数据快但相互影响大；进程间地址空间隔离，通常要通过管道、消息队列、共享内存等 IPC 通信。
- 操作系统的互斥只解决临界区并发访问。跨服务器订单“只处理一次”、数据库多表一致更新，还要依靠事务、唯一订单号和幂等校验，不能只答“加线程锁”。

场景题先找被争用的对象，再判断要求：若强调“同一时刻只能一个访问”，答互斥；若强调“必须先 A 后 B”，答同步；若双方互相等待对方持有的资源，才进一步分析死锁。

## 四、同步、互斥与死锁
<!-- exam: A | 单选、判断、多选、场景题、简答 -->

### 1. 同步与互斥

临界资源是一次只允许一个进程使用的资源；访问临界资源的代码段称临界区。互斥强调“不能同时使用”，同步强调“必须按先后顺序协作”。

信号量只能通过 P、V 操作改变。P 表示申请资源或等待条件，V 表示释放资源或发出通知。互斥信号量通常初值为 1，资源信号量初值通常等于资源数量。

常见模型：

- 生产者—消费者：生产者等待空缓冲区，消费者等待满缓冲区，访问缓冲区时还要互斥。
- 读者—写者：多个读者可以同时读，写者与其他读写必须互斥。
- 管程：把共享数据和操作封装起来，由系统保证管程内部互斥。

### 2. 死锁

死锁是多个进程因竞争资源而永久等待，若无外力不能继续运行。

四个必要条件：互斥、请求并保持、不剥夺、循环等待。破坏其中任意一个可预防死锁。常见处理方式包括预防、避免、检测和解除。

安全状态表示存在一种进程完成顺序，使所有进程都可以依次获得资源并结束。安全状态一定没有死锁；不安全状态只是有可能发生死锁，不等于已经死锁。

## 五、存储管理
<!-- exam: A | 单选、判断、填空、计算、场景题 -->

### 1. 连续分配与碎片

连续分配把一个进程放入连续内存区域。首次适应选择第一个足够大的空闲区，最佳适应选择最小的足够区，最坏适应选择最大的空闲区。

内部碎片是已分配区域内部没有被利用的空间；外部碎片是空闲空间总量足够但分散不连续。

### 2. 分页与分段

分页把逻辑空间分成固定大小的页，把物理内存分成同样大小的页框。页表记录页号到页框号的映射。分页便于离散分配，主要产生内部碎片。

分段按程序的逻辑部分划分，如代码段、数据段和栈段，每段长度可以不同。段表记录段基址和段长。分段便于共享和保护，但可能产生外部碎片。

页表快表 TLB 是缓存近期页表项的高速存储器。TLB 未命中只表示要查页表，并不等于页面不在内存；只有页表表明页面未装入时才发生缺页。

### 3. 虚拟存储

虚拟存储利用局部性，只把当前需要的部分装入内存，使程序看到的逻辑空间可以大于物理内存。

- 时间局部性：最近访问的内容可能再次访问。
- 空间局部性：某地址附近的内容可能很快被访问。

常见页面置换方法：

- 最佳 OPT：淘汰未来最久不用的页，只作理论比较。
- 先进先出 FIFO：淘汰最早进入的页，可能出现增加页框反而缺页更多的现象。
- 最近最少使用 LRU：淘汰过去最长时间未访问的页。
- CLOCK：循环检查访问位，近似实现 LRU。

抖动是进程频繁缺页换页、实际执行效率很低的现象，常与分配页框不足或并发程度过高有关。

### 4. 覆盖、交换与虚拟存储辨析

覆盖由程序员或编译工具事先把互斥使用的程序段放到同一内存区域，主要解决早期内存不足；交换由操作系统把整个进程暂时移到外存再调回；虚拟存储由系统和硬件按页或段自动调入所需部分。三者都能缓解内存限制，但管理粒度和透明程度不同。

逻辑地址是程序使用的地址，物理地址是主存单元的实际地址。动态地址转换由硬件在程序运行时完成，使程序装入不同物理位置仍可执行，并为保护、共享和虚拟存储提供基础。

内存保护常通过基址/限长、页表权限位等实现。保护的目的不是禁止一切共享，而是让共享只能通过受控区域和权限进行。

## 六、文件系统
<!-- exam: B | 单选、判断、场景题 -->

文件是具有名称的一组相关信息。文件控制块 FCB 记录文件属性和存储位置；类 Unix 系统常由目录项保存文件名和 inode 号，inode 本身通常不保存文件名。

文件物理分配方式：

- 连续分配：顺序和随机访问快，但扩展困难。
- 链接分配：扩展方便，但随机访问差。
- 索引分配：通过索引块保存数据块地址，支持随机访问，但需要额外索引空间。

目录可分单级、两级、树形和图形目录。绝对路径从根目录开始，相对路径从当前目录开始。

硬链接让多个目录项指向同一个 inode，通常不能跨文件系统；符号链接保存目标路径，可以跨文件系统，但目标删除后可能失效。

## 七、设备与磁盘管理
<!-- exam: A | 单选、判断、场景题、简答 -->

### 1. I/O 控制

- 程序查询：CPU 不断检查设备状态，控制简单但浪费 CPU 时间。
- 中断方式：设备完成后通知 CPU，提高 CPU 与设备并行程度。
- DMA：控制器直接在设备与主存间成块传送，CPU 负责初始化和结束处理。

缓冲用于协调 CPU 与设备速度差、减少中断次数，但不会改变设备本身速度。

### 2. 磁盘调度

- FCFS：按请求到达顺序，公平简单。
- SSTF：先服务离当前磁头最近的请求，平均寻道较短，但远端请求可能长期等待。
- SCAN：磁头像电梯一样沿一个方向移动，到端点后反向。
- LOOK：只移动到本方向最远请求，不必走到物理端点。
- C-SCAN：只沿一个方向服务，回程不服务，等待更均匀。

机械磁盘访问时间主要由寻道时间、旋转延迟和传输时间组成；固态硬盘没有机械寻道过程。

### 3. 设备管理中的易混概念

设备驱动程序是操作系统控制具体设备的软件，向上提供统一接口、向下处理设备寄存器和中断。设备控制器是硬件部件，负责执行主机发来的 I/O 命令。二者不能混为一种软件或硬件。

中断使 CPU 不必一直等待慢速设备，但每次数据处理仍可能需要 CPU 参与；DMA 更适合成块高速传输。脱机 I/O、假脱机 SPOOLing 则利用外存把独占设备改造成逻辑上的共享设备，打印队列是常见例子。

通道是具有较强 I/O 控制能力的专用处理部件，能够执行通道程序并进一步减轻 CPU 负担。基础题常按 CPU 介入程度比较“查询—中断—DMA—通道”。
