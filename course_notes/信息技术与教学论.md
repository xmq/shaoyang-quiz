# 📚 信息科技与教学论

使用说明：🔴红色 = 课程标准、核心考点与判断改错点；🔵蓝色 = 简答题得分词、设计步骤与评价术语；⚫️黑色 = 适用条件、案例和不同教材口径。本章同时涉及《义务教育信息科技课程标准（2022年版）》和《普通高中信息技术课程标准（2017年版2020年修订）》，答题必须先辨明学段与课标版本。

第一章：课程标准与课程性质

考点1：学段、课程名称与现行依据

🔴【义务教育】：2022 年秋季起执行《义务教育信息科技课程标准（2022年版）》，课程名称为“信息科技”，并从综合实践活动中独立设置。

🔴【普通高中】：现行国家课标通常指《普通高中信息技术课程标准（2017年版2020年修订）》，课程名称为“信息技术”。

⚫️【口径陷阱】：旧教材、地方教材和不同学段可能使用“信息技术课程五大性质”等归纳。不能把“基础性、综合性、实践性、层次性、工具性”写成所有现行国家课标唯一固定表述，更不能断言“工具性是唯一的本质属性”。

考点2：义务教育信息科技课程性质与理念

🔴【课程性质】：义务教育信息科技课程是现代科学技术领域的重要组成部分，主要研究以数字形式表达的信息及其应用中的科学原理、思维方法、处理过程和工程实现；课程具有基础性、实践性和综合性。

🔵【课程理念得分词】：反映数字时代正确育人方向；构建逻辑关联的课程结构；遴选科学原理和实践应用并重的课程内容；倡导真实性学习；强化素养导向的多元评价。

⚫️【判断陷阱】：信息科技课程不等于软件操作培训。操作技能是载体，还要理解数据、算法、网络、系统等原理，发展计算思维、创新能力和信息社会责任。

考点3：核心素养

🔴【四项核心素养】：信息意识、计算思维、数字化学习与创新、信息社会责任。四者相互支持、相互渗透，共同促进数字素养与技能提升。

🔵信息意识：对信息的敏感度和信息价值的判断力；能评估来源、可靠性和时效性，主动利用信息解决问题。

🔵计算思维：运用计算机科学领域的思想方法，通过抽象、分解、建模、算法设计、自动化等活动形成问题解决方案。

🔵数字化学习与创新：合理选择数字工具和资源，管理学习过程，协同学习并创造性解决问题、形成作品。

🔵信息社会责任：遵守法律规范与伦理道德，保护隐私和安全，尊重知识产权，负责任地发布和使用信息。

⚫️【判断陷阱】：计算思维不等于“只会编程”；数字化学习不等于“用了电脑上课”；信息意识不等于信息越多越好。

第二章：学习理论与教学观

考点4：主要学习理论

🔴【行为主义】：关注刺激—反应、强化、练习和可观察行为；适合基础操作训练，但若只模仿步骤，可能忽视理解与迁移。

🔴【认知主义】：关注信息加工、知识结构、先备知识和认知策略；教学可运用先行组织者、概念图、分块与精加工。

🔴【建构主义】：学习者在一定情境中借助协作和必要帮助，利用学习资源主动建构意义；强调情境、协作、会话和意义建构。

⚫️【教师角色】：强调学生主体不等于教师退出。教师仍需设计环境、提供支架、组织反馈并纠正关键误解。

🔴【最近发展区与支架】：任务应略高于学生独立完成水平，在帮助下可完成；随着能力提升逐步撤去支架。

第三章：教学设计

考点5：教学设计基本流程

🔵【常用答题骨架】：分析学习需要与课标 → 分析学习者 → 分析内容和任务 → 制定素养导向目标 → 选择策略、资源与环境 → 设计活动和评价 → 实施、反馈与修订。

⚫️【系统观】：目标、活动和评价应一致。若目标要求“设计并实现算法”，评价却只考术语记忆，就没有实现教—学—评一致性。

考点6：学习者分析

🔴【分析维度】：起点知识与技能、认知发展、学习动机与兴趣、学习风格和个体差异、数字设备经验、特殊学习需要。

🔵【起点能力确认】：可用访谈、问卷、前测、作品分析、课堂观察等，不应只凭教师主观猜测。

考点7：教学目标

🔴【现行导向】：目标应指向核心素养，并转化为可观察、可评价的学习表现。传统“三维目标”可用于理解知识技能、过程方法、情感价值等维度，但在现行课标答题中不应取代核心素养导向。

🔴【ABCD 表述】：A 学习者，B 可观察行为，C 条件，D 达成程度。不是每句话都必须机械写齐四项，但行为主体、表现和标准应尽量清楚。

🔵【行为动词】：识别、描述、比较、解释、设计、调试、评价、创作等；“了解、掌握、培养、提高”若无表现证据，难以直接测量。

⚫️【示例】：在给定的校园数据表中（C），学生（A）能使用合适图表呈现趋势并解释选择理由（B），关键字段和结论完整（D）。

考点8：重点与难点

🔴【教学重点】：由课标要求、内容结构和育人价值决定，是本课必须掌握的核心内容。

🔴【教学难点】：由内容特性和具体学生的认知基础共同决定，是理解或操作障碍所在，具有相对性和动态性。

⚫️【判断陷阱】：重点不一定就是难点；不能只按教师觉得难来确定，也不能把每个环节都列为重点。

第四章：教学方法与学习方式

考点9：任务驱动教学

🔴【定义】：把知识与技能融入具有意义的任务，学生在分析、实施和评价任务的过程中学习。

🔵【常用流程】：创设情境 → 呈现并明确任务 → 分析任务、形成方案 → 自主/协作探究与实施 → 展示交流 → 评价反思与迁移。

🔵【设计得分词】：真实性、目标一致、难度适切、层次递进、开放选择、成果可评价、安全可实施。

⚫️【“核心”口径】：不同教材对“任务设计是核心”“怎样驱动是关键”等表述可能不同。简答题应写清任务如何承载目标、学生如何主动解决问题、教师如何提供支架，而不是死背一句口号。

⚫️【常见失误】：把任务写成“照着教师演示操作”；情境热闹但与目标无关；只评价作品外观；任务过大却没有子任务和支架。

考点10：项目式学习与问题解决

🔴【项目式学习】：围绕真实、复杂且可探究的问题，持续开展设计、制作、迭代与展示，形成公开成果；项目比单一课堂任务更综合、周期更长。

🔵【流程骨架】：提出驱动性问题 → 规划与分工 → 调研/设计 → 制作与测试 → 迭代改进 → 展示答辩 → 反思评价。

🔴【探究式学习】：学生经历提出问题、作出假设、收集与分析证据、形成解释、交流评价等过程；教师需控制探究开放度并提供必要支架。

考点11：讲授、演示、练习与讨论

🔴【讲授法】：适合系统解释概念、原理和背景，效率高；应结合提问、案例、可视化和练习，避免长时间单向灌输。

🔴【演示—练习】：教师示范关键操作并口述思路，学生模仿、独立练习和迁移。演示前应确认屏幕可见，步骤应分段，并预留学生操作时间。

🔴【讨论法】：适合有多种观点、需要比较论证的问题。低年级或新知识阶段并非“绝对不能用”，但问题难度、语言要求、先备知识和组织方式要适合学生。

考点12：合作学习

🔴【基本特征】：共同目标、积极相互依赖、个体责任、面对面促进性互动、合作技能与小组反思。

🔵【组织要点】：异质分组、角色明确、成果与个人责任并重、过程巡视、及时反馈。4～6 人是常见建议，不是所有任务的硬性人数标准。

⚫️【防止“搭便车”】：设置个人可追踪任务、随机汇报、同伴互评与个人反思；仅把学生围坐在一起不等于合作学习。

考点13：WebQuest、微课与翻转课堂

🔴【WebQuest】：以网络资源为主要信息来源的探究活动。经典结构常包括导言、任务、过程、资源、评价和结论；不同模板可能把资源并入过程或增加教师页，不能脱离教材断言只有一种绝对结构。

🔴【微课】：围绕相对聚焦的学习目标，以短小数字化学习资源为核心并配套学习活动。视频是常见载体，但“微课=一个短视频”不完整。

⚫️【时长口径】：5～10 分钟是常见经验值，不是所有学段、内容和国家标准的法定上限；应以目标聚焦、认知负荷适当和学习效果为准。

🔴【翻转课堂】：把部分直接讲授或知识获取前移到课前，课堂时间更多用于问题解决、协作和个别化指导。它不是简单地“课前看视频、课上做作业”，也不意味着教师作用减弱。

第五章：教学评价

考点14：按评价时机和参照分类

🔴【诊断性评价】：教学前或新单元开始时了解起点和困难，如前测、访谈、先备作品分析。

🔴【形成性评价】：教学过程中收集证据、反馈并改进学习，如观察、提问、调试记录、阶段作品和学习单。

🔴【总结性评价】：阶段或课程结束后判断目标达成，如期末考试、综合作品与答辩。

🔴【目标/标准参照评价】：对照预定标准判断达成程度；【常模参照评价】：与群体表现比较；【个体内差异评价】：与学习者自己过去的表现比较。

⚫️【判断陷阱】：一次测验属于哪类，不只看形式，还看使用目的。相同测验用于课前诊断、课中反馈或期末认定，性质可能不同。

考点15：多元评价与评价工具

🔴【评价原则】：素养导向、目标一致、过程与结果结合、定性与定量结合、主体多元、关注差异、以评促学。

🔵【信息科技常用证据】：纸笔测验、上机操作、项目作品、程序运行结果、设计文档、过程日志、答辩、档案袋。不能断言“纸笔+上机”是所有情境唯一最合适的总结性评价。

🔴【量规 Rubric】：把评价维度和不同水平的表现描述清楚。优秀量规应对应目标、维度可区分、描述可观察，避免只用“好/一般/差”。

🔴【作品评价】：既看功能正确性、算法/技术合理性，也看创意、规范、安全责任、协作过程和表达；评价维度应与任务目标匹配。

考点16：测验质量指标

🔴【信度】：测量结果的一致性和稳定性；【效度】：是否真正测到了预期目标。信度是效度的必要但非充分条件。

🔴【难度】：题目通过率越高，通常表示题目越容易（采用不同难度指数定义时要看公式）；【区分度】：区分不同水平学习者的能力。

🔵【客观题命制】：题干清楚、只有一个最佳答案、干扰项有迷惑性且同质、避免无意线索和双重否定；判断题避免使用“永远、全部、绝不”等无条件绝对词。

第六章：课堂实施与案例分析

考点17：信息科技课堂组织

🔵【上机课流程】：设备与账号检查 → 明确目标和安全规则 → 分段示范/任务发布 → 学生实践、教师巡视 → 保存与提交 → 展示评价 → 退出账号和整理设备。

🔴【故障应对】：先保证人身和数据安全，再定位电源、连接、账号、网络、软件、文件路径等；准备离线资源、备用设备或同伴协作方案。不能把技术故障时间全部变成无目标自由活动。

🔴【差异化教学】：提供基础任务、进阶任务和挑战任务；给予步骤卡、半成品、示例数据等支架，同时为高水平学生提供开放拓展。

考点18：案例分析答题框架

🔵【先判断】：指出做法是否符合素养导向、学生主体、目标一致和安全伦理。

🔵【再依据】：结合课标、学习理论、教学方法或评价原则解释原因。

🔵【后改进】：从目标、任务、支架、互动、资源、评价和课堂管理提出可执行措施。

🔵【补证据】：说明如何观察改进是否有效，如作品质量、调试过程、学生解释、量规得分或反思记录。

⚫️【高频问题】：教师包办操作、任务脱离真实问题、只重结果不看过程、评价主体单一、忽视版权隐私、目标不可测、合作学习无个人责任。

口诀速记

“义教信息科技二二课标，高中信息技术一七二〇修订；核心素养信计数社；目标、活动、评价要一致；任务真实有梯度，项目规划做测改；讨论不是任何阶段都禁用，合作人数不是硬规定；诊断看起点，形成促改进，总结判达成；课前先分析，课中给支架，课后用证据评价。”
