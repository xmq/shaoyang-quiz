# 信息技术与教学论基础

本讲义面向事业单位基础专业知识复习，重点掌握信息技术教育的基本概念、课程特点、学习理论、教学设计要素、常用教学方法、教学媒体、评价方式和教师基本职责。复习时以定义、分类、作用、区别和简答得分词为主。

## 一、信息技术教育概述

### 1. 基本含义

信息技术教育不仅教授软件操作，还包括信息获取与处理、数据与算法、网络与系统、数字化工具应用，以及利用技术解决问题的意识、方法和责任。

其基本作用可以概括为：帮助学习者形成信息意识，掌握信息处理方法，发展计算思维和实践能力，养成安全、合法、负责任地使用技术的习惯。

### 2. 基本特点

- 基础性：为学习、工作和生活提供基本信息能力。
- 实践性：强调动手操作、问题解决和作品形成。
- 综合性：涉及技术、社会、伦理和其他学科知识。
- 发展性：内容随技术和社会需求变化。
- 创新性：鼓励设计、表达、改进和迁移应用。

判断陷阱：信息技术课不是单纯的计算机操作训练；实践性也不等于只让学生自由上机。

### 3. 基本素养

常见表述包括信息意识、计算思维、数字化学习与创新、信息社会责任。

- 信息意识：主动发现信息需求，判断信息价值和可靠性。
- 计算思维：用分解、抽象、建模、算法等方法解决问题。
- 数字化学习与创新：选择工具、管理资源、协作学习和创作。
- 信息社会责任：遵守法律伦理，保护隐私安全，尊重知识产权。

## 二、主要学习理论

### 1. 行为主义

行为主义关注刺激、反应、练习和强化，重视可观察行为。适合基本操作、规则和熟练技能训练。教学中可使用明确示范、分步练习和及时反馈。

### 2. 认知主义

认知主义关注学习者如何接收、加工、存储和提取信息，强调先备知识、认知结构和学习策略。教学中可使用先行组织者、分块、概念图、比较和精加工。

### 3. 建构主义

建构主义强调学习者在一定情境中，借助协作、会话、资源和教师支持主动建构意义。常用真实任务、问题情境、合作学习和支架。

### 4. 人本主义

人本主义重视人的需要、情感、尊重、选择和自我实现，强调有意义学习与和谐师生关系。

### 5. 最近发展区与支架

最近发展区是学生独立完成任务的现实水平与在帮助下可达到水平之间的区域。支架是教师或同伴提供的临时帮助，如步骤卡、示例、提示和半成品，学生能力提高后应逐步撤除。

高频区别：行为主义重外显行为，认知主义重信息加工，建构主义重情境中的主动建构，人本主义重完整人格和情感需要。任何理论都不是所有教学内容的唯一方法。

## 三、教学设计基础

### 1. 教学设计的含义

教学设计是为实现学习目标，对学习者、内容、活动、方法、资源、环境和评价进行系统安排并持续改进的过程。

基本流程：分析学习者和内容 → 确定目标 → 选择方法与资源 → 设计活动和评价 → 实施 → 根据反馈修改。

### 2. 学习者分析

应分析先备知识与技能、年龄和认知特点、学习动机、兴趣经验、个体差异、设备条件及特殊学习需要。证据可以来自前测、访谈、观察、作品和学习记录，不能只凭教师印象。

### 3. 教学目标

目标应明确、具体、可观察、可评价。可用“学习者—行为—条件—标准”组织。

例如：“在给定数据表的条件下，学生能使用排序和筛选完成信息查找，结果正确。”其中“能完成排序和筛选”比“掌握表格知识”更容易评价。

判断陷阱：“了解、掌握、培养、提高”如果没有对应行为和评价标准，通常过于笼统。

### 4. 重点与难点

重点由课程要求、内容结构和教育价值决定；难点是学生理解或操作中可能遇到的困难，受到内容特点和学生基础共同影响。重点不一定就是难点。

### 5. 教、学、评一致

教学目标说明学到什么，学习活动提供达成目标的经历，评价收集目标是否达成的证据。三者应互相对应，不能目标要求“设计作品”，课堂只让学生听讲，评价却只考名词记忆。

## 四、常用教学方法

### 1. 讲授法

教师系统说明概念、原理和方法，效率较高，适合基础知识。应结合提问、示例、图示和练习，避免长时间单向灌输。

### 2. 演示练习法

教师展示操作并说明关键思路，学生模仿、独立练习并迁移到新任务。演示应步骤清楚、速度适当，让学生看得到并有动手机会。

### 3. 任务驱动法

把知识技能放进有意义的任务中，学生在分析和完成任务的过程中学习。基本环节为创设情境、明确任务、分析方案、自主或协作实施、展示评价、总结迁移。

### 4. 项目式学习

围绕真实而较复杂的问题，持续经历规划、分工、调研、设计、制作、测试、改进和展示。项目式学习周期通常较长，成果和过程都要评价。

### 5. 探究学习

围绕问题提出假设，收集和分析证据，形成解释并交流评价。教师要提供必要的资源、方法和安全指导。

### 6. 合作学习

合作学习要求共同目标、积极相互依赖、个体责任、互动和小组反思。只是把学生分组或坐在一起，不等于有效合作。

高频区别：任务驱动通常围绕一个较明确任务；项目式学习更综合、周期更长；探究学习强调证据和解释；合作学习强调成员相互依赖和个体责任。

## 五、信息技术课堂组织

### 1. 上机课基本流程

设备和账号检查 → 说明目标与安全规则 → 发布任务或分段演示 → 学生实践、教师巡视 → 保存提交 → 展示评价 → 退出账号、整理设备。

### 2. 差异化教学

可设计基础、提高和拓展任务。对基础薄弱学生提供步骤卡、示例和半成品；对能力较强学生提供开放问题和改进任务。

### 3. 课堂管理

既要管理人员和时间，也要关注设备、账号、文件、网络、数据和用电安全。发生故障时先保证人身和数据安全，再判断是个别还是普遍问题，启用备用方案并做好记录。

### 4. 教师作用

教师是教学的组织者、引导者、支持者、评价者和安全责任落实者。强调学生主体，不等于教师不讲解、不指导或不评价。

## 六、教学媒体与数字资源

### 1. 教学媒体

教学媒体是承载和传递教学信息的工具或材料，如文字、图片、音频、视频、课件、网络平台和实物设备。

选择媒体应考虑目标、内容、学习者、教学方法、环境条件、成本、易用性、无障碍、安全和版权，而不是越新越好。

### 2. 多媒体使用原则

- 相关文字和图形尽量邻近呈现。
- 突出关键信息，减少无关装饰。
- 内容适当分段，避免一次呈现过多。
- 让媒体服务学习活动，不替代思考和实践。

### 3. 常见数字化教学形态

- 微课：围绕较小目标的短小数字学习资源及活动，不等于只有一段视频。
- 翻转课堂：把部分知识学习前移，课堂更多用于问题解决、协作和反馈。
- 混合式学习：按目标有机结合线上与线下活动，不是简单叠加平台和课堂。

## 七、教学评价

### 1. 按时间和作用分类

- 诊断性评价：教学前了解起点和困难。
- 形成性评价：教学过程中发现问题、提供反馈并改进。
- 总结性评价：阶段结束后判断目标达成情况。

### 2. 按比较标准分类

- 标准参照：与预定标准比较。
- 常模参照：与群体表现比较。
- 个体内差异：与学习者过去表现比较。

### 3. 评价证据

信息技术课程可以使用纸笔测验、上机任务、程序运行结果、作品、过程记录、答辩和档案袋。评价应兼顾知识、过程、作品和规范责任。

### 4. 信度、效度与反馈

信度表示结果是否稳定一致；效度表示是否测到了预期目标。信度高是效度高的重要条件，但不等于效度一定高。

有效反馈应及时、具体，指出当前表现、与目标的差距以及下一步改进方法。只说“很好”或“继续努力”信息不足。

## 八、基础简答题答法

### 1. 特点或作用题

先写概念，再分点写特点或作用，每点用“关键词＋一句解释”。例如回答信息技术教育的特点，可写基础性、实践性、综合性、发展性和创新性，并分别解释。

### 2. 方法比较题

按“定义—适用内容—教师作用—学生活动—关键区别”比较。不要只写一个方法好、另一个方法不好。

### 3. 教学设计题

按“学情与内容—目标—重点难点—活动方法—资源—评价—安全”作答，重点检查目标、活动和评价是否一致。

### 4. 案例分析题

按“判断做法—说明理论依据—分析对学习的影响—提出具体改进—说明如何评价”作答。避免只写“激发兴趣、培养能力”。

## 九、高频判断纠错

1. “信息技术教育就是训练办公软件操作。”错误。它还包括原理、方法、问题解决和责任伦理。
2. “建构主义要求教师完全退出课堂。”错误。教师仍需创设情境、提供资源和支架。
3. “重点一定也是难点。”错误。二者确定依据不同。
4. “用了视频和网络平台就是高质量信息化教学。”错误。媒体必须服务目标和活动。
5. “把学生分组就形成了合作学习。”错误。还需共同目标和个体责任。
6. “项目式学习就是完成一个作品。”错误。它还包含持续的问题解决、规划、测试和改进。
7. “形成性评价只能用测验。”错误。观察、提问、作品和过程记录也可用于形成性评价。
8. “信度高就一定效度高。”错误。稳定地测错内容仍可能效度低。
9. “学生主体意味着教师不讲授。”错误。教学方法应根据目标和内容选择。
10. “评价只看最终作品外观。”错误。还应看功能、过程、方法、规范和目标达成证据。
