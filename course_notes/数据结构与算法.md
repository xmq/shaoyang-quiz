# 📚 数据结构与算法

使用说明：🔴红色= 一级考点 / 核心术语 / 判断改错点（背下来）🔵蓝色= 简答题踩分关键词 / 时间复杂度 / 算法特征（考试时写上去就是分）⚫️黑色= 具体定义解释、例子、选择题常见正确/错误描述（理解即可）
第一章：数据结构概述
考点1：数据结构的基本概念
🔴【数据】：信息的载体，是描述客观事物的符号。
🔴【数据元素】：数据的基本单位，在计算机中通常作为一个整体进行考虑和处理。
🔴【数据项】：数据的最小单位，是数据元素中不可再分的单元。
🔴【数据对象】：具有相同性质的数据元素的集合。
⚫️【坑点】：数据的最小单位是数据项，不是数据元素。数据元素是数据的基本单位。
考点2：数据结构的三个层次
🔴【逻辑结构】：数据元素之间的逻辑关系，与计算机无关。
分类：线性结构（线性表、栈、队列、串）和非线性结构（树、图、集合）。
🔴【存储结构（物理结构）】：数据在计算机中的存储表示。
分类：顺序存储结构（连续空间）、链式存储结构（非连续空间）、索引存储、散列存储。
🔴【数据的运算】：对数据进行的操作（插入、删除、查找、排序等）。
⚫️【坑点】：
研究数据结构就是研究数据的逻辑结构、存储结构及其基本操作。
数据的逻辑结构与数据元素本身的内容和形式无关。
与所使用的计算机无关的是数据的逻辑结构。
数据结构主要分为线性结构和非线性结构两类。
考点3：算法
🔴【算法的定义】：为解决特定问题而规定的、有限的指令序列；算法是对求解步骤的抽象描述，不等同于某一种程序设计语言编写的程序。
🔴【算法的五大特性】：
有穷性：必须在有限步内终止。
确定性：每一步的含义无歧义。
可行性：每一步操作可执行。
输入：可以有0个或多个输入。
输出：必须有1个或多个输出。
🔴【算法效率的度量】：
时间复杂度：以输入规模 n 为自变量，分析基本操作执行次数的数量级，常用大 O 记号表示；它不是某台机器上的实际运行秒数。
空间复杂度：分析算法所需存储空间随输入规模增长的数量级。题目说“辅助空间”时只计算输入数据之外的额外空间；说“总空间”时还应计入输入存储。
⚫️【坑点】：
算法必须有输出，但可以没有输入（0个输入）。
算法的可行性是指每一步都能精确运行，不是指指令没有二义性（那是确定性）。
算法的时间复杂度关注基本操作次数随输入规模增长的数量级，不等于程序实际运行时间；同一数量级的算法在常数、缓存和数据分布上仍可能有差别。
第二章：线性表
考点4：线性表的定义与特点
🔴【定义】：n 个具有相同性质的数据元素组成的有限序列（n≥0）。“相同性质”是抽象数据结构口径，不必机械等同于某种语言里字节长度完全相同。
🔴【特点】：数据元素之间是一对一的关系。除了第一个和最后一个，每个元素有且仅有一个直接前驱和一个直接后继。
⚫️【坑点】：
线性表可以为空表（n=0）。
线性表中的元素之间是一对一的关系，不是一对多。
线性表是n个数据元素的有限序列。
考点5：顺序存储结构（顺序表）
🔴【定义】：用连续的存储单元依次存储线性表的数据元素。
🔴【特点】：
逻辑相邻的，物理上也相邻。
可随机存取（通过首地址+下标直接访问任意元素）。
存储密度高（接近1）。
插入/删除需要移动大量元素，效率低。
🔵【插入时间复杂度】：平均移动n/2个元素，时间复杂度O(n)。
🔵【删除时间复杂度】：平均移动(n-1)/2个元素，时间复杂度O(n)。
⚫️【坑点】：
顺序存储的线性表可以实现随机存取。
顺序存储方式的优点是存储密度大，缺点是插入、删除效率低。
一个顺序表第一个元素地址90，每个元素长度2，第6个元素地址 = 90 + 5×2 =100。
考点6：链式存储结构（链表）
🔴【定义】：用任意的存储单元存储数据元素，元素间通过指针链接。
🔴【特点】：
逻辑相邻的，物理上不一定相邻。
顺序存取（不能随机访问）。
在已知插入/删除位置及其相关结点指针时，不需要成批移动元素，只需修改链接，局部操作可为 O(1)；若需先从表头查找位置，整体仍可能为 O(n)。
存储密度小于1（需要额外的指针空间）。
🔴【单链表】：每个节点包含数据域和指针域（指向下一个节点）。
🔴【循环链表】：最后一个节点的指针指向头节点，形成环。
🔴【双向链表】：每个节点包含前驱指针和后继指针。
⚫️【坑点】：
线性表采用链式存储时，地址连续与否均可。
链表的存储密度小于1。
链表不需成批移动元素且容量调整灵活，但访问第 i 个元素通常要顺链查找；若未给定位置指针，插入/删除的定位成本仍可能是 O(n)。
带头结点的链表最主要的好处是使空表和非空表的处理统一。
在单链表指针为p的结点之后插入指针为s的结点：s->next = p->next; p->next = s;（必须先连后断）。

第三章：栈和队列
考点8：栈（Stack）
🔴【定义】：限定只在表尾（栈顶）进行插入和删除操作的线性表。
🔴【特点】：后进先出（LIFO，Last In First Out）。
🔴【基本操作】：入栈（Push）、出栈（Pop）、取栈顶（Top）。
🔴【栈的应用】：
函数调用（递归）。
表达式求值。
括号匹配。
深度优先搜索（DFS）。
🔴【链式栈 vs 顺序栈】：链式栈按需动态申请结点，不受固定数组容量限制，但仍可能因可用内存耗尽而入栈失败。
⚫️【坑点】：
栈是线性结构，不是非线性结构。
栈的特点是后进先出，队列的特点是先进先出。
入栈顺序为1,2,3，全部入栈后出栈顺序为3,2,1。
顺序栈和链式栈在已知栈顶时，入栈、出栈通常都是 O(1)；顺序栈可能受固定容量限制，链式栈有指针和动态分配开销。
考点9：队列（Queue）
🔴【定义】：限定在一端（队尾）插入，在另一端（队头）删除的线性表。
🔴【特点】：先进先出（FIFO，First In First Out）。
🔴【基本操作】：入队（Enqueue）、出队（Dequeue）。
🔴【队列的应用】：
广度优先搜索（BFS）。
缓冲区管理（打印任务队列）。
二叉树层序遍历（将根节点入队，出队时子节点入队）。
⚫️【坑点】：队列是一种先进先出的线性表，不是后进先出。
考点10：循环队列
🔴【定义】：将队列的存储空间首尾相接，形成逻辑上的环。
🔴【常见约定】：若 front 指向队头元素、rear 指向下一个可入队位置，并**牺牲一个存储单元**区分空与满，则：
🔵【判空条件】：front == rear。
🔵【判满条件】：(rear + 1) % MAXSIZE == front。
🔵【元素个数】：(rear - front + MAXSIZE) % MAXSIZE；长度为 MAXSIZE 的数组最多存 MAXSIZE-1 个队列元素。
⚫️【坑点】：
循环队列是队列的一种顺序存储结构（不是链式存储结构！）。
循环队列中元素的个数由队头指针和队尾指针共同决定。
循环队列的判空、判满公式取决于指针含义和实现方案。使用计数器或标志位时可利用全部 n 个单元，判满条件与上述“牺牲一格”公式不同；做题必须先看题设约定。
第四章：树与二叉树
考点11：树的基本概念
🔴【树的定义】：树是 n（n≥0）个结点的有限集合。n=0 时为空树；非空树有且仅有一个根结点，其余结点可分为若干互不相交、且本身也是树的子树。
🔴【树的性质】：每个节点最多有一个直接前驱，可以有多个直接后继。
🔴【树的度】：节点拥有的子树个数的最大值。
🔴【叶子节点（叶节点）】：度为0的节点（没有子节点）。
🔴【树的深度（高度）】：节点所在的最大层次。
⚫️【坑点】：树形结构中每个节点最多只有一个直接前驱。根节点没有前驱。
考点12：二叉树（Binary Tree）
🔴【定义】：每个节点最多有两棵子树（左子树和右子树），子树有左右之分，不能颠倒。
🔴【二叉树 ≠ 度为2的树】：二叉树允许空树，度为2的树至少有3个节点。
🔴【满二叉树】：所有非叶子节点的度都为2，且所有叶子节点都在同一层。
🔴【完全二叉树】：除最后一层外，其他层都满，最后一层节点从左到右连续排列。
🔴【二叉树与度为2的树的区别】：
二叉树中结点的度可以为0、1、2；“度为2的树”是指整棵树的最大结点度为2，并不表示每个结点的度都固定为2。
二叉树的子树有左右之分，度为2的树没有顺序之分。
⚫️【坑点】：空二叉树没有根结点；**非空二叉树恰有一个根结点**。
考点13：二叉树的性质（计算必考）
🔴【性质1】：在二叉树的第i层上至多有2^(i-1)个节点（i≥1）。
🔴【性质2】：深度为k的二叉树至多有2^k - 1个节点（k≥1）。至少有k个节点。
🔴【性质3】：对任意二叉树T，叶子节点数 n0 = 度为2的节点数 n2 + 1。
🔴【性质4】：具有n个节点的完全二叉树，深度为⌊log₂n⌋ + 1。
🔴【性质5】：按层序从 1 开始编号的完全二叉树中，结点 i 的双亲为 ⌊i/2⌋（i>1）；若 2i≤n，左孩子为 2i；若 2i+1≤n，右孩子为 2i+1。若数组从 0 编号，公式将改变。
⚫️【坑点】：
一棵二叉树中，度为1的节点数为9 → 叶子节点的数目无法确定（需要知道度为2的节点数）。
设树T的度为4，其中度为1,2,3,4的节点个数分别为4,2,1,1 → 叶子数 = n2+2n3+3n4+1 = 2+2+3+1 =8。
一个有30个节点的二叉树深度至少为5（1+2+4+8+15=30，深度5）。
考点14：二叉树的遍历（极高频）
🔴【前序遍历（Pre-order）】：根 → 左 → 右（递归：先访问根，再遍历左子树，再遍历右子树）。
🔴【中序遍历（In-order）】：左 → 根 → 右（递归：先遍历左子树，再访问根，再遍历右子树）。
🔴【后序遍历（Post-order）】：左 → 右 → 根（递归：先遍历左子树，再遍历右子树，再访问根）。
🔴【层序遍历（Level-order）】：从上到下，从左到右逐层访问。使用队列实现。
🔴【重要结论】：已知前序+中序或后序+中序可唯一确定一棵二叉树。仅知道前序+后序不能唯一确定。
⚫️【坑点】：
若一个树叶是某二叉树的中序遍历的最后一个节点，则它也一定是该二叉树的前序遍历最后一个节点（正确）。
若一个节点是某二叉树的前序遍历最后一个节点，则它不一定是中序遍历最后一个节点。
某二叉树中序序列DBAFCE，层次序列FDECBA → 后序序列为ABDCEF。
结点按中序遍历为SYZ的二叉树可能有5种（卡特兰数：C₃ = 5）。
考点15：二叉树的构造
🔴【由前序+中序构造】：
前序第一个为根节点。
在中序中找到根节点位置，左侧为左子树，右侧为右子树。
递归构造。
🔴【由后序+中序构造】：
后序最后一个为根节点。
在中序中找到根节点位置，左侧为左子树，右侧为右子树。
递归构造。
⚫️【坑点】：已知一棵二叉树的前序遍历和中序遍历序列，可以构造出这棵二叉树。
第五章：图
考点16：图的基本概念
🔴【图的定义】：由顶点（Vertex）和边（Edge）组成的数据结构。Graph = (V, E)。
🔴【无向图】：边无方向（如：好友关系）。
🔴【有向图】：边有方向（如：关注关系）。
🔴【完全图】：任意两个顶点之间都有边相连。
无向完全图有n(n-1)/2条边。
有向完全图有n(n-1)条边。
🔴【顶点的度】：
无向图：度 = 与该顶点相连的边数。
有向图：入度（指向该顶点的边）+出度（从该顶点出发的边）。
🔵【度数公式】：无向图所有顶点度数之和等于边数的 2 倍；有向图所有顶点入度之和 = 出度之和 = 边数。
⚫️【坑点】：图中任何两个节点之间都可能有逻辑关系，不是一对一或一对多。
考点17：图的存储与连通性（高频补充）
🔴【邻接矩阵】：用 n×n 矩阵表示顶点间是否有边，判断两点是否相邻为 O(1)，空间 O(n²)，适合稠密图。无向图的邻接矩阵关于主对角线对称。
🔴【邻接表】：每个顶点保存其邻接点链表/集合，空间通常为 O(V+E)，适合稀疏图；检查一条边的成本与相关顶点的度有关。
🔵【连通图】：无向图任意两顶点间都有路径称连通；有向图任意两顶点间彼此可达称强连通。
🔵【生成树】：含图中全部 n 个顶点且连通、无环的子图，边数恰为 n-1。连通无向图才存在生成树；最小生成树常用 Prim 或 Kruskal 算法。
🔵【最短路径】Dijkstra 适用于边权非负的单源最短路径；含负权边时不能直接套用。Floyd 可求所有顶点对最短路径。
考点18：图的遍历
🔴【深度优先搜索（DFS）】：使用栈（递归实现）。类似树的前序遍历。
🔴【广度优先搜索（BFS）】：使用队列。类似树的层序遍历。
⚫️【坑点】：深度优先搜索实现时需要用的数据结构是栈，广度优先搜索使用队列。
🔵【复杂度前提】：采用邻接表时，DFS、BFS 的时间复杂度均为 O(V+E)；采用邻接矩阵时通常为 O(V²)。非连通图要从尚未访问的顶点重新启动，才能遍历全部顶点。
第六章：查找与排序
考点19：查找算法
🔴【顺序查找】：
适用：任意线性表（无序有序都可以）。
时间复杂度：O(n)。
在各元素查找概率相等且查找成功的前提下，平均查找长度为 (n+1)/2；查找失败的比较次数取决于实现和表结构。
🔴【折半查找（二分查找）】：
适用：有序的顺序存储表（要求随机存取）。
时间复杂度：O(log₂n)。
每次比较使搜索范围缩小一半。
不能用于链表（链表不能随机存取）。
🔵【最大比较次数】：含 n 个元素的有序顺序表，成功或失败查找的比较次数上界通常为 ⌊log₂n⌋+1（不同循环边界写法可能使失败统计口径略有差异）。
⚫️【坑点】：
二分查找要求线性表必须以顺序方式存储，且节点按关键字有序排序。
折半查找（二分查找）也称二分搜索、对数搜索。
考点20：树表查找与散列查找（高频补充）
🔴【二叉排序树 BST】：对任一结点，左子树关键字通常小于该结点，右子树关键字通常大于该结点（重复关键字如何处理由实现约定）；中序遍历得到有序序列。
🔵【查找复杂度】：BST 查找、插入、删除均为 O(h)，h 为树高。树较平衡时约 O(log n)，退化成单链时为 O(n)，不能一概写成 O(log n)。
🔴【平衡二叉树 AVL】：任一结点左右子树高度差的绝对值不超过 1，结点平衡因子只能为 -1、0、1；通过旋转维持平衡，查找为 O(log n)。
🔴【堆】：堆是一棵完全二叉树。大根堆中每个父结点关键字不小于孩子，根最大；小根堆相反。堆只保证父子间偏序，**中序遍历不一定有序**。
🔴【散列表 Hash Table】：散列函数把关键字映射到表地址；不同关键字映射到同一地址称冲突。常用处理方法有开放定址法（线性探测、二次探测等）和链地址法。
🔵【散列效率前提】：散列查找平均可接近 O(1)，但依赖散列函数、装填因子和冲突处理；冲突严重时最坏可退化为 O(n)。散列表通常不支持按关键字有序遍历。
考点21：排序算法（高频对比）
1. 插入类排序
🔴【直接插入排序】：将元素逐个插入已排序的有序序列中。
时间复杂度：最好 O(n)，最坏 O(n²)，平均 O(n²)。
稳定排序。
2. 交换类排序
🔴【冒泡排序（Bubble Sort）】：两两比较，反序则交换，直到没有反序记录为止。
时间复杂度：带“本趟无交换则提前结束”标志的改进实现最好 O(n)；朴素固定趟数实现即使已有序仍可能 O(n²)。最坏、平均 O(n²)。
稳定排序。
记忆口诀：大的往下沉，小的往上冒。
🔴【快速排序（Quick Sort）】：选一个基准元素，将序列划分为“不大于/不小于”基准的区域（具体等号归属取决于划分算法），再递归排序子区间。
时间复杂度：最好/平均O(nlog₂n)，最坏O(n²)。
当总选首元素或尾元素为基准时，初始序列正序或反序会造成极不平衡划分，是典型最坏情况；随机选基准、三数取中或三路划分可降低该风险，因此不能说所有快速排序实现遇到有序序列都必然最坏。
不稳定排序。
原地实现、缓存友好，实践中平均性能常很好，但不能绝对表述为“任何数据和平台上最快”。
3. 选择类排序
🔴【简单选择排序】：每趟找出最小（或最大）元素，放到序列起始位置。
时间复杂度：始终O(n²)。
不稳定排序。
🔴【堆排序】：利用堆的数据结构进行排序。
时间复杂度：O(nlog₂n)。
不稳定排序。
4. 其他排序
🔴【希尔排序（Shell Sort）】：插入类排序法。按一定间隔分组，分别进行直接插入排序。
属于插入类排序，不是交换类或选择类。
时间复杂度与增量序列有关，不能脱离增量统一给出最好、平均、最坏界；常见基础教材按所指定的增量分析。希尔排序不稳定。
5. 归并排序
🔴【归并排序（Merge Sort）】：把序列递归分成子序列，分别排序后再合并。
🔵【复杂度与性质】：最好、平均、最坏时间复杂度均为 O(nlog₂n)；常见数组实现需要 O(n) 辅助空间；稳定。适合外部排序及链表排序。
⚫️【坑点】：
希尔排序属于插入类排序法。
冒泡排序的基本思想是两两比较，反序交换。
“快速排序在正序或反序时最坏”需要以固定选首/尾元素作基准等划分策略为前提。
选择排序第一次交换：从“2，6，4，1”从大到小选最大6与第一个交换 →6，2，4，1。
📊 终极复杂度/性质记忆表（考前过一遍）
⚫️【稳定排序总结】：
稳定的：直接插入排序、冒泡排序、归并排序。（口诀：冒泡直接归并稳）
不稳定的：希尔、选择、堆、快速。（口诀：希选快堆都不稳）

1. 简述算法的五个重要特性。
🔵【踩分点】：有穷性（有限步终止）、确定性（无二义）、可行性（可执行）、输入（0个或多个）、输出（1个或多个）。
2. 简述数组和链表的区别。
🔵【踩分点】：① 数组/顺序表占用连续内存空间，链表不要求连续。② 数组随机存取，链表顺序存取。③ 顺序表插入/删除通常需移动元素；链表在已定位时只需修改链接。④ 顺序表存储密度高，链表有指针开销。⑤ 二者操作复杂度都要结合“是否已定位”和具体实现说明。
3. 简述栈和队列的区别。
🔵【踩分点】：① 栈后进先出（LIFO），队列先进先出（FIFO）。② 栈只在一端操作，队列在两端操作。③ 栈常用于函数调用、递归，队列常用于缓冲区、层序遍历。
4. 简述二叉树的前序、中序、后序遍历方法。
🔵【踩分点】：①前序：根 → 左 → 右。②中序：左 → 根 → 右。③后序：左 → 右 → 根。
5. 简述折半查找的基本思想。
🔵【踩分点】：在有序表中，取中间元素与目标比较。相等则查找成功；目标小于中间值，则在左半区继续查找；目标大于中间值，则在右半区继续查找。每次比较使搜索范围缩小一半。
口诀速记
“算法五要有穷确，可行输入输出全；线性表序和链表，顺序存密链插简；栈后进先出递归用，队列先进先出层序游；二叉树前根左右，中左右根，后左右根；堆排快排不稳当，冒泡插入最稳当；二分查找O(log n)，有序顺序是前提。”
