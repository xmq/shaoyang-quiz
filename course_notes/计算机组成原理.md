# 📚 计算机组成原理

使用说明：🔴红色= 一级考点 / 核心术语 / 判断改错点（背下来）🔵蓝色= 简答题踩分关键词 / 计算公式 / 操作步骤动词（考试时写上去就是分）⚫️黑色= 具体定义解释、例子、选择题常见正确/错误描述（理解即可）
第一章：计算机发展史
考点1：早期电子计算机（题干限定词决定答案）
🔴【ENIAC】：1946年公开运行于美国宾夕法尼亚大学，通常按我国基础教材口径称“世界上第一台通用电子数字计算机”，主要使用电子管。
🔴【ENIAC的局限性】：早期主要通过插接线和开关设置程序，并非今天意义上的存储程序计算机，改换任务很不方便。
🔴【存储程序计算机要分清】：EDVAC是冯·诺依曼等参与设计的早期存储程序方案；Manchester Baby（1948）通常被认为第一台运行存储程序的电子计算机；EDSAC（1949）是较早投入实际使用的存储程序计算机。
⚫️【坑点】：题目若只问基础教材中的“第一台电子数字计算机”，通常答ENIAC；若题干出现“通用”“运行存储程序”“投入实际使用”等限定词，答案可能不同。ENIAC并不是由冯·诺依曼独立设计的。
考点2：计算机的四代划分
🔴【划分依据】：电子元器件（不是按体积、速度、价格！）。
🔴【各代对应】：
第1代（1946-1957）：电子管。机器语言，体积大、耗电多、速度慢。
第2代（1958-1964）：晶体管。高级语言开始出现（FORTRAN、COBOL）。
第3代（1965-1970）：中小规模集成电路。操作系统雏形出现。
第4代（1971-至今）：大规模/超大规模集成电路。微型计算机诞生。
⚫️【坑点】：判断题说“大规模集成电路是第三代计算机的标志” →错！是第四代！
考点3：计算机之父的尊称（极易混淆）
🔴【冯·诺依曼】：“现代计算机之父”（提出存储程序+程序控制，确定五大部件结构）。
🔴【图灵】：“计算机科学之父”、“人工智能之父”（提出图灵机模型、图灵测试）。
🔴【查尔斯·巴贝奇】：“通用计算机之父”（差分机/分析机的设计者）。
🔴【阿塔纳索夫】：与贝里研制ABC计算机，它是早期电子数字计算装置，但属于专用、不可编程设备；“电子计算机之父”并非所有教材统一采用的唯一称号。
⚫️【坑点】：题目问“计算机科学的奠基人/图灵机提出者”通常选图灵；问“存储程序思想或冯·诺依曼体系结构”通常选冯·诺依曼。尊称题必须服从题干和教材口径，不能只凭“某某之父”判断。
考点4：计算机的特点与分类
🔴【常见特点】：运算速度快、计算精度高、存储容量大、具有逻辑判断能力、自动化程度高、通用性强。
🔴【自动工作的基础】：在存储程序控制下，计算机能按程序连续执行；“最关键特点”属于具体教材口径，不宜脱离题干绝对化。
🔴【传统教材按规模/性能分类】：巨型机、大型机、中型机、小型机、微型机；现代设备边界已不完全由体积或单项速度决定。
巨型机：“神威·太湖之光”、“天河二号”。
微型机：PC（个人计算机），以微处理器为核心。
⚫️【坑点】：世界上第一台计算机ENIAC是巨型机，不是个人计算机。个人计算机（PC）是第四代才出现的。
第二章：冯·诺依曼体系结构
考点5：核心设计思想（简答题必背！）
🔴【思想名称】：“存储程序”与“程序控制”（合称“存储程序控制原理”）。
🔵【三大核心要点】：
计算机内部采用二进制表示数据和指令。
将编好的程序和数据事先存入内存储器（存储程序）。
计算机在工作时，控制器自动、逐条地从内存中取出指令并加以执行（程序控制）。
⚫️【坑点】：计算机之所以能“自动连续”工作，最直接的原因是采用了“存储程序控制”，而不是“运算速度快”或“有逻辑判断能力”。
考点6：硬件五大部件（极高频填空/多选）
🔴【五大部件】：运算器、控制器、存储器、输入设备、输出设备。
🔴【CPU的基本组成】：运算器 + 控制器；从实际结构看还包含寄存器组，现代CPU通常集成多级Cache等部件。基础选择题常按“运算器+控制器”作答。
🔴【主机的组成】：CPU + 内存（主存储器）。（注意：主机不包括外存！）
🔵【各部件核心功能】：
⚫️【坑点】：
运算器不包括译码器（译码器属于控制器）。
话筒是输入设备（把声音变成电信号），不是输出设备！
触摸屏既是输入设备（触摸操作），又是输出设备（显示画面）。
磁盘驱动器（硬盘/U盘）既是输入设备又是输出设备（可读可写）。
第三章：CPU与指令系统
考点7：CPU的性能指标
🔴【机器字长】：CPU定点运算器一次能处理的二进制数据位数（常见32位、64位）。在其他条件相近时，字长会影响整数表示范围、运算精度和寻址/处理能力，但不能单独决定整机性能。
🔴【主频】：CPU时钟频率（常用GHz）。同一微体系结构且其他条件相近时，主频提高通常可缩短CPU时间；跨架构比较时不能据此断定谁更快。
🔴【性能指标】：可用执行时间、吞吐率、CPI、IPC、MIPS等描述。MIPS只适合在指令系统和测试负载相近时参考，不能脱离程序直接横向比较。
🔵【MIPS计算公式】：MIPS = 主频(Hz) / (CPI × 10⁶)，其中CPI是执行一条指令所需的平均时钟周期数。
🔵【CPU执行时间】：CPU时间 = 指令数 × CPI × 时钟周期 = 指令数 × CPI / 主频。
⚫️【坑点】：整机性能由CPU、内存层次、I/O、软件和工作负载共同决定，不能写成“只由CPU或主频决定”。“Pentium 4”是处理器系列名称。
考点8：CPU不能直接访问的存储器
🔴【核心结论】：CPU不能直接访问外存储器（硬盘、U盘、光盘等）。
🔵【正确流程】：外存中的数据必须先调入内存（RAM），CPU才能通过数据总线访问。
⚫️【坑点】：判断题说“CPU可以直接读取硬盘中的程序并执行” →错！必须先调入内存。
考点9：指令的执行过程
🔴【三大步骤】：每条指令的执行分为三个阶段：
取指令：控制器从内存中取出指令，送入指令寄存器（IR）。
分析指令：指令译码器对操作码进行译码，确定要执行什么操作。
执行指令：控制器发出控制信号，指挥各部件完成操作。
🔴【指令的组成】：操作码（规定做什么）+地址码/操作数（规定对谁做）。
⚫️【坑点】：在冯·诺依曼结构中，指令和数据都以二进制形式存入存储器；某段二进制被当作指令还是数据，取决于程序执行上下文、地址和控制器的解释。不能简单说“仅靠取指/执行两个时间阶段区分”，执行阶段也可能不访存。
考点10：CISC 与 RISC（拔高题）
🔴【CISC（复杂指令集）】：
通常指令种类和寻址方式较丰富，单条指令可完成较复杂功能；指令长度可能可变。
代表：Intel x86 架构（PC电脑的CPU）。
传统实现常采用微程序控制，但现代CISC处理器内部也大量采用硬布线、流水线和微操作转换；“一定功耗高、对编译器要求低”不是定义性结论。
🔴【RISC（精简指令集）】：
指令系统精简高效，指令长度固定，格式规整。
代表：ARM 架构（手机、平板、嵌入式设备）。
典型特点：指令格式规整、寻址方式较少、强调寄存器操作和Load/Store结构，利于流水线；不等于每条指令都单周期，也不保证任何RISC都比CISC更快或更省电。
⚫️【坑点】：RISC的指令种类少（精简），不是多！说“RISC指令种类多”是错的。
第四章：存储器系统（极高频，细节极多）
考点11：存储器的层次结构
🔴【速度由快到慢】：寄存器（CPU内部） > Cache（高速缓存） > 内存（主存） > 外存（硬盘/U盘）。
🔴【容量由小到大】：寄存器 < Cache < 内存 < 外存。
🔴【价格由高到低】：寄存器 > Cache > 内存 > 外存。
🔴【引入Cache的目的】：解决CPU 与 内存（主存）之间速度不匹配的矛盾。
⚫️【坑点】：Cache是为了解决CPU和内存的速度矛盾，不是解决内存和外存的矛盾！也不是为了解决容量矛盾！
考点12：内存（主存） vs 外存（辅存）—— 必考对比
🔴【内存（主存）】：
速度快、容量小、价格高。
其中RAM通常具有易失性，断电后其数据消失；主存概念也可能包括少量ROM，因此不能把“内存”与“RAM”在所有语境下完全等同。
存放正在运行的程序和数据。
CPU可以直接访问。
🔴【外存（辅存）】：
速度慢、容量大、价格低。
断电后数据保存。
存放暂时不用的程序和数据。
CPU不能直接访问（需先调入内存）。
⚫️【坑点】：相同容量的内存比外存贵得多，但速度快得多。判断题说“相同容量的内存和外存价格差不多” →错！
考点13：ROM vs RAM（选择填空必出）
🔴【RAM（随机存取存储器）】：
特点：可读可写，断电丢失数据（易失性）。
作用：存放当前正在运行的程序和数据（就是我们的内存条）。
SRAM（静态RAM）：速度快，用作Cache（高速缓存）。
DRAM（动态RAM）：速度较慢，用作主存（需要周期性刷新）。
🔴【ROM（只读存储器）】：
基本特征是非易失、正常运行时以读为主。掩膜ROM出厂固化；PROM可一次编程，EPROM/EEPROM/Flash可按相应方式擦写，因此“所有ROM永远不能写”是错误表述。
作用：存放BIOS（基本输入输出系统）等固定程序。
注：PROM只能编程一次；EPROM可用紫外线擦除后重写；EEPROM可电擦写。基础题中的“只读”强调正常使用方式，不代表所有ROM技术物理上绝对不可改写。
⚫️【特别警示】：
BIOS（基本输入输出系统）存放在ROM中。
CMOS（互补金属氧化物半导体）是主板上的一块RAM芯片，靠电池供电保存日期/时间/密码等设置信息。CMOS设置程序在BIOS中。
判断题说“BIOS存放在CMOS中” →错！BIOS在ROM中，CMOS是存设置参数的。
考点14：存储容量计算
🔴【基本单位】：字节（Byte，简称B）是存储容量的基本单位。位（bit，简称b）是数据的最小单位。
🔵【二进制容量换算（传统考试口径）】：1B = 8bit；1KB = 1024B；1MB = 1024KB；1GB = 1024MB；1TB = 1024GB。严格的IEC记号中1024B=1KiB，而SI中的1kB=1000B；做题按题干口径使用。
🔵【汉字存储空间计算】：
一个16×16点阵汉字占：16×16/8 = 32字节。
一个24×24点阵汉字占：24×24/8 = 72字节。
一个32×32点阵汉字占：32×32/8 = 128字节。
公式：存储空间(字节) = (点阵行数 × 点阵列数) / 8。
🔴【字符占用空间要看编码】：GB2312/GBK常用双字节表示汉字，标准ASCII字符占1字节；UTF-8中常用汉字通常占3字节，UTF-16中常见汉字通常占2字节、部分字符占4字节。
⚫️【坑点】：只有题目明确采用“双字节汉字机内码”时，2KB才可按1024个汉字计算；不能把“任意汉字在任何编码中固定占2字节”当作通则。
考点15：地址总线与内存容量的关系
🔴【关键结论】：地址总线的宽度（根数）决定了CPU的寻址范围（最大内存容量）。
🔵【计算公式】：最大内存容量（字节）= 2^(地址总线根数)（按字节编址时）。
⚫️【坑点】：公式必须写清“按字节编址”；若按字编址，最大可寻址容量还要乘每个编址单位的字节数。地址总线宽度给出的是可寻址上限，实际安装主存还受芯片组、主板、操作系统等限制。数据总线宽度影响一次并行传送的数据位数。
考点15A：Cache命中率与平均访问时间（高频计算）
🔴【局部性基础】：Cache利用时间局部性和空间局部性，把近期可能使用的主存块副本保存在更快的存储器中。
🔵【命中率】：命中率 h = Cache命中次数 / 总访问次数；失效率 = 1-h。
🔵【平均访问时间】：若先查Cache、失效后再访问主存，可按题目时序写为 T = h×Tc + (1-h)×(Tc+Tm)；若题目规定Cache与主存并行访问，公式会不同，必须看题干。
⚫️【映射方法】：直接映射实现简单但冲突较多；全相联冲突少但硬件复杂；组相联是二者折中。写策略常见写直达（Write Through）与写回（Write Back）。
第五章：I/O设备与总线
考点16：输入/输出设备分类（务必分清）
🔴【常见输入设备】：键盘、鼠标、扫描仪、话筒、摄像机、条码阅读器、触摸屏（兼输出）。
🔴【常见输出设备】：显示器、打印机、音箱、绘图仪、投影仪。
🔴【兼有输入/输出功能】：触摸显示屏既显示又接收触控；可读写存储设备在I/O数据流角度可输入和输出，但硬盘、U盘通常归入外存/存储设备，分类要看题干口径。
⚫️【坑点】：
扫描仪是输入设备（纸上的图→电子版），不是输出设备！
投影仪是输出设备（电脑画面→投影幕布），不是输入设备！
打印机用DPI（每英寸点数）衡量打印精度，DPI越高打印越清晰。
喷墨打印机耗材是墨盒；激光打印机耗材是硒鼓。
DPI是衡量打印分辨能力的重要指标之一；打印质量还受墨粉/墨水、色彩、纸张、机械精度等影响，不能只由DPI决定。
考点17：显示器性能指标
🔴【常考指标】：分辨率（屏幕像素矩阵，如1920×1080）是重要指标之一；显示效果还与尺寸、像素密度、刷新率、色域、亮度等有关。
🔴【其他指标】：点距（越小越清晰）、刷新率（越高越流畅）、颜色质量（色深）。
⚫️【坑点】：显示系统的主要性能指标是分辨率、颜色质量、刷新速度，不包括内存容量。
考点18：系统总线（极高频）
🔴【总线的英文】：BUS（注意不是UBS，也不是UPS！UPS是不间断电源，UBS是公司名）。
🔴【系统总线分类（按传输信息）】：
数据总线（DB）：传输数据。双向。宽度（位数）决定了CPU一次能传输的数据量。
地址总线（AB）：传输地址。单向（由CPU发出）。宽度决定了寻址范围（最大内存容量）。
控制总线（CB）：传输读/写、中断、总线请求/响应等控制与状态信号；各条线有各自方向，作为一组通常包含双向的信息流。
🔵【理想峰值带宽】：带宽 = 每次传输字节数 × 每秒传输次数。若每个时钟只传一次，才可写为（总线宽度/8）×总线频率；DDR或突发传输还要乘每周期传输次数等系数。
例：32位总线，33MHz → 带宽 = (32/8) × 33 = 4×33 = 132 MB/s。
⚫️【坑点】：
决定CPU与外部进行一次数据传输信息量的是数据总线宽度。
决定主存储器大小的是地址总线宽度。
数据、地址、控制是“系统总线按所传信息”的三类；按位置或用途还可分片内总线、系统总线、I/O总线等，不能说所有总线在任何分类下都只有三种。
第六章：I/O数据传送控制方式
考点19：三种控制方式（高频多选/判断）
🔴【方式1：程序控制I/O】：无条件传送适用于外设总能按约定及时就绪的简单场景；程序查询方式由CPU反复读取状态并在就绪后传送，两者不宜完全等同。
CPU主动、不断查询外设状态（“忙不忙？好了没？”）。
缺点：CPU大量时间浪费在查询等待上，效率极低。
🔴【方式2：中断方式】：
外设准备好后，主动向CPU发中断请求，CPU暂停当前工作去处理。
优点：CPU利用率提高。缺点：适合少量数据传输，频繁中断也会消耗CPU资源。
🔴【方式3：直接存储器存取方式（DMA）】（极高频考点）：
核心：DMA控制器在取得总线控制权后，使I/O设备与主存之间成批传送数据，CPU不逐字节搬运。
适用场景：大批量数据传送（如磁盘读写、视频数据传输）。
特点：传送过程中CPU可以执行其他任务，传送结束后才向CPU发中断报告。
⚫️【坑点判断题】：
“DMA方式传输数据时，完全不需要CPU干预。” → 严格说传送过程中不需要CPU，但开始和结束仍需CPU参与配置和中断响应。如果选项说“完全不需要CPU”，通常是错的！
“中断方式适合大批量数据传输” →错！适合小批量。大批量用DMA。
第七章：数据的表示与编码
考点20：原码、反码、补码（必考计算！）
🔴【核心结论】：现代计算机中的有符号整数通常用补码表示；无符号整数、浮点数、字符等采用各自编码，不能统称“所有数据都用补码”。补码使加减法可统一使用加法器，并把零表示唯一化。
🔵【三码转换规则（以8位为例）】：
正数：原码 = 反码 = 补码（三者相同）。
例：+5 → 原码00000101，反码00000101，补码00000101。
负数：符号位为1（最高位），其余位：
原码：数值部分为绝对值的二进制。
例：-5 → 原码10000101。
反码：符号位不变，其余位按位取反。
例：-5 → 反码11111010。
补码：反码 + 1（末位加1）。
例：-5 → 补码11111011。
⚫️【特殊值】：8位补码能表示的范围是-128 ~ +127。-128的补码是10000000（这是个必须记住的特殊值）。
⚫️【坑点】：8位无符号二进制能表示的最大十进制数是255（2⁸-1），不是256。
考点21：ASCII码
🔴【ASCII码】：美国信息交换标准代码，7位二进制编码（存储时占1个字节，即8位）。
🔴【编码数量】：共128个（0-127）。
🔴【关键数值（必须记住！）】：
数字'0'→ 48（十进制）→00110000（二进制）
大写'A'→ 65 →1000001
小写'a'→ 97 →1100001
🔴【ASCII码大小排序】：数字（0-9）< 大写字母（A-Z）< 小写字母（a-z）。
⚫️【坑点】：大写字母和小写字母的ASCII码差32。'A'=65，'a'=97，差32。'a'比'A'大32。
考点22：汉字编码
🔴【四个层次】：
输入码（外码）：用于输入汉字，如五笔（形码）、拼音（音码）。
交换码（国标码）：传统教材常以GB2312国标码说明信息交换；GBK是对GB2312的扩展编码，Unicode则是现代跨平台字符编码标准。
机内码（内码）：传统GB2312教材中，汉字机内码由国标码两个字节各加80H得到，两个字节最高位均为1；“占2字节”只适用于该编码口径。
字形码（字模码）：用于输出显示/打印，用点阵表示（如16×16、24×24）。
⚫️【坑点】：
五笔字型属于输入码（外码），不是机内码！
传统教材中对汉字处理、存储常说使用机内码；现代系统具体采用UTF-8、UTF-16、GBK等编码，应以题干为准。
输出汉字字形的清晰度与汉字点阵的规模有关（点阵越大越清晰）。
在GB2312传统双字节口径下，汉字机内码的两个字节分别比国标码高80H，整体常表述为相差8080H。
考点22A：浮点数表示（常考概念）
🔴【基本组成】：浮点数通常由符号、阶码和尾数（有效数）组成；IEEE 754单精度为1位符号、8位阶码、23位小数部分，双精度为1+11+52位。
🔵【规格化目的】：在约定表示下使有效数字尽可能充分，保证表示唯一性并提高精度。
⚫️【坑点】：浮点数的表示范围主要受阶码位数影响，精度主要受尾数位数影响；浮点运算会有舍入误差，十进制小数不一定能被二进制精确表示。
第八章：进制转换（计算题必过）
考点23：R进制转十进制
🔵【方法】：“按权展开，相加求和”。
🔵【公式】：(N)R = Σ(每一位数字 × R^(该位所在位置))，小数点左第一位指数为0，向左递增，向右递减为负。
⚫️【例】：(1101.1)₂ = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ + 1×2⁻¹ = 8+4+0+1+0.5 = 13.5
考点24：十进制转R进制
🔵【整数部分】：“除R取余，倒序输出”（一直除到商为0，余数从下往上读）。
🔵【小数部分】：“乘R取整，正序输出”（一直乘到积为0或达到精度，整数从上往下读）。
⚫️【例】：(53.125)₁₀ = (110101.001)₂
考点25：二进制转八进制/十六进制（速算法）
🔵【二转八】：“三位并一位”。以小数点为界，左右每3位一组，不足补0，每组转成1位八进制数。
例：(10100010.0011)₂ = (242.14)₈
🔵【二转十六】：“四位并一位”。以小数点为界，左右每4位一组，不足补0，每组转成1位十六进制数。
例：(11101111)₂ = (EF)₁₆
⚫️【坑点】：十六进制数码包括0-9和A-F（A=10，B=11，C=12，D=13，E=14，F=15）。

第九章：故障与日常维护（常识应用）
考点27：常见故障原因
🔴【CMOS设置丢失/时间反复重置】：常见原因是主板实时时钟/CMOS供电电池电量不足，但也可能是固件设置、主板故障等，不能仅凭现象唯一断定。
🔴【显示器“花屏”/玩3D游戏死机】：通常怀疑显卡问题（特别是玩3D游戏时死机，重点检查显卡性能和散热）。
🔴【开机报警不启动】：报警码含义取决于BIOS/UEFI和主板厂商，内存接触不良只是常见原因之一。应先断电并按说明书查报警码，再检查内存、显卡、供电等；不应把“用橡皮擦金手指”写成通用标准步骤。
🔴【投影仪不显示电脑画面】：可能性：①未按笔记本切换键（Fn+F4/F5等）；②分辨率不匹配；③连接线松动或损坏。
🔴【计算机无声】：可能原因：声卡损坏、扬声器损坏、音量静音。显示器处于屏保状态不是无声的原因！
⚫️【坑点】：键盘进水后，仅仅擦干表面不能保证键盘恢复正常，水可能已渗入内部电路造成短路。

🔥 本章高频简答题汇总
1. 冯·诺依曼计算机的基本特点是什么？
🔵【踩分点】：① 由运算器、控制器、存储器、输入设备、输出设备五大部件组成。② 计算机内部采用二进制表示数据和指令。③ 采用**“存储程序、程序控制”**技术（将程序事先存入主存，计算机自动逐条取出并执行）。
2. 简述内存和外存的区别。
🔵【踩分点】：①内存速度快、容量小、价格高，断电后数据消失，CPU可直接访问，存放正在运行的程序和数据。②外存速度慢、容量大、价格低，断电后数据保留，CPU不能直接访问（需先调入内存），存放暂时不用的程序和数据。
3. 简述CPU的组成及各部分功能。
🔵【踩分点】：① CPU由运算器和控制器组成（Cache有时也在CPU内部）。②运算器：进行算术运算和逻辑运算，核心是ALU。③控制器：产生控制信号，指挥协调全机工作，核心是PC（程序计数器）和IR（指令寄存器）。
4. 简述总线的分类及各自功能。
🔵【踩分点】：①数据总线（DB）：传输数据，双向，宽度决定一次传输的数据量。②地址总线（AB）：传输地址，单向（CPU发出），宽度决定寻址范围（最大内存容量）。③控制总线（CB）：传输控制信号，双向。
📈 本章计算题汇总
1. 点阵汉字存储空间
公式：字节数 = (点阵行数 × 点阵列数) / 8
16×16点阵 = 32字节；24×24点阵 = 72字节；32×32点阵 = 128字节。
2. 进制转换
二转十：按权展开。十转二：整数除2倒取余，小数乘2正取整。
二转八：三位并一位。二转十六：四位并一位。
3. 总线带宽
公式：带宽 = (总线宽度 / 8) × 工作频率
单位：字节/秒（B/s）
4. 补码计算
负数补码 = 原码除符号位外逐位取反 + 1
