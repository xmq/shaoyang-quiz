## 第一部分：Visual Basic（VB）语言
### 考点1：VB概述
- **🔴【适用版本】**：本节按教师招聘试题最常见的 **Visual Basic 6.0（VB6）** 口径整理；VB.NET 的类型系统、工程文件和面向对象机制与 VB6 不同，不能混答。
- **🔴【VB全称】**：Visual Basic，Microsoft 公司开发的可视化、事件驱动程序设计语言。VB6 通常称为**基于对象**，不能据此断言它具备 VB.NET 那样完整的继承等面向对象机制。
- **🔴【VB最突出的特点】**：**事件驱动编程机制**（与传统程序设计语言相比的最大区别）。
- **🔴【VB的工程文件】**：
- 工程文件：`.vbp`（VBP = Visual Basic Project）
- 窗体文件：`.frm`
- 标准模块：`.bas`
- **⚫️【坑点】**：VB应用程序既可以**解释方式**执行（在IDE中），也可以**编译方式**执行（生成.exe文件）。说“只能以解释方式执行”是错的。
### 考点2：VB变量命名规则（极高频判断）
- **🔴【命名规则】**：
1. 按常见 VB6 教材口径，必须以**字母**开头；中文开发环境通常也允许以汉字等字母字符命名，但考试若未说明环境，优先采用“英文字母开头”的标准示例。
2. 后续可由**字母、数字、下划线**等组成（中文环境可使用汉字），不能含小数点、空格及运算符等特殊符号。
3. 长度**不超过255个字符**。
4. **不能使用VB关键字**（如：Dim、Sub、If、End、For、Next、Public、Private、Integer、String等）。
5. **不区分大小写**。
- **⚫️【坑点】**：
- `x.yz` → **不合法**（含小数点）。
- `a_b` → **合法**（下划线允许）。
- `Andif` → **合法**（不是VB关键字，And和If分开是关键字，但Andif不是）。
- `Dim` → **不合法**（是关键字）。
### 考点3：VB数据类型
- **🔴【基本数据类型】**：
- **Integer**：整型（2字节，-32768~32767）。
- **Long**：长整型（4字节）。
- **Single**：单精度浮点型（4字节）。
- **Double**：双精度浮点型（8字节）。
- **String**：**字符串型**。
- **Boolean**：布尔型（True/False，True转为数值为**-1**，False转为数值为**0**）。
- **Date**：日期/时间型，日期字面量用 `#` 括起来。数字日期的月/日次序可能受区域格式影响；需要无歧义时使用 `DateSerial(2017, 1, 1)`。
- **⚫️【坑点】**：
- Boolean类型的True转换为数值是 **-1**（不是1！），False是0。
- `Dim X, Y As Integer` → X被定义为**变体类型（Variant）**，Y被定义为**整型**。（注意：只有Y是Integer，X未声明类型默认为Variant）
### 考点4：VB运算符与表达式
- **🔴【算术运算符优先级】**（由高到低）：
1. `^`（乘方）
2. `-`（取负）
3. `*`（乘）、`/`（除，结果为浮点数）
4. `\`（整除，结果为整数）
5. `Mod`（取余）
6. `+`（加）、`-`（减）
- **🔴【常用函数】**：
- `Abs(x)`：**绝对值**。
- `Int(x)`：**向下取整**（取不大于x的最大整数）。`Int(5.49)=5`，`Int(-7.8)=-8`。
- `Fix(x)`：**取整数部分**（直接截断）。`Fix(-7.8)=-7`。
- `Sqr(x)`：**平方根**。
- `Rnd`：产生[0,1)之间的**随机数**。
- `Val("123")`：将**数字字符串**转换为**数值**，返回123。
- `Str(123)`：将数值转换为字符串，但正数结果通常保留一个前导空格（`Str(123)` 为 `" 123"`）；不需要前导空格时常用 `CStr(123)`，结果为 `"123"`。
- `Len("ABC")`：返回**字符串长度**，返回3。
- `Mid("ABCDE", 2, 3)`：从第2个字符开始取3个，返回"BCD"。
- `Right("Beijing", 4)`：从右边取4个字符，返回"jing"。
- **⚫️【坑点】**：
- `Int(-7.8)` = **-8**（向下取整，取不大于-7.8的最大整数）。
- `10 + Int(5.49)` = 10 + 5 = **15**。
- 表达式 `6 + 10 Mod 10 \ 9 / 3 + 2 ^ 3` = **15**（按优先级逐步计算）。
- `156 \ 10 Mod 7` = 15 Mod 7 = **1**（整除优先级高于Mod）。
- `Abs(-3.8) = 3.8`，`Sqr(49) = 7`，`Int(-7.8) = -8`。
### 考点5：VB控制结构（程序阅读必考）
- **🔴【If语句】**：
```vb
If 条件 Then
语句1
Else
语句2
End If
```
单行格式：`If 条件 Then 语句`（无End If）
- **🔴【Select Case语句】**（多分支选择）：
```vb
Select Case 表达式
Case 值1: 语句1
Case 值2: 语句2
Case Else: 语句3
End Select
```
*注意：VB用`=`比较相等，C用`==`，Python用`==`*
- **🔴【For循环】**：
```vb
For i = 1 To 10 Step 2   ' i从1到10，步长为2（1,3,5,7,9）
循环体
Next i
```
- **🔴【Do While循环】**：
```vb
Do While 条件
循环体
Loop
```
- **⚫️【程序阅读真题】**：
- For i = 1 To 11 Step 3 → i取值：1,4,7,10 → 循环体执行 **4** 次。
- `s=0: i=0: Do While s<10: i=i+1: s=s+i*i: Loop` → 循环体执行 **3** 次（i从1到3，s=1+4+9=14，第4次条件不满足）。
### 考点6：VB数组
- **🔴【数组声明】**：未写 `Option Base 1` 时，`Dim a(5) As Integer` 通常表示 a(0) 到 a(5)，共 **6** 个元素；若模块使用 `Option Base 1`，则默认下限变为 1。最稳妥的写法是显式指定下限。
- **🔴【指定下限】**：`Dim a(1 To 5) As Integer` → a(1)到a(5)，共 **5** 个元素。
- **🔴【二维数组】**：`Dim a(0 To 5, 1 To 5) As Integer` → 6×5 = **30** 个元素。
- **⚫️【坑点】**：VB6/VBA 数组的默认下限通常为 0，但可受模块级 `Option Base 0/1` 影响；`Dim a(1 To 5)` 不受该设置影响。
### 考点7：VB对象、属性、方法与事件（操作题高频）
- **🔴【四个概念】**：对象是被操作的实体；属性描述对象状态；方法是对象能够执行的操作；事件是对象能够识别并响应的外部动作。
- **🔵【常见属性】**：`Name` 是程序代码中引用控件的名称；`Caption` 常表示窗体、标签、按钮上显示的标题；文本框主要使用 `Text`。不要把对象名称与界面显示文字混为一谈。
- **🔵【常见事件】**：`Form_Load` 在窗体装载时发生，`CommandButton_Click` 在按钮被单击时发生，`TextBox_Change` 在文本内容改变时发生。事件过程名称通常由“对象名_事件名”构成。
- **⚫️【判断陷阱】**：事件由用户操作或系统状态触发，事件过程才是响应事件执行的代码；“Click 是按钮的方法”在 VB6 常见试题口径下通常错误。
## 第二部分：C语言
### 考点8：C语言概述
- **🔴【C程序结构】**：按标准的宿主环境 C 程序口径，一个程序由一个或多个函数组成，并有一个程序入口 `main`。推荐写作 `int main(void)` 或 `int main(int argc, char *argv[])`；`void main()` 不是 ISO C 的标准形式。嵌入式等独立环境可以不以 `main` 为入口，考试未说明时按宿主环境作答。
- **🔴【C程序运行步骤】**：**编辑 → 编译 → 连接 → 运行**。
- **⚫️【坑点】**：`main()`函数可以在程序文件的**任何位置**，不一定在开头或结尾。
### 考点9：C语言标识符规则
- **🔴【命名规则】**：
1. 只能由**字母、数字、下划线**组成。
2. 第一个字符**必须是字母或下划线**（不能是数字）。
3. **区分大小写**（`abc`和`ABC`是不同的变量）。
4. **不能使用C关键字**（如：if、case、switch、int、float、return等）。
- **⚫️【坑点】**：
- `_int&int` → **不合法**（含`&`特殊符号）。
- `_Int` → **合法**（下划线开头）。
- `intint` → **合法**（不是关键字）。
- `case` → **不合法**（是关键字）。
### 考点10：C语言数据类型
- **🔴【基本类型】**：
- `char`：字符型，按 C 语言定义恒为 **1 个 C 字节**；一个字节至少 8 位。
- `short`、`int`、`long` 的精确字节数由实现决定，标准只保证 `sizeof(char) ≤ sizeof(short) ≤ sizeof(int) ≤ sizeof(long)` 以及各自的最小取值范围。
- 在多数当前 32/64 位教学环境中，`short` 常为 2 字节、`int` 常为 4 字节、`float` 常为 4 字节、`double` 常为 8 字节；题目指定编译器或存储模型时以题设为准。
- **🔴【常量的合法判断】**：
- `1.234e04` → **合法**（1.234×10⁴）。
- `1.234e0.4` → **不合法**（指数必须是整数）。
- `1.234e4` → **合法**。
- `1.234e0` → **合法**。
- **⚫️【坑点】**：在试题常用实现中，`double` 一般为 64 位（8 字节），但不能把实现相关的 `sizeof` 结论写成所有 C 平台都必须如此。
### 考点11：C语言运算符
- **🔴【自增自减运算符】**：
- `i++`：表达式的值是递增前的 i，随后产生 i 加 1 的副作用。
- `++i`：先使 i 加 1，表达式的值是递增后的 i。
- *例*：`int i=1; j=-i++;` → i=**2**，j=**-1**（先取i=1，j=-1，再i自增为2）。
- **🔴【逗号表达式】**：取**最右边表达式**的值。`a=2*6, a*3, a+5` → a=12，表达式值为12+5=**17**。
- **🔴【条件运算符（三目运算符）】**：`表达式1 ? 表达式2 : 表达式3`。`int a=7,b=9; t=(a>b)?a:b;` → t=**9**（a>b为假，取b）。
- **🔴【逻辑运算符】**：
- `&&`（与）：两个都为真才为真。
- `||`（或）：一个为真即为真。
- `!`（非）：取反。
- **🔵【短路求值】**：`a && b` 在 a 为假时不再求值 b；`a || b` 在 a 为真时不再求值 b。可用于防止非法运算，如 `p != NULL && p->value > 0`。
- **🔵【整数除法】**：两个整数相除仍为整数；C99 及以后标准规定商向 0 截断，如 `-7/3 == -2`，余数满足 `(a/b)*b + a%b == a`，因此 `-7%3 == -1`。
- **⚫️【坑点】**：
- 赋值运算符`=`的结合方向是**“自右至左”**。
- `strcmp(a, b) == 0` 才表示两个字符串内容相等；`if (strcmp(a,b))` 在字符串**不相等**时条件为真。
- `scanf("%d%d%d", &x, &y, &z)` → 正确的输入语句，`&`取地址。
- 不要在同一个未规定求值顺序的表达式中多次修改同一标量，例如 `i = i++ + ++i` 会产生未定义行为，不能机械套用“先用后加”求唯一答案。
- 赋值、条件运算符以及多数一元运算符等均具有从右到左的结合性；不能说“只有赋值运算符从右结合”。结合性也不等同于操作数实际求值顺序。
### 考点12：C语言流程控制
- **🔴【if-else语句】**：
```c
if (条件) {
语句1;
} else {
语句2;
}
```
- **🔴【switch语句】**：
```c
switch(表达式) {
case 值1: 语句1; break;
case 值2: 语句2; break;
default: 语句3;
}
```
- **🔴【循环结构】**：
- `for(i=0; i<5; i++)` → 循环体执行 **5** 次（i=0,1,2,3,4）。
- `while(条件)` → 先判断后执行。
- `do-while(条件)` → 先执行后判断，循环体**至少执行一次**。
- **⚫️【程序阅读真题】**：
- `for(x=0,y=0; (y!=1234) && (x<5); x++)` → 执行 **5** 次（x=0~4）。
- `switch(i){ case 9: i+=1; case 10: i+=1; case 11: i+=1; default: i+=1; }` → i=10，无break，依次执行case10、case11、default → i=**13**。
### 考点13：C语言函数
- **🔴【函数的定义】**：`返回值类型 函数名(参数列表) { 函数体 }`。
- **🔴【形参与实参】**：形参是函数定义时的参数，实参是调用时传入的值。形参的修改**不影响**实参（值传递）。
- **🔵【指针传参辨析】**：C 始终按值传递。传入指针时复制的是地址值，函数可通过该地址修改调用者对象；这不是语言层面的“引用传递”。数组形参在函数参数位置会调整为指针类型。
- **🔴【递归函数】**：函数调用**自身**。
```c
int x(int n) {
if (n <= 3) return 1;
else return x(n-2) + x(n-4) + 1;
}
```
- **⚫️【程序阅读真题】**：
- `void fun(int x,int y,int z){ z=x*x+y*y; } int main(void){ int a=31; fun(5,2,a); printf("%d",a); return 0; }` → 输出 **31**（形参 z 的修改不影响实参 a）。旧教材省略函数返回类型的写法不符合 C99 及以后标准。
- 递归函数`x(x(8))`需要计算 **18** 次x函数。
- **⚫️【递归前提】**：递归必须有可到达的终止条件，并使问题规模趋近终止条件；否则可能无限递归并耗尽调用栈。
### 考点14：C数组、指针与字符串（岗位考试高频）
- **🔴【数组】**：同一数组的元素类型相同，并按下标顺序连续存放；长度为 n 的数组合法下标为 0～n-1。越界访问属于未定义行为，编译器不一定报错。
- **🔵【数组名退化】**：在大多数表达式中，数组名会转换为指向首元素的指针；但作为 `sizeof`、一元 `&` 等运算的操作数时不发生该转换。数组名本身不是可修改的指针变量。
- **🔴【指针】**：`&x` 取得 x 的地址，`*p` 访问 p 所指对象。使用未初始化、已释放或空指针解引用都会产生未定义行为。
- **🔵【字符串】**：C 字符串是以空字符 `\0` 结尾的字符序列。字符数组容量必须为正文字符数加 1，例如 `char s[]="abc"` 占 4 个字节。
- **⚫️【输入陷阱】**：`scanf("%s", s)` 遇空白停止且必须限制宽度以防溢出；读取整行更适合用 `fgets`。字符串内容比较使用 `strcmp`，不能用 `==` 比较字符数组内容。
- **🔵【变量作用域与生存期】**：局部变量通常具有块作用域；文件作用域变量常称全局变量。局部 `static` 变量只初始化一次，生存期贯穿程序执行，但作用域仍限于所在块。
## 第三部分：Python语言
### 考点15：Python概述
- **🔴【Python特点】**：
- Python 通常以解释器运行，但“解释型/编译型”取决于实现：CPython 会先把源代码编译为字节码，再由虚拟机执行。因此“Python 完全不编译”是错的。
- 可调用C语言编写的底层代码（**黏性扩展**）。
- **平台无关**（可在Windows、Linux、Mac等平台运行）。
- 通过缩进表示代码块，强调可读性。
- **变量无需声明**，赋值即创建。
- **⚫️【坑点】**：不能脱离实现和任务笼统断言 Python 一定比所有编译型语言慢。Python 广泛用于 Web、数据分析、人工智能、自动化和教学，也能参与系统工具开发；“只能用于某一领域”通常是错误表述。
### 考点16：Python语法规则
- **🔴【缩进】**：Python 用缩进表示代码块。同一代码块必须保持一致；PEP 8 **建议**每级 4 个空格，但“语法强制只能用4个空格”是错的。Tab 与空格不一致可能触发 `TabError` 或改变层级。
- **🔴【注释】**：
- 单行注释：`# 注释内容`
- 严格说 Python 没有专门的“多行注释”语法；`'''...'''` 或 `"""..."""` 是三引号字符串。未赋给变量的独立字符串常被当作说明文字，模块、类、函数开头的三引号字符串是文档字符串（docstring）。
- **🔴【变量】**：名字通常无需预先声明，赋值会建立名字与对象的绑定。**对象有类型，变量名本身不固定绑定某一种类型**，因此同一名字可先后绑定不同类型对象。
- **⚫️【坑点】**：
- 所有代码都必须在行前至少加一个空格 → **错**！只有需要缩进的代码块才需要缩进。
- 已知`x=1.5`，赋值`x='china'`**不会报错**（Python变量可以改变类型）。
### 考点17：Python数据类型与运算
- **🔴【基本数据类型】**：
- `int`：整数。
- `float`：浮点数。
- `bool`：布尔值（True/False）。
- `complex`：复数。
- `str`：字符串。
- **🔴【运算符】**：
- `/`：除法（结果为浮点数）。`3.0/5 = 0.6`
- `//`：向负无穷取整的商（floor division），不是简单“截去小数”。`3//5 = 0`，但 `-3//2 = -2`。
- `%`：取余。
- `**`：乘方。
- **🔴【序列类型】**：元素具有先后次序，可通过整数索引访问，并通常支持切片。
- 字符串（str）、**元组（tuple）**、**列表（list）**、`range` 都是常见序列类型。
- **字典（dict）** 是映射类型，**不是**序列类型！
- **🔴【列表操作】**：
- 列表中的元素**可以是不同数据类型**。
- 列表是**可变对象**，`append()` 原地修改同一个列表，因此列表对象的 `id` 不变；但其内部存储区可能重新分配，不能表述为“所有内存地址都不变”。
- **⚫️【程序阅读真题】**：
- `x = 12 + 3 * ((5 * 8) - 14) // 6; print(x)` → 25。
- `for i in range(1,6): if i%4==0: continue; else: print(i)` → 输出 **1,2,3,5**（4被跳过）。
- `x = 'R\OS\OT'; print(len(x))`：`\O` 不是 Python 定义的转义序列；按当前常见实现保留反斜杠，长度为 **7**，并可能产生无效转义警告。表达 Windows 路径宜写原始字符串 `r'R\OS\OT'` 或转义反斜杠 `'R\\OS\\OT'`，二者长度同为 7。
### 考点18：Python流程控制
- **🔴【条件判断】**：
```python
if 条件:
语句块
elif 条件:
语句块
else:
语句块
```
Python支持链式比较：`4 <= 5 <= 6` → **True**。
- **🔴【循环】**：
```python
for i in range(1, 10, 2):   # i = 1,3,5,7,9
循环体
while 条件:
循环体
```
- **🔴【break与continue】**：
- `break`：**终止整个循环**。
- `continue`：**结束本次循环**，进入下一次循环。
- **⚫️【坑点】**：`break`用来结束当前循环语句，**但跳出当前循环体**（是跳出，不是不跳出）。
### 考点19：Python容器、切片与函数（高频补充）
- **🔴【可变与不可变】**：列表、字典、集合通常可变；数字、字符串、元组、`frozenset` 不可变。元组不可变是指元组保存的引用序列不能改，元组内部若引用可变对象，该对象本身仍可能被修改。
- **🔵【切片规则】**：`s[start:stop:step]` 左闭右开，不包含 `stop`；`s[::-1]` 常用于逆序。索引越界通常抛出 `IndexError`，普通切片越过边界会自动截断。
- **🔵【字典与集合】**：字典按键访问，键必须可哈希；集合元素互异且可哈希。Python 3.7 起语言规范保证字典保持插入顺序，但“集合按插入顺序输出”没有保证。
- **🔵【函数参数】**：Python 采用“对象引用按值传递/共享传参”。函数可原地修改传入的可变对象，但给形参重新赋值不会让调用者变量改绑。
- **⚫️【默认参数陷阱】**：默认参数在函数定义时求值一次。避免 `def f(a=[]): ...` 造成多次调用共享同一列表，常写成 `def f(a=None): a = [] if a is None else a`。
- **⚫️【作用域口诀】**：名字查找遵循 LEGB：Local（局部）→ Enclosing（闭包外层）→ Global（全局）→ Built-in（内置）。函数内给全局变量赋值需声明 `global`；修改外层函数变量绑定使用 `nonlocal`。
### 考点20：跨语言易混对照（判断题高频）
- **🔴【相等与赋值】**：VB6 中赋值和相等比较都使用 `=`，由语境区分；C/Python 赋值用 `=`，相等比较用 `==`。
- **🔵【逻辑运算】**：VB6 常用 `And/Or/Not`；C 用 `&&/||/!`；Python 用 `and/or/not`。Python 的 `and/or` 返回参与运算的对象之一，不保证只返回 `True/False`。
- **🔵【下标与区间】**：C 数组从 0 开始；Python 序列从 0 开始且支持负索引；VB6 数组下限可显式设置，并可能受 `Option Base` 影响。
- **⚫️【版本前提】**：Python 题应留意 Python 2 与 Python 3 的差异（如 `/` 和 `print`）；当前未注明版本时通常按 Python 3。C 题留意编译器、标准版本及数据模型；VB 题留意 VB6 与 VB.NET。
