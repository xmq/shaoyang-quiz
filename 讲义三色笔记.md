# 信息科技教师招聘·讲义三色笔记（初稿）

> 颜色约定：🔴 **重点**（必须掌握）；🔵 **易错**（高频混淆）；🟢 **记忆**（口诀/速记）。
> 本笔记依据 `source` 目录讲义 PDF 的目录页、正文题例和可提取文本整理；页码为 PDF 页码（从 1 开始）。扫描页文字可能存在 OCR 缺失，复习时应以原 PDF 对照。

## 一、程序设计、算法与数据结构

### 1. 程序设计语言
- 🔴 **重点**：程序设计语言讲义覆盖 LOGO、Scratch、VB、C、Python 五类语言（`02、第二篇-第一章-0101（晚）.pdf`，p1-2）。掌握变量、表达式、分支、循环、数组/列表、函数与调试。
- 🔵 **易错**：区分“语法错误”（不符合语言规则，运行前/编译时发现）、“运行时错误”（执行中异常）和“逻辑错误”（结果不符合预期但程序可运行）。
- 🟢 **记忆**：算法三特征可记为“有穷、确定、可行”；流程图判断框用菱形，处理框用矩形，输入/输出用平行四边形。

### 2. 数据结构与算法
- 🔴 **重点**：线性表、栈、队列、树、图及查找/排序是核心内容（`05、第二篇-0108_.pdf`，p1-3）。栈遵循后进先出（LIFO），队列遵循先进先出（FIFO）。
- 🔵 **易错**：二分查找要求数据**有序**；树的遍历顺序（先序/中序/后序）不要与层次遍历混淆。
- 🟢 **记忆**：栈“进出同端”，队列“入队尾、出队头”；快速排序平均 O(n log n)，最坏 O(n²)。

### 3. 软件工程
- 🔴 **重点**：生命周期通常包括需求分析、设计、实现、测试、运行维护；测试类型包括单元、集成、系统、验收测试（`06、第二篇-0110_.pdf`，p1-3）。
- 🔵 **易错**：瀑布模型阶段顺序固定、变更成本高；敏捷强调迭代交付、持续反馈；“验证（Verification）”关注是否正确构造产品，“确认（Validation）”关注是否构造了正确产品。
- 🟢 **记忆**：白盒看内部逻辑，黑盒看输入输出；软件质量属性可记“功、靠、易、效、维、移”（功能、可靠、易用、效率、可维护、可移植）。

## 二、计算机组成、网络、数据库与操作系统

### 1. 计算机组成
- 🔴 **重点**：CPU 由运算器、控制器等组成；存储器分层为寄存器、Cache、内存、外存（`06、第三篇-0110_.pdf`，p1-3）。
- 🔵 **易错**：容量单位换算以 1024 为进位（教材语境）；RAM 断电丢失，ROM/闪存通常具有非易失性。
- 🟢 **记忆**：存储层次“越上越快、越贵、容量越小”。

### 2. 计算机网络
- 🔴 **重点**：OSI 七层自下而上：物理、数据链路、网络、传输、会话、表示、应用；TCP/UDP 属于传输层，IP 属于网络层（`08、第三篇-0115_.pdf`，p1-3）。
- 🔵 **易错**：交换机主要工作在数据链路层，路由器工作在网络层；LAN/MAN/WAN 分别按覆盖范围划分。
- 🟢 **记忆**：TCP“可靠有连接”，UDP“快速无连接”；ARP 是 IP 地址到 MAC 地址的解析。

### 3. 数据库
- 🔴 **重点**：数据库三级模式为外模式、概念模式、内模式；关系模型以表、行、列组织数据（`09、第三篇-0117_.pdf`，p1-3）。
- 🔵 **易错**：主键唯一且非空；外键可重复但引用另一表主键/候选键；`WHERE` 过滤行，`HAVING` 过滤分组。
- 🟢 **记忆**：事务 ACID：原子性、一致性、隔离性、持久性；规范化目标是减少冗余和更新异常。

### 4. 操作系统
- 🔴 **重点**：操作系统提供人机接口、资源管理和程序运行环境；进程是资源分配基本单位，线程是调度基本单位（`10、第三篇-0119_.pdf`，p1-3）。
- 🔵 **易错**：死锁四条件：互斥、占有且等待、不可剥夺、循环等待；缺一不可。
- 🟢 **记忆**：页面置换常见 FIFO、LRU、OPT；LRU 即“最近最少使用”。

## 三、办公软件与多媒体

### 1. Word
- 🔴 **重点**：样式用于统一格式；分节符控制页面布局；邮件合并由主文档、数据源和合并域组成（`11、第四篇第一章第一节-Word.pdf`，p1-10）。
- 🔵 **易错**：分页符只换页，分节符可改变页眉页脚、纸张和方向；“保存”覆盖原文档，“另存为”生成新文件。
- 🟢 **记忆**：Ctrl+C 复制、Ctrl+X 剪切、Ctrl+V 粘贴、Ctrl+S 保存。

### 2. Excel
- 🔴 **重点**：相对引用、绝对引用（`$A$1`）和混合引用；常用函数 SUM、AVERAGE、IF、COUNTIF、VLOOKUP（`12、第四篇第一章第二节-Excel_.pdf`，p1-10）。
- 🔵 **易错**：数字前加英文单引号可按文本保存；排序会改变记录顺序，筛选只隐藏不符合条件的行。
- 🟢 **记忆**：绝对引用“行列都锁”；填充序列先输入两个样本值更稳妥。

### 3. PowerPoint 与 WPS
- 🔴 **重点**：母版控制统一版式；幻灯片浏览视图适合调整顺序；动画分进入、强调、退出和动作路径（`12、第四篇第一章第三节-PowerPoint_.pdf`，p1-8；`13、第四篇第二章-WPS.pdf`，p1-5）。
- 🔵 **易错**：切换效果作用于幻灯片之间，动画效果作用于对象；“隐藏幻灯片”不等于删除。

### 4. Photoshop、Flash、Windows
- 🔴 **重点**：PS 图层蒙版中黑色隐藏、白色显示、灰色半透明（`14、第四篇第四章-flash_.pdf`，p1）；Flash 元件分图形、影片剪辑、按钮（`14、第四篇第四章-flash_.pdf`，p1-10）；Windows 负责硬件资源管理和提供操作界面（`14、第四篇第五章-Windows_.pdf`，p1-3）。
- 🔵 **易错**：蒙版涂黑不会删除原图；Flash 运动补间要求起止关键帧对象为同一元件；Alt+PrintScreen 截取当前活动窗口。
- 🟢 **记忆**：PS 蒙版“黑遮白显灰半透”；Flash “图形随时间轴、影片剪辑独立播放、按钮响应交互”。

## 四、信息科技教学论

- 🔴 **重点**：信息科技课程强调信息意识、计算思维、数字化学习与创新、信息社会责任等核心素养（`15、第五篇-信息科技教学论_.pdf`，p1-10）。
- 🔵 **易错**：教学目标应可观察、可测量；过程性评价关注学习过程，终结性评价关注阶段结果；项目式学习须有真实问题、持续探究和成果展示。
- 🟢 **记忆**：教学流程可记“情境—问题—探究—展示—评价—迁移”。

## 五、大数据与新补充方向

- 🔴 **重点**：大数据 5V：Volume（规模）、Velocity（速度）、Variety（多样）、Value（价值）、Veracity（真实性）；HDFS 负责分布式存储，MapReduce 负责批处理计算，YARN 负责资源管理。
- 🔵 **易错**：HDFS NameNode 管元数据，DataNode 存数据块；Spark 的 RDD 具备不可变、可分区、可并行计算特性；Kafka 的 topic 可分 partition，consumer group 内同一分区同一时刻只由一个消费者消费。
- 🟢 **记忆**：Lambda 架构“批处理+流处理”；数据仓库 ETL 是“抽取（Extract）—转换（Transform）—加载（Load）”。
