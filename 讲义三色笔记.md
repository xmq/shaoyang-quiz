# 计算机岗位考试 · 三色笔记完整版

## Office软件操作

### 第一章：Word 文字处理软件

#### 考点1：文件、视图与编辑状态

<span style="color:#c62828;font-weight:700">🔴【常见格式】：Word 2007 及以后默认格式为 `.docx`，Word 97—2003 常用 `.doc`；还可另存或导出为 `.pdf`、`.txt`、网页等。`.dotx` 是不含宏的模板，`.dotm` 是启用宏的模板。</span>

<span style="color:#c62828;font-weight:700">🔴【常用视图】：</span>

页面/打印布局视图：按打印页面显示页边距、页眉页脚等，适合排版；多数桌面版默认采用该视图。

阅读模式：侧重阅读，部分编辑工具会收起；不同版本名称和布局略有差异。

Web 版式视图：按网页方式连续显示，内容随窗口宽度重排。

大纲视图：按标题级别查看、折叠和调整长文档结构。

草稿视图：侧重快速编辑正文，不按完整打印页面呈现页眉页脚等对象。

⚫️【版本条件】：不能把“默认视图”“按钮位置”写成跨版本不变的结论；Word 2003、2010、2016、Microsoft 365 的界面和默认设置可能不同。

<span style="color:#c62828;font-weight:700">🔴【插入与改写】：插入模式在插入点处加入字符；改写模式会覆盖插入点后的字符。`Insert` 是否直接切换改写模式取决于 Word 选项和键盘，较新版本可能需要先启用该功能。</span>

#### 考点2：文本选择、删除与移动

<span style="color:#c62828;font-weight:700">🔴【鼠标操作必须说明位置】：</span>

<span style="color:#1565c0;font-weight:700">🔵正文文字区域：单击定位插入点；双击通常选中一个词；在同一段落内快速三击通常选中整个段落。</span>

<span style="color:#1565c0;font-weight:700">🔵页面左侧选定栏（正文左边距）：单击选中所指行；双击选中相邻段落；三击选中整篇文档。</span>

<span style="color:#1565c0;font-weight:700">🔵拖动可选择连续文本；按住 `Ctrl` 再选择可选取多个不连续区域；`Ctrl+A` 选中全文。</span>

⚫️【判断陷阱】：题目只写“单击、双击、三击”，却不说明鼠标位于正文还是选定栏，结论是不完整的。触摸设备、Web 版 Word 的手势也不宜直接套用桌面版鼠标规则。

<span style="color:#c62828;font-weight:700">🔴【Backspace 与 Delete】：没有选中文本时，`Backspace` 通常删除插入点左侧字符，`Delete` 通常删除右侧字符；已经选中文本时，两者都可删除选中内容。</span>

<span style="color:#c62828;font-weight:700">🔴【移动与复制】：拖动选中文本通常是移动；按住 `Ctrl` 拖动通常是复制。`Ctrl+X/C/V` 分别是剪切、复制、粘贴。</span>

<span style="color:#c62828;font-weight:700">🔴【字号】：中文字号中一般“初号”大于“一号”，“二号”大于“四号”；以磅为单位时，磅值越大，字越大。不能混用两套序列直接比较而不看换算值。</span>

#### 考点3：段落、页面与节

<span style="color:#c62828;font-weight:700">🔴【段落格式】：对齐、缩进、段前/段后间距、行距属于段落格式；字体、字号、颜色、字符间距属于字符格式。</span>

<span style="color:#1565c0;font-weight:700">🔵【行距辨析】：单倍、1.5 倍、多倍行距会随字体等因素计算；“固定值”按指定磅值设置，字号过大时可能出现字符被遮挡；“最小值”至少保证指定值，必要时会自动增大。</span>

<span style="color:#c62828;font-weight:700">🔴【分页符与分节符】：</span>

分页符只强制从下一页开始，不创建新的节。

分节符把文档分成不同的节，可分别设置页边距、纸张方向、分栏、页码格式以及页眉页脚等。

<span style="color:#1565c0;font-weight:700">🔵【不同页眉页脚步骤】：插入分节符 → 进入后一节页眉/页脚 → 关闭“链接到前一节” → 分别编辑。仅插入分页符不能实现独立设置。</span>

⚫️【首页不同/奇偶页不同】：若只是同一节的首页或奇偶页需要不同页眉页脚，可启用相应选项，不一定要新增分节符。

<span style="color:#c62828;font-weight:700">🔴【分栏】：分栏属于页面布局，既可应用于选中文本，也可应用于当前节或整篇文档。可设置栏宽、间距和分隔线。</span>

⚫️【数量陷阱】：可设栏数受 Word 版本、页面宽度、页边距和栏间距约束；“最多 11 栏”只可能是特定教材环境或默认页面参数下的口径，不应当作所有版本的绝对上限。

#### 考点4：图文混排与对象

<span style="color:#c62828;font-weight:700">🔴【嵌入型与浮动对象】：嵌入型图片像一个字符一样随文本排列；其他环绕方式属于浮动对象，可相对页面、段落或字符定位。</span>

<span style="color:#c62828;font-weight:700">🔴【常见环绕方式】：四周型按对象外接矩形环绕；紧密型更贴近对象轮廓；衬于文字下方常用于背景；浮于文字上方可能遮挡文字。</span>

⚫️【默认方式有条件】：很多桌面版 Word 的“插入图片”默认采用嵌入型，但用户可在选项中修改默认环绕方式，粘贴来源也会影响结果。题干未指定环境时，应说“通常/默认设置下”。

<span style="color:#c62828;font-weight:700">🔴【等比例绘制】：拖动矩形或椭圆时按住 `Shift` 通常约束为正方形或正圆；按住 `Ctrl` 常用于从中心绘制。不同工具、触控方式可能略有差异。</span>

<span style="color:#c62828;font-weight:700">🔴【组合条件】：通常只能组合浮动对象；嵌入型对象需先改为非嵌入型环绕。组合后可整体移动、缩放和设置格式。</span>

#### 考点5：表格

<span style="color:#c62828;font-weight:700">🔴【选择对象决定操作结果】：</span>

选中单元格内容后按 `Delete`，通常只清除内容，不删除单元格结构。

使用“删除单元格/行/列/表格”命令，才会改变表格结构；选中整张表并执行“删除表格”可删除表格和内容。

<span style="color:#1565c0;font-weight:700">🔵【拆分与合并】：合并单元格是把相邻单元格合为一个；拆分单元格是把一个单元格分成多个；拆分表格会在指定行前把一张表分成两张。两张相邻表格能否重新连接，取决于它们之间的段落标记、环绕方式和结构，不能笼统写成“拆分后一定可直接合并”。</span>

<span style="color:#c62828;font-weight:700">🔴【表格计算】：Word 表格可使用 `SUM`、`AVERAGE` 等简单公式，例如 `=SUM(ABOVE)`；修改源数据后往往需要手动更新域（如选中后按 `F9`），不像 Excel 那样普遍实时重算。</span>

<span style="color:#c62828;font-weight:700">🔴【重复标题行】：跨页表格可将首行设置为标题行重复出现；若表格被手动分页或嵌套，效果可能受限。</span>

#### 考点6：查找替换、样式与自动化

<span style="color:#c62828;font-weight:700">🔴【查找与替换】：可查找/替换文本、格式、通配符和多种特殊字符。桌面版还可用特殊代码查找某些嵌入式图形；浮动形状及复杂对象的查找能力有限。因此“只能替换文字”或“绝对不能查找图形”都不严谨。</span>

<span style="color:#c62828;font-weight:700">🔴【格式刷】：单击格式刷通常使用一次；双击可连续应用；按 `Esc` 或再次单击退出。格式刷复制的是格式，不复制正文内容。</span>

<span style="color:#c62828;font-weight:700">🔴【样式】：样式是一组命名格式，可统一标题与正文外观，也是自动目录、大纲导航和长文档结构化的基础。手工把字放大加粗不等于设置了“标题样式”。</span>

<span style="color:#c62828;font-weight:700">🔴【目录】：先正确应用标题样式，再插入自动目录；标题或页码变化后需要更新目录，而不是手工逐项修改。</span>

<span style="color:#c62828;font-weight:700">🔴【修订与批注】：修订记录插入、删除和格式更改，审阅者可接受或拒绝；批注用于讨论和备注，通常不直接改正文。删除批注不等于接受修订。</span>

<span style="color:#c62828;font-weight:700">🔴【邮件合并】：由主文档、数据源和合并域构成，可批量生成格式相同而个别字段不同的信函、标签、证书等。</span>

<span style="color:#c62828;font-weight:700">🔴【宏】：宏是为自动执行一系列操作而记录或编写的命令序列，常以 VBA 过程实现；“宏是一段应用程序”过于笼统。宏功能本身不是病毒，但不可信文档中的宏可能携带恶意代码。</span>

### 第二章：Excel 电子表格软件

#### 考点1：工作簿、工作表与单元格

<span style="color:#c62828;font-weight:700">🔴【常见格式】：Excel 2007 及以后默认 `.xlsx`；旧格式为 `.xls`；含 VBA 宏的工作簿常用 `.xlsm`；模板常用 `.xltx`。</span>

<span style="color:#c62828;font-weight:700">🔴【工作簿与工作表】：工作簿是文件，工作表是工作簿内的表页。新建工作簿默认工作表数量可由版本和用户设置决定：Excel 2013 及以后通常默认 1 张，较早版本常见 3 张，不能统一写成“永远 3 张”。</span>

⚫️【数量上限】：现代 Excel 可添加的工作表数量主要受可用内存限制；“最多 255 张”不是现代版本的固定上限。旧版设置中“新工作簿内工作表数”可选到 255，也不等于所有工作簿只能有 255 张。

<span style="color:#c62828;font-weight:700">🔴【单元格地址】：`B3` 表示 B 列第 3 行。活动单元格是当前输入位置，通常只有一个；一个选区可以同时包含许多单元格。</span>

<span style="color:#c62828;font-weight:700">🔴【现代工作表规模】：Excel 2007 及以后每张工作表为 1,048,576 行、16,384 列，最后一列为 `XFD`；Excel 2003 及以前为 65,536 行、256 列。</span>

#### 考点2：数据输入、显示与填充

<span style="color:#c62828;font-weight:700">🔴【长编号】：身份证号、学号等不参与运算的编号应按文本存储，可先设为“文本”，或在输入时加英文单引号。Excel 对数值仅保证约 15 位有效数字，直接当数值输入可能导致末位变为 0。</span>

<span style="color:#c62828;font-weight:700">🔴【`######`】：最常见原因是列宽不足；日期或时间结果为负数等情况下也可能显示一串 `#`。因此应先结合单元格内容和格式判断，不能只认定为列宽问题。</span>

<span style="color:#c62828;font-weight:700">🔴【单元格内换行】：Windows 桌面版通常用 `Alt+Enter`；还可启用“自动换行”。</span>

<span style="color:#c62828;font-weight:700">🔴【自动填充】：可复制常量和公式，也可扩展日期、星期、等差序列及自定义序列。公式被填充时，引用会按相对/绝对规则变化。“不能填充公式”是错误说法。</span>

⚫️【自动识别有环境条件】：输入 `1/2` 可能按当前区域设置识别为日期，也可能按格式或区域设置产生不同结果；要保留字符串应设为文本。公式必须以 `=` 开头，`=2/5` 才执行除法。

#### 考点3：引用、运算符与错误值

<span style="color:#c62828;font-weight:700">🔴【相对引用】：`A1`，复制公式时行列按相对位置改变。</span>

<span style="color:#c62828;font-weight:700">🔴【绝对引用】：`$A$1`，复制公式时行列都锁定。</span>

<span style="color:#c62828;font-weight:700">🔴【混合引用】：`$A1` 锁列不锁行；`A$1` 锁行不锁列。编辑公式时可用 `F4` 在引用形式间切换（部分笔记本需配合 `Fn`）。</span>

<span style="color:#c62828;font-weight:700">🔴【跨表引用】：`Sheet2!$A$4` 表示 Sheet2 中 A4 的绝对引用；工作表名含空格时需写成 `'期末成绩'!A1`。</span>

<span style="color:#c62828;font-weight:700">🔴【运算优先级】：引用运算 → 负号/百分号/幂 → 乘除 → 加减 → 连接 `&` → 比较；有疑问时用括号明确顺序。</span>

<span style="color:#c62828;font-weight:700">🔴【常见错误值】：`#DIV/0!` 除数为 0 或空；`#VALUE!` 数据类型不合适；`#NAME?` 名称或函数名无法识别；`#REF!` 引用无效；`#N/A` 没有可用值；`#SPILL!` 是动态数组结果区域被占用等（新版 Excel）。</span>

#### 考点4：函数与公式

<span style="color:#c62828;font-weight:700">🔴【公式规则】：公式以 `=` 开头；函数由函数名、括号和参数组成。文本条件通常需要英文双引号。</span>

<span style="color:#1565c0;font-weight:700">🔵【高频函数】：</span>

`SUM(B2:B5)` 求和；`AVERAGE(B2:B5)` 求算术平均；`MAX/MIN` 求极值；`COUNT` 统计数值单元格个数；`COUNTA` 统计非空单元格个数。

`COUNTIF(B2:B8,">=60")` 单条件计数；`SUMIF` 单条件求和；`COUNTIFS/SUMIFS` 处理多条件。

`IF(B2>=60,"及格","不及格")` 条件判断；新版可使用 `IFS`，但旧版本不支持。

`ROUND(A1,2)` 四舍五入保留两位小数；`LEFT/RIGHT/MID` 提取文本；`LEN` 统计字符数；`TODAY()` 返回当前日期。

⚫️【范围边界】：`A1:A4` 包含 A1、A2、A3、A4；冒号表示连续区域，逗号在多数环境中分隔参数（受区域设置影响，部分环境使用分号）。

#### 考点5：排序、筛选、汇总与透视表

<span style="color:#c62828;font-weight:700">🔴【排序】：可按一个或多个关键字排序。排序连续数据区域时应让相关列一起扩展选择，否则可能破坏记录之间的对应关系。</span>

<span style="color:#c62828;font-weight:700">🔴【筛选】：筛选通常隐藏不符合条件的整行，不删除记录；清除筛选条件可恢复显示。高级筛选可处理较复杂的条件区域并可将结果复制到别处。</span>

<span style="color:#c62828;font-weight:700">🔴【分类汇总】：使用传统“分类汇总”命令前，应先按分类字段排序，使同类记录相邻；“全部删除”可移除分类汇总。是否能撤销取决于后续操作和应用状态，不能写成“删除后绝对不能撤销”。</span>

<span style="color:#c62828;font-weight:700">🔴【数据透视表】：用于对大量明细按字段分组、汇总、交叉分析；源数据变化后通常需要刷新数据透视表，不能与普通图表“自动实时更新”混为一谈。</span>

<span style="color:#c62828;font-weight:700">🔴【数据有效性/数据验证】：可限制允许输入的数据类型或范围、创建下拉列表并设置出错警告；它不能代替密码保护。</span>

#### 考点6：图表、打印与保护

<span style="color:#c62828;font-weight:700">🔴【图表选择】：柱形/条形图适合比较类别；折线图适合展示随时间变化的趋势；饼图适合少量类别构成比例，且通常只表达一个数据系列；散点图适合研究两个数值变量关系。</span>

⚫️【图表联动】：普通图表引用的源数据变化后通常会更新；若新增数据不在原引用区域内，图表未必自动扩展。把数据区域转换为“表格”可提高自动扩展能力。

<span style="color:#c62828;font-weight:700">🔴【冻结窗格】：用于滚动时保持标题行/列可见；不是锁定编辑。保护工作表用于限制对锁定单元格等内容的修改，但不等同于强加密。</span>

<span style="color:#c62828;font-weight:700">🔴【打印】：打印区域、分页、页边距、缩放和重复打印标题行是高频设置；“将工作表调整为一页”可能使内容过小，应区分“1 页宽”和“1 页高”。</span>

### 第三章：PowerPoint 演示文稿软件

#### 考点1：文件、页面与视图

<span style="color:#c62828;font-weight:700">🔴【常见格式】：`.pptx` 为普通演示文稿，`.ppt` 为旧格式，`.ppsx` 打开时通常直接进入放映，`.potx` 为模板；含宏文件分别常用 `.pptm/.ppsm/.potm`。</span>

<span style="color:#c62828;font-weight:700">🔴【演示文稿与幻灯片】：演示文稿是文件，幻灯片是其中的一页。单页可复制到新演示文稿，也可导出为图片或 PDF 页面，因此“幻灯片绝对不能单独保存”不严谨。</span>

<span style="color:#c62828;font-weight:700">🔴【常见视图】：普通视图用于编辑；幻灯片浏览视图适合整体排序；备注页视图编辑讲者备注；阅读视图在窗口中播放。不同版本是否单列“大纲视图”有所差异，较新版本常在普通视图的大纲窗格中使用。</span>

#### 考点2：编辑与放映

<span style="color:#c62828;font-weight:700">🔴【`Ctrl+A` 有对象范围】：焦点在幻灯片缩略图窗格时选中所有幻灯片；焦点在当前页编辑区时选中当前页的全部对象；焦点在文本框内时选中该文本框文字。不能只写“Ctrl+A = 全选幻灯片”。</span>

<span style="color:#c62828;font-weight:700">🔴【放映快捷键】：Windows 桌面版通常 `F5` 从头放映，`Shift+F5` 从当前幻灯片放映，`Esc` 退出。笔记本可能需要配合 `Fn`，网页版/移动版快捷键可能不同。</span>

<span style="color:#c62828;font-weight:700">🔴【复制幻灯片】：可用复制粘贴或“复制幻灯片”；在缩略图/浏览视图中按住 `Ctrl` 拖动通常可复制，单纯拖动通常是移动。</span>

<span style="color:#c62828;font-weight:700">🔴【隐藏幻灯片】：放映时跳过该页，但页面仍保留在文件中；并非删除。</span>

#### 考点3：主题、母版与版式

<span style="color:#c62828;font-weight:700">🔴【主题】：统一定义颜色、字体、效果和背景样式；模板是在主题基础上还可包含版式、占位符和示例内容的文件。</span>

<span style="color:#c62828;font-weight:700">🔴【幻灯片母版】：控制一组版式的共同格式和占位符。母版不是 `.potx` 文件；`.potx` 是演示文稿模板的扩展名。</span>

⚫️【母版修改范围】：修改母版通常影响使用该母版/版式且未被单页覆盖的幻灯片；单页手工格式、被删除的占位符或多母版演示文稿可能不会完全同步。因此“母版一改，所有对象无条件变化”是错误说法。

<span style="color:#c62828;font-weight:700">🔴【多主题/多母版】：一个演示文稿可以包含多个幻灯片母版，从而让不同部分使用不同主题或版式。</span>

#### 考点4：动画、切换与交互

<span style="color:#c62828;font-weight:700">🔴【动画与切换】：动画作用于一张幻灯片内的对象，常见类别为进入、强调、退出、动作路径；切换作用于幻灯片之间。二者对象不同，不能混淆。</span>

<span style="color:#1565c0;font-weight:700">🔵【动画触发方式】：单击时、与上一动画同时、上一动画之后；还可设置持续时间、延迟、重复和触发器。动画窗格用于调整顺序和参数。</span>

<span style="color:#c62828;font-weight:700">🔴【超链接】：超链接可附着在文字、图片、形状等对象上；目标可以是网页、文件、电子邮件地址、本演示文稿中的指定幻灯片，也可在支持的桌面版本中定位到另一演示文稿中的特定幻灯片。原稿“不能链接到某个对象/另一演示文稿某一页”表述错误。</span>

<span style="color:#c62828;font-weight:700">🔴【动作按钮】：可设置跳转、运行程序/宏、播放声音等动作；“鼠标单击”和“鼠标移过”是不同触发方式。</span>

#### 考点5：媒体、打包与演示规范

<span style="color:#c62828;font-weight:700">🔴【音视频】：现代 PowerPoint 可插入音频和视频，并设置自动播放、单击播放、跨幻灯片播放、循环等；可用格式与编解码器、Office 版本和操作系统有关。</span>

<span style="color:#c62828;font-weight:700">🔴【字体与链接资源】：把文件交给另一台电脑时，应检查特殊字体、外部链接的视频/音频和文件路径；较新版本通常会嵌入部分媒体，但外部链接资源仍可能丢失。可使用“打包演示文稿”等功能降低缺失风险。</span>

<span style="color:#c62828;font-weight:700">🔴【演示者视图】：演讲者可查看当前页、下一页、备注和计时，观众只看到放映画面；需要正确配置多显示器。</span>

口诀速记

“鼠标点击看位置，删除操作看对象；分页只换页，分节才分格式；表格 Delete 清内容，删表要用结构命令；Excel 默认表数看版本，填充可以复制公式；井号先查列宽，也查负日期时间；相对 A1 绝对 `$A$1`，分类汇总先排序；PPT 全选先看焦点，F5 从头、Shift+F5 从当前；主题管风格，母版管版式，动画管对象、切换管页面。”

## 信息技术与教学论

### 第一章：课程标准与课程性质

#### 考点1：学段、课程名称与现行依据

<span style="color:#c62828;font-weight:700">🔴【义务教育】：2022 年秋季起执行《义务教育信息科技课程标准（2022年版）》，课程名称为“信息科技”，并从综合实践活动中独立设置。</span>

<span style="color:#c62828;font-weight:700">🔴【普通高中】：现行国家课标通常指《普通高中信息技术课程标准（2017年版2020年修订）》，课程名称为“信息技术”。</span>

⚫️【口径陷阱】：旧教材、地方教材和不同学段可能使用“信息技术课程五大性质”等归纳。不能把“基础性、综合性、实践性、层次性、工具性”写成所有现行国家课标唯一固定表述，更不能断言“工具性是唯一的本质属性”。

#### 考点2：义务教育信息科技课程性质与理念

<span style="color:#c62828;font-weight:700">🔴【课程性质】：义务教育信息科技课程是现代科学技术领域的重要组成部分，主要研究以数字形式表达的信息及其应用中的科学原理、思维方法、处理过程和工程实现；课程具有基础性、实践性和综合性。</span>

<span style="color:#1565c0;font-weight:700">🔵【课程理念得分词】：反映数字时代正确育人方向；构建逻辑关联的课程结构；遴选科学原理和实践应用并重的课程内容；倡导真实性学习；强化素养导向的多元评价。</span>

⚫️【判断陷阱】：信息科技课程不等于软件操作培训。操作技能是载体，还要理解数据、算法、网络、系统等原理，发展计算思维、创新能力和信息社会责任。

#### 考点3：核心素养

<span style="color:#c62828;font-weight:700">🔴【四项核心素养】：信息意识、计算思维、数字化学习与创新、信息社会责任。四者相互支持、相互渗透，共同促进数字素养与技能提升。</span>

<span style="color:#1565c0;font-weight:700">🔵信息意识：对信息的敏感度和信息价值的判断力；能评估来源、可靠性和时效性，主动利用信息解决问题。</span>

<span style="color:#1565c0;font-weight:700">🔵计算思维：运用计算机科学领域的思想方法，通过抽象、分解、建模、算法设计、自动化等活动形成问题解决方案。</span>

<span style="color:#1565c0;font-weight:700">🔵数字化学习与创新：合理选择数字工具和资源，管理学习过程，协同学习并创造性解决问题、形成作品。</span>

<span style="color:#1565c0;font-weight:700">🔵信息社会责任：遵守法律规范与伦理道德，保护隐私和安全，尊重知识产权，负责任地发布和使用信息。</span>

⚫️【判断陷阱】：计算思维不等于“只会编程”；数字化学习不等于“用了电脑上课”；信息意识不等于信息越多越好。

### 第二章：学习理论与教学观

#### 考点4：主要学习理论

<span style="color:#c62828;font-weight:700">🔴【行为主义】：关注刺激—反应、强化、练习和可观察行为；适合基础操作训练，但若只模仿步骤，可能忽视理解与迁移。</span>

<span style="color:#c62828;font-weight:700">🔴【认知主义】：关注信息加工、知识结构、先备知识和认知策略；教学可运用先行组织者、概念图、分块与精加工。</span>

<span style="color:#c62828;font-weight:700">🔴【建构主义】：学习者在一定情境中借助协作和必要帮助，利用学习资源主动建构意义；强调情境、协作、会话和意义建构。</span>

⚫️【教师角色】：强调学生主体不等于教师退出。教师仍需设计环境、提供支架、组织反馈并纠正关键误解。

<span style="color:#c62828;font-weight:700">🔴【最近发展区与支架】：任务应略高于学生独立完成水平，在帮助下可完成；随着能力提升逐步撤去支架。</span>

### 第三章：教学设计

#### 考点5：教学设计基本流程

<span style="color:#1565c0;font-weight:700">🔵【常用答题骨架】：分析学习需要与课标 → 分析学习者 → 分析内容和任务 → 制定素养导向目标 → 选择策略、资源与环境 → 设计活动和评价 → 实施、反馈与修订。</span>

⚫️【系统观】：目标、活动和评价应一致。若目标要求“设计并实现算法”，评价却只考术语记忆，就没有实现教—学—评一致性。

#### 考点6：学习者分析

<span style="color:#c62828;font-weight:700">🔴【分析维度】：起点知识与技能、认知发展、学习动机与兴趣、学习风格和个体差异、数字设备经验、特殊学习需要。</span>

<span style="color:#1565c0;font-weight:700">🔵【起点能力确认】：可用访谈、问卷、前测、作品分析、课堂观察等，不应只凭教师主观猜测。</span>

#### 考点7：教学目标

<span style="color:#c62828;font-weight:700">🔴【现行导向】：目标应指向核心素养，并转化为可观察、可评价的学习表现。传统“三维目标”可用于理解知识技能、过程方法、情感价值等维度，但在现行课标答题中不应取代核心素养导向。</span>

<span style="color:#c62828;font-weight:700">🔴【ABCD 表述】：A 学习者，B 可观察行为，C 条件，D 达成程度。不是每句话都必须机械写齐四项，但行为主体、表现和标准应尽量清楚。</span>

<span style="color:#1565c0;font-weight:700">🔵【行为动词】：识别、描述、比较、解释、设计、调试、评价、创作等；“了解、掌握、培养、提高”若无表现证据，难以直接测量。</span>

⚫️【示例】：在给定的校园数据表中（C），学生（A）能使用合适图表呈现趋势并解释选择理由（B），关键字段和结论完整（D）。

#### 考点8：重点与难点

<span style="color:#c62828;font-weight:700">🔴【教学重点】：由课标要求、内容结构和育人价值决定，是本课必须掌握的核心内容。</span>

<span style="color:#c62828;font-weight:700">🔴【教学难点】：由内容特性和具体学生的认知基础共同决定，是理解或操作障碍所在，具有相对性和动态性。</span>

⚫️【判断陷阱】：重点不一定就是难点；不能只按教师觉得难来确定，也不能把每个环节都列为重点。

### 第四章：教学方法与学习方式

#### 考点9：任务驱动教学

<span style="color:#c62828;font-weight:700">🔴【定义】：把知识与技能融入具有意义的任务，学生在分析、实施和评价任务的过程中学习。</span>

<span style="color:#1565c0;font-weight:700">🔵【常用流程】：创设情境 → 呈现并明确任务 → 分析任务、形成方案 → 自主/协作探究与实施 → 展示交流 → 评价反思与迁移。</span>

<span style="color:#1565c0;font-weight:700">🔵【设计得分词】：真实性、目标一致、难度适切、层次递进、开放选择、成果可评价、安全可实施。</span>

⚫️【“核心”口径】：不同教材对“任务设计是核心”“怎样驱动是关键”等表述可能不同。简答题应写清任务如何承载目标、学生如何主动解决问题、教师如何提供支架，而不是死背一句口号。

⚫️【常见失误】：把任务写成“照着教师演示操作”；情境热闹但与目标无关；只评价作品外观；任务过大却没有子任务和支架。

#### 考点10：项目式学习与问题解决

<span style="color:#c62828;font-weight:700">🔴【项目式学习】：围绕真实、复杂且可探究的问题，持续开展设计、制作、迭代与展示，形成公开成果；项目比单一课堂任务更综合、周期更长。</span>

<span style="color:#1565c0;font-weight:700">🔵【流程骨架】：提出驱动性问题 → 规划与分工 → 调研/设计 → 制作与测试 → 迭代改进 → 展示答辩 → 反思评价。</span>

<span style="color:#c62828;font-weight:700">🔴【探究式学习】：学生经历提出问题、作出假设、收集与分析证据、形成解释、交流评价等过程；教师需控制探究开放度并提供必要支架。</span>

#### 考点11：讲授、演示、练习与讨论

<span style="color:#c62828;font-weight:700">🔴【讲授法】：适合系统解释概念、原理和背景，效率高；应结合提问、案例、可视化和练习，避免长时间单向灌输。</span>

<span style="color:#c62828;font-weight:700">🔴【演示—练习】：教师示范关键操作并口述思路，学生模仿、独立练习和迁移。演示前应确认屏幕可见，步骤应分段，并预留学生操作时间。</span>

<span style="color:#c62828;font-weight:700">🔴【讨论法】：适合有多种观点、需要比较论证的问题。低年级或新知识阶段并非“绝对不能用”，但问题难度、语言要求、先备知识和组织方式要适合学生。</span>

#### 考点12：合作学习

<span style="color:#c62828;font-weight:700">🔴【基本特征】：共同目标、积极相互依赖、个体责任、面对面促进性互动、合作技能与小组反思。</span>

<span style="color:#1565c0;font-weight:700">🔵【组织要点】：异质分组、角色明确、成果与个人责任并重、过程巡视、及时反馈。4～6 人是常见建议，不是所有任务的硬性人数标准。</span>

⚫️【防止“搭便车”】：设置个人可追踪任务、随机汇报、同伴互评与个人反思；仅把学生围坐在一起不等于合作学习。

#### 考点13：WebQuest、微课与翻转课堂

<span style="color:#c62828;font-weight:700">🔴【WebQuest】：以网络资源为主要信息来源的探究活动。经典结构常包括导言、任务、过程、资源、评价和结论；不同模板可能把资源并入过程或增加教师页，不能脱离教材断言只有一种绝对结构。</span>

<span style="color:#c62828;font-weight:700">🔴【微课】：围绕相对聚焦的学习目标，以短小数字化学习资源为核心并配套学习活动。视频是常见载体，但“微课=一个短视频”不完整。</span>

⚫️【时长口径】：5～10 分钟是常见经验值，不是所有学段、内容和国家标准的法定上限；应以目标聚焦、认知负荷适当和学习效果为准。

<span style="color:#c62828;font-weight:700">🔴【翻转课堂】：把部分直接讲授或知识获取前移到课前，课堂时间更多用于问题解决、协作和个别化指导。它不是简单地“课前看视频、课上做作业”，也不意味着教师作用减弱。</span>

### 第五章：教学评价

#### 考点14：按评价时机和参照分类

<span style="color:#c62828;font-weight:700">🔴【诊断性评价】：教学前或新单元开始时了解起点和困难，如前测、访谈、先备作品分析。</span>

<span style="color:#c62828;font-weight:700">🔴【形成性评价】：教学过程中收集证据、反馈并改进学习，如观察、提问、调试记录、阶段作品和学习单。</span>

<span style="color:#c62828;font-weight:700">🔴【总结性评价】：阶段或课程结束后判断目标达成，如期末考试、综合作品与答辩。</span>

<span style="color:#c62828;font-weight:700">🔴【目标/标准参照评价】：对照预定标准判断达成程度；【常模参照评价】：与群体表现比较；【个体内差异评价】：与学习者自己过去的表现比较。</span>

⚫️【判断陷阱】：一次测验属于哪类，不只看形式，还看使用目的。相同测验用于课前诊断、课中反馈或期末认定，性质可能不同。

#### 考点15：多元评价与评价工具

<span style="color:#c62828;font-weight:700">🔴【评价原则】：素养导向、目标一致、过程与结果结合、定性与定量结合、主体多元、关注差异、以评促学。</span>

<span style="color:#1565c0;font-weight:700">🔵【信息科技常用证据】：纸笔测验、上机操作、项目作品、程序运行结果、设计文档、过程日志、答辩、档案袋。不能断言“纸笔+上机”是所有情境唯一最合适的总结性评价。</span>

<span style="color:#c62828;font-weight:700">🔴【量规 Rubric】：把评价维度和不同水平的表现描述清楚。优秀量规应对应目标、维度可区分、描述可观察，避免只用“好/一般/差”。</span>

<span style="color:#c62828;font-weight:700">🔴【作品评价】：既看功能正确性、算法/技术合理性，也看创意、规范、安全责任、协作过程和表达；评价维度应与任务目标匹配。</span>

#### 考点16：测验质量指标

<span style="color:#c62828;font-weight:700">🔴【信度】：测量结果的一致性和稳定性；【效度】：是否真正测到了预期目标。信度是效度的必要但非充分条件。</span>

<span style="color:#c62828;font-weight:700">🔴【难度】：题目通过率越高，通常表示题目越容易（采用不同难度指数定义时要看公式）；【区分度】：区分不同水平学习者的能力。</span>

<span style="color:#1565c0;font-weight:700">🔵【客观题命制】：题干清楚、只有一个最佳答案、干扰项有迷惑性且同质、避免无意线索和双重否定；判断题避免使用“永远、全部、绝不”等无条件绝对词。</span>

### 第六章：课堂实施与案例分析

#### 考点17：信息科技课堂组织

<span style="color:#1565c0;font-weight:700">🔵【上机课流程】：设备与账号检查 → 明确目标和安全规则 → 分段示范/任务发布 → 学生实践、教师巡视 → 保存与提交 → 展示评价 → 退出账号和整理设备。</span>

<span style="color:#c62828;font-weight:700">🔴【故障应对】：先保证人身和数据安全，再定位电源、连接、账号、网络、软件、文件路径等；准备离线资源、备用设备或同伴协作方案。不能把技术故障时间全部变成无目标自由活动。</span>

<span style="color:#c62828;font-weight:700">🔴【差异化教学】：提供基础任务、进阶任务和挑战任务；给予步骤卡、半成品、示例数据等支架，同时为高水平学生提供开放拓展。</span>

#### 考点18：案例分析答题框架

<span style="color:#1565c0;font-weight:700">🔵【先判断】：指出做法是否符合素养导向、学生主体、目标一致和安全伦理。</span>

<span style="color:#1565c0;font-weight:700">🔵【再依据】：结合课标、学习理论、教学方法或评价原则解释原因。</span>

<span style="color:#1565c0;font-weight:700">🔵【后改进】：从目标、任务、支架、互动、资源、评价和课堂管理提出可执行措施。</span>

<span style="color:#1565c0;font-weight:700">🔵【补证据】：说明如何观察改进是否有效，如作品质量、调试过程、学生解释、量规得分或反思记录。</span>

⚫️【高频问题】：教师包办操作、任务脱离真实问题、只重结果不看过程、评价主体单一、忽视版权隐私、目标不可测、合作学习无个人责任。

口诀速记

“义教信息科技二二课标，高中信息技术一七二〇修订；核心素养信计数社；目标、活动、评价要一致；任务真实有梯度，项目规划做测改；讨论不是任何阶段都禁用，合作人数不是硬规定；诊断看起点，形成促改进，总结判达成；课前先分析，课中给支架，课后用证据评价。”

## 多媒体技术

### 第一章：多媒体基础

#### 考点1：概念与主要特征

<span style="color:#c62828;font-weight:700">🔴【多媒体技术】：以计算机为核心，对文本、图形、图像、声音、动画、视频等多种媒体信息进行数字化获取、处理、存储、集成、控制和交互呈现的技术。</span>

<span style="color:#c62828;font-weight:700">🔴【高频特征】：集成性、交互性、数字化是多数教材的核心表述；有的教材还强调实时性。题目问“三大特征”或“本质特征”时必须依据所用教材口径和选项判断，不能同时断言“交互性”和“数字化”都是唯一的本质特征。</span>

<span style="color:#1565c0;font-weight:700">🔵集成性：对多种媒体统一组织与协同处理。</span>

<span style="color:#1565c0;font-weight:700">🔵交互性：用户能主动选择、控制信息呈现并得到反馈。</span>

<span style="color:#1565c0;font-weight:700">🔵数字化：各种媒体在计算机中以数字形式表示和处理。</span>

<span style="color:#1565c0;font-weight:700">🔵实时性：音视频播放或交互系统需在规定时间内连续响应。</span>

⚫️【超文本与超媒体】：超文本以链接组织文本节点；节点扩展到图像、声音、视频等多种媒体时称为超媒体。超链接是实现非线性组织的一种机制。

#### 考点2：媒体分类

<span style="color:#c62828;font-weight:700">🔴【感觉媒体】：直接作用于感官的信息形态，如声音、图像、文字。</span>

<span style="color:#c62828;font-weight:700">🔴【表示媒体】：为交换和处理而规定的编码形式，如 ASCII、JPEG 编码。</span>

<span style="color:#c62828;font-weight:700">🔴【表现媒体】：输入或输出信息的物理设备，如键盘、摄像头、显示器、扬声器。</span>

<span style="color:#c62828;font-weight:700">🔴【存储媒体】：保存信息的介质，如磁盘、光盘、闪存。</span>

<span style="color:#c62828;font-weight:700">🔴【传输媒体】：承载信号传输的介质，如双绞线、光纤、无线电波。</span>

### 第二章：图形与图像

#### 考点3：位图与矢量图

<span style="color:#c62828;font-weight:700">🔴【位图】：以像素为基本单元；适合照片和色彩层次丰富的图像；按像素放大通常出现锯齿或模糊。常见格式有 BMP、JPEG、PNG、GIF、TIFF，PSD 是 Photoshop 常用源文件。</span>

<span style="color:#c62828;font-weight:700">🔴【矢量图】：以点、线、曲线、形状及其属性描述；适合标志、图标、示意图；在重新计算的条件下缩放通常不失真。常见格式有 SVG、AI、EPS、WMF。</span>

⚫️【判断条件】：矢量文件不一定总比位图小，复杂路径也会很大；SWF 是可包含矢量图、位图、音频和脚本的容器/发布格式，不能简单等同于“纯矢量图格式”；Office 剪贴画的实际类型取决于资源文件，不能一概认定都是 WMF。

#### 考点4：常见图像格式

<span style="color:#c62828;font-weight:700">🔴【BMP】：常见位图格式；可采用不压缩或无损压缩，文件通常较大，但不能一概写成“BMP 永远无压缩”。</span>

<span style="color:#c62828;font-weight:700">🔴【JPEG/JPG】：常用于照片，通常采用有损压缩；不支持常规透明通道，不适合反复编辑保存文字线稿。</span>

<span style="color:#c62828;font-weight:700">🔴【GIF】：索引色，单帧最多 256 色；支持动画和 1 位透明（透明/不透明），不支持 PNG 那样的完整半透明 alpha 通道；LZW 压缩为无损压缩。</span>

<span style="color:#c62828;font-weight:700">🔴【PNG】：无损压缩，支持透明和 alpha 半透明，适合网页图标、截图和线条图；标准 PNG 本身不等同于 GIF 动画。</span>

<span style="color:#c62828;font-weight:700">🔴【PSD】：可保留图层、蒙版、通道等编辑信息；导出为 JPEG 等格式通常会失去可编辑图层结构。</span>

⚫️【压缩陷阱】：格式转换后的大小和质量还取决于尺寸、颜色、压缩参数及原始内容，不能只凭扩展名断言一定缩小多少。“JPEG2000 比 JPEG 高 30%”不是通用固定值，应删除。

#### 考点5：分辨率、颜色与容量

<span style="color:#c62828;font-weight:700">🔴【像素尺寸】：如 1920×1080，表示横向和纵向像素数；打印分辨率常用 PPI/DPI 表述，必须结合物理尺寸理解。仅写“分辨率”时要看题目是在问像素尺寸、显示分辨率还是打印精度。</span>

<span style="color:#c62828;font-weight:700">🔴【颜色深度】：每像素使用的二进制位数。若不考虑调色板和保留色，n 位最多表示 `2^n` 种状态。24 位 RGB 通常为 R、G、B 各 8 位，可表示约 1677 万种颜色。</span>

<span style="color:#c62828;font-weight:700">🔴【RGB】：加色模型；常见 8 位通道范围为 0～255。`(0,0,0)` 为黑，`(255,255,255)` 为白。印刷常用 CMYK 减色模型，不能和 RGB 通道直接混淆。</span>

<span style="color:#1565c0;font-weight:700">🔵【未压缩位图容量】：`字节数 = 宽度像素 × 高度像素 × 每像素位数 ÷ 8`。</span>

⚫️【公式适用范围】：该式估算未压缩像素数据，不含文件头、调色板、元数据、行填充和压缩影响。题目说“RGB 每通道 8 bit”时，每像素位数是 `3×8=24 bit`，不能再额外重复乘色彩数。

#### 考点6：OCR 与图像采集

<span style="color:#c62828;font-weight:700">🔴【OCR】：把图像中的印刷或手写文字识别为可检索、可编辑字符。典型流程包括图像采集、预处理、版面分析/字符识别、校对与导出。</span>

⚫️【条件辨析】：倾斜校正、去噪并非每张图片都必做，但可提高复杂扫描件识别率；支持哪些输入格式由 OCR 软件决定，不能用“PSD 一定不能识别”作跨软件结论；识别后的文件也不必然比原图小，若保留原图、版式和字体，文件可能更大。

### 第三章：音频技术

#### 考点7：音频数字化

<span style="color:#c62828;font-weight:700">🔴【基本过程】：采样 → 量化 → 编码。</span>

<span style="color:#1565c0;font-weight:700">🔵采样频率：每秒采样次数；在理想带限条件下，为无失真重建，采样频率应至少大于最高频率的 2 倍（奈奎斯特采样定理）。</span>

<span style="color:#1565c0;font-weight:700">🔵量化位数：每个样本的二进制位数，位数越高通常量化级数越多、动态范围越大，数据量也越大。</span>

<span style="color:#1565c0;font-weight:700">🔵声道数：单声道为 1，立体声通常为 2。</span>

#### 考点8：音频格式与容量

<span style="color:#c62828;font-weight:700">🔴【WAV】：容器格式，可保存未压缩 PCM，也可装载压缩音频；不能简单写成“WAV 一定无损且文件大”。</span>

<span style="color:#c62828;font-weight:700">🔴【MP3/AAC/WMA】：常见有损编码格式，目标是以感知冗余换取较小数据量；具体音质取决于码率、编码器和源文件。</span>

<span style="color:#c62828;font-weight:700">🔴【FLAC/ALAC】：无损压缩，解码后可还原原 PCM 数据。</span>

<span style="color:#c62828;font-weight:700">🔴【MIDI】：记录音符、时值、力度、音色控制等演奏事件，不记录麦克风采集的人声波形。它可驱动音源播放人声采样音色，但 MIDI 文件本身并不等于人声音频录音。</span>

<span style="color:#1565c0;font-weight:700">🔵【未压缩 PCM 容量】：`字节数 = 采样频率 × 量化位数 × 声道数 × 时间(秒) ÷ 8`。</span>

⚫️【CD 常见口径】：音频 CD 的 PCM 参数通常为 44.1 kHz、16 bit、双声道。5 分钟未压缩数据约为 `44100×16×2×300÷8 ≈ 52.92 MB（十进制）`，若按 `1024²` 换算约 50.47 MiB。选择题要看 MB 的换算口径。

⚫️【压缩音频容量】：已知码率时用 `文件大小 ≈ 码率 × 时间 ÷ 8`；MP3 等压缩文件不能再直接套用 PCM 采样公式。

### 第四章：视频与动画

#### 考点9：视频数字化与容量

<span style="color:#c62828;font-weight:700">🔴【视频数字化】：对连续画面进行时间采样和空间采样，再量化、编码；实际系统通常还要进行压缩、封装、存储或传输。把流程写成“采样→A/D→压缩→存储→解压→D/A”混合了采集端与播放端，不是规范的单向数字化步骤。</span>

<span style="color:#1565c0;font-weight:700">🔵【未压缩视频容量】：`字节数 = 宽×高×每像素位数×每秒帧数×时间÷8`。</span>

<span style="color:#1565c0;font-weight:700">🔵【压缩视频容量】：`文件大小 ≈ 总码率（视频码率+音频码率）×时间÷8`。</span>

⚫️【制式口径】：传统 PAL 常按 25 帧/秒考查；传统 NTSC 实际常为约 29.97 帧/秒，基础教材可能近似写 30 帧/秒。数字视频也可能使用 24、30、50、60 等其他帧率。

#### 考点10：容器、编码与格式

<span style="color:#c62828;font-weight:700">🔴【容器不等于编码】：MP4、AVI、MOV、MKV、WebM 等多为封装容器；H.264/AVC、H.265/HEVC、AV1、VP9 等是视频编码。扩展名相同的文件可能采用不同编码，能否播放取决于播放器是否支持相应解码器。</span>

<span style="color:#c62828;font-weight:700">🔴【流媒体】：强调边下载边播放和连续传输；FLV 曾广泛用于网页视频，但已不是现代网页视频的唯一或“必然主流”格式。现代平台常使用 MP4/WebM 及 HLS、DASH 等传输方案。</span>

⚫️【动画与视频】：GIF、APNG、SWF 可承载动画，但不能因此统称为普通视频格式；`.exe` 是可执行文件，不是视频容器。

### 第五章：Photoshop 与图像处理

#### 考点11：图层、选区与蒙版

<span style="color:#c62828;font-weight:700">🔴【图层】：分层组织内容便于独立编辑；PSD/PSB 常用于保留图层结构。背景图层在传统 Photoshop 中默认锁定且不含透明区域，但可解锁/转换为普通图层后移动、调透明度或应用更多效果，不能写成“背景图层永远不能改变透明度”。</span>

<span style="color:#c62828;font-weight:700">🔴【选区工具】：魔棒按颜色相似度选区；容差越大通常选取的颜色范围越宽。套索适合自由选区，多边形套索适合直线边界，磁性套索依据边缘对比自动吸附。</span>

<span style="color:#c62828;font-weight:700">🔴【仿制图章】：Windows 下通常按 `Alt` 单击设取样点，再涂抹复制像素；是否能在当前图层、所有图层或调整图层上取样取决于“样本”设置和目标图层类型。</span>

<span style="color:#c62828;font-weight:700">🔴【减淡/加深/海绵】：减淡使局部变亮，加深使局部变暗；海绵工具可按模式提高或降低饱和度，原稿“只能减少饱和度”错误。</span>

<span style="color:#c62828;font-weight:700">🔴【Alpha 通道】：选择范围保存为通道时，默认白色通常代表选中、黑色代表未选、灰色代表部分选中；若反转通道或改变显示选项，视觉表现可不同。</span>

<span style="color:#c62828;font-weight:700">🔴【图层蒙版】：默认白显、黑隐、灰色部分显示；它是非破坏性控制可见性的常用方法，不直接删除原像素。</span>

⚫️【版本条件】：现代 Photoshop 具有时间轴，可处理视频帧和帧动画，也可导出 GIF；“Photoshop 绝对不能播放动画/视频”只适用于某些旧版教材环境。

#### 考点12：常用快捷键

<span style="color:#1565c0;font-weight:700">🔵Windows 桌面版常用：`Ctrl+D` 取消选择，`Ctrl+Shift+I` 反选，`Ctrl+U` 色相/饱和度，`D` 恢复默认前景/背景色，`X` 交换前景/背景色，`Alt+Delete` 填充前景色，`Ctrl+Delete` 填充背景色。</span>

⚫️【快捷键条件】：macOS 使用 `Command/Option` 等按键；快捷键可能被系统、工作区或用户自定义修改。题干指定版本/平台时以题干为准。

### 第六章：Flash/Animate 动画基础

#### 考点13：帧、元件和补间

<span style="color:#c62828;font-weight:700">🔴【历史背景】：Adobe Flash Player 已停止支持，但教师招聘旧教材仍会考查 Flash 创作概念；现行 Adobe Animate 的界面和术语可能与 Flash 旧版不同。</span>

<span style="color:#c62828;font-weight:700">🔴【帧操作】：传统 Flash 中 `F5` 插入普通帧，`F6` 插入关键帧，`F7` 插入空白关键帧。普通帧延续前一关键帧状态；关键帧定义内容变化节点。</span>

<span style="color:#c62828;font-weight:700">🔴【元件类型】：图形、影片剪辑、按钮。影片剪辑有独立时间轴；图形元件时间轴通常受主时间轴控制；按钮经典四帧为“弹起、指针经过、按下、点击”，其中“点击”帧定义感应区域，不是第四个可见播放状态。</span>

<span style="color:#c62828;font-weight:700">🔴【逐帧动画】：关键画面逐帧制作，控制细但数据量和工作量通常较大。</span>

<span style="color:#c62828;font-weight:700">🔴【传统补间】：传统动作补间通常作用于元件实例、组或文本等可补间对象；形状补间作用于未组合的矢量形状。元件实例若要做形状补间，需要先分离为可编辑形状。</span>

<span style="color:#c62828;font-weight:700">🔴【引导层与遮罩层】：传统引导层规定运动路径，发布时引导线通常不显示；遮罩层形状控制被遮罩层的可见区域。</span>

⚫️【帧频条件】：旧版 Flash 新文档常见默认 12 fps，Animate 新版默认值可能不同；允许范围也随版本变化。题目未指定软件版本时，不应把“默认 12、最大 120”作为永恒结论。

#### 考点14：源文件与发布文件

<span style="color:#c62828;font-weight:700">🔴【FLA】：可编辑源文件；【SWF】：传统发布文件，可包含矢量、位图、音频、动画和脚本。GIF 是动画图像，AVI/MOV/MP4 是视频容器；把 GIF、SWF 全列成“视频格式”不严谨。</span>

口诀速记

“多媒体口径看教材，集成、交互、数字化；位图像素，矢量路径，复杂矢量未必小；JPEG 有损，GIF 256 色，PNG 无损带半透；音频采样量化编码，PCM 算采样参数，压缩音频算码率；视频容器不等于编码，压缩容量看总码率；PS 背景可转普通层，海绵既可增也可减；Flash 题先看版本，动作补间与形状补间对象不同。”

## 操作系统原理

### 第一章：操作系统概述

#### 考点1：操作系统的定义与地位

<span style="color:#c62828;font-weight:700">🔴【定义】：操作系统（OS）是管理计算机软硬件资源、控制程序执行并为用户和应用提供服务的系统软件。传统教材常说它运行在裸机之上；现代系统还可能位于固件、虚拟机监控器之上。</span>
<span style="color:#c62828;font-weight:700">🔴【地位】：</span>
操作系统是用户与计算机硬件之间的接口。
操作系统是对硬件的第一次扩充。
操作系统是其他所有软件运行的基础平台。
⚫️【坑点】：操作系统是核心系统软件，不是一般应用软件。操作系统的功能不包括“高级程序设计语言的编译器”（编译器属于语言处理程序）。
#### 考点2：操作系统的功能（五大管理）

<span style="color:#c62828;font-weight:700">🔴【五大功能】：</span>
处理器（处理机）管理：CPU的分配和调度。
存储器管理：内存的分配、回收和保护。
设备管理：I/O设备的分配和管理。
文件管理（信息管理）：文件的存储、检索、共享和保护。
作业/用户接口管理：组织和控制作业，并通过命令接口、程序接口（系统调用）等提供服务。不同教材可能概括为四大管理、五大管理或另列用户接口。
⚫️【坑点】：存储器管理主要负责内存的分配与回收，不是外存（外存属于设备管理或文件管理）。
#### 考点3：操作系统的分类（必考判断）

<span style="color:#c62828;font-weight:700">🔴【批处理操作系统】：</span>
单道批处理：内存中只有一道程序。特点：单道性、顺序性、自动性。
多道批处理：内存中同时存放多道程序，共享CPU。特点：资源利用率高、系统吞吐量大，但无交互能力（用户无法干预）。设计时首先考虑资源利用率和吞吐量。
<span style="color:#c62828;font-weight:700">🔴【分时操作系统】：</span>
将CPU时间划分为时间片，轮流分配给各终端用户。
四大特点：多路性、独立性、及时性、交互性。（口诀：多独及时交）
坑点：“自治性”不是分时操作系统的特点！
单核计算机实现多任务“同时”执行假象的机制是时间片轮转。
<span style="color:#c62828;font-weight:700">🔴【实时操作系统】：</span>
对外部事件在规定时间内做出响应。
特点：正确性同时取决于逻辑结果与规定时限，强调可预测性和可靠性。实时系统可以具有很强的人机交互，“交互性弱”不是定义性特征。
应用：飞机控制、工业控制等硬实时场景，以及部分通信、媒体等软实时场景。订票系统更常归为联机事务处理，是否算实时系统取决于教材语境。
输出结果的正确性不仅依赖于结果的正确性，还依赖于产生结果的时间（实时性要求）。
⚫️【坑点】：
分时操作系统的时间片不是越小越好（过小会导致频繁切换，系统开销增大）。
批处理系统没有交互能力。
实时系统与分时系统的区别：实时系统交互性弱，响应时间敏感度更高。
#### 考点4：操作系统的特征

<span style="color:#c62828;font-weight:700">🔴【四大特征】：并发、共享、虚拟、异步。（口诀：并共虚异）</span>
<span style="color:#c62828;font-weight:700">🔴【并发】：两个或多个事件在同一时间间隔内发生（宏观同时，微观交替）。并发性是指若干事件在同一时间间隔发生。</span>
<span style="color:#c62828;font-weight:700">🔴【共享】：系统资源供多个并发执行的进程共同使用。</span>
<span style="color:#c62828;font-weight:700">🔴【虚拟】：把一个物理实体变成若干个逻辑上的对应物（如虚拟内存）。</span>
<span style="color:#c62828;font-weight:700">🔴【异步】：进程以不可预知的速度向前推进（走走停停）。</span>
⚫️【坑点】：
“并发”与“并行”的区别：并发是同一时间间隔（单核），并行是同一时刻（多核）。
多个进程的实体存在于同一内存中，在一段时间内都得到运行 → 这种性质称作并发性。
### 第二章：进程管理

#### 考点5：进程与程序的区别（绝对高频）

<span style="color:#c62828;font-weight:700">🔴【程序】：静态的，是指令的有序集合，存储在外存中。</span>
<span style="color:#c62828;font-weight:700">🔴【进程】：动态的，是程序的一次执行活动；其地址空间可部分驻留内存、部分位于外存，不能简单定义为“进程存储在内存”。</span>
<span style="color:#1565c0;font-weight:700">🔵【本质区别】：</span>
进程是动态的，程序是静态的。
进程是资源分配的基本单位（有生命周期）。
进程 =程序 + PCB + 数据集合。
⚫️【坑点】：
进程是动态的，程序是静态的。这是它们的本质区别。
可执行程序通常以文件形式保存在外存，执行时创建进程；进程的运行状态由PCB等维护，其地址空间不要求全部常驻内存。
不同进程所执行的代码可以相同（如多个用户同时运行同一个程序）。
#### 考点6：进程控制块（PCB）

<span style="color:#c62828;font-weight:700">🔴【PCB定义】：进程控制块，是进程存在的唯一标志。</span>
<span style="color:#c62828;font-weight:700">🔴【PCB内容】：进程标识符、进程状态、程序计数器、寄存器、内存信息、资源清单等。</span>
<span style="color:#c62828;font-weight:700">🔴【PCB作用】：系统根据PCB感知进程的存在，通过PCB控制和管理进程。</span>
⚫️【坑点】：创建/撤销进程的核心操作包括建立/撤销PCB。PCB由内核维护，用户进程通常不能直接读写；应用可通过系统调用查询操作系统允许公开的部分进程信息。
#### 考点7：进程的三种基本状态及其转换（必考）

<span style="color:#c62828;font-weight:700">🔴【三种状态】：</span>
就绪态（Ready）：除CPU外，所有资源都已获得，等待CPU调度。
运行态（Running）：已获得CPU，正在执行。
阻塞态（Blocked/Waiting）：因等待某事件（如I/O完成）而暂停执行。
<span style="color:#1565c0;font-weight:700">🔵【状态转换条件】：</span>
⚫️【坑点】：
时间片用完 → 进程从运行态变为就绪态。
等待某一事件 → 进程从运行态变为阻塞态。
进程申请CPU得不到满足 → 处于就绪态，不是阻塞态。
单处理器系统中，同时存在12个进程，处于就绪队列的进程数量最多为11个（1个在运行）。
#### 考点7A：进程与线程（高频辨析）

<span style="color:#c62828;font-weight:700">🔴【进程】：传统教材中是资源分配和保护的基本单位，拥有独立地址空间及资源。</span>
<span style="color:#c62828;font-weight:700">🔴【线程】：是处理器调度的基本单位；同一进程内线程共享代码、数据和打开文件等进程资源，但拥有各自的程序计数器、寄存器和栈。</span>
⚫️【坑点】：线程既有共享的进程资源，也有私有运行现场；“线程之间完全不共享资源”和“线程没有私有状态”都错误。
#### 考点8：进程调度算法

<span style="color:#c62828;font-weight:700">🔴【先来先服务（FCFS）】：按就绪/到达先后服务，通常为非抢占，实现简单；长作业在前会使短作业久等，产生护航效应，但平均周转时间并非在任何任务集合中都必然最长。</span>
<span style="color:#c62828;font-weight:700">🔴【短作业优先（SJF）】：选择运行时间最短的进程先执行。平均周转时间最短（所有作业同时到达时）。需要预知运行时间。</span>
<span style="color:#c62828;font-weight:700">🔴【高响应比优先（HRRN）】：响应比 = (等待时间 + 要求服务时间) / 要求服务时间。兼顾等待时间和执行时间，既照顾短作业，又不使长作业长期等待。</span>
<span style="color:#c62828;font-weight:700">🔴【时间片轮转（RR）】：分时系统使用。每个进程轮流获得固定时间片的CPU。</span>
<span style="color:#c62828;font-weight:700">🔴【优先级调度】：选择优先级最高的进程执行。若使当前运行进程总是优先级最高的进程，应选择高优先级优先调度算法。</span>
⚫️【坑点】：
所有作业同时到达时，平均周转时间最短的是短作业优先。
响应比高优先算法中：等待时间相同时，运行时间短的优先；运行时间相同时，等待时间长的优先。
某些调度策略会提高I/O密集型或交互型任务优先级，以改善响应并让I/O与CPU并行；优先级仍取决于系统目标，不能把“计算型必低于I/O型”当作通则。
#### 考点9：进程同步与互斥

<span style="color:#c62828;font-weight:700">🔴【同步】：多个进程因合作而产生的直接制约关系（如流水线上下工序）。</span>
<span style="color:#c62828;font-weight:700">🔴【互斥】：多个进程因竞争同一资源而产生的间接制约关系（如多个进程争抢打印机）。</span>
<span style="color:#c62828;font-weight:700">🔴【临界资源】：一次只允许一个进程访问的资源（如打印机）。</span>
<span style="color:#c62828;font-weight:700">🔴【临界区】：访问临界资源的那段程序代码。</span>
<span style="color:#c62828;font-weight:700">🔴【信号量】：用于进程同步与互斥的机制。P操作（申请资源）和V操作（释放资源）是原子操作。</span>
在部分教材的计数信号量定义中，值大于0表示可用资源数，值小于0时绝对值表示等待进程数；有些实现不允许用户直接观察负值，因此该解释依赖具体P/V定义。
⚫️【坑点】：
临界区是一段程序，不是缓冲区或数据区！
P、V操作既能实现进程互斥，也能实现进程同步。
按“P操作先减1、负值入队”的教材定义，有3个进程且最多允许2个进入时，信号量可出现2、1、0、-1；题目若采用不同信号量语义，不能机械套用。
信号量初值为3，当前为1 → 可用资源数M=1，等待进程数N=0。
#### 考点10：死锁（极高频）

<span style="color:#c62828;font-weight:700">🔴【定义】：两个或多个进程因互相等待对方占有的资源而无限期阻塞的现象。</span>
<span style="color:#c62828;font-weight:700">🔴【死锁产生的四个必要条件】（全部满足才可能死锁）：</span>
互斥：资源一次只能分配给一个进程。
请求与保持：进程占有一些资源的同时，请求新的资源。
不可剥夺：进程已获得的资源在使用完毕前不能被强行剥夺。
环路等待：若干进程形成头尾相接的循环等待资源链。
<span style="color:#c62828;font-weight:700">🔴【死锁的预防】：在资源分配规则上破坏四个必要条件之一。</span>
破坏“请求与保持”：一次性申请所有资源。
破坏“环路等待”：资源有序分配（给资源类型编号，按规定顺序申请）。它是常见预防方法之一，但可能降低资源利用率或增加编程约束。
<span style="color:#c62828;font-weight:700">🔴【死锁的避免】：银行家算法。系统分配资源前先计算是否会导致不安全状态。</span>
<span style="color:#c62828;font-weight:700">🔴【死锁与安全状态的关系】：</span>
死锁状态一定是不安全状态。
不安全状态不一定会死锁（但可能导致死锁）。
安全状态一定不会死锁。
⚫️【坑点】：
死锁的预防方法中，不太可能的一种是摒弃互斥条件（很多资源本身就是互斥的，无法摒弃）。
可共享资源通常能减少互斥竞争，但硬盘等设备还涉及控制块、缓冲区和锁，不能绝对说“不会引起死锁”。
死锁发生后可通过终止进程、资源剥夺、回滚等恢复；“不能被中断或唤醒”不是死锁定义，不可剥夺条件只表示资源在正常规则下不能被强制收回。
系统中出现环路不一定死锁（还需要其他条件）。
### 第三章：存储管理

#### 考点11：存储管理的目的

<span style="color:#c62828;font-weight:700">🔴【主要功能】：内存的分配、回收、地址转换、存储保护和内存扩充（虚拟内存）。</span>
<span style="color:#c62828;font-weight:700">🔴【地址转换（重定位）】：</span>
静态重定位：程序装入时一次性完成地址转换。
动态重定位：程序执行过程中借助地址转换硬件完成逻辑地址到物理地址的转换。现代通用操作系统通常采用动态地址转换，但不能写成“所有现代操作系统只采用这一方式”。
⚫️【坑点】：静态重定位的时机是程序装入时，动态重定位的时机是程序运行时。
#### 考点12：连续分配方式

<span style="color:#c62828;font-weight:700">🔴【单一连续分配】：内存分为系统区和用户区，只能用于单用户、单任务操作系统。最简单。</span>
<span style="color:#c62828;font-weight:700">🔴【固定分区分配】：内存划分为多个固定大小的分区，每个分区可装入一道程序。分区大小可以不同，但预先固定。有内部碎片。</span>
<span style="color:#c62828;font-weight:700">🔴【动态分区分配】：根据进程需要动态分配内存空间。有外部碎片。可通过“紧凑”解决。</span>
<span style="color:#c62828;font-weight:700">🔴【首次适应算法】：按地址递增的顺序分配空闲分区。</span>
⚫️【坑点】：固定分区中，分区大小可以不同但预先固定。首次适应算法要求空闲区按地址递增登记。
#### 考点13：分页存储管理（离散分配）

<span style="color:#c62828;font-weight:700">🔴【核心思想】：将进程的逻辑地址空间分成页，将内存的物理地址空间分成块。页和块的大小相等。</span>
<span style="color:#c62828;font-weight:700">🔴【页表】：记录页号到物理块号的映射关系。每个进程拥有一张页表。</span>
<span style="color:#c62828;font-weight:700">🔴【地址结构】：逻辑地址 =页号 + 页内地址。物理地址 =块号 + 块内地址（块内地址 = 页内地址）。</span>
<span style="color:#c62828;font-weight:700">🔴【访问内存次数】：未引入快表时，每次存取指令或数据需2次访问内存（第一次查页表，第二次存取数据）。</span>
⚫️【坑点】：
块内位移量等于页内位移量，是因为页和块的大小相等。
每个进程拥有自己的页表，不是所有进程共享一张页表。
8个页面，每页1024字节 → 逻辑地址需要3（页号）+ 10（页内地址）= 13位二进制表示。
#### 考点14：分段存储管理

<span style="color:#c62828;font-weight:700">🔴【核心思想】：将进程的地址空间按逻辑信息（代码段、数据段等）分成若干段。每段从0开始编址，大小不固定。</span>
<span style="color:#c62828;font-weight:700">🔴【地址结构】：逻辑地址 =段号 + 段内地址。</span>
<span style="color:#c62828;font-weight:700">🔴【与分页的区别】：</span>
页是进程逻辑地址空间的固定大小划分，页框/块是物理内存的固定大小划分；分页主要出于系统管理需要，对用户通常不可见。段按程序逻辑信息划分，对用户可见。
页大小固定，段大小不固定。
分页是一维地址空间，分段是二维地址空间（段名+段内地址）。
#### 考点15：段页式存储管理

<span style="color:#c62828;font-weight:700">🔴【核心思想】：分段 + 分页的结合。用分段方法管理用户地址空间，用分页方法管理物理存储空间。</span>
⚫️【坑点】：段页式管理的实现原理是：用分段方法来分配和管理用户地址空间，用分页方法来管理物理存储空间。
#### 考点16：虚拟存储管理（核心考点）

<span style="color:#c62828;font-weight:700">🔴【虚拟存储的定义】：基于局部性原理，将进程的部分程序和数据装入内存，其余放在外存，逻辑上扩充内存容量。</span>
<span style="color:#c62828;font-weight:700">🔴【局部性原理】：程序执行时对内存访问的不均匀性（时间局部性：某条指令被执行后不久可能再次被执行；空间局部性：某存储单元被访问后不久其附近的存储单元也可能被访问）。</span>
<span style="color:#c62828;font-weight:700">🔴【虚拟存储的实现】：常见为请求分页，也可采用请求分段或段页式；请求分页需要页表、缺页异常处理、地址变换和页面置换等支持。</span>
⚫️【坑点】：虚拟存储的目的是逻辑上扩充主存容量（让用户感觉内存更大），不是物理上扩大主存。
#### 考点17：页面置换算法（计算高频）

<span style="color:#c62828;font-weight:700">🔴【最佳置换算法（OPT）】：选择以后永不使用或最长时间内不再使用的页面淘汰。理论最优，但无法实现（无法预知未来）。</span>
<span style="color:#c62828;font-weight:700">🔴【先进先出置换算法（FIFO）】：淘汰最早进入内存的页面。可能出现Belady异常（分配的物理块数增加，缺页次数反而增加）。</span>
<span style="color:#c62828;font-weight:700">🔴【最近最久未使用置换算法（LRU）】：淘汰过去最长时间未被访问的页面，通常表现较好，但不是理论最优；精确实现开销较大，实际系统常采用近似LRU策略。</span>
<span style="color:#c62828;font-weight:700">🔴【Clock/Second-Chance】：利用访问位循环检查页面，是近似LRU算法；基础Clock主要看访问位。增强Clock还结合修改位，通常优先选择访问位=0、修改位=0的页。</span>
⚫️【坑点】：
精确LRU需要维护访问次序，通常需硬件访问位、时间戳或栈等支持，成本较高；实际常用Clock等近似实现。
系统发生抖动（频繁的页面换入换出）会使系统效率降低。
页面置换计算必须明确页框初始是否为空、页面序列和算法细节；应逐次记录页框状态，不能脱离前提死记某个缺页次数。
### 第四章：设备管理

#### 考点18：I/O设备分类

<span style="color:#c62828;font-weight:700">🔴【块设备】：以数据块为单位传输，可随机访问。如：磁盘。I/O控制主要采取DMA方式。</span>
<span style="color:#c62828;font-weight:700">🔴【字符设备】：以字符为单位传输，顺序访问。如：键盘、鼠标、打印机。</span>
⚫️【坑点】：磁盘属于块设备，键盘属于字符设备。
#### 考点19：缓冲技术

<span style="color:#c62828;font-weight:700">🔴【缓冲的作用】：</span>
缓和CPU与I/O设备之间速度不匹配的矛盾。
减少对CPU的中断频率。
协调数据处理单位与传输单位不匹配的问题。
<span style="color:#c62828;font-weight:700">🔴【缓冲的实现】：软件缓冲通常是在内存中分配一段空间来实现的（不是在磁盘上）。</span>
⚫️【坑点】：判断题说“软件缓冲通常是在磁盘上分配一段空间” →错！是在内存中。
#### 考点20：SPOOLing技术（假脱机技术）

<span style="color:#c62828;font-weight:700">🔴【定义】：将独占设备改造为共享设备的技术（如打印机共享）。</span>
<span style="color:#c62828;font-weight:700">🔴【核心】：利用磁盘作为缓冲区（输入井/输出井），在磁盘上形成队列。</span>
<span style="color:#c62828;font-weight:700">🔴【作用】：实现虚拟设备功能。</span>
⚫️【坑点】：SPOOLing技术的核心是把独占设备变为共享设备。
#### 考点21：磁盘调度算法

<span style="color:#c62828;font-weight:700">🔴【先来先服务（FCFS）】：按请求顺序服务。</span>
<span style="color:#c62828;font-weight:700">🔴【最短寻道时间优先（SSTF）】：选择距离当前磁头位置最近的磁道访问。每次移动距离最短。</span>
<span style="color:#c62828;font-weight:700">🔴【扫描算法（SCAN，电梯算法）】：磁头按当前方向移动并服务沿途请求，到磁盘端点后反向；LOOK在当前方向无更远请求时即反向，不必走到物理端点。</span>
<span style="color:#c62828;font-weight:700">🔴【循环扫描（C-SCAN）】：只在一个方向上服务请求，到端点后快速返回另一端再继续；C-LOOK则在最远请求处回绕。</span>
⚫️【坑点】：
磁盘访问时间 =寻道时间 + 旋转延迟时间 + 读写时间。不包括CPU调度时间。
最短寻道时间优先算法会导致远处的请求饥饿。
### 第五章：文件管理

#### 考点22：文件与文件系统

<span style="color:#c62828;font-weight:700">🔴【文件】：具有文件名的信息集合。</span>
<span style="color:#c62828;font-weight:700">🔴【文件控制块（FCB）】：文件存在的标志，记录文件的属性信息（文件名、大小、地址、权限等）。</span>
<span style="color:#c62828;font-weight:700">🔴【目录结构】：</span>
单级目录：所有文件在同一目录下（有重名问题）。
二级目录：主目录 + 用户目录。
多级目录（树型目录）：最常用，允许不同用户的文件同名。形式为树型结构。
⚫️【坑点】：为了允许不同用户的文件具有相同的文件名，文件系统中采用多级目录。
#### 考点23：文件的逻辑结构

<span style="color:#c62828;font-weight:700">🔴【流式文件（无结构文件）】：文件中的信息是字节流，无结构。如：文本文件（.txt）。</span>
<span style="color:#c62828;font-weight:700">🔴【记录式文件（有结构文件）】：文件由一组记录组成。如：数据库文件。</span>
⚫️【坑点】：数据库文件的逻辑结构形式是记录式文件。
#### 考点24：文件的物理结构（分配方式）

<span style="color:#c62828;font-weight:700">🔴【连续分配】：文件占用连续磁盘块，支持顺序和随机访问，访问快；难以动态增长并会产生外部碎片。</span>
<span style="color:#c62828;font-weight:700">🔴【链接分配】：文件占用不连续的磁盘块，通过指针链接。存取时必须按指针顺序进行，效率较低。支持顺序存取，不支持随机存取。</span>
<span style="color:#c62828;font-weight:700">🔴【索引分配】：文件可占用不连续磁盘块，索引块保存各数据块地址，支持直接访问。这里的“索引分配”是文件物理块索引，不等同于数据库中“关键字→记录地址”的内容索引。</span>
⚫️【坑点】：
对文件的存取时必须按指针进行，效率较低 →链接文件。
文件的逻辑结构、存取方法和物理分配方式是不同层次：顺序存取不要求文件一定连续分配，链接分配也可以顺序访问。
同一文件的不同副本可以采用相同或不同物理结构，取决于文件系统和介质；不存在“必须不同”的通则。
#### 考点25：文件的存取方法

<span style="color:#c62828;font-weight:700">🔴【顺序存取】：按文件中的记录顺序依次访问（磁带、链接文件）。</span>
<span style="color:#c62828;font-weight:700">🔴【随机存取（直接存取）】：可访问任意位置的记录（磁盘、索引文件）。</span>

🔥 本章高频简答题汇总
1. 简述进程和程序的区别。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：① 进程是动态的，程序是静态的。② 进程有生命周期（创建到撤销），程序是永久的。③ 进程 =程序 + PCB + 数据集合。④ 一个程序可对应多个进程。</span>
2. 简述进程的三种基本状态及其转换。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：①就绪态：等待CPU。②运行态：占有CPU执行。③阻塞态：等待某事件（如I/O）。转换：就绪→运行（调度选中），运行→就绪（时间片完），运行→阻塞（等待事件），阻塞→就绪（事件完成）。</span>
3. 简述死锁产生的必要条件及预防方法。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：必要条件：互斥、请求与保持、不可剥夺、环路等待。预防通过破坏至少一个必要条件；资源有序分配法破坏环路等待，是常见方法。</span>
4. 简述分页和分段的区别。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：① 页是物理单位（为管理内存），段是逻辑单位（为用户需求）。② 页大小固定，段大小不固定。③ 分页是一维地址空间，分段是二维地址空间。</span>
5. 简述虚拟存储的原理。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：基于局部性原理（程序执行时对内存访问不均匀），将程序的部分装入内存，部分放在外存，在逻辑上扩充内存容量。当访问的页面不在内存时，产生缺页中断，将其调入内存。</span>
6. 简述SPOOLing技术的作用。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：将独占设备（如打印机）改造为共享设备，实现虚拟设备功能，提高了设备利用率。</span>

## 数据库技术

### 第一章：数据库系统概述

#### 考点1：数据库相关概念（选择/填空必考）

<span style="color:#c62828;font-weight:700">🔴【数据（Data）】：描述事物的符号记录，是数据库中存储的基本对象。</span>
<span style="color:#c62828;font-weight:700">🔴【数据库（DB）】：长期存储在计算机内、有组织、可共享并相互关联的数据集合；其数据按一定模型组织，具有尽可能小的冗余并可供多个应用使用。</span>
<span style="color:#c62828;font-weight:700">🔴【数据库管理系统（DBMS）】：位于用户与操作系统之间的一层数据管理软件。是数据库系统的核心。</span>
<span style="color:#c62828;font-weight:700">🔴【数据库系统（DBS）】：通常包括数据库、DBMS、应用程序、数据库管理员和用户，并依托计算机硬件、操作系统等运行环境；不同教材的枚举范围略有差异。</span>
<span style="color:#c62828;font-weight:700">🔴【数据库管理员（DBA）】：负责模式定义、权限与安全、备份恢复、性能监控和日常维护等。应用开发主要由开发人员承担，但小型项目中角色可兼任，不能把“不编写任何程序”当作定义。</span>
⚫️【坑点】：
DBMS是建立在操作系统之上的系统软件，不是建立在硬件系统上。
DBA的职责不包括编写应用程序（那是程序员的工作）。
数据库系统是在文件系统的基础上发展起来的。
#### 考点2：数据管理技术的发展（三个阶段）

<span style="color:#c62828;font-weight:700">🔴【人工管理阶段】：数据不保存，程序管理数据，数据不共享，数据与程序不具有独立性。</span>
<span style="color:#c62828;font-weight:700">🔴【文件系统阶段】：数据可长期保存，由文件系统管理。数据共享性差、冗余度大、独立性差。</span>
<span style="color:#c62828;font-weight:700">🔴【数据库系统阶段】：数据结构化、共享性高、冗余度低、数据独立性高。共享性最好的是数据库系统阶段。</span>
⚫️【坑点】：
数据管理技术发展的三个阶段中，数据共享性最好的是数据库系统阶段。
文件系统阶段的数据独立性差，不是强。
数据库避免了一切数据的重复→错！数据库只是减少了数据冗余，不可能完全避免（如为了查询效率，有时会保留少量冗余）。
#### 考点3：数据库系统的特点

<span style="color:#c62828;font-weight:700">🔴【三大特点】：</span>
数据结构化（主要特点之一）。
数据共享性高、冗余度低。
数据独立性高（物理独立性和逻辑独立性）。
⚫️【坑点】：数据库的主要特点不包括“程序的标准化”。
#### 考点4：数据独立性（重要概念）

<span style="color:#c62828;font-weight:700">🔴【物理独立性】：用户的应用程序与数据库的物理存储相互独立。当内模式（存储结构）改变时，模式不变，应用程序不需要修改。</span>
<span style="color:#c62828;font-weight:700">🔴【逻辑独立性】：模式发生一定范围变化时，可通过调整外模式/模式映像使外模式尽量不变，从而使基于外模式的应用程序不必修改。它通常比物理独立性更难实现。</span>
⚫️【坑点】：
逻辑独立性不是说模式怎样改变都能自动保持应用不变，而是由外模式/模式映像屏蔽允许范围内的逻辑变化。
物理独立性是指内模式改变，模式和应用程序不变。
数据库三级模式体系结构的划分，有利于保持数据库的数据独立性。
#### 考点5：数据库三级模式结构

<span style="color:#c62828;font-weight:700">🔴【外模式（用户模式/子模式）】：用户看到的局部数据逻辑结构。一个数据库可以有多个外模式。</span>
<span style="color:#c62828;font-weight:700">🔴【模式（概念模式/逻辑模式）】：数据库中全体数据的全局逻辑结构和特征的描述。一个数据库只有一个模式。</span>
<span style="color:#c62828;font-weight:700">🔴【内模式（存储模式）】：数据物理存储结构的描述。一个数据库只有一个内模式。最接近物理存储设备的结构是内模式。</span>
⚫️【坑点】：
数据的全局逻辑结构用模式描述，不是外模式。
三级模式中，外模式（用户模式）可以有多个，模式（概念模式）和内模式各只有一个。
最接近物理存储设备的结构是内模式。
### 第二章：数据模型

#### 考点6：数据模型的三要素

<span style="color:#c62828;font-weight:700">🔴【三要素】：</span>
数据结构：描述系统的静态特性（数据的组织方式）。
数据操作：描述系统的动态特性（对数据的增、删、改、查）。
完整性约束：保证数据的正确性、有效性和相容性。
⚫️【坑点】：数据模型的三要素不包括“并发控制”。并发控制是数据库保护的内容。
#### 考点7：概念模型（E-R模型）

<span style="color:#c62828;font-weight:700">🔴【E-R图三要素】：实体（矩形）、属性（椭圆）、联系（菱形）。</span>
<span style="color:#c62828;font-weight:700">🔴【实体】：客观存在且可相互区别的事物。</span>
<span style="color:#c62828;font-weight:700">🔴【属性】：实体所具有的特性。如学生的学号、姓名。</span>
<span style="color:#c62828;font-weight:700">🔴【联系】：实体之间的对应关系。</span>
<span style="color:#c62828;font-weight:700">🔴【联系类型】：</span>
1：1（一对一）：如丈夫与妻子、人与身份证、学生与学籍档案。
1：N（一对多）：如班级与学生、部门与员工。
M：N（多对多）：如学生与课程、供应商与零件。
⚫️【坑点】：
用E-R图表示实体类型、属性和联系是在概念设计阶段。
建立E-R模型的正确步骤：确定实体 → 确定实体的属性 → 确定实体之间的联系类型 → 设计出E-R图。
实体供应商和实体零件之间的联系是多对多（一个供应商可提供多种零件，一种零件可由多个供应商提供）。
#### 考点8：逻辑模型（三种数据模型）

<span style="color:#c62828;font-weight:700">🔴【层次模型】：用树形结构表示实体及实体间的联系。一对多。通过指针实现。</span>
<span style="color:#c62828;font-weight:700">🔴【网状模型】：用网状结构表示实体及实体间的联系。多对多。通过指针实现。</span>
<span style="color:#c62828;font-weight:700">🔴【关系模型】：用二维表表示实体及实体间的联系。目前最常用。用表（关系）表示数据。</span>
⚫️【坑点】：
用树形结构来表示实体及实体间联系的是层次模型。
通过指针链接来表示和实现实体之间联系的模型是层次模型和网状模型（关系模型用表，不用指针）。
数据库类型是按照数据模型来划分的。
概念模型（E-R模型）不是逻辑模型。
#### 考点9：关系模型的基本术语

<span style="color:#c62828;font-weight:700">🔴【关系（Relation）】：一张二维表。</span>
<span style="color:#c62828;font-weight:700">🔴【元组（Tuple）】：二维表中的一行（一条记录）。</span>
<span style="color:#c62828;font-weight:700">🔴【属性（Attribute）】：二维表中的一列（一个字段）。</span>
<span style="color:#c62828;font-weight:700">🔴【域（Domain）】：属性的取值范围。</span>
<span style="color:#c62828;font-weight:700">🔴【码/键（Key）】：能唯一标识一个元组的属性或属性组合。</span>
<span style="color:#c62828;font-weight:700">🔴【超码（Superkey）】：能唯一标识元组的属性集合，可能包含多余属性。</span>
<span style="color:#c62828;font-weight:700">🔴【候选码（Candidate Key）】：能唯一标识元组且不含多余属性的最小超码，可以有多个。</span>
<span style="color:#c62828;font-weight:700">🔴【主码（Primary Key）】：从候选码中选定一个作为主码。主码不能为空。</span>
<span style="color:#c62828;font-weight:700">🔴【外码（Foreign Key）】：关系中的一个属性组引用本关系或另一关系的候选码/具有唯一性约束的键；外码可以是复合属性，也不必引用“另一个表的主码”这一种情况。</span>
⚫️【坑点】：
二维表的行对应元组（记录），列对应属性（字段）。
一个表中可以有多个候选关键字，但只能有一个主键。
二维表的列可以任意交换，行也可以任意交换（关系模型的性质）。
关系中的行、列在逻辑上无序；关系不能有重复元组，每个属性值应来自同一域。SQL表在未写ORDER BY时也不保证输出顺序。
二维表中的每一列代表一个属性，不是实体。
关系模型以二维表结构来表示实体与实体之间的联系。
### 第三章：关系代数与SQL

#### 考点10：关系代数基本运算

<span style="color:#c62828;font-weight:700">🔴【五种基本运算】：并（∪）、差（−）、笛卡尔积（×）、选择（σ）、投影（π）。</span>
<span style="color:#c62828;font-weight:700">🔴【选择（σ）】：从行的角度进行筛选（水平方向）。选出满足条件的元组。</span>
<span style="color:#c62828;font-weight:700">🔴【投影（π）】：从列的角度进行筛选（垂直方向）。选出指定的属性。</span>
<span style="color:#c62828;font-weight:700">🔴【连接（⋈）】：从两个关系的笛卡尔积中选出满足条件的元组。</span>
<span style="color:#c62828;font-weight:700">🔴【除（÷）】：设R(X,Y)÷S(Y)，结果为满足“对S中的每个Y值，(X,Y)都在R中”的X值集合，常用于表达“至少具有全部……”的查询。</span>
⚫️【坑点】：
对R进行投影得到S时，因列被删除且结果要去除重复元组，所以|S|≤|R|；这一结论不能套用于任意两个无关关系R、S。
集合R与S的交可以用R − (R − S)表示。
传统集合运算是从行（水平）角度进行的，专门的关系运算是从行和列角度进行的。
#### 考点11：SQL语言概述

<span style="color:#c62828;font-weight:700">🔴【SQL全称】：Structured Query Language，结构化查询语言。</span>
<span style="color:#c62828;font-weight:700">🔴【SQL类型】：关系数据库语言。</span>
<span style="color:#c62828;font-weight:700">🔴【SQL功能分类】：DDL负责定义结构，DML负责数据操作，DCL负责权限控制；有些教材把SELECT单列为DQL，有些SQL标准/教材把查询纳入DML，答题以题目分类口径为准。</span>
⚫️【坑点】：
DELETE是删除数据（记录），DROP是删除表结构。
SQL语言是关系数据库语言，不是层次或网络数据库语言。
删除经销商1018的数据记录应使用：DELETE FROM distributors WHERE distri_num = 1018;
删除Student表的Birthday列，同时删除引用该列的视图和约束：ALTER TABLE Student DROP Birthday CASCADE;
将查询关系S的权限授予用户WANG，并允许其转授：GRANT SELECT ON S TO WANG WITH GRANT OPTION;
#### 考点12：SELECT语句（极高频）

<span style="color:#1565c0;font-weight:700">🔵【基本语法模板】：SELECT [DISTINCT] 目标列 FROM 表名 [WHERE 行条件] [GROUP BY 分组列] [HAVING 组条件] [ORDER BY 排序列 [ASC|DESC]];</span>
<span style="color:#c62828;font-weight:700">🔴【DISTINCT】：去掉查询结果中的重复行。</span>
<span style="color:#c62828;font-weight:700">🔴【WHERE】：筛选元组（行）的条件。</span>
<span style="color:#c62828;font-weight:700">🔴【GROUP BY】：按某列分组。</span>
<span style="color:#c62828;font-weight:700">🔴【HAVING】：对分组后的结果进行筛选。常与 GROUP BY 配合使用。</span>
<span style="color:#c62828;font-weight:700">🔴【ORDER BY】：对结果进行排序。ASC升序（默认），DESC降序。</span>
<span style="color:#c62828;font-weight:700">🔴【LIKE】：模糊查询。%（百分号）匹配任意长度的字符串；_（下划线）匹配单个字符。</span>
<span style="color:#c62828;font-weight:700">🔴【IN】：判断值是否在集合中。</span>
<span style="color:#c62828;font-weight:700">🔴【BETWEEN…AND…】：判断值是否在区间内（包含边界）。</span>
⚫️【坑点】：
SELECT语句的执行结果是集合（表）。
SELECT * FROM book WHERE book_num NOT BETWEEN 200 AND 300;等价于SELECT * FROM book WHERE book_num < 200 OR book_num > 300;
LIKE '_A%'表示第二个字符为A且至少2个字符；大小写是否敏感取决于DBMS、字符集与排序规则，不能写成“SQL通常不区分大小写”的通则。
HAVING用于分组后的条件筛选，WHERE用于分组前的条件筛选。
返回结果集中不包含相同的行，使用关键字DISTINCT。
SELECT INTO创建新表的语法和含义依DBMS而异（SQL Server/Access常见，MySQL通常使用CREATE TABLE ... SELECT或INSERT ... SELECT），必须看题设平台。
#### 考点13：SQL常用函数

<span style="color:#c62828;font-weight:700">🔴【COUNT()】：统计行数。</span>
<span style="color:#c62828;font-weight:700">🔴【SUM()】：求总和（数值列）。</span>
<span style="color:#c62828;font-weight:700">🔴【AVG()】：求平均值（数值列）。</span>
<span style="color:#c62828;font-weight:700">🔴【MAX()】：求最大值。</span>
<span style="color:#c62828;font-weight:700">🔴【MIN()】：求最小值。</span>
⚫️【NULL规则】：COUNT(*)统计结果行数；COUNT(列)、SUM、AVG、MAX、MIN通常忽略NULL。AVG(列)不会把NULL当0。聚合函数支持的数据类型依函数和DBMS而定，MAX/MIN常可用于可排序的字符、日期列。
#### 考点13A：连接、子查询与NULL（高频SQL）

<span style="color:#c62828;font-weight:700">🔴【连接】：INNER JOIN只保留匹配行；LEFT JOIN保留左表全部行，右表无匹配处补NULL；RIGHT JOIN反之；FULL OUTER JOIN保留两边未匹配行（并非所有DBMS直接支持）。</span>
<span style="color:#1565c0;font-weight:700">🔵【模板】：SELECT ... FROM A JOIN B ON A.key=B.key WHERE ...；连接条件应写清，否则可能产生笛卡尔积。</span>
<span style="color:#c62828;font-weight:700">🔴【NULL】：NULL表示未知/缺失，不等于0、空串或任何具体值。判断空值用IS NULL或IS NOT NULL，不能用“= NULL”。</span>
⚫️【三值逻辑】：涉及NULL的普通比较结果通常为UNKNOWN；WHERE只保留条件为TRUE的行。NOT IN子查询若含NULL可能得到意外空结果，考试中要特别辨析。
### 第四章：数据库完整性

#### 考点14：三种完整性约束

<span style="color:#c62828;font-weight:700">🔴【实体完整性（Entity Integrity）】：</span>
主码（主键）的值必须唯一且不能为空。
通过PRIMARY KEY定义。
要求userID作为Users表中元组的唯一标识→ 属于实体完整性规则。
<span style="color:#c62828;font-weight:700">🔴【参照完整性（Referential Integrity）】：</span>
外码值的非空部分必须与被参照键的某个值匹配；能否为NULL还取决于该外码列是否允许NULL。外键引用的通常是主键或UNIQUE候选键。
通过FOREIGN KEY定义。
不允许引用不存在的实体。
<span style="color:#c62828;font-weight:700">🔴【用户自定义完整性（User-defined Integrity）】：</span>
用户根据实际需要定义的约束。如：年龄必须在0-150之间、性别只能是“男”或“女”。
通过CHECK定义。
⚫️【坑点】：
PRIMARY KEY是实体完整性，FOREIGN KEY是参照完整性。
NOT NULL、UNIQUE、CHECK等属于SQL声明性约束；在“实体/参照/用户定义完整性”的教材分类中，非主键列上的取值限制常归用户定义完整性，但具体归类要看约束对象，不能机械地仅凭关键字判断。
主码子句和外码子句属于DBS的完整性措施，不是安全性措施。
### 第五章：范式理论

#### 考点15：范式（Normal Form）

<span style="color:#c62828;font-weight:700">🔴【1NF（第一范式）】：属性不可再分（原子性）。所有关系模式都满足1NF。</span>
<span style="color:#c62828;font-weight:700">🔴【2NF（第二范式）】：满足1NF，且每个非主属性完全函数依赖于每个候选码，即不存在非主属性对候选码真子集的部分依赖。</span>
<span style="color:#c62828;font-weight:700">🔴【3NF（第三范式）】：满足2NF，且非主属性不传递依赖于任何候选码。形式化地，对每个非平凡函数依赖X→A，X是超码或A是主属性。</span>
<span style="color:#c62828;font-weight:700">🔴【BCNF】：对关系模式中的每个非平凡函数依赖X→Y，决定因素X都必须是超码。BCNF严格强于3NF，BCNF一定是3NF，3NF不一定是BCNF。</span>
<span style="color:#c62828;font-weight:700">🔴【部分依赖】：某属性依赖于复合候选码的一部分；只有复合候选码才可能产生这类部分依赖，不能只检查被选为“主码”的一个候选码。</span>
<span style="color:#c62828;font-weight:700">🔴【传递依赖】：非主属性依赖于另一个非主属性，而该非主属性又依赖于主码。</span>
⚫️【坑点】：
如果关系模式R是2NF，且每个非主属性都不传递依赖于R的候选键，那么R的模式是3NF。
属于BCNF的关系模式必属于3NF。
关系规范化中的删除操作异常是指不该删除的数据被删除。
规范化理论中，关系必须满足：其每一属性都是不可分解的（1NF要求）。
### 第六章：事务与并发控制

#### 考点16：事务（Transaction）

<span style="color:#c62828;font-weight:700">🔴【定义】：事务是用户定义的一个数据库操作序列，构成单一逻辑工作单元，也是并发控制与恢复处理的重要单位；通常以COMMIT提交或ROLLBACK回滚结束。</span>
<span style="color:#c62828;font-weight:700">🔴【事务的ACID特性】：</span>
原子性（Atomicity）：事务中的操作要么全做，要么全不做，不可分割。
一致性（Consistency）：事务执行前后，数据库的完整性约束不被破坏。
隔离性（Isolation）：一个事务的执行不受其他事务干扰。
持续性（Durability）：事务一旦提交，对数据库的改变是永久的。
⚫️【坑点】：
“一个事务中所有对DB的操作是一个不可分割的操作序列” → 这是原子性。
事务的4个特征是：原子性、一致性、隔离性、持久性（Durability，部分旧译“持续性”）。
日志文件用来记录对数据库中数据进行的每一次更新操作（用于恢复）。
#### 考点17：并发操作的三大问题

<span style="color:#c62828;font-weight:700">🔴【丢失修改】：两个事务同时修改同一数据，后提交的覆盖了先提交的。</span>
<span style="color:#c62828;font-weight:700">🔴【不可重复读】：一个事务两次读取同一数据，在此期间被另一个事务修改，导致两次读到的值不同。</span>
<span style="color:#c62828;font-weight:700">🔴【读“脏”数据】：一个事务读取了另一个未提交事务修改的数据。</span>
<span style="color:#c62828;font-weight:700">🔴【幻读】：一个事务按同一条件两次查询一组行，另一个已提交事务在其间插入、删除或改变符合条件的行，导致结果集合发生变化。</span>
⚫️【坑点】：题干中“T1读A=10，T2读A=10并修改为20写回，T1再读A=20” →不可重复读。注意区分：读脏数据是读了未提交的数据，不可重复读是读了已提交但被修改的数据。
#### 考点18：封锁机制

<span style="color:#c62828;font-weight:700">🔴【X锁（排它锁/写锁）】：</span>
事务T对数据加X锁后，T可读可写该数据。
按常见锁协议，其他事务不能取得与X锁冲突的S锁或X锁，因而需等待；实际读取行为也受隔离级别、多版本并发控制等实现影响。
<span style="color:#c62828;font-weight:700">🔴【S锁（共享锁/读锁）】：</span>
事务T对数据加S锁后，T只能读不能写。
其他事务可以加S锁（共享读），但不能加X锁。
⚫️【坑点】：
如果事务T2对数据对象D施加了排它锁X，则事务T2对D可读可写。
加了X锁后，其他事务不能加任何锁。
<span style="color:#c62828;font-weight:700">🔴【两段锁协议（2PL）】：事务先处于增长阶段只加锁不解锁，后处于缩减阶段只解锁不加锁；基本2PL可保证冲突可串行化。严格2PL通常把写锁持有到提交/回滚，可避免级联回滚。</span>
<span style="color:#c62828;font-weight:700">🔴【隔离级别】：由低到高常见为读未提交、读已提交、可重复读、可串行化；级别越高通常并发异常越少、并发开销越大，但各DBMS的MVCC实现细节可能不同。</span>
### 第七章：数据库设计

#### 考点19：数据库设计步骤

<span style="color:#c62828;font-weight:700">🔴【六个阶段】：</span>
需求分析：调查业务、数据及处理需求，形成需求说明；数据流图和数据字典是常用分析成果，但具体方法可采用用例、业务规则等，并非所有项目都必须只产出这两项。
概念结构设计：将用户需求抽象为概念模型（E-R图）。产出：E-R图。
逻辑结构设计：将E-R图转换为关系模型（二维表）。
物理结构设计：确定数据的存储结构和存取路径（如建立索引）。选出存取路径。
数据库实施：建立数据库，编制应用程序，组织数据入库。
数据库运行与维护：对数据库进行评价、调整与修改。
⚫️【坑点】：
E-R图是在概念结构设计阶段使用的。
逻辑结构设计是将E-R图转换为关系模式。
物理结构设计是选择存取路径（建立索引等）。
需求分析阶段产生需求规格说明书。
### 第八章：Access数据库

#### 考点19A：视图与索引（常考对象）

<span style="color:#c62828;font-weight:700">🔴【视图（View）】：由查询定义的虚表，通常只保存定义而不保存查询结果（物化视图除外）。可简化查询、提供逻辑数据独立性并限制用户可见的行列。</span>
⚫️【可更新性】：不是所有视图都可直接更新；包含聚合、分组、DISTINCT、复杂连接等的视图通常受限，具体规则依DBMS而定。
<span style="color:#c62828;font-weight:700">🔴【索引（Index）】：辅助存取结构，可提高特定查询、连接和排序效率，但会占用存储空间，并增加INSERT、UPDATE、DELETE的维护成本。</span>
⚫️【坑点】：建立索引通常不改变表的逻辑内容；“索引越多越好”“索引一定提高所有操作速度”都错误。主键/唯一约束是否自动创建索引取决于DBMS实现。
#### 考点19B：恢复技术（高频辨析）

<span style="color:#c62828;font-weight:700">🔴【故障类型】：事务故障、系统故障、介质故障。事务/系统故障常依靠日志撤销或重做，介质故障通常需要备份配合日志恢复。</span>
<span style="color:#1565c0;font-weight:700">🔵【WAL预写日志】：修改后的数据页写入磁盘前，相应日志记录必须先持久化；提交成功前提交记录也应持久化，这是日志恢复的重要基础。</span>
<span style="color:#c62828;font-weight:700">🔴【检查点】：缩短故障恢复时需要扫描日志的范围，但不是“有了检查点就不需要日志或备份”。</span>
⚫️【备份区别】：全量备份保存全部选定数据；增量备份保存自上次任意备份后的变化；差异备份保存自上次全量备份后的变化。恢复链与时间成本不同。
#### 考点20：Access基础

<span style="color:#c62828;font-weight:700">🔴【Access类型】：关系数据库管理系统。</span>
<span style="color:#c62828;font-weight:700">🔴【Access文件扩展名】：.mdb（2003以前）、.accdb（2007以后）。</span>
<span style="color:#c62828;font-weight:700">🔴【Access对象要看版本】：Access常见核心对象为表、查询、窗体、报表、宏、模块六类；“数据访问页”是Access 2000/2003等旧版本对象，Access 2007起已不再作为受支持对象。旧教材题若明确旧版本，可能按7类作答。</span>
表：最基础的对象，用于存储数据。
查询：按条件检索或操作数据。选择查询的结果可视为动态结果集/虚表；Access还存在追加、更新、删除、生成表等操作查询，不能把所有查询都只定义为“虚拟表”。
窗体：用户交互界面。
报表：打印输出数据。
⚫️【坑点】：
Access是关系数据库，不是层次数据库或网状数据库。
旧版Access的数据访问页通常对应独立.htm文件；新版题目不要把它列为现行Access对象。
查询是Access数据库中的重要对象，是一种虚拟表。

## 数据结构与算法

### 第一章：数据结构概述

#### 考点1：数据结构的基本概念

<span style="color:#c62828;font-weight:700">🔴【数据】：信息的载体，是描述客观事物的符号。</span>
<span style="color:#c62828;font-weight:700">🔴【数据元素】：数据的基本单位，在计算机中通常作为一个整体进行考虑和处理。</span>
<span style="color:#c62828;font-weight:700">🔴【数据项】：数据的最小单位，是数据元素中不可再分的单元。</span>
<span style="color:#c62828;font-weight:700">🔴【数据对象】：具有相同性质的数据元素的集合。</span>
⚫️【坑点】：数据的最小单位是数据项，不是数据元素。数据元素是数据的基本单位。
#### 考点2：数据结构的三个层次

<span style="color:#c62828;font-weight:700">🔴【逻辑结构】：数据元素之间的逻辑关系，与计算机无关。</span>
分类：线性结构（线性表、栈、队列、串）和非线性结构（树、图、集合）。
<span style="color:#c62828;font-weight:700">🔴【存储结构（物理结构）】：数据在计算机中的存储表示。</span>
分类：顺序存储结构（连续空间）、链式存储结构（非连续空间）、索引存储、散列存储。
<span style="color:#c62828;font-weight:700">🔴【数据的运算】：对数据进行的操作（插入、删除、查找、排序等）。</span>
⚫️【坑点】：
研究数据结构就是研究数据的逻辑结构、存储结构及其基本操作。
数据的逻辑结构与数据元素本身的内容和形式无关。
与所使用的计算机无关的是数据的逻辑结构。
数据结构主要分为线性结构和非线性结构两类。
#### 考点3：算法

<span style="color:#c62828;font-weight:700">🔴【算法的定义】：为解决特定问题而规定的、有限的指令序列；算法是对求解步骤的抽象描述，不等同于某一种程序设计语言编写的程序。</span>
<span style="color:#c62828;font-weight:700">🔴【算法的五大特性】：</span>
有穷性：必须在有限步内终止。
确定性：每一步的含义无歧义。
可行性：每一步操作可执行。
输入：可以有0个或多个输入。
输出：必须有1个或多个输出。
<span style="color:#c62828;font-weight:700">🔴【算法效率的度量】：</span>
时间复杂度：以输入规模 n 为自变量，分析基本操作执行次数的数量级，常用大 O 记号表示；它不是某台机器上的实际运行秒数。
空间复杂度：分析算法所需存储空间随输入规模增长的数量级。题目说“辅助空间”时只计算输入数据之外的额外空间；说“总空间”时还应计入输入存储。
⚫️【坑点】：
算法必须有输出，但可以没有输入（0个输入）。
算法的可行性是指每一步都能精确运行，不是指指令没有二义性（那是确定性）。
算法的时间复杂度关注基本操作次数随输入规模增长的数量级，不等于程序实际运行时间；同一数量级的算法在常数、缓存和数据分布上仍可能有差别。
### 第二章：线性表

#### 考点4：线性表的定义与特点

<span style="color:#c62828;font-weight:700">🔴【定义】：n 个具有相同性质的数据元素组成的有限序列（n≥0）。“相同性质”是抽象数据结构口径，不必机械等同于某种语言里字节长度完全相同。</span>
<span style="color:#c62828;font-weight:700">🔴【特点】：数据元素之间是一对一的关系。除了第一个和最后一个，每个元素有且仅有一个直接前驱和一个直接后继。</span>
⚫️【坑点】：
线性表可以为空表（n=0）。
线性表中的元素之间是一对一的关系，不是一对多。
线性表是n个数据元素的有限序列。
#### 考点5：顺序存储结构（顺序表）

<span style="color:#c62828;font-weight:700">🔴【定义】：用连续的存储单元依次存储线性表的数据元素。</span>
<span style="color:#c62828;font-weight:700">🔴【特点】：</span>
逻辑相邻的，物理上也相邻。
可随机存取（通过首地址+下标直接访问任意元素）。
存储密度高（接近1）。
插入/删除需要移动大量元素，效率低。
<span style="color:#1565c0;font-weight:700">🔵【插入时间复杂度】：平均移动n/2个元素，时间复杂度O(n)。</span>
<span style="color:#1565c0;font-weight:700">🔵【删除时间复杂度】：平均移动(n-1)/2个元素，时间复杂度O(n)。</span>
⚫️【坑点】：
顺序存储的线性表可以实现随机存取。
顺序存储方式的优点是存储密度大，缺点是插入、删除效率低。
一个顺序表第一个元素地址90，每个元素长度2，第6个元素地址 = 90 + 5×2 =100。
#### 考点6：链式存储结构（链表）

<span style="color:#c62828;font-weight:700">🔴【定义】：用任意的存储单元存储数据元素，元素间通过指针链接。</span>
<span style="color:#c62828;font-weight:700">🔴【特点】：</span>
逻辑相邻的，物理上不一定相邻。
顺序存取（不能随机访问）。
在已知插入/删除位置及其相关结点指针时，不需要成批移动元素，只需修改链接，局部操作可为 O(1)；若需先从表头查找位置，整体仍可能为 O(n)。
存储密度小于1（需要额外的指针空间）。
<span style="color:#c62828;font-weight:700">🔴【单链表】：每个节点包含数据域和指针域（指向下一个节点）。</span>
<span style="color:#c62828;font-weight:700">🔴【循环链表】：最后一个节点的指针指向头节点，形成环。</span>
<span style="color:#c62828;font-weight:700">🔴【双向链表】：每个节点包含前驱指针和后继指针。</span>
⚫️【坑点】：
线性表采用链式存储时，地址连续与否均可。
链表的存储密度小于1。
链表不需成批移动元素且容量调整灵活，但访问第 i 个元素通常要顺链查找；若未给定位置指针，插入/删除的定位成本仍可能是 O(n)。
带头结点的链表最主要的好处是使空表和非空表的处理统一。
在单链表指针为p的结点之后插入指针为s的结点：s->next = p->next; p->next = s;（必须先连后断）。

### 第三章：栈和队列

#### 考点8：栈（Stack）

<span style="color:#c62828;font-weight:700">🔴【定义】：限定只在表尾（栈顶）进行插入和删除操作的线性表。</span>
<span style="color:#c62828;font-weight:700">🔴【特点】：后进先出（LIFO，Last In First Out）。</span>
<span style="color:#c62828;font-weight:700">🔴【基本操作】：入栈（Push）、出栈（Pop）、取栈顶（Top）。</span>
<span style="color:#c62828;font-weight:700">🔴【栈的应用】：</span>
函数调用（递归）。
表达式求值。
括号匹配。
深度优先搜索（DFS）。
<span style="color:#c62828;font-weight:700">🔴【链式栈 vs 顺序栈】：链式栈按需动态申请结点，不受固定数组容量限制，但仍可能因可用内存耗尽而入栈失败。</span>
⚫️【坑点】：
栈是线性结构，不是非线性结构。
栈的特点是后进先出，队列的特点是先进先出。
入栈顺序为1,2,3，全部入栈后出栈顺序为3,2,1。
顺序栈和链式栈在已知栈顶时，入栈、出栈通常都是 O(1)；顺序栈可能受固定容量限制，链式栈有指针和动态分配开销。
#### 考点9：队列（Queue）

<span style="color:#c62828;font-weight:700">🔴【定义】：限定在一端（队尾）插入，在另一端（队头）删除的线性表。</span>
<span style="color:#c62828;font-weight:700">🔴【特点】：先进先出（FIFO，First In First Out）。</span>
<span style="color:#c62828;font-weight:700">🔴【基本操作】：入队（Enqueue）、出队（Dequeue）。</span>
<span style="color:#c62828;font-weight:700">🔴【队列的应用】：</span>
广度优先搜索（BFS）。
缓冲区管理（打印任务队列）。
二叉树层序遍历（将根节点入队，出队时子节点入队）。
⚫️【坑点】：队列是一种先进先出的线性表，不是后进先出。
#### 考点10：循环队列

<span style="color:#c62828;font-weight:700">🔴【定义】：将队列的存储空间首尾相接，形成逻辑上的环。</span>
<span style="color:#c62828;font-weight:700">🔴【常见约定】：若 front 指向队头元素、rear 指向下一个可入队位置，并**牺牲一个存储单元**区分空与满，则：</span>
<span style="color:#1565c0;font-weight:700">🔵【判空条件】：front == rear。</span>
<span style="color:#1565c0;font-weight:700">🔵【判满条件】：(rear + 1) % MAXSIZE == front。</span>
<span style="color:#1565c0;font-weight:700">🔵【元素个数】：(rear - front + MAXSIZE) % MAXSIZE；长度为 MAXSIZE 的数组最多存 MAXSIZE-1 个队列元素。</span>
⚫️【坑点】：
循环队列是队列的一种顺序存储结构（不是链式存储结构！）。
循环队列中元素的个数由队头指针和队尾指针共同决定。
循环队列的判空、判满公式取决于指针含义和实现方案。使用计数器或标志位时可利用全部 n 个单元，判满条件与上述“牺牲一格”公式不同；做题必须先看题设约定。
### 第四章：树与二叉树

#### 考点11：树的基本概念

<span style="color:#c62828;font-weight:700">🔴【树的定义】：树是 n（n≥0）个结点的有限集合。n=0 时为空树；非空树有且仅有一个根结点，其余结点可分为若干互不相交、且本身也是树的子树。</span>
<span style="color:#c62828;font-weight:700">🔴【树的性质】：每个节点最多有一个直接前驱，可以有多个直接后继。</span>
<span style="color:#c62828;font-weight:700">🔴【树的度】：节点拥有的子树个数的最大值。</span>
<span style="color:#c62828;font-weight:700">🔴【叶子节点（叶节点）】：度为0的节点（没有子节点）。</span>
<span style="color:#c62828;font-weight:700">🔴【树的深度（高度）】：节点所在的最大层次。</span>
⚫️【坑点】：树形结构中每个节点最多只有一个直接前驱。根节点没有前驱。
#### 考点12：二叉树（Binary Tree）

<span style="color:#c62828;font-weight:700">🔴【定义】：每个节点最多有两棵子树（左子树和右子树），子树有左右之分，不能颠倒。</span>
<span style="color:#c62828;font-weight:700">🔴【二叉树 ≠ 度为2的树】：二叉树允许空树，度为2的树至少有3个节点。</span>
<span style="color:#c62828;font-weight:700">🔴【满二叉树】：所有非叶子节点的度都为2，且所有叶子节点都在同一层。</span>
<span style="color:#c62828;font-weight:700">🔴【完全二叉树】：除最后一层外，其他层都满，最后一层节点从左到右连续排列。</span>
<span style="color:#c62828;font-weight:700">🔴【二叉树与度为2的树的区别】：</span>
二叉树中结点的度可以为0、1、2；“度为2的树”是指整棵树的最大结点度为2，并不表示每个结点的度都固定为2。
二叉树的子树有左右之分，度为2的树没有顺序之分。
⚫️【坑点】：空二叉树没有根结点；**非空二叉树恰有一个根结点**。
#### 考点13：二叉树的性质（计算必考）

<span style="color:#c62828;font-weight:700">🔴【性质1】：在二叉树的第i层上至多有2^(i-1)个节点（i≥1）。</span>
<span style="color:#c62828;font-weight:700">🔴【性质2】：深度为k的二叉树至多有2^k - 1个节点（k≥1）。至少有k个节点。</span>
<span style="color:#c62828;font-weight:700">🔴【性质3】：对任意二叉树T，叶子节点数 n0 = 度为2的节点数 n2 + 1。</span>
<span style="color:#c62828;font-weight:700">🔴【性质4】：具有n个节点的完全二叉树，深度为⌊log₂n⌋ + 1。</span>
<span style="color:#c62828;font-weight:700">🔴【性质5】：按层序从 1 开始编号的完全二叉树中，结点 i 的双亲为 ⌊i/2⌋（i>1）；若 2i≤n，左孩子为 2i；若 2i+1≤n，右孩子为 2i+1。若数组从 0 编号，公式将改变。</span>
⚫️【坑点】：
一棵二叉树中，度为1的节点数为9 → 叶子节点的数目无法确定（需要知道度为2的节点数）。
设树T的度为4，其中度为1,2,3,4的节点个数分别为4,2,1,1 → 叶子数 = n2+2n3+3n4+1 = 2+2+3+1 =8。
一个有30个节点的二叉树深度至少为5（1+2+4+8+15=30，深度5）。
#### 考点14：二叉树的遍历（极高频）

<span style="color:#c62828;font-weight:700">🔴【前序遍历（Pre-order）】：根 → 左 → 右（递归：先访问根，再遍历左子树，再遍历右子树）。</span>
<span style="color:#c62828;font-weight:700">🔴【中序遍历（In-order）】：左 → 根 → 右（递归：先遍历左子树，再访问根，再遍历右子树）。</span>
<span style="color:#c62828;font-weight:700">🔴【后序遍历（Post-order）】：左 → 右 → 根（递归：先遍历左子树，再遍历右子树，再访问根）。</span>
<span style="color:#c62828;font-weight:700">🔴【层序遍历（Level-order）】：从上到下，从左到右逐层访问。使用队列实现。</span>
<span style="color:#c62828;font-weight:700">🔴【重要结论】：已知前序+中序或后序+中序可唯一确定一棵二叉树。仅知道前序+后序不能唯一确定。</span>
⚫️【坑点】：
若一个树叶是某二叉树的中序遍历的最后一个节点，则它也一定是该二叉树的前序遍历最后一个节点（正确）。
若一个节点是某二叉树的前序遍历最后一个节点，则它不一定是中序遍历最后一个节点。
某二叉树中序序列DBAFCE，层次序列FDECBA → 后序序列为ABDCEF。
结点按中序遍历为SYZ的二叉树可能有5种（卡特兰数：C₃ = 5）。
#### 考点15：二叉树的构造

<span style="color:#c62828;font-weight:700">🔴【由前序+中序构造】：</span>
前序第一个为根节点。
在中序中找到根节点位置，左侧为左子树，右侧为右子树。
递归构造。
<span style="color:#c62828;font-weight:700">🔴【由后序+中序构造】：</span>
后序最后一个为根节点。
在中序中找到根节点位置，左侧为左子树，右侧为右子树。
递归构造。
⚫️【坑点】：已知一棵二叉树的前序遍历和中序遍历序列，可以构造出这棵二叉树。
### 第五章：图

#### 考点16：图的基本概念

<span style="color:#c62828;font-weight:700">🔴【图的定义】：由顶点（Vertex）和边（Edge）组成的数据结构。Graph = (V, E)。</span>
<span style="color:#c62828;font-weight:700">🔴【无向图】：边无方向（如：好友关系）。</span>
<span style="color:#c62828;font-weight:700">🔴【有向图】：边有方向（如：关注关系）。</span>
<span style="color:#c62828;font-weight:700">🔴【完全图】：任意两个顶点之间都有边相连。</span>
无向完全图有n(n-1)/2条边。
有向完全图有n(n-1)条边。
<span style="color:#c62828;font-weight:700">🔴【顶点的度】：</span>
无向图：度 = 与该顶点相连的边数。
有向图：入度（指向该顶点的边）+出度（从该顶点出发的边）。
<span style="color:#1565c0;font-weight:700">🔵【度数公式】：无向图所有顶点度数之和等于边数的 2 倍；有向图所有顶点入度之和 = 出度之和 = 边数。</span>
⚫️【坑点】：图中任何两个节点之间都可能有逻辑关系，不是一对一或一对多。
#### 考点17：图的存储与连通性（高频补充）

<span style="color:#c62828;font-weight:700">🔴【邻接矩阵】：用 n×n 矩阵表示顶点间是否有边，判断两点是否相邻为 O(1)，空间 O(n²)，适合稠密图。无向图的邻接矩阵关于主对角线对称。</span>
<span style="color:#c62828;font-weight:700">🔴【邻接表】：每个顶点保存其邻接点链表/集合，空间通常为 O(V+E)，适合稀疏图；检查一条边的成本与相关顶点的度有关。</span>
<span style="color:#1565c0;font-weight:700">🔵【连通图】：无向图任意两顶点间都有路径称连通；有向图任意两顶点间彼此可达称强连通。</span>
<span style="color:#1565c0;font-weight:700">🔵【生成树】：含图中全部 n 个顶点且连通、无环的子图，边数恰为 n-1。连通无向图才存在生成树；最小生成树常用 Prim 或 Kruskal 算法。</span>
<span style="color:#1565c0;font-weight:700">🔵【最短路径】Dijkstra 适用于边权非负的单源最短路径；含负权边时不能直接套用。Floyd 可求所有顶点对最短路径。</span>
#### 考点18：图的遍历

<span style="color:#c62828;font-weight:700">🔴【深度优先搜索（DFS）】：使用栈（递归实现）。类似树的前序遍历。</span>
<span style="color:#c62828;font-weight:700">🔴【广度优先搜索（BFS）】：使用队列。类似树的层序遍历。</span>
⚫️【坑点】：深度优先搜索实现时需要用的数据结构是栈，广度优先搜索使用队列。
<span style="color:#1565c0;font-weight:700">🔵【复杂度前提】：采用邻接表时，DFS、BFS 的时间复杂度均为 O(V+E)；采用邻接矩阵时通常为 O(V²)。非连通图要从尚未访问的顶点重新启动，才能遍历全部顶点。</span>
### 第六章：查找与排序

#### 考点19：查找算法

<span style="color:#c62828;font-weight:700">🔴【顺序查找】：</span>
适用：任意线性表（无序有序都可以）。
时间复杂度：O(n)。
在各元素查找概率相等且查找成功的前提下，平均查找长度为 (n+1)/2；查找失败的比较次数取决于实现和表结构。
<span style="color:#c62828;font-weight:700">🔴【折半查找（二分查找）】：</span>
适用：有序的顺序存储表（要求随机存取）。
时间复杂度：O(log₂n)。
每次比较使搜索范围缩小一半。
不能用于链表（链表不能随机存取）。
<span style="color:#1565c0;font-weight:700">🔵【最大比较次数】：含 n 个元素的有序顺序表，成功或失败查找的比较次数上界通常为 ⌊log₂n⌋+1（不同循环边界写法可能使失败统计口径略有差异）。</span>
⚫️【坑点】：
二分查找要求线性表必须以顺序方式存储，且节点按关键字有序排序。
折半查找（二分查找）也称二分搜索、对数搜索。
#### 考点20：树表查找与散列查找（高频补充）

<span style="color:#c62828;font-weight:700">🔴【二叉排序树 BST】：对任一结点，左子树关键字通常小于该结点，右子树关键字通常大于该结点（重复关键字如何处理由实现约定）；中序遍历得到有序序列。</span>
<span style="color:#1565c0;font-weight:700">🔵【查找复杂度】：BST 查找、插入、删除均为 O(h)，h 为树高。树较平衡时约 O(log n)，退化成单链时为 O(n)，不能一概写成 O(log n)。</span>
<span style="color:#c62828;font-weight:700">🔴【平衡二叉树 AVL】：任一结点左右子树高度差的绝对值不超过 1，结点平衡因子只能为 -1、0、1；通过旋转维持平衡，查找为 O(log n)。</span>
<span style="color:#c62828;font-weight:700">🔴【堆】：堆是一棵完全二叉树。大根堆中每个父结点关键字不小于孩子，根最大；小根堆相反。堆只保证父子间偏序，**中序遍历不一定有序**。</span>
<span style="color:#c62828;font-weight:700">🔴【散列表 Hash Table】：散列函数把关键字映射到表地址；不同关键字映射到同一地址称冲突。常用处理方法有开放定址法（线性探测、二次探测等）和链地址法。</span>
<span style="color:#1565c0;font-weight:700">🔵【散列效率前提】：散列查找平均可接近 O(1)，但依赖散列函数、装填因子和冲突处理；冲突严重时最坏可退化为 O(n)。散列表通常不支持按关键字有序遍历。</span>
#### 考点21：排序算法（高频对比）

1. 插入类排序
<span style="color:#c62828;font-weight:700">🔴【直接插入排序】：将元素逐个插入已排序的有序序列中。</span>
时间复杂度：最好 O(n)，最坏 O(n²)，平均 O(n²)。
稳定排序。
2. 交换类排序
<span style="color:#c62828;font-weight:700">🔴【冒泡排序（Bubble Sort）】：两两比较，反序则交换，直到没有反序记录为止。</span>
时间复杂度：带“本趟无交换则提前结束”标志的改进实现最好 O(n)；朴素固定趟数实现即使已有序仍可能 O(n²)。最坏、平均 O(n²)。
稳定排序。
记忆口诀：大的往下沉，小的往上冒。
<span style="color:#c62828;font-weight:700">🔴【快速排序（Quick Sort）】：选一个基准元素，将序列划分为“不大于/不小于”基准的区域（具体等号归属取决于划分算法），再递归排序子区间。</span>
时间复杂度：最好/平均O(nlog₂n)，最坏O(n²)。
当总选首元素或尾元素为基准时，初始序列正序或反序会造成极不平衡划分，是典型最坏情况；随机选基准、三数取中或三路划分可降低该风险，因此不能说所有快速排序实现遇到有序序列都必然最坏。
不稳定排序。
原地实现、缓存友好，实践中平均性能常很好，但不能绝对表述为“任何数据和平台上最快”。
3. 选择类排序
<span style="color:#c62828;font-weight:700">🔴【简单选择排序】：每趟找出最小（或最大）元素，放到序列起始位置。</span>
时间复杂度：始终O(n²)。
不稳定排序。
<span style="color:#c62828;font-weight:700">🔴【堆排序】：利用堆的数据结构进行排序。</span>
时间复杂度：O(nlog₂n)。
不稳定排序。
4. 其他排序
<span style="color:#c62828;font-weight:700">🔴【希尔排序（Shell Sort）】：插入类排序法。按一定间隔分组，分别进行直接插入排序。</span>
属于插入类排序，不是交换类或选择类。
时间复杂度与增量序列有关，不能脱离增量统一给出最好、平均、最坏界；常见基础教材按所指定的增量分析。希尔排序不稳定。
5. 归并排序
<span style="color:#c62828;font-weight:700">🔴【归并排序（Merge Sort）】：把序列递归分成子序列，分别排序后再合并。</span>
<span style="color:#1565c0;font-weight:700">🔵【复杂度与性质】：最好、平均、最坏时间复杂度均为 O(nlog₂n)；常见数组实现需要 O(n) 辅助空间；稳定。适合外部排序及链表排序。</span>
⚫️【坑点】：
希尔排序属于插入类排序法。
冒泡排序的基本思想是两两比较，反序交换。
“快速排序在正序或反序时最坏”需要以固定选首/尾元素作基准等划分策略为前提。
选择排序第一次交换：从“2，6，4，1”从大到小选最大6与第一个交换 →6，2，4，1。
📊 终极复杂度/性质记忆表（考前过一遍）
⚫️【稳定排序总结】：
稳定的：直接插入排序、冒泡排序、归并排序。（口诀：冒泡直接归并稳）
不稳定的：希尔、选择、堆、快速。（口诀：希选快堆都不稳）

🔥 本章高频简答题汇总
1. 简述算法的五个重要特性。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：有穷性（有限步终止）、确定性（无二义）、可行性（可执行）、输入（0个或多个）、输出（1个或多个）。</span>
2. 简述数组和链表的区别。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：① 数组/顺序表占用连续内存空间，链表不要求连续。② 数组随机存取，链表顺序存取。③ 顺序表插入/删除通常需移动元素；链表在已定位时只需修改链接。④ 顺序表存储密度高，链表有指针开销。⑤ 二者操作复杂度都要结合“是否已定位”和具体实现说明。</span>
3. 简述栈和队列的区别。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：① 栈后进先出（LIFO），队列先进先出（FIFO）。② 栈只在一端操作，队列在两端操作。③ 栈常用于函数调用、递归，队列常用于缓冲区、层序遍历。</span>
4. 简述二叉树的前序、中序、后序遍历方法。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：①前序：根 → 左 → 右。②中序：左 → 根 → 右。③后序：左 → 右 → 根。</span>
5. 简述折半查找的基本思想。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：在有序表中，取中间元素与目标比较。相等则查找成功；目标小于中间值，则在左半区继续查找；目标大于中间值，则在右半区继续查找。每次比较使搜索范围缩小一半。</span>
口诀速记
“算法五要有穷确，可行输入输出全；线性表序和链表，顺序存密链插简；栈后进先出递归用，队列先进先出层序游；二叉树前根左右，中左右根，后左右根；堆排快排不稳当，冒泡插入最稳当；二分查找O(log n)，有序顺序是前提。”

### 第一部分：Visual Basic（VB）语言

#### 考点1：VB概述

- **🔴【适用版本】**：本节按教师招聘试题最常见的 **Visual Basic 6.0（VB6）** 口径整理；VB.NET 的类型系统、工程文件和面向对象机制与 VB6 不同，不能混答。
- **🔴【VB全称】**：Visual Basic，Microsoft 公司开发的可视化、事件驱动程序设计语言。VB6 通常称为**基于对象**，不能据此断言它具备 VB.NET 那样完整的继承等面向对象机制。
- **🔴【VB最突出的特点】**：**事件驱动编程机制**（与传统程序设计语言相比的最大区别）。
- **🔴【VB的工程文件】**：
- 工程文件：`.vbp`（VBP = Visual Basic Project）
- 窗体文件：`.frm`
- 标准模块：`.bas`
- **⚫️【坑点】**：VB应用程序既可以**解释方式**执行（在IDE中），也可以**编译方式**执行（生成.exe文件）。说“只能以解释方式执行”是错的。
#### 考点2：VB变量命名规则（极高频判断）

- **🔴【命名规则】**：
1. 按常见 VB6 教材口径，必须以**字母**开头；中文开发环境通常也允许以汉字等字母字符命名，但考试若未说明环境，优先采用“英文字母开头”的标准示例。
2. 后续可由**字母、数字、下划线**等组成（中文环境可使用汉字），不能含小数点、空格及运算符等特殊符号。
3. 长度**不超过255个字符**。
4. **不能使用VB关键字**（如：Dim、Sub、If、End、For、Next、Public、Private、Integer、String等）。
5. **不区分大小写**。
- **⚫️【坑点】**：
- `x.yz` → **不合法**（含小数点）。
- `a_b` → **合法**（下划线允许）。
- `Andif` → **合法**（不是VB关键字，And和If分开是关键字，但Andif不是）。
- `Dim` → **不合法**（是关键字）。
#### 考点3：VB数据类型

- **🔴【基本数据类型】**：
- **Integer**：整型（2字节，-32768~32767）。
- **Long**：长整型（4字节）。
- **Single**：单精度浮点型（4字节）。
- **Double**：双精度浮点型（8字节）。
- **String**：**字符串型**。
- **Boolean**：布尔型（True/False，True转为数值为**-1**，False转为数值为**0**）。
- **Date**：日期/时间型，日期字面量用 `#` 括起来。数字日期的月/日次序可能受区域格式影响；需要无歧义时使用 `DateSerial(2017, 1, 1)`。
- **⚫️【坑点】**：
- Boolean类型的True转换为数值是 **-1**（不是1！），False是0。
- `Dim X, Y As Integer` → X被定义为**变体类型（Variant）**，Y被定义为**整型**。（注意：只有Y是Integer，X未声明类型默认为Variant）
#### 考点4：VB运算符与表达式

- **🔴【算术运算符优先级】**（由高到低）：
1. `^`（乘方）
2. `-`（取负）
3. `*`（乘）、`/`（除，结果为浮点数）
4. `\`（整除，结果为整数）
5. `Mod`（取余）
6. `+`（加）、`-`（减）
- **🔴【常用函数】**：
- `Abs(x)`：**绝对值**。
- `Int(x)`：**向下取整**（取不大于x的最大整数）。`Int(5.49)=5`，`Int(-7.8)=-8`。
- `Fix(x)`：**取整数部分**（直接截断）。`Fix(-7.8)=-7`。
- `Sqr(x)`：**平方根**。
- `Rnd`：产生[0,1)之间的**随机数**。
- `Val("123")`：将**数字字符串**转换为**数值**，返回123。
- `Str(123)`：将数值转换为字符串，但正数结果通常保留一个前导空格（`Str(123)` 为 `" 123"`）；不需要前导空格时常用 `CStr(123)`，结果为 `"123"`。
- `Len("ABC")`：返回**字符串长度**，返回3。
- `Mid("ABCDE", 2, 3)`：从第2个字符开始取3个，返回"BCD"。
- `Right("Beijing", 4)`：从右边取4个字符，返回"jing"。
- **⚫️【坑点】**：
- `Int(-7.8)` = **-8**（向下取整，取不大于-7.8的最大整数）。
- `10 + Int(5.49)` = 10 + 5 = **15**。
- 表达式 `6 + 10 Mod 10 \ 9 / 3 + 2 ^ 3` = **15**（按优先级逐步计算）。
- `156 \ 10 Mod 7` = 15 Mod 7 = **1**（整除优先级高于Mod）。
- `Abs(-3.8) = 3.8`，`Sqr(49) = 7`，`Int(-7.8) = -8`。

- **⚫️【程序阅读真题】**：
- For i = 1 To 11 Step 3 → i取值：1,4,7,10 → 循环体执行 **4** 次。
- `s=0: i=0: Do While s<10: i=i+1: s=s+i*i: Loop` → 循环体执行 **3** 次（i从1到3，s=1+4+9=14，第4次条件不满足）。
#### 考点6：VB数组

- **🔴【数组声明】**：未写 `Option Base 1` 时，`Dim a(5) As Integer` 通常表示 a(0) 到 a(5)，共 **6** 个元素；若模块使用 `Option Base 1`，则默认下限变为 1。最稳妥的写法是显式指定下限。
- **🔴【指定下限】**：`Dim a(1 To 5) As Integer` → a(1)到a(5)，共 **5** 个元素。
- **🔴【二维数组】**：`Dim a(0 To 5, 1 To 5) As Integer` → 6×5 = **30** 个元素。
- **⚫️【坑点】**：VB6/VBA 数组的默认下限通常为 0，但可受模块级 `Option Base 0/1` 影响；`Dim a(1 To 5)` 不受该设置影响。
#### 考点7：VB对象、属性、方法与事件（操作题高频）

- **🔴【四个概念】**：对象是被操作的实体；属性描述对象状态；方法是对象能够执行的操作；事件是对象能够识别并响应的外部动作。
- **🔵【常见属性】**：`Name` 是程序代码中引用控件的名称；`Caption` 常表示窗体、标签、按钮上显示的标题；文本框主要使用 `Text`。不要把对象名称与界面显示文字混为一谈。
- **🔵【常见事件】**：`Form_Load` 在窗体装载时发生，`CommandButton_Click` 在按钮被单击时发生，`TextBox_Change` 在文本内容改变时发生。事件过程名称通常由“对象名_事件名”构成。
- **⚫️【判断陷阱】**：事件由用户操作或系统状态触发，事件过程才是响应事件执行的代码；“Click 是按钮的方法”在 VB6 常见试题口径下通常错误。
### 第二部分：C语言

#### 考点8：C语言概述

- **🔴【C程序结构】**：按标准的宿主环境 C 程序口径，一个程序由一个或多个函数组成，并有一个程序入口 `main`。推荐写作 `int main(void)` 或 `int main(int argc, char *argv[])`；`void main()` 不是 ISO C 的标准形式。嵌入式等独立环境可以不以 `main` 为入口，考试未说明时按宿主环境作答。
- **🔴【C程序运行步骤】**：**编辑 → 编译 → 连接 → 运行**。
- **⚫️【坑点】**：`main()`函数可以在程序文件的**任何位置**，不一定在开头或结尾。
#### 考点9：C语言标识符规则

- **🔴【命名规则】**：
1. 只能由**字母、数字、下划线**组成。
2. 第一个字符**必须是字母或下划线**（不能是数字）。
3. **区分大小写**（`abc`和`ABC`是不同的变量）。
4. **不能使用C关键字**（如：if、case、switch、int、float、return等）。
- **⚫️【坑点】**：
- `_int&int` → **不合法**（含`&`特殊符号）。
- `_Int` → **合法**（下划线开头）。
- `intint` → **合法**（不是关键字）。
- `case` → **不合法**（是关键字）。
#### 考点10：C语言数据类型

- **🔴【基本类型】**：
- `char`：字符型，按 C 语言定义恒为 **1 个 C 字节**；一个字节至少 8 位。
- `short`、`int`、`long` 的精确字节数由实现决定，标准只保证 `sizeof(char) ≤ sizeof(short) ≤ sizeof(int) ≤ sizeof(long)` 以及各自的最小取值范围。
- 在多数当前 32/64 位教学环境中，`short` 常为 2 字节、`int` 常为 4 字节、`float` 常为 4 字节、`double` 常为 8 字节；题目指定编译器或存储模型时以题设为准。
- **🔴【常量的合法判断】**：
- `1.234e04` → **合法**（1.234×10⁴）。
- `1.234e0.4` → **不合法**（指数必须是整数）。
- `1.234e4` → **合法**。
- `1.234e0` → **合法**。
- **⚫️【坑点】**：在试题常用实现中，`double` 一般为 64 位（8 字节），但不能把实现相关的 `sizeof` 结论写成所有 C 平台都必须如此。
#### 考点11：C语言运算符

- **🔴【自增自减运算符】**：
- `i++`：表达式的值是递增前的 i，随后产生 i 加 1 的副作用。
- `++i`：先使 i 加 1，表达式的值是递增后的 i。
- *例*：`int i=1; j=-i++;` → i=**2**，j=**-1**（先取i=1，j=-1，再i自增为2）。
- **🔴【逗号表达式】**：取**最右边表达式**的值。`a=2*6, a*3, a+5` → a=12，表达式值为12+5=**17**。
- **🔴【条件运算符（三目运算符）】**：`表达式1 ? 表达式2 : 表达式3`。`int a=7,b=9; t=(a>b)?a:b;` → t=**9**（a>b为假，取b）。
- **🔴【逻辑运算符】**：
- `&&`（与）：两个都为真才为真。
- `||`（或）：一个为真即为真。
- `!`（非）：取反。
- **🔵【短路求值】**：`a && b` 在 a 为假时不再求值 b；`a || b` 在 a 为真时不再求值 b。可用于防止非法运算，如 `p != NULL && p->value > 0`。
- **🔵【整数除法】**：两个整数相除仍为整数；C99 及以后标准规定商向 0 截断，如 `-7/3 == -2`，余数满足 `(a/b)*b + a%b == a`，因此 `-7%3 == -1`。
- **⚫️【坑点】**：
- 赋值运算符`=`的结合方向是**“自右至左”**。
- `strcmp(a, b) == 0` 才表示两个字符串内容相等；`if (strcmp(a,b))` 在字符串**不相等**时条件为真。
- `scanf("%d%d%d", &x, &y, &z)` → 正确的输入语句，`&`取地址。
- 不要在同一个未规定求值顺序的表达式中多次修改同一标量，例如 `i = i++ + ++i` 会产生未定义行为，不能机械套用“先用后加”求唯一答案。
- 赋值、条件运算符以及多数一元运算符等均具有从右到左的结合性；不能说“只有赋值运算符从右结合”。结合性也不等同于操作数实际求值顺序。
#### 考点12：C语言流程控制

- **🔴【if-else语句】**：
```c
if (条件) {
语句1;
} else {
语句2;
}
```
- **🔴【switch语句】**：
```c
switch(表达式) {
case 值1: 语句1; break;
case 值2: 语句2; break;
default: 语句3;
}
```
- **🔴【循环结构】**：
- `for(i=0; i<5; i++)` → 循环体执行 **5** 次（i=0,1,2,3,4）。
- `while(条件)` → 先判断后执行。
- `do-while(条件)` → 先执行后判断，循环体**至少执行一次**。
- **⚫️【程序阅读真题】**：
- `for(x=0,y=0; (y!=1234) && (x<5); x++)` → 执行 **5** 次（x=0~4）。
- `switch(i){ case 9: i+=1; case 10: i+=1; case 11: i+=1; default: i+=1; }` → i=10，无break，依次执行case10、case11、default → i=**13**。
#### 考点13：C语言函数

- **🔴【函数的定义】**：`返回值类型 函数名(参数列表) { 函数体 }`。
- **🔴【形参与实参】**：形参是函数定义时的参数，实参是调用时传入的值。形参的修改**不影响**实参（值传递）。
- **🔵【指针传参辨析】**：C 始终按值传递。传入指针时复制的是地址值，函数可通过该地址修改调用者对象；这不是语言层面的“引用传递”。数组形参在函数参数位置会调整为指针类型。
- **🔴【递归函数】**：函数调用**自身**。
```c
int x(int n) {
if (n <= 3) return 1;
else return x(n-2) + x(n-4) + 1;
}
```
- **⚫️【程序阅读真题】**：
- `void fun(int x,int y,int z){ z=x*x+y*y; } int main(void){ int a=31; fun(5,2,a); printf("%d",a); return 0; }` → 输出 **31**（形参 z 的修改不影响实参 a）。旧教材省略函数返回类型的写法不符合 C99 及以后标准。
- 递归函数`x(x(8))`需要计算 **18** 次x函数。
- **⚫️【递归前提】**：递归必须有可到达的终止条件，并使问题规模趋近终止条件；否则可能无限递归并耗尽调用栈。
#### 考点14：C数组、指针与字符串（岗位考试高频）

- **🔴【数组】**：同一数组的元素类型相同，并按下标顺序连续存放；长度为 n 的数组合法下标为 0～n-1。越界访问属于未定义行为，编译器不一定报错。
- **🔵【数组名退化】**：在大多数表达式中，数组名会转换为指向首元素的指针；但作为 `sizeof`、一元 `&` 等运算的操作数时不发生该转换。数组名本身不是可修改的指针变量。
- **🔴【指针】**：`&x` 取得 x 的地址，`*p` 访问 p 所指对象。使用未初始化、已释放或空指针解引用都会产生未定义行为。
- **🔵【字符串】**：C 字符串是以空字符 `\0` 结尾的字符序列。字符数组容量必须为正文字符数加 1，例如 `char s[]="abc"` 占 4 个字节。
- **⚫️【输入陷阱】**：`scanf("%s", s)` 遇空白停止且必须限制宽度以防溢出；读取整行更适合用 `fgets`。字符串内容比较使用 `strcmp`，不能用 `==` 比较字符数组内容。
- **🔵【变量作用域与生存期】**：局部变量通常具有块作用域；文件作用域变量常称全局变量。局部 `static` 变量只初始化一次，生存期贯穿程序执行，但作用域仍限于所在块。
### 第三部分：Python语言

#### 考点15：Python概述

- **🔴【Python特点】**：
- Python 通常以解释器运行，但“解释型/编译型”取决于实现：CPython 会先把源代码编译为字节码，再由虚拟机执行。因此“Python 完全不编译”是错的。
- 可调用C语言编写的底层代码（**黏性扩展**）。
- **平台无关**（可在Windows、Linux、Mac等平台运行）。
- 通过缩进表示代码块，强调可读性。
- **变量无需声明**，赋值即创建。
- **⚫️【坑点】**：不能脱离实现和任务笼统断言 Python 一定比所有编译型语言慢。Python 广泛用于 Web、数据分析、人工智能、自动化和教学，也能参与系统工具开发；“只能用于某一领域”通常是错误表述。
#### 考点16：Python语法规则

- **🔴【缩进】**：Python 用缩进表示代码块。同一代码块必须保持一致；PEP 8 **建议**每级 4 个空格，但“语法强制只能用4个空格”是错的。Tab 与空格不一致可能触发 `TabError` 或改变层级。
- **🔴【注释】**：
- 单行注释：`# 注释内容`
- 严格说 Python 没有专门的“多行注释”语法；`'''...'''` 或 `"""..."""` 是三引号字符串。未赋给变量的独立字符串常被当作说明文字，模块、类、函数开头的三引号字符串是文档字符串（docstring）。
- **🔴【变量】**：名字通常无需预先声明，赋值会建立名字与对象的绑定。**对象有类型，变量名本身不固定绑定某一种类型**，因此同一名字可先后绑定不同类型对象。
- **⚫️【坑点】**：
- 所有代码都必须在行前至少加一个空格 → **错**！只有需要缩进的代码块才需要缩进。
- 已知`x=1.5`，赋值`x='china'`**不会报错**（Python变量可以改变类型）。
#### 考点17：Python数据类型与运算

- **🔴【基本数据类型】**：
- `int`：整数。
- `float`：浮点数。
- `bool`：布尔值（True/False）。
- `complex`：复数。
- `str`：字符串。
- **🔴【运算符】**：
- `/`：除法（结果为浮点数）。`3.0/5 = 0.6`
- `//`：向负无穷取整的商（floor division），不是简单“截去小数”。`3//5 = 0`，但 `-3//2 = -2`。
- `%`：取余。
- `**`：乘方。
- **🔴【序列类型】**：元素具有先后次序，可通过整数索引访问，并通常支持切片。
- 字符串（str）、**元组（tuple）**、**列表（list）**、`range` 都是常见序列类型。
- **字典（dict）** 是映射类型，**不是**序列类型！
- **🔴【列表操作】**：
- 列表中的元素**可以是不同数据类型**。
- 列表是**可变对象**，`append()` 原地修改同一个列表，因此列表对象的 `id` 不变；但其内部存储区可能重新分配，不能表述为“所有内存地址都不变”。
- **⚫️【程序阅读真题】**：
- `x = 12 + 3 * ((5 * 8) - 14) // 6; print(x)` → 25。
- `for i in range(1,6): if i%4==0: continue; else: print(i)` → 输出 **1,2,3,5**（4被跳过）。
- `x = 'R\OS\OT'; print(len(x))`：`\O` 不是 Python 定义的转义序列；按当前常见实现保留反斜杠，长度为 **7**，并可能产生无效转义警告。表达 Windows 路径宜写原始字符串 `r'R\OS\OT'` 或转义反斜杠 `'R\\OS\\OT'`，二者长度同为 7。
#### 考点18：Python流程控制

- **🔴【条件判断】**：
```python
if 条件:
语句块
elif 条件:
语句块
else:
语句块
```
Python支持链式比较：`4 <= 5 <= 6` → **True**。
- **🔴【循环】**：
```python
for i in range(1, 10, 2):   # i = 1,3,5,7,9
循环体
while 条件:
循环体
```
- **🔴【break与continue】**：
- `break`：**终止整个循环**。
- `continue`：**结束本次循环**，进入下一次循环。
- **⚫️【坑点】**：`break`用来结束当前循环语句，**但跳出当前循环体**（是跳出，不是不跳出）。
#### 考点19：Python容器、切片与函数（高频补充）

- **🔴【可变与不可变】**：列表、字典、集合通常可变；数字、字符串、元组、`frozenset` 不可变。元组不可变是指元组保存的引用序列不能改，元组内部若引用可变对象，该对象本身仍可能被修改。
- **🔵【切片规则】**：`s[start:stop:step]` 左闭右开，不包含 `stop`；`s[::-1]` 常用于逆序。索引越界通常抛出 `IndexError`，普通切片越过边界会自动截断。
- **🔵【字典与集合】**：字典按键访问，键必须可哈希；集合元素互异且可哈希。Python 3.7 起语言规范保证字典保持插入顺序，但“集合按插入顺序输出”没有保证。
- **🔵【函数参数】**：Python 采用“对象引用按值传递/共享传参”。函数可原地修改传入的可变对象，但给形参重新赋值不会让调用者变量改绑。
- **⚫️【默认参数陷阱】**：默认参数在函数定义时求值一次。避免 `def f(a=[]): ...` 造成多次调用共享同一列表，常写成 `def f(a=None): a = [] if a is None else a`。
- **⚫️【作用域口诀】**：名字查找遵循 LEGB：Local（局部）→ Enclosing（闭包外层）→ Global（全局）→ Built-in（内置）。函数内给全局变量赋值需声明 `global`；修改外层函数变量绑定使用 `nonlocal`。
#### 考点20：跨语言易混对照（判断题高频）

- **🔴【相等与赋值】**：VB6 中赋值和相等比较都使用 `=`，由语境区分；C/Python 赋值用 `=`，相等比较用 `==`。
- **🔵【逻辑运算】**：VB6 常用 `And/Or/Not`；C 用 `&&/||/!`；Python 用 `and/or/not`。Python 的 `and/or` 返回参与运算的对象之一，不保证只返回 `True/False`。
- **🔵【下标与区间】**：C 数组从 0 开始；Python 序列从 0 开始且支持负索引；VB6 数组下限可显式设置，并可能受 `Option Base` 影响。
- **⚫️【版本前提】**：Python 题应留意 Python 2 与 Python 3 的差异（如 `/` 和 `print`）；当前未注明版本时通常按 Python 3。C 题留意编译器、标准版本及数据模型；VB 题留意 VB6 与 VB.NET。
### 口诀速记

> **“VB变量字母汉字开头，不含点空格和特殊；**
> **整数取整Int向下，负七点八返回负八；**
> **VB名字不分大小，C语言区分大小写；**
> **C的char是一字节，其他字节看实现；**
> **i++先用后自增，++i先增后使用；**
> **Python缩进表示块，对象有类型名可换；**
> **列表可变原地改，字典是映射非序列。”**

## 计算机组成原理

### 第一章：计算机发展史

#### 考点1：早期电子计算机（题干限定词决定答案）

<span style="color:#c62828;font-weight:700">🔴【ENIAC】：1946年公开运行于美国宾夕法尼亚大学，通常按我国基础教材口径称“世界上第一台通用电子数字计算机”，主要使用电子管。</span>
<span style="color:#c62828;font-weight:700">🔴【ENIAC的局限性】：早期主要通过插接线和开关设置程序，并非今天意义上的存储程序计算机，改换任务很不方便。</span>
<span style="color:#c62828;font-weight:700">🔴【存储程序计算机要分清】：EDVAC是冯·诺依曼等参与设计的早期存储程序方案；Manchester Baby（1948）通常被认为第一台运行存储程序的电子计算机；EDSAC（1949）是较早投入实际使用的存储程序计算机。</span>
⚫️【坑点】：题目若只问基础教材中的“第一台电子数字计算机”，通常答ENIAC；若题干出现“通用”“运行存储程序”“投入实际使用”等限定词，答案可能不同。ENIAC并不是由冯·诺依曼独立设计的。
#### 考点2：计算机的四代划分

<span style="color:#c62828;font-weight:700">🔴【划分依据】：电子元器件（不是按体积、速度、价格！）。</span>
<span style="color:#c62828;font-weight:700">🔴【各代对应】：</span>
第1代（1946-1957）：电子管。机器语言，体积大、耗电多、速度慢。
第2代（1958-1964）：晶体管。高级语言开始出现（FORTRAN、COBOL）。
第3代（1965-1970）：中小规模集成电路。操作系统雏形出现。
第4代（1971-至今）：大规模/超大规模集成电路。微型计算机诞生。
⚫️【坑点】：判断题说“大规模集成电路是第三代计算机的标志” →错！是第四代！
#### 考点3：计算机之父的尊称（极易混淆）

<span style="color:#c62828;font-weight:700">🔴【冯·诺依曼】：“现代计算机之父”（提出存储程序+程序控制，确定五大部件结构）。</span>
<span style="color:#c62828;font-weight:700">🔴【图灵】：“计算机科学之父”、“人工智能之父”（提出图灵机模型、图灵测试）。</span>
<span style="color:#c62828;font-weight:700">🔴【查尔斯·巴贝奇】：“通用计算机之父”（差分机/分析机的设计者）。</span>
<span style="color:#c62828;font-weight:700">🔴【阿塔纳索夫】：与贝里研制ABC计算机，它是早期电子数字计算装置，但属于专用、不可编程设备；“电子计算机之父”并非所有教材统一采用的唯一称号。</span>
⚫️【坑点】：题目问“计算机科学的奠基人/图灵机提出者”通常选图灵；问“存储程序思想或冯·诺依曼体系结构”通常选冯·诺依曼。尊称题必须服从题干和教材口径，不能只凭“某某之父”判断。
#### 考点4：计算机的特点与分类

<span style="color:#c62828;font-weight:700">🔴【常见特点】：运算速度快、计算精度高、存储容量大、具有逻辑判断能力、自动化程度高、通用性强。</span>
<span style="color:#c62828;font-weight:700">🔴【自动工作的基础】：在存储程序控制下，计算机能按程序连续执行；“最关键特点”属于具体教材口径，不宜脱离题干绝对化。</span>
<span style="color:#c62828;font-weight:700">🔴【传统教材按规模/性能分类】：巨型机、大型机、中型机、小型机、微型机；现代设备边界已不完全由体积或单项速度决定。</span>
巨型机：“神威·太湖之光”、“天河二号”。
微型机：PC（个人计算机），以微处理器为核心。
⚫️【坑点】：世界上第一台计算机ENIAC是巨型机，不是个人计算机。个人计算机（PC）是第四代才出现的。
### 第二章：冯·诺依曼体系结构

#### 考点5：核心设计思想（简答题必背！）

<span style="color:#c62828;font-weight:700">🔴【思想名称】：“存储程序”与“程序控制”（合称“存储程序控制原理”）。</span>
<span style="color:#1565c0;font-weight:700">🔵【三大核心要点】：</span>
计算机内部采用二进制表示数据和指令。
将编好的程序和数据事先存入内存储器（存储程序）。
计算机在工作时，控制器自动、逐条地从内存中取出指令并加以执行（程序控制）。
⚫️【坑点】：计算机之所以能“自动连续”工作，最直接的原因是采用了“存储程序控制”，而不是“运算速度快”或“有逻辑判断能力”。
#### 考点6：硬件五大部件（极高频填空/多选）

<span style="color:#c62828;font-weight:700">🔴【五大部件】：运算器、控制器、存储器、输入设备、输出设备。</span>
<span style="color:#c62828;font-weight:700">🔴【CPU的基本组成】：运算器 + 控制器；从实际结构看还包含寄存器组，现代CPU通常集成多级Cache等部件。基础选择题常按“运算器+控制器”作答。</span>
<span style="color:#c62828;font-weight:700">🔴【主机的组成】：CPU + 内存（主存储器）。（注意：主机不包括外存！）</span>
<span style="color:#1565c0;font-weight:700">🔵【各部件核心功能】：</span>
⚫️【坑点】：
运算器不包括译码器（译码器属于控制器）。
话筒是输入设备（把声音变成电信号），不是输出设备！
触摸屏既是输入设备（触摸操作），又是输出设备（显示画面）。
磁盘驱动器（硬盘/U盘）既是输入设备又是输出设备（可读可写）。
### 第三章：CPU与指令系统

#### 考点7：CPU的性能指标

<span style="color:#c62828;font-weight:700">🔴【机器字长】：CPU定点运算器一次能处理的二进制数据位数（常见32位、64位）。在其他条件相近时，字长会影响整数表示范围、运算精度和寻址/处理能力，但不能单独决定整机性能。</span>
<span style="color:#c62828;font-weight:700">🔴【主频】：CPU时钟频率（常用GHz）。同一微体系结构且其他条件相近时，主频提高通常可缩短CPU时间；跨架构比较时不能据此断定谁更快。</span>
<span style="color:#c62828;font-weight:700">🔴【性能指标】：可用执行时间、吞吐率、CPI、IPC、MIPS等描述。MIPS只适合在指令系统和测试负载相近时参考，不能脱离程序直接横向比较。</span>
<span style="color:#1565c0;font-weight:700">🔵【MIPS计算公式】：MIPS = 主频(Hz) / (CPI × 10⁶)，其中CPI是执行一条指令所需的平均时钟周期数。</span>
<span style="color:#1565c0;font-weight:700">🔵【CPU执行时间】：CPU时间 = 指令数 × CPI × 时钟周期 = 指令数 × CPI / 主频。</span>
⚫️【坑点】：整机性能由CPU、内存层次、I/O、软件和工作负载共同决定，不能写成“只由CPU或主频决定”。“Pentium 4”是处理器系列名称。
#### 考点8：CPU不能直接访问的存储器

<span style="color:#c62828;font-weight:700">🔴【核心结论】：CPU不能直接访问外存储器（硬盘、U盘、光盘等）。</span>
<span style="color:#1565c0;font-weight:700">🔵【正确流程】：外存中的数据必须先调入内存（RAM），CPU才能通过数据总线访问。</span>
⚫️【坑点】：判断题说“CPU可以直接读取硬盘中的程序并执行” →错！必须先调入内存。
#### 考点9：指令的执行过程

<span style="color:#c62828;font-weight:700">🔴【三大步骤】：每条指令的执行分为三个阶段：</span>
取指令：控制器从内存中取出指令，送入指令寄存器（IR）。
分析指令：指令译码器对操作码进行译码，确定要执行什么操作。
执行指令：控制器发出控制信号，指挥各部件完成操作。
<span style="color:#c62828;font-weight:700">🔴【指令的组成】：操作码（规定做什么）+地址码/操作数（规定对谁做）。</span>
⚫️【坑点】：在冯·诺依曼结构中，指令和数据都以二进制形式存入存储器；某段二进制被当作指令还是数据，取决于程序执行上下文、地址和控制器的解释。不能简单说“仅靠取指/执行两个时间阶段区分”，执行阶段也可能不访存。
#### 考点10：CISC 与 RISC（拔高题）

<span style="color:#c62828;font-weight:700">🔴【CISC（复杂指令集）】：</span>
通常指令种类和寻址方式较丰富，单条指令可完成较复杂功能；指令长度可能可变。
代表：Intel x86 架构（PC电脑的CPU）。
传统实现常采用微程序控制，但现代CISC处理器内部也大量采用硬布线、流水线和微操作转换；“一定功耗高、对编译器要求低”不是定义性结论。
<span style="color:#c62828;font-weight:700">🔴【RISC（精简指令集）】：</span>
指令系统精简高效，指令长度固定，格式规整。
代表：ARM 架构（手机、平板、嵌入式设备）。
典型特点：指令格式规整、寻址方式较少、强调寄存器操作和Load/Store结构，利于流水线；不等于每条指令都单周期，也不保证任何RISC都比CISC更快或更省电。
⚫️【坑点】：RISC的指令种类少（精简），不是多！说“RISC指令种类多”是错的。
### 第四章：存储器系统（极高频，细节极多）

#### 考点11：存储器的层次结构

<span style="color:#c62828;font-weight:700">🔴【速度由快到慢】：寄存器（CPU内部） > Cache（高速缓存） > 内存（主存） > 外存（硬盘/U盘）。</span>
<span style="color:#c62828;font-weight:700">🔴【容量由小到大】：寄存器 < Cache < 内存 < 外存。</span>
<span style="color:#c62828;font-weight:700">🔴【价格由高到低】：寄存器 > Cache > 内存 > 外存。</span>
<span style="color:#c62828;font-weight:700">🔴【引入Cache的目的】：解决CPU 与 内存（主存）之间速度不匹配的矛盾。</span>
⚫️【坑点】：Cache是为了解决CPU和内存的速度矛盾，不是解决内存和外存的矛盾！也不是为了解决容量矛盾！
#### 考点12：内存（主存） vs 外存（辅存）—— 必考对比

<span style="color:#c62828;font-weight:700">🔴【内存（主存）】：</span>
速度快、容量小、价格高。
其中RAM通常具有易失性，断电后其数据消失；主存概念也可能包括少量ROM，因此不能把“内存”与“RAM”在所有语境下完全等同。
存放正在运行的程序和数据。
CPU可以直接访问。
<span style="color:#c62828;font-weight:700">🔴【外存（辅存）】：</span>
速度慢、容量大、价格低。
断电后数据保存。
存放暂时不用的程序和数据。
CPU不能直接访问（需先调入内存）。
⚫️【坑点】：相同容量的内存比外存贵得多，但速度快得多。判断题说“相同容量的内存和外存价格差不多” →错！
#### 考点13：ROM vs RAM（选择填空必出）

<span style="color:#c62828;font-weight:700">🔴【RAM（随机存取存储器）】：</span>
特点：可读可写，断电丢失数据（易失性）。
作用：存放当前正在运行的程序和数据（就是我们的内存条）。
SRAM（静态RAM）：速度快，用作Cache（高速缓存）。
DRAM（动态RAM）：速度较慢，用作主存（需要周期性刷新）。
<span style="color:#c62828;font-weight:700">🔴【ROM（只读存储器）】：</span>
基本特征是非易失、正常运行时以读为主。掩膜ROM出厂固化；PROM可一次编程，EPROM/EEPROM/Flash可按相应方式擦写，因此“所有ROM永远不能写”是错误表述。
作用：存放BIOS（基本输入输出系统）等固定程序。
注：PROM只能编程一次；EPROM可用紫外线擦除后重写；EEPROM可电擦写。基础题中的“只读”强调正常使用方式，不代表所有ROM技术物理上绝对不可改写。
⚫️【特别警示】：
BIOS（基本输入输出系统）存放在ROM中。
CMOS（互补金属氧化物半导体）是主板上的一块RAM芯片，靠电池供电保存日期/时间/密码等设置信息。CMOS设置程序在BIOS中。
判断题说“BIOS存放在CMOS中” →错！BIOS在ROM中，CMOS是存设置参数的。
#### 考点14：存储容量计算

<span style="color:#c62828;font-weight:700">🔴【基本单位】：字节（Byte，简称B）是存储容量的基本单位。位（bit，简称b）是数据的最小单位。</span>
<span style="color:#1565c0;font-weight:700">🔵【二进制容量换算（传统考试口径）】：1B = 8bit；1KB = 1024B；1MB = 1024KB；1GB = 1024MB；1TB = 1024GB。严格的IEC记号中1024B=1KiB，而SI中的1kB=1000B；做题按题干口径使用。</span>
<span style="color:#1565c0;font-weight:700">🔵【汉字存储空间计算】：</span>
一个16×16点阵汉字占：16×16/8 = 32字节。
一个24×24点阵汉字占：24×24/8 = 72字节。
一个32×32点阵汉字占：32×32/8 = 128字节。
公式：存储空间(字节) = (点阵行数 × 点阵列数) / 8。
<span style="color:#c62828;font-weight:700">🔴【字符占用空间要看编码】：GB2312/GBK常用双字节表示汉字，标准ASCII字符占1字节；UTF-8中常用汉字通常占3字节，UTF-16中常见汉字通常占2字节、部分字符占4字节。</span>
⚫️【坑点】：只有题目明确采用“双字节汉字机内码”时，2KB才可按1024个汉字计算；不能把“任意汉字在任何编码中固定占2字节”当作通则。
#### 考点15：地址总线与内存容量的关系

<span style="color:#c62828;font-weight:700">🔴【关键结论】：地址总线的宽度（根数）决定了CPU的寻址范围（最大内存容量）。</span>
<span style="color:#1565c0;font-weight:700">🔵【计算公式】：最大内存容量（字节）= 2^(地址总线根数)（按字节编址时）。</span>
⚫️【坑点】：公式必须写清“按字节编址”；若按字编址，最大可寻址容量还要乘每个编址单位的字节数。地址总线宽度给出的是可寻址上限，实际安装主存还受芯片组、主板、操作系统等限制。数据总线宽度影响一次并行传送的数据位数。
#### 考点15A：Cache命中率与平均访问时间（高频计算）

<span style="color:#c62828;font-weight:700">🔴【局部性基础】：Cache利用时间局部性和空间局部性，把近期可能使用的主存块副本保存在更快的存储器中。</span>
<span style="color:#1565c0;font-weight:700">🔵【命中率】：命中率 h = Cache命中次数 / 总访问次数；失效率 = 1-h。</span>
<span style="color:#1565c0;font-weight:700">🔵【平均访问时间】：若先查Cache、失效后再访问主存，可按题目时序写为 T = h×Tc + (1-h)×(Tc+Tm)；若题目规定Cache与主存并行访问，公式会不同，必须看题干。</span>
⚫️【映射方法】：直接映射实现简单但冲突较多；全相联冲突少但硬件复杂；组相联是二者折中。写策略常见写直达（Write Through）与写回（Write Back）。
### 第五章：I/O设备与总线

#### 考点16：输入/输出设备分类（务必分清）

<span style="color:#c62828;font-weight:700">🔴【常见输入设备】：键盘、鼠标、扫描仪、话筒、摄像机、条码阅读器、触摸屏（兼输出）。</span>
<span style="color:#c62828;font-weight:700">🔴【常见输出设备】：显示器、打印机、音箱、绘图仪、投影仪。</span>
<span style="color:#c62828;font-weight:700">🔴【兼有输入/输出功能】：触摸显示屏既显示又接收触控；可读写存储设备在I/O数据流角度可输入和输出，但硬盘、U盘通常归入外存/存储设备，分类要看题干口径。</span>
⚫️【坑点】：
扫描仪是输入设备（纸上的图→电子版），不是输出设备！
投影仪是输出设备（电脑画面→投影幕布），不是输入设备！
打印机用DPI（每英寸点数）衡量打印精度，DPI越高打印越清晰。
喷墨打印机耗材是墨盒；激光打印机耗材是硒鼓。
DPI是衡量打印分辨能力的重要指标之一；打印质量还受墨粉/墨水、色彩、纸张、机械精度等影响，不能只由DPI决定。
#### 考点17：显示器性能指标

<span style="color:#c62828;font-weight:700">🔴【常考指标】：分辨率（屏幕像素矩阵，如1920×1080）是重要指标之一；显示效果还与尺寸、像素密度、刷新率、色域、亮度等有关。</span>
<span style="color:#c62828;font-weight:700">🔴【其他指标】：点距（越小越清晰）、刷新率（越高越流畅）、颜色质量（色深）。</span>
⚫️【坑点】：显示系统的主要性能指标是分辨率、颜色质量、刷新速度，不包括内存容量。
#### 考点18：系统总线（极高频）

<span style="color:#c62828;font-weight:700">🔴【总线的英文】：BUS（注意不是UBS，也不是UPS！UPS是不间断电源，UBS是公司名）。</span>
<span style="color:#c62828;font-weight:700">🔴【系统总线分类（按传输信息）】：</span>
数据总线（DB）：传输数据。双向。宽度（位数）决定了CPU一次能传输的数据量。
地址总线（AB）：传输地址。单向（由CPU发出）。宽度决定了寻址范围（最大内存容量）。
控制总线（CB）：传输读/写、中断、总线请求/响应等控制与状态信号；各条线有各自方向，作为一组通常包含双向的信息流。
<span style="color:#1565c0;font-weight:700">🔵【理想峰值带宽】：带宽 = 每次传输字节数 × 每秒传输次数。若每个时钟只传一次，才可写为（总线宽度/8）×总线频率；DDR或突发传输还要乘每周期传输次数等系数。</span>
例：32位总线，33MHz → 带宽 = (32/8) × 33 = 4×33 = 132 MB/s。
⚫️【坑点】：
决定CPU与外部进行一次数据传输信息量的是数据总线宽度。
决定主存储器大小的是地址总线宽度。
数据、地址、控制是“系统总线按所传信息”的三类；按位置或用途还可分片内总线、系统总线、I/O总线等，不能说所有总线在任何分类下都只有三种。
### 第六章：I/O数据传送控制方式

#### 考点19：三种控制方式（高频多选/判断）

<span style="color:#c62828;font-weight:700">🔴【方式1：程序控制I/O】：无条件传送适用于外设总能按约定及时就绪的简单场景；程序查询方式由CPU反复读取状态并在就绪后传送，两者不宜完全等同。</span>
CPU主动、不断查询外设状态（“忙不忙？好了没？”）。
缺点：CPU大量时间浪费在查询等待上，效率极低。
<span style="color:#c62828;font-weight:700">🔴【方式2：中断方式】：</span>
外设准备好后，主动向CPU发中断请求，CPU暂停当前工作去处理。
优点：CPU利用率提高。缺点：适合少量数据传输，频繁中断也会消耗CPU资源。
<span style="color:#c62828;font-weight:700">🔴【方式3：直接存储器存取方式（DMA）】（极高频考点）：</span>
核心：DMA控制器在取得总线控制权后，使I/O设备与主存之间成批传送数据，CPU不逐字节搬运。
适用场景：大批量数据传送（如磁盘读写、视频数据传输）。
特点：传送过程中CPU可以执行其他任务，传送结束后才向CPU发中断报告。
⚫️【坑点判断题】：
“DMA方式传输数据时，完全不需要CPU干预。” → 严格说传送过程中不需要CPU，但开始和结束仍需CPU参与配置和中断响应。如果选项说“完全不需要CPU”，通常是错的！
“中断方式适合大批量数据传输” →错！适合小批量。大批量用DMA。
### 第七章：数据的表示与编码

#### 考点20：原码、反码、补码（必考计算！）

<span style="color:#c62828;font-weight:700">🔴【核心结论】：现代计算机中的有符号整数通常用补码表示；无符号整数、浮点数、字符等采用各自编码，不能统称“所有数据都用补码”。补码使加减法可统一使用加法器，并把零表示唯一化。</span>
<span style="color:#1565c0;font-weight:700">🔵【三码转换规则（以8位为例）】：</span>
正数：原码 = 反码 = 补码（三者相同）。
例：+5 → 原码00000101，反码00000101，补码00000101。
负数：符号位为1（最高位），其余位：
原码：数值部分为绝对值的二进制。
例：-5 → 原码10000101。
反码：符号位不变，其余位按位取反。
例：-5 → 反码11111010。
补码：反码 + 1（末位加1）。
例：-5 → 补码11111011。
⚫️【特殊值】：8位补码能表示的范围是-128 ~ +127。-128的补码是10000000（这是个必须记住的特殊值）。
⚫️【坑点】：8位无符号二进制能表示的最大十进制数是255（2⁸-1），不是256。
#### 考点21：ASCII码

<span style="color:#c62828;font-weight:700">🔴【ASCII码】：美国信息交换标准代码，7位二进制编码（存储时占1个字节，即8位）。</span>
<span style="color:#c62828;font-weight:700">🔴【编码数量】：共128个（0-127）。</span>
<span style="color:#c62828;font-weight:700">🔴【关键数值（必须记住！）】：</span>
数字'0'→ 48（十进制）→00110000（二进制）
大写'A'→ 65 →1000001
小写'a'→ 97 →1100001
<span style="color:#c62828;font-weight:700">🔴【ASCII码大小排序】：数字（0-9）< 大写字母（A-Z）< 小写字母（a-z）。</span>
⚫️【坑点】：大写字母和小写字母的ASCII码差32。'A'=65，'a'=97，差32。'a'比'A'大32。
#### 考点22：汉字编码

<span style="color:#c62828;font-weight:700">🔴【四个层次】：</span>
输入码（外码）：用于输入汉字，如五笔（形码）、拼音（音码）。
交换码（国标码）：传统教材常以GB2312国标码说明信息交换；GBK是对GB2312的扩展编码，Unicode则是现代跨平台字符编码标准。
机内码（内码）：传统GB2312教材中，汉字机内码由国标码两个字节各加80H得到，两个字节最高位均为1；“占2字节”只适用于该编码口径。
字形码（字模码）：用于输出显示/打印，用点阵表示（如16×16、24×24）。
⚫️【坑点】：
五笔字型属于输入码（外码），不是机内码！
传统教材中对汉字处理、存储常说使用机内码；现代系统具体采用UTF-8、UTF-16、GBK等编码，应以题干为准。
输出汉字字形的清晰度与汉字点阵的规模有关（点阵越大越清晰）。
在GB2312传统双字节口径下，汉字机内码的两个字节分别比国标码高80H，整体常表述为相差8080H。
#### 考点22A：浮点数表示（常考概念）

<span style="color:#c62828;font-weight:700">🔴【基本组成】：浮点数通常由符号、阶码和尾数（有效数）组成；IEEE 754单精度为1位符号、8位阶码、23位小数部分，双精度为1+11+52位。</span>
<span style="color:#1565c0;font-weight:700">🔵【规格化目的】：在约定表示下使有效数字尽可能充分，保证表示唯一性并提高精度。</span>
⚫️【坑点】：浮点数的表示范围主要受阶码位数影响，精度主要受尾数位数影响；浮点运算会有舍入误差，十进制小数不一定能被二进制精确表示。
### 第八章：进制转换（计算题必过）

#### 考点23：R进制转十进制

<span style="color:#1565c0;font-weight:700">🔵【方法】：“按权展开，相加求和”。</span>
<span style="color:#1565c0;font-weight:700">🔵【公式】：(N)R = Σ(每一位数字 × R^(该位所在位置))，小数点左第一位指数为0，向左递增，向右递减为负。</span>
⚫️【例】：(1101.1)₂ = 1×2³ + 1×2² + 0×2¹ + 1×2⁰ + 1×2⁻¹ = 8+4+0+1+0.5 = 13.5
#### 考点24：十进制转R进制

<span style="color:#1565c0;font-weight:700">🔵【整数部分】：“除R取余，倒序输出”（一直除到商为0，余数从下往上读）。</span>
<span style="color:#1565c0;font-weight:700">🔵【小数部分】：“乘R取整，正序输出”（一直乘到积为0或达到精度，整数从上往下读）。</span>
⚫️【例】：(53.125)₁₀ = (110101.001)₂
#### 考点25：二进制转八进制/十六进制（速算法）

<span style="color:#1565c0;font-weight:700">🔵【二转八】：“三位并一位”。以小数点为界，左右每3位一组，不足补0，每组转成1位八进制数。</span>
例：(10100010.0011)₂ = (242.14)₈
<span style="color:#1565c0;font-weight:700">🔵【二转十六】：“四位并一位”。以小数点为界，左右每4位一组，不足补0，每组转成1位十六进制数。</span>
例：(11101111)₂ = (EF)₁₆
⚫️【坑点】：十六进制数码包括0-9和A-F（A=10，B=11，C=12，D=13，E=14，F=15）。

#### 考点27：常见故障原因

<span style="color:#c62828;font-weight:700">🔴【CMOS设置丢失/时间反复重置】：常见原因是主板实时时钟/CMOS供电电池电量不足，但也可能是固件设置、主板故障等，不能仅凭现象唯一断定。</span>

<span style="color:#c62828;font-weight:700">🔴【投影仪不显示电脑画面】：可能性：①未按笔记本切换键（Fn+F4/F5等）；②分辨率不匹配；③连接线松动或损坏。</span>
<span style="color:#c62828;font-weight:700">🔴【计算机无声】：可能原因：声卡损坏、扬声器损坏、音量静音。显示器处于屏保状态不是无声的原因！</span>
⚫️【坑点】：键盘进水后，仅仅擦干表面不能保证键盘恢复正常，水可能已渗入内部电路造成短路。

🔥 本章高频简答题汇总
1. 冯·诺依曼计算机的基本特点是什么？
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：① 由运算器、控制器、存储器、输入设备、输出设备五大部件组成。② 计算机内部采用二进制表示数据和指令。③ 采用**“存储程序、程序控制”**技术（将程序事先存入主存，计算机自动逐条取出并执行）。</span>
2. 简述内存和外存的区别。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：①内存速度快、容量小、价格高，断电后数据消失，CPU可直接访问，存放正在运行的程序和数据。②外存速度慢、容量大、价格低，断电后数据保留，CPU不能直接访问（需先调入内存），存放暂时不用的程序和数据。</span>
3. 简述CPU的组成及各部分功能。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：① CPU由运算器和控制器组成（Cache有时也在CPU内部）。②运算器：进行算术运算和逻辑运算，核心是ALU。③控制器：产生控制信号，指挥协调全机工作，核心是PC（程序计数器）和IR（指令寄存器）。</span>
4. 简述总线的分类及各自功能。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：①数据总线（DB）：传输数据，双向，宽度决定一次传输的数据量。②地址总线（AB）：传输地址，单向（CPU发出），宽度决定寻址范围（最大内存容量）。③控制总线（CB）：传输控制信号，双向。</span>

## 计算机网络

### 第一章：计算机网络概述

#### 考点1：计算机网络的定义

<span style="color:#c62828;font-weight:700">🔴【定义】：将地理位置不同的、具有独立功能的多台计算机及其外部设备，通过通信线路连接起来，在网络操作系统、网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。</span>
<span style="color:#1565c0;font-weight:700">🔵【三大要素】：计算机系统 + 通信线路与通信设备 + 网络协议。</span>
⚫️【坑点】：计算机网络是计算机技术和通信技术相结合的产物，不是计算机技术和传感技术！
#### 考点2：计算机网络的功能（简答题必背）

<span style="color:#c62828;font-weight:700">🔴【三大功能】：</span>
数据通信（最基本功能）：实现计算机之间、计算机与终端之间的信息传输。如：电子邮件、文件传输。
资源共享（重要功能）：共享硬件（打印机）、软件服务和数据（文件、数据库内容）。数据和信息资源本身也可以成为共享对象。
分布处理（协同工作）：将大型任务分配给多台计算机并行处理。
⚫️【坑点】：学校网站上有很多微视频供同学们下载学习，主要体现了网络的资源共享功能，不是数据通信！
#### 考点3：计算机网络的分类

<span style="color:#c62828;font-weight:700">🔴【按覆盖范围分类】：</span>
局域网（LAN）：覆盖范围最小（几公里内），如校园网、机房。传输速率高、误码率低。
城域网（MAN）：覆盖一个城市（几十公里）。
广域网（WAN）：跨地区连接多个局域网或城域网，如运营商骨干网；Internet是全球互联网络，不宜简单等同于某一个WAN。现代数据网络以分组交换为主，传统电话网是电路交换通信网，不应直接作为计算机WAN例子。
<span style="color:#c62828;font-weight:700">🔴【按拓扑结构分类】：总线型、星型、环型、树型、网状型。</span>
<span style="color:#c62828;font-weight:700">🔴【按工作模式分类】：</span>
C/S模式（客户机/服务器）：由专用客户端向服务器请求服务，通常需要安装或部署客户端程序，如邮件客户端、数据库客户端。
B/S模式（浏览器/服务器）：客户端主要通过浏览器访问Web服务器，业务更新通常集中在服务器端，如网页版邮箱、在线教务系统。
⚫️【坑点】：判断C/S或B/S看系统交互架构，不只看产品名称；许多现代应用是Web、原生客户端和云服务混合架构。“使用浏览器访问Web版”通常按B/S作答，使用专用客户端通常按C/S作答。
#### 考点4：计算机网络的组成（逻辑划分）

<span style="color:#c62828;font-weight:700">🔴【从功能上划分】：</span>
资源子网：负责数据处理和资源共享。包括：主机、终端、外设、软件、数据。核心是主机。
通信子网：负责数据传输。包括：通信线路、网络连接设备（路由器、交换机等）、通信协议。核心是通信设备和线路。
⚫️【坑点】：转发器、交换机属于通信子网。主机属于资源子网。判断题说“资源子网负责计算机间的数据通信” →错！那是通信子网的功能！
#### 考点5：网络拓扑结构（对比记忆）

<span style="color:#c62828;font-weight:700">🔴【总线型】：</span>
特点：所有节点连接在一条总线上。结构简单、成本低、易于扩展。
缺点：总线故障导致全网瘫痪，故障检测困难。
代表：早期的以太网。
<span style="color:#c62828;font-weight:700">🔴【星型】：</span>
特点：所有节点通过中心节点（集线器/交换机）连接。结构简单、易维护、故障隔离容易。
缺点：中心节点故障导致全网瘫痪，电缆成本高。
代表：家用路由器网络、绝大多数现代局域网。
<span style="color:#c62828;font-weight:700">🔴【环型】：</span>
特点：节点首尾相连形成闭合环。实时性好。
缺点：可扩充性差，局部节点故障可能导致全网瘫痪，故障检测困难。
<span style="color:#c62828;font-weight:700">🔴【网状型】：</span>
特点：节点之间有多条路径相连。可靠性高，路径选择灵活。
缺点：线路成本高、不易维护。
代表：Internet的拓扑结构。
⚫️【坑点】：一旦中心节点出现故障则整个网络瘫痪的是星型结构。Internet属于网状结构。
### 第二章：网络体系结构与OSI模型

#### 考点6：网络协议

<span style="color:#c62828;font-weight:700">🔴【定义】：网络协议是计算机网络中进行数据交换而建立的规则、标准或约定的集合。是不同结点对等实体之间进行通信的规则。</span>
<span style="color:#c62828;font-weight:700">🔴【三要素】：</span>
语法（Syntax）：规定数据的格式和编码。→ “如何讲”。
语义（Semantics）：规定控制信息的含义和功能。→ “讲什么”。
时序/同步（Timing）：规定事件执行的顺序和速度匹配。→ “何时讲/讲话次序”。
⚫️【坑点】：判断题说“语法确定通信双方讲什么” →错！语法是“如何讲”，语义才是“讲什么”。
#### 考点7：OSI七层模型（绝对高频，滚瓜烂熟）

<span style="color:#c62828;font-weight:700">🔴【记忆口诀】：“物、数、网、传、会、表、应”（由下到上）。</span>
<span style="color:#1565c0;font-weight:700">🔵【各层核心功能与设备】：</span>
⚫️【坑点】：
路由器是网络层设备，交换机是数据链路层设备，集线器是物理层设备。
TCP提供可靠的端到端传输，UDP提供不可靠的端到端传输。
数据链路层的PDU叫帧，网络层常称分组/数据报，TCP的传输层PDU常称报文段，UDP称用户数据报；不能把所有传输层PDU一律写成“报文段”。
#### 考点8：OSI各层特性详解

<span style="color:#c62828;font-weight:700">🔴【物理层四大特性】：</span>
机械特性：接口的形状、尺寸、引脚数。
电气特性：电压范围、信号电平。
功能特性：各引脚的功能定义。
过程特性：事件发生的顺序。
坑点：描述“完成每种功能的事件发生顺序”的是过程特性，不是功能特性！
<span style="color:#c62828;font-weight:700">🔴【数据链路层】：在相邻节点之间以帧为单位传输，完成成帧、介质访问控制和差错检测等；是否提供确认与重传取决于具体链路协议，不能笼统说一定“无差错”。以太网MAC地址通常为48位，用十六进制表示。</span>
<span style="color:#c62828;font-weight:700">🔴【网络层】：提供逻辑寻址和路由选择，使分组跨网络到达目的主机。IP、ICMP属于网际层/网络层；ARP用于同一链路上把IPv4地址解析为链路层地址，在不同分层教材中可归到网络接口层或网络层边界，答题按教材口径。</span>
⚫️【坑点】：以太网物理地址（MAC地址）长度是48位，不是32位。IPv4地址是32位，IPv6地址是128位。
### 第三章：TCP/IP协议族

#### 考点9：TCP/IP参考模型

<span style="color:#c62828;font-weight:700">🔴【层次结构（4层）】：</span>
应用层（对应OSI的应用层+表示层+会话层）
传输层（对应OSI的传输层）
网络层/网际层（对应OSI的网络层）
网络接口层（对应OSI的物理层+数据链路层）
⚫️【坑点】：OSI是7层，TCP/IP是4层。TCP/IP的网络接口层对应OSI的物理层和数据链路层。
#### 考点10：IP协议（网络层核心）

<span style="color:#c62828;font-weight:700">🔴【IP协议功能】：无连接、不可靠的数据报传输。负责对数据包进行寻址和路由选择。</span>
⚫️【坑点】：IP协议是不可靠的，它只负责尽力而为地传输，不保证数据包不丢失、不乱序。可靠性由TCP在传输层保证。
#### 考点11：ICMP协议

<span style="color:#c62828;font-weight:700">🔴【全称】：Internet Control Message Protocol，网际控制报文协议。</span>
<span style="color:#c62828;font-weight:700">🔴【功能】：传递差错报告和控制信息。用于网络诊断。</span>
<span style="color:#c62828;font-weight:700">🔴【Ping命令】：基于ICMP协议，用于测试两台主机之间的连通性。</span>
⚫️【坑点】：Ping命令使用的是ICMP协议，不是TCP或UDP。ICMP属于网络层协议。
#### 考点12：ARP与RARP协议

<span style="color:#c62828;font-weight:700">🔴【ARP（地址解析协议）】：将主机的IP地址解析为对应的MAC地址（物理地址）。</span>
<span style="color:#c62828;font-weight:700">🔴【RARP（逆地址解析协议）】：早期用于主机依据MAC地址获取IPv4地址，现已基本被BOOTP/DHCP取代。</span>
⚫️【坑点】：DNS主要完成域名与资源记录的查询（常见为域名→IP），ARP是在本地链路解析“下一跳IPv4地址→MAC地址”；RARP是早期MAC→IP机制。ARP不能跨路由器解析远端主机的MAC。
#### 考点13：传输层协议（TCP vs UDP）

<span style="color:#c62828;font-weight:700">🔴【TCP（传输控制协议）】：</span>
面向连接（需建立连接，三次握手）。
可靠传输（有确认、重传机制）。
提供流量控制和拥塞控制。
传统协议栈中FTP、HTTP/1.1和HTTP/2、SMTP通常使用TCP；HTTP/3基于QUIC（运行在UDP之上），所以不要把“HTTP永远使用TCP”写成绝对结论。
核心功能：完成对数据包的确认、流量控制和网络拥塞；自动检测数据包并提供错误重发功能。
<span style="color:#c62828;font-weight:700">🔴【UDP（用户数据报协议）】：</span>
无连接（无需建立连接）。
不可靠传输（不保证到达，不保证顺序）。
传输效率高，开销小。
适用于：视频会议、IP电话、在线直播、DNS查询。
⚫️【坑点】：
TCP是面向连接的，UDP是无连接的。
视频会议、IP电话使用UDP，因为实时性要求高，可以容忍少量丢包。
TCP和UDP都属于传输层协议。
说“UDP比TCP更可靠” →错！TCP可靠，UDP不可靠。
#### 考点14：应用层常用协议（端口号必记）

<span style="color:#c62828;font-weight:700">🔴【HTTP】：超文本传输协议，端口80。用于浏览器与Web服务器之间的通信。</span>
<span style="color:#c62828;font-weight:700">🔴【HTTPS】：HTTP通过TLS建立受保护的通信，默认端口443，可提供机密性、完整性和服务器身份认证；安全性仍依赖证书验证、TLS配置和应用本身。</span>
<span style="color:#c62828;font-weight:700">🔴【FTP】：文件传输协议，控制连接默认端口21；主动模式的数据连接常由服务器端口20发起，被动模式的数据端口由协商决定。FTP明文传输，安全传输要区分FTPS和SFTP（SFTP基于SSH，并非FTP加密版）。</span>
<span style="color:#c62828;font-weight:700">🔴【DNS】：域名系统，默认端口53，常用UDP，响应较大、区域传送等情况可用TCP。它查询多种资源记录，最常见功能是域名解析为IP地址。</span>
<span style="color:#c62828;font-weight:700">🔴【SMTP】：简单邮件传输协议，端口25。用于发送电子邮件。</span>
<span style="color:#c62828;font-weight:700">🔴【POP3】：邮局协议第3版，端口110。用于接收电子邮件。</span>
<span style="color:#c62828;font-weight:700">🔴【Telnet】：远程登录协议，端口23。</span>
⚫️【坑点】：
SMTP是发邮件，POP3是收邮件，不要记反！
SMTP实现的是将邮件从发件人发送到收件人的邮件服务器，不是直接发送到收件人的电脑。
HTTP是应用层协议，不是传输层协议。
FTP的21是控制端口；“数据端口固定为20”只适用于经典主动模式，不能套用于被动模式。
#### 考点15：常用命令

<span style="color:#c62828;font-weight:700">🔴【Ping】：测试两台主机之间的连通性。基于ICMP协议。</span>
<span style="color:#c62828;font-weight:700">🔴【Ipconfig】：查看本机的IP地址、子网掩码、默认网关等网络配置信息。</span>
<span style="color:#c62828;font-weight:700">🔴【Tracert】：显示数据包到达目的主机所经过的路径（路由追踪）。</span>
<span style="color:#c62828;font-weight:700">🔴【Netstat】：显示网络连接状态、路由表、接口统计等信息。</span>
⚫️【坑点】：查看本机IP地址的命令是Ipconfig，不是Ping。Ping是测试连通性。
### 第四章：IP地址与子网划分

#### 考点16：IP地址基本概念

<span style="color:#c62828;font-weight:700">🔴【IP地址长度】：32位（二进制），分为4组，每组8位，用“点分十进制”表示。</span>
<span style="color:#c62828;font-weight:700">🔴【IP地址组成】：网络号（Net-ID）+ 主机号（Host-ID）。</span>
<span style="color:#c62828;font-weight:700">🔴【历史分类编址】：IPv4曾按A、B、C、D、E分类，D类用于组播，E类保留。现代互联网地址分配和路由采用CIDR前缀，不再以A/B/C类作为实际划分基础；分类地址仍是基础考试常考口径。</span>
⚫️【坑点】：D类地址用于组播（224.0.0.0-239.255.255.255），不是用于普通主机通信。
#### 考点17：A、B、C类地址范围（必须烂熟于心）

<span style="color:#c62828;font-weight:700">🔴【A类】：1 ~ 126（0和127保留）。默认子网掩码：255.0.0.0。网络号占1字节，主机号占3字节。每个网络可容纳约1677万台主机。</span>
范围：1.0.0.0 ~ 126.255.255.255
<span style="color:#c62828;font-weight:700">🔴【B类】：128 ~ 191。默认子网掩码：255.255.0.0。网络号占2字节，主机号占2字节。每个网络可容纳65534台主机。</span>
范围：128.0.0.0 ~ 191.255.255.255
<span style="color:#c62828;font-weight:700">🔴【C类】：192 ~ 223。默认子网掩码：255.255.255.0。网络号占3字节，主机号占1字节。每个网络可容纳254台主机。</span>
范围：192.0.0.0 ~ 223.255.255.255
⚫️【坑点】：
判断IP地址类别，只看第一个十进制数！
192.168.0.12 → 第一个数是192，属于C类地址。
127.0.0.0/8保留为回送地址块，127.0.0.1是最常用回送地址。按首位模式它落在历史A类范围，但不能分配给普通网络接口或作为公网主机地址；“完全不属于A类”并不严谨。
C类地址中，主机号全0表示网络号，全1表示广播地址，都不能分配给主机。所以C类每个网络最多254台主机（2⁸-2=254）。
#### 考点18：特殊IP地址

<span style="color:#c62828;font-weight:700">🔴【127.0.0.1】：本机回送地址，用于主机自我测试。不能作为互联网上的有效IP地址。</span>
<span style="color:#c62828;font-weight:700">🔴【主机号全0】：代表本网络（网络地址）。如：192.168.1.0。</span>
<span style="color:#c62828;font-weight:700">🔴【主机号全1】：代表广播地址。如：192.168.1.255。</span>
<span style="color:#c62828;font-weight:700">🔴【0.0.0.0】：作为主机地址时常表示“未指定地址/本主机尚无地址”；路由表中的0.0.0.0/0才表示默认路由，二者不能混为一个含义。</span>
⚫️【坑点】：不能作为互联网可用的IP地址的是127.0.0.1（回送地址）。192.168.1.0是网络地址，不能分配给主机。
#### 考点19：子网掩码

<span style="color:#c62828;font-weight:700">🔴【定义】：用于区分IP地址中的网络号和主机号。1对应网络位，0对应主机位。</span>
<span style="color:#c62828;font-weight:700">🔴【默认子网掩码】：</span>
A类：255.0.0.0
B类：255.255.0.0
C类：255.255.255.0
<span style="color:#c62828;font-weight:700">🔴【子网划分的目的】：把一个地址前缀划成多个更小的广播域，便于分层管理、隔离广播和更合理地分配地址；提高地址利用率只是常见收益之一。</span>
<span style="color:#1565c0;font-weight:700">🔵【判断同网段】：两台主机的IP地址分别与子网掩码进行“与（AND）”运算，结果相同则在同一子网，可直接通信；结果不同则不能直接通信，需要通过路由器。</span>
⚫️【坑点】：
子网掩码必须是连续的1后面跟连续的0。255.241.0.0不可能是子网掩码（241转二进制11110001，不连续）。
子网掩码不能单独存在，必须结合IP地址一起使用。
主机A的IP为202.96.128.130，子网掩码255.255.255.128 → 网络号为202.96.128.128。
#### 考点20：IPv6

<span style="color:#c62828;font-weight:700">🔴【长度】：128位，分为8组，每组16位，用十六进制表示。</span>
<span style="color:#c62828;font-weight:700">🔴【地址总数】：2¹²⁸个。</span>
<span style="color:#c62828;font-weight:700">🔴【表示方式】：用冒号分隔的十六进制数，如：2001:0db8:85a3:0000:0000:8a2e:0370:7334。连续零可压缩为::。</span>
<span style="color:#c62828;font-weight:700">🔴【IPv6出现的原因】：解决IPv4地址枯竭问题。</span>
⚫️【压缩规则】：每组可省略前导0；一段或多段连续的全0组可用“::”压缩，但一个地址中“::”最多出现一次。IPv6没有广播，使用组播和任播等方式。
⚫️【坑点】：IPv6由16个字节（128位）表示，不是16位或8个字节。IPv4由4个字节（32位）表示。
#### 考点20A：CIDR与子网计算（岗位考试高频）

<span style="color:#c62828;font-weight:700">🔴【前缀长度】：/n表示前n位是网络前缀。例如255.255.255.192的二进制有26个连续1，因此等价于/26。</span>
<span style="color:#1565c0;font-weight:700">🔵【地址总数】：一个IPv4 /n地址块共有2^(32-n)个地址。传统普通子网题中可分配主机数通常按2^(32-n)-2计算，扣除网络地址和定向广播地址；/31点到点链路、/32主机路由是特殊情形。</span>
<span style="color:#1565c0;font-weight:700">🔵【求网络地址】：IP地址与子网掩码逐位AND；广播地址是在该网络地址基础上把主机位全置1。</span>
⚫️【例】：192.168.1.130/26中，每段块大小为64，130落在128～191，因此网络地址192.168.1.128，广播地址192.168.1.191，传统可用主机范围129～190。
#### 考点20B：DHCP与NAT（常考功能配对）

<span style="color:#c62828;font-weight:700">🔴【DHCP】：动态主机配置协议，可向客户端分配IP地址、子网掩码、默认网关、DNS服务器等参数；典型IPv4过程为Discover、Offer、Request、ACK（DORA），使用UDP 67/68端口。</span>
<span style="color:#c62828;font-weight:700">🔴【NAT】：在网络边界转换内部地址和外部地址；NAPT/PAT还利用端口号让多台内网主机共享少量公网IPv4地址。</span>
⚫️【坑点】：DHCP负责“自动配置地址参数”，DNS负责“名称解析”，ARP负责“同链路IPv4到MAC解析”，NAT负责“地址/端口转换”，四者不可互换。
### 第五章：网络设备与传输介质

#### 考点21：网络互联设备（工作层级对比）

<span style="color:#c62828;font-weight:700">🔴【中继器（Repeater）】：物理层。放大信号，延长传输距离。不能识别数据内容。</span>
<span style="color:#c62828;font-weight:700">🔴【集线器（Hub）】：物理层。多端口的中继器，将信号广播到所有端口。共享式设备，同一时刻只能一台设备发送数据。不能识别MAC地址。</span>
<span style="color:#c62828;font-weight:700">🔴【网桥（Bridge）】：数据链路层。连接两个同类型局域网，能识别MAC地址，可以隔离冲突域。</span>
<span style="color:#c62828;font-weight:700">🔴【交换机（Switch）】：数据链路层。多端口的网桥，能识别MAC地址，根据MAC地址表定向转发。隔离冲突域，但不能隔离广播域。</span>
<span style="color:#c62828;font-weight:700">🔴【路由器（Router）】：网络层。连接不同网络，能识别IP地址，进行路由选择。隔离冲突域和广播域。</span>
<span style="color:#c62828;font-weight:700">🔴【网关（Gateway）】：广义上指连接不同网络、承担转发或协议转换的实体；“默认网关”通常就是本子网通向其他网络的路由器接口。旧式OSI教材常把执行高层协议转换的网关归于高层，但不能据此断言所有网关都只工作在传输层以上。</span>
⚫️【坑点】：
中继器和集线器属于物理层设备。
交换机和网桥属于数据链路层设备。
路由器属于网络层设备。
集线器（Hub）不能识别目的地址，交换机可以识别MAC地址。
路由器可以实现有线网络和无线网络的连接。
#### 考点22：传输介质

<span style="color:#c62828;font-weight:700">🔴【有线传输介质】：</span>
双绞线：最常用、最便宜的传输介质。传输距离最长100米。分为非屏蔽（UTP）和屏蔽（STP）。
同轴电缆：抗干扰能力强。粗缆传输距离500米，细缆185米。
光纤：带宽高、传输距离远、抗电磁干扰、误码率低，常用于骨干和高速链路；接入终端和局域网中双绞线同样广泛，不能说光纤在所有场景“应用最广泛”。成本应综合光纤、光模块、施工和维护，不能只写“价格一定贵”。
<span style="color:#c62828;font-weight:700">🔴【无线传输介质/信号】：常见电磁波形式包括无线电波、地面/卫星微波、红外线和激光；蓝牙是使用2.4GHz频段的短距离无线通信技术/协议，不是独立的物理介质。</span>
微波通信：卫星通信是微波接力通信向空间的扩展。
蓝牙：短距离数据交换（10米左右），功耗低。
红外线：短距离，需要直线传输。
⚫️【坑点】：
“性能最好”需说明比较维度；若题目强调带宽、距离和抗干扰，通常选光纤，若强调以太网水平布线的普及和成本，通常是双绞线。
常见铜缆以太网信道长度上限为100米（典型为90米永久链路+两端跳线），具体仍取决于线缆类别和以太网标准。
粗同轴电缆支持的最大电缆长度是500米，细同轴电缆185米。
“价格便宜”不是光纤相对于双绞线的固有技术优势；成本比较要结合光模块、施工距离和维护场景。
#### 考点22A：以太网与无线局域网（高频补充）

<span style="color:#c62828;font-weight:700">🔴【以太网】：IEEE 802.3标准，使用MAC地址标识链路接口。传统共享式半双工以太网采用CSMA/CD；现代交换式全双工以太网不存在冲突，不使用CSMA/CD处理冲突。</span>
<span style="color:#c62828;font-weight:700">🔴【无线局域网】：IEEE 802.11系列，常采用CSMA/CA以尽量避免冲突；SSID是无线网络名称，不是密码或MAC地址。</span>
<span style="color:#c62828;font-weight:700">🔴【冲突域与广播域】：交换机每个端口通常形成独立冲突域；同一VLAN仍属于同一广播域。路由器或三层设备可分隔广播域，VLAN也能在二层逻辑划分广播域。</span>
⚫️【坑点】：交换机“不能隔离广播域”只适用于未划分VLAN、同一二层广播域的传统题设；交换机可通过VLAN划分多个广播域。
### 第六章：网络应用与服务

#### 考点23：域名系统（DNS）

<span style="color:#c62828;font-weight:700">🔴【域名结构】：由标号序列组成，从右到左级别升高。如：www.sina.com.cn。</span>
cn：顶级域名（国家域名）
com：二级域名（机构域名，商业机构）
sina：三级域名
www：四级域名（Web服务器）
<span style="color:#c62828;font-weight:700">🔴【常见顶级域名】：</span>
.com：商业机构
.edu：教育机构
.gov：政府机构
.mil：军事机构
.org：非盈利组织
.net：网络机构
<span style="color:#c62828;font-weight:700">🔴【域名解析】：客户端通过递归/迭代查询获得DNS资源记录，常见是将域名解析为IPv4/IPv6地址（A/AAAA记录）；反向解析使用PTR记录。DNS不只保存IP地址。</span>
<span style="color:#c62828;font-weight:700">🔴【负责中国域名管理的机构】：CNNIC（中国互联网络信息中心）。</span>
⚫️【坑点】：
域名不分大小写。
http://www.China.edu.cn中，代表国家的顶级域名是cn，代表教育机构的二级域名是edu。
请求域名解析的客户不需要知道根域名服务器和本地域名服务器父结点的名字。
#### 考点24：电子邮件（E-mail）

<span style="color:#c62828;font-weight:700">🔴【E-mail地址格式】：用户名@域名（如：qwer@163.com，用户名为qwer）。</span>
<span style="color:#c62828;font-weight:700">🔴【发送/转发邮件】：客户端提交邮件以及邮件服务器之间转发通常使用SMTP。</span>
<span style="color:#c62828;font-weight:700">🔴【读取邮件】：客户端可使用POP3下载邮件，也可用IMAP在服务器上同步文件夹和状态；Web邮箱则由浏览器通过HTTP(S)访问前端，服务器内部仍使用邮件协议处理投递。</span>
<span style="color:#c62828;font-weight:700">🔴【邮件发送特点】：</span>
可以同时发送给多个收件人。
发送邮件不需要对方开机。
发送电子邮件时，可以不输入邮件主题（但建议输入）。
E-mail采用存储转发，投递时间一般不由地理距离直接决定，但会受网络、服务器队列、反垃圾策略和故障影响，不能保证“即发即收”。
⚫️【坑点】：
电子邮件地址@之后表示邮件服务器名。
电子邮件不是即时通讯工具（即时通讯如QQ、微信）。
收件箱内带有回形针标志的邮件表示该邮件有附件。
电子邮件服务的功能不包括制作动画。
#### 考点25：WWW（万维网）

<span style="color:#c62828;font-weight:700">🔴【全称】：World Wide Web，全球信息网。</span>
<span style="color:#c62828;font-weight:700">🔴【核心协议】：HTTP（超文本传输协议）。</span>
<span style="color:#c62828;font-weight:700">🔴【URL格式】：协议名://主机名/目录名/文件名</span>
例：http://www.ye21.cn/news/index.aspx→ http（服务标志）、www.ye21.cn（主机域名）、news（目录名）、index.aspx（文件名）。
<span style="color:#c62828;font-weight:700">🔴【收藏夹功能】：保存网页地址（URL），不是保存网页内容。</span>
<span style="color:#c62828;font-weight:700">🔴【Web服务器】：以网页形式存储信息资源的服务器。www通常指Web服务器。</span>
⚫️【坑点】：HTML是超文本标记语言，不是协议。HTTP是协议。
### 第七章：网络安全

#### 考点26：计算机病毒

<span style="color:#c62828;font-weight:700">🔴【定义】：人为编制的一组计算机指令或程序代码，能破坏计算机功能或数据，能自我复制。</span>
<span style="color:#c62828;font-weight:700">🔴【病毒特征】：寄生性、传染性、潜伏性、隐蔽性、可触发性、破坏性、不可预见性。</span>
坑点：病毒没有“智能性”和“免疫性”特征。
<span style="color:#c62828;font-weight:700">🔴【病毒分类】：</span>
引导型病毒：感染主引导记录（MBR）或引导扇区。
文件型病毒：感染可执行文件（.exe、.com）。如：CIH病毒。
宏病毒：感染Office文档（.doc、.xls）。通过文档中的宏传播。
混合型病毒：同时感染引导区和文件。
蠕虫病毒：独立程序，通过网络主动传播。如：冲击波、震荡波。
木马病毒：伪装成正常软件，不自我复制，通过欺骗安装。
⚫️【坑点】：
CIH病毒属于文件型病毒。
宏病毒主要感染.doc和.xls文件。
蠕虫病毒不需要寄生宿主程序，是独立的。
特洛伊木马不具有自我复制能力。
计算机病毒不会传染给人类。
#### 考点27：病毒传播途径

<span style="color:#c62828;font-weight:700">🔴【主要途径】：</span>
移动存储设备（U盘、移动硬盘）。
网络（电子邮件附件、网页链接、下载文件、即时通讯）。
⚫️【坑点】：使用扫描仪不会感染病毒。病毒不能通过硬件设备本身传播。
#### 考点28：网络安全技术

<span style="color:#c62828;font-weight:700">🔴【防火墙】：</span>
功能：隔离内部网络和外部网络，过滤网络通信，防止外部攻击。
类型：硬件防火墙和软件防火墙。
传统边界防火墙难以防住绕过其路径的内部攻击、允许流量中的应用漏洞或终端已感染恶意软件；下一代防火墙可提供应用识别、入侵防御等能力，但仍不能替代终端防护、身份控制和安全管理。
防火墙的核心职责是按安全策略过滤和控制流量，路由器的核心职责是选路和转发；实际安全网关常同时具备路由、NAT、防火墙等功能，不能说二者在设备形态上绝不重合。
防火墙不能过滤掉一切攻击，只能起到防护作用。
<span style="color:#c62828;font-weight:700">🔴【数字签名】：</span>
功能：保证信息的完整性、身份认证、不可抵赖性。
不能防止计算机病毒入侵。
<span style="color:#c62828;font-weight:700">🔴【身份认证技术】：如银行取款输入密码，属于身份识别/身份认证。</span>
<span style="color:#c62828;font-weight:700">🔴【加密方式】：</span>
对称加密：加密和解密使用同一个密钥（速度快，密钥管理困难）。
非对称加密：加密和解密使用一对密钥（公钥和私钥）。如：U盾。工行U盾使用的是非对称加密算法。
⚫️【坑点】：
防火墙设置的位置应该在学校局域网与外网连接处。
防火墙是网络边界防护的重要技术之一；“最基本/唯一”属于旧教材口径，完整防护还需身份认证、补丁、最小权限、加密、备份、监测与响应。
数字签名的主要功能是防止交易中的抵赖行为。
U盾/USB Key常安全保存私钥和数字证书，利用公钥密码完成身份认证或数字签名，也可能配合对称加密等机制；具体实现不能简化为“只使用一种非对称算法”。
### 第八章：网络基础知识补充

#### 考点29：交换技术

<span style="color:#c62828;font-weight:700">🔴【电路交换】：建立物理连接，独占线路（如传统电话）。实时性好，但线路利用率低。</span>
<span style="color:#c62828;font-weight:700">🔴【报文交换】：存储-转发，整个报文一起发送（如老式电报）。延迟大，对节点缓存要求高。</span>
<span style="color:#c62828;font-weight:700">🔴【分组交换】：采用存储-转发，把报文拆成分组并可统计复用链路（Internet采用）。链路利用率通常较高，但存在排队时延、丢包和时延抖动；不能无条件说“延迟一定最小”或“利用率永远最高”。</span>
⚫️【坑点】：目前广域网主要采用分组交换技术，不是电路交换。
#### 考点30：数据通信基础

<span style="color:#c62828;font-weight:700">🔴【调制/解调】：</span>
调制：数字信号 → 模拟信号（发送端）。
解调：模拟信号 → 数字信号（接收端）。
电话拨号上网需要调制解调器（Modem）。
<span style="color:#c62828;font-weight:700">🔴【数据传输方式】：</span>
单工：单向传输（如：电视广播）。
半双工：双向交替传输（如：对讲机）。
全双工：双向同时传输（如：电话）。
<span style="color:#c62828;font-weight:700">🔴【波特率 vs 比特率】：</span>
波特率：每秒传输的码元数（码元速率，单位Bd），不等同于“波形变化次数”。
比特率：每秒钟传输的比特数（数据传输速率）。
<span style="color:#1565c0;font-weight:700">🔵【二者关系】：若每个码元有M种可区分状态且等概率携带log₂M位，则理想情况下比特率 = 波特率 × log₂M；二进制调制时二者数值可能相等。</span>
⚫️【坑点】：波特率不是每秒传输的比特数，也不应简单定义为“信号变化次数”。
#### 考点31：Internet基础知识

<span style="color:#c62828;font-weight:700">🔴【Internet雏形】：ARPANET（阿帕网），美国国防部1969年创建。</span>
<span style="color:#c62828;font-weight:700">🔴【TCP/IP协议】：Internet采用的核心协议。</span>
<span style="color:#c62828;font-weight:700">🔴【ISP（互联网服务提供商）】：提供Internet接入服务的公司（如中国电信、中国联通）。个人用户注册后，ISP提供上网账号、上网口令和DNS地址。</span>
⚫️【坑点】：Internet的最早雏形是ARPANET，不是NSFNET或CERNET。

## 软件工程

### 第一章：软件工程概述

#### 考点1：软件的定义与特点

<span style="color:#c62828;font-weight:700">🔴【软件的定义】：软件 =程序 + 数据 + 相关文档的完整集合。</span>
<span style="color:#c62828;font-weight:700">🔴【软件的特点】：</span>
软件是一种逻辑产品，不是物理实体。
软件的开发主要是研制，不是生产制造。
软件不会像硬件那样因物理磨损失效，但持续修改可能引入缺陷、破坏结构，使其出现“退化”；运行环境变化也会导致原本正确的软件不再适用。
软件的开发和运行受计算机系统限制。
⚫️【坑点】：
程序不等于软件（软件还包括数据和文档）。
软件既是逻辑实体，不是物理实体。
软件安装的越多，计算机性能不一定越好（过多软件会占用系统资源）。
“裸机”是指不装备任何软件的计算机。
#### 考点2：软件危机（Software Crisis）

<span style="color:#c62828;font-weight:700">🔴【定义】：在软件开发和维护过程中遇到的一系列严重问题。</span>
<span style="color:#c62828;font-weight:700">🔴【主要表现】：</span>
进度滞后：软件开发进度难以预测。
成本超支：软件开发成本难以控制。
质量不可控：软件产品质量无法保证。
维护困难：软件产品难以维护。
文档缺乏：缺少适当的文档资料。
<span style="color:#c62828;font-weight:700">🔴【产生原因】：</span>
用户需求描述不准确。
缺乏有力的组织和管理。
缺乏有力的方法学和工具支持。
软件规模越来越大，复杂度越来越高。
⚫️【坑点】：
软件危机不是“电脑坏了”或“硬件故障”，而是软件开发生产力跟不上硬件发展。
软件危机的表现不包括“软件依赖于硬件”（那是软件的特性之一）。
#### 考点3：软件工程（Software Engineering）

<span style="color:#c62828;font-weight:700">🔴【定义】：采用工程的概念、原理、技术和方法来开发和维护软件，以经济地开发出高质量的软件并有效地维护它。</span>
<span style="color:#c62828;font-weight:700">🔴【软件工程三要素】：方法（技术手段）+工具（自动化支撑）+过程（质量管理的纽带）。（口诀：方工过）</span>
<span style="color:#c62828;font-weight:700">🔴【软件工程的目标】：在给定成本和进度的前提下，开发出具有适用性、有效性、可修改性、可靠性、可理解性、可维护性、可重用性、可移植性、可追踪性、可互操作性的软件产品。</span>
⚫️【坑点】：软件工程的核心是管理复杂性，不是单纯地写代码。
### 第二章：软件生命周期

#### 考点4：软件生命周期（SDLC）

<span style="color:#c62828;font-weight:700">🔴【定义】：软件从产生到报废的整个周期。</span>
<span style="color:#c62828;font-weight:700">🔴【常见教材口径】：定义期（问题定义、可行性研究、需求分析）、开发期（概要设计、详细设计、编码、测试）、运行维护期。生命周期阶段的名称和划分会因过程标准与模型不同而变化，并非只能固定为“三期八阶段”。</span>
⚫️【坑点】：
维护通常持续时间很长、累计成本占比较高，但具体比例随系统寿命、类型和统计口径而变，不能把60%～70%当作通用定律。
性能优化可能发生在设计、编码、测试和维护中，是软件工程活动的一部分；优化前应有可测量目标并避免牺牲正确性和可维护性。
详细设计属于软件开发期，不属于定义期或维护期。
### 第三章：软件开发模型（极高频）

#### 考点5：瀑布模型（Waterfall Model）

<span style="color:#c62828;font-weight:700">🔴【核心特征】：阶段化、文档驱动，阶段成果经评审后进入后续阶段；经典瀑布强调顺序和基线控制，但实际可有反馈与返工，并非物理上绝对不能回退。</span>
<span style="color:#c62828;font-weight:700">🔴【优点】：文档驱动，每个阶段都有明确的产出，易于管理和审查。</span>
<span style="color:#c62828;font-weight:700">🔴【缺点】：需求变更极难，必须等全部做完才能看到效果。不适合需求不明确的项目。</span>
<span style="color:#c62828;font-weight:700">🔴【适用场景】：需求较明确稳定、技术成熟、强调阶段审查和可追踪文档的项目。军工、银行等项目是否采用瀑布不能仅凭行业名称判断。</span>
⚫️【坑点】：瀑布模型的最大优点是将软件开发各个阶段划分得十分清晰。
#### 考点6：原型模型（Prototyping Model）

<span style="color:#c62828;font-weight:700">🔴【核心特征】：快速构建原型以澄清需求、验证方案并获取反馈。抛弃式原型在需求明确后废弃；演化式原型在工程控制下逐步演化为产品，不能混为“边看边改直到交付”。</span>
<span style="color:#c62828;font-weight:700">🔴【优点】：能有效解决需求模糊的问题，降低风险，促进用户与开发人员的沟通。</span>
<span style="color:#c62828;font-weight:700">🔴【缺点】：若不加控制，容易导致设计草率、结构混乱，原型可能被直接当作最终产品交付。</span>
<span style="color:#c62828;font-weight:700">🔴【适用场景】：需求不明确、用户交互复杂的系统（如：UI界面设计、新概念验证）。</span>
⚫️【坑点】：快速原型模型可以有效地适应用户需求的动态变化。
#### 考点7：增量模型（Incremental Model）

<span style="color:#c62828;font-weight:700">🔴【核心特征】：分批次交付，每次交付一个可运行的子集（增量），逐步完善。</span>
<span style="color:#c62828;font-weight:700">🔴【优点】：用户可以尽早拿到核心功能，分散风险，先交付核心功能。</span>
<span style="color:#c62828;font-weight:700">🔴【适用场景】：大型项目，且核心功能需尽快上线的场景。</span>
#### 考点8：螺旋模型（Spiral Model）

<span style="color:#c62828;font-weight:700">🔴【核心特征】：迭代演化并由风险驱动，每轮通常经历目标确定、风险分析与化解、开发验证、下一轮计划。它综合了原型和系统化开发思想，突出风险分析。</span>
<span style="color:#c62828;font-weight:700">🔴【优点】：强调风险规避，适合大型高风险项目。</span>
<span style="color:#c62828;font-weight:700">🔴【缺点】：过分依赖风险评估经验，成本较高，对风险分析人员要求高。</span>
<span style="color:#c62828;font-weight:700">🔴【适用场景】：内部开发的大型、高风险、复杂项目。</span>
⚫️【坑点】：风险分析是螺旋模型的标志性活动，但其他过程也可以做风险管理，不能理解成“只有螺旋模型才允许分析风险”。
#### 考点9：敏捷开发（Agile Development）

<span style="color:#c62828;font-weight:700">🔴【核心特征】：迭代增量交付、快速反馈、响应变化、重视人员协作和可工作的软件。敏捷强调“够用且有价值的文档”，不是不要文档。</span>
<span style="color:#c62828;font-weight:700">🔴【代表性方法】：</span>
极限编程（XP）：测试驱动开发、结对编程、持续集成。
Scrum：冲刺（Sprint）、每日站会、产品待办列表。
<span style="color:#c62828;font-weight:700">🔴【优点】：响应变化快，客户满意度高，交付迅速。</span>
<span style="color:#c62828;font-weight:700">🔴【局限】：对协作、持续反馈和工程实践要求较高；规模扩大时还要处理跨团队协调、架构治理和合规文档。大型项目也可采用规模化敏捷或混合过程，不能笼统说“不适合”。</span>
<span style="color:#c62828;font-weight:700">🔴【适用场景】：需求快速变化的互联网产品（如：APP开发、网站开发）。</span>
⚫️【坑点】：敏捷开发强调**“拥抱变化”**，不是拒绝变化。

### 第四章：需求分析

#### 考点11：需求分析的任务与产出

<span style="color:#c62828;font-weight:700">🔴【核心任务】：准确回答系统**“必须做什么”**（WHAT），而不是“怎样做”（HOW）。</span>
<span style="color:#c62828;font-weight:700">🔴【主要工作】：</span>
问题识别：明确功能需求、性能需求、环境需求、用户界面需求。
分析与综合：导出软件的逻辑模型。
编写文档：产出需求规格说明书（SRS）。
<span style="color:#c62828;font-weight:700">🔴【最终产物】：需求规格说明书（SRS）。</span>
⚫️【坑点】：需求分析不仅确定功能，还要明确性能、安全、接口、数据、约束和验收标准。需求应正确、完整、一致、可验证、可追踪，并与利益相关者确认。
#### 考点12：需求的分类（极高频判断/选择）

<span style="color:#c62828;font-weight:700">🔴【功能需求】：系统必须做什么（具体的业务功能）。</span>
例：“用户输入账号密码，系统验证登录。”
例：“系统能够完成数据的增删改查。”
<span style="color:#c62828;font-weight:700">🔴【非功能需求】：系统做得好不好（质量属性、性能、安全、可靠性、可维护性）。</span>
例：“系统支持500人并发访问。”
例：“页面响应时间小于3秒。”
例：“数据必须加密传输。”
<span style="color:#c62828;font-weight:700">🔴【约束需求】：开发过程中的限制条件（外部硬性规定）。</span>
例：“必须使用MySQL数据库。”
例：“开发语言必须是Java。”
例：“必须在Windows环境下运行。”
⚫️【坑点】：
看到**“必须使用XX语言/平台/工具”** → 选约束需求，不是功能需求！
看到**“响应时间/吞吐量/并发数”** → 选非功能需求！
“系统需要支持1000个用户同时在线查询” →非功能需求（性能指标）。
#### 考点13：需求获取方式（简答题）

<span style="color:#1565c0;font-weight:700">🔵【常用方式】：</span>
问卷调查法：向用户发放调查问卷收集需求。
会议讨论法：开发方和用户方召开需求讨论会议。
界面原型法：通过“界面原型”这一载体与用户沟通需求。
可运行原型系统法：在以往类似项目基础上修改得到可运行系统，让用户试用以挖掘需求。
访谈与现场观察：通过访谈利益相关者、观察真实业务流程识别显性与隐性需求。
文档分析与工作坊：分析现有规章、表单和系统资料，组织JAD/需求工作坊协调冲突并形成共识。
### 第五章：软件设计

#### 考点14：模块设计原则（极高频）

<span style="color:#c62828;font-weight:700">🔴【核心原则】：“高内聚、低耦合”。</span>
<span style="color:#c62828;font-weight:700">🔴【内聚（Cohesion）】：一个模块内部各元素之间结合的紧密程度。内聚越高越好（模块内部团结一致）。</span>
内聚强度从低到高：偶然内聚 < 逻辑内聚 < 时间内聚 < 过程内聚 < 通信内聚 < 顺序内聚 <功能内聚（最强）。
<span style="color:#c62828;font-weight:700">🔴【耦合（Coupling）】：不同模块之间相互依赖的程度。耦合越低越好（模块之间独立性强）。</span>
耦合强度从低到高：非直接耦合 <数据耦合< 标记耦合 < 控制耦合 < 公共耦合 < 内容耦合（最强）。
降低耦合不仅是“传得少”，还要让接口简单稳定、只传必要数据，避免共享全局数据、控制标志和直接访问另一模块内部实现。
⚫️【坑点】：
数据耦合、标记耦合、控制耦合、公共耦合的耦合性从低到高：数据 < 标记 < 控制 < 公共。
要减少模块间的耦合，应减少直接传递的信息量。
模块设计应追求低耦合、高内聚。
#### 考点15：概要设计（总体设计）

<span style="color:#c62828;font-weight:700">🔴【主要任务】：</span>
确定软件的体系结构（模块结构）。
模块划分。
确定全局数据结构。
定义模块之间的接口。
<span style="color:#c62828;font-weight:700">🔴【产出文档】：模块结构图和模块说明书。</span>
#### 考点16：详细设计

<span style="color:#c62828;font-weight:700">🔴【主要任务】：确定每个模块的算法和使用的数据结构。</span>
<span style="color:#c62828;font-weight:700">🔴【常用工具】：程序流程图（PFD）、N-S图（盒图）、PAD图、伪代码。</span>
<span style="color:#c62828;font-weight:700">🔴【程序流程图中的箭头】：代表控制流（不是数据流！）。</span>
⚫️【坑点】：程序流程图中的箭头代表控制流，系统流程图中的箭头代表数据流。
### 第六章：软件测试（极高频，题型多样）

#### 考点17：测试的目的

<span style="color:#c62828;font-weight:700">🔴【测试目的】：发现软件中的错误，不是为了证明软件正确。</span>
<span style="color:#c62828;font-weight:700">🔴【经典名言】：测试是为了发现错误而执行程序的过程。好的测试用例是能发现迄今为止尚未发现的错误的用例。</span>
⚫️【坑点】：软件测试的目的是发现软件的错误，不是“找出软件的所有错误”或“证明软件是正确的”（后者不可能实现）。
#### 考点18：黑盒测试（Black-box Testing）

<span style="color:#c62828;font-weight:700">🔴【定义】：不看内部代码，只关注输入和输出（把程序当黑盒子）。又称功能测试。</span>
<span style="color:#c62828;font-weight:700">🔴【常用方法】：</span>
等价类划分：将输入数据划分为若干有效等价类和无效等价类，从每类中选一个代表测试。
边界值分析：重点测试边界及其邻近值（如整数范围1～100可测0、1、2、99、100、101）。它对边界缺陷很有效，但不存在对所有缺陷都“能力最强”的单一黑盒方法。
错误猜测法：根据经验猜测可能出错的情况。
因果图：描述多种条件组合的测试用例。
⚫️【坑点】：
黑盒测试的测试用例不是根据程序内部逻辑设计的（那是白盒测试）。
边界值分析尤其擅长发现边界处理错误，应与等价类、判定表、状态迁移等方法组合，不能绝对称为任何场景都最强。
等价类划分法不是白盒测试技术，是黑盒测试技术。
常用的黑盒测试方法包括：等价类划分、边界值分析、错误猜测、因果图。（α测试和β测试是测试阶段，不是测试方法）
#### 考点19：白盒测试（White-box Testing）

<span style="color:#c62828;font-weight:700">🔴【定义】：看内部逻辑结构，检查代码路径是否走通。又称结构测试。</span>
<span style="color:#c62828;font-weight:700">🔴【常见覆盖标准】：</span>
语句覆盖（最弱）：每条可执行语句至少执行一次。
判定覆盖（分支覆盖）：每个判断的真/假分支至少执行一次。
条件覆盖：每个判断中每个条件的真/假至少出现一次。
判定/条件覆盖：同时满足判定覆盖和条件覆盖。
条件组合覆盖：每个判断中所有条件的组合至少出现一次。
路径覆盖（最强）：程序中所有可能的执行路径都至少走一遍。
⚫️【坑点】：
判定覆盖通常包含语句覆盖；条件覆盖与判定覆盖一般互不蕴含，所以不能简单写成“语句<判定<条件<路径”的全序。判定/条件覆盖同时满足二者，条件组合覆盖更强。
条件覆盖不一定包含判定覆盖！（极高频坑点：两个独立条件都出现了，但整体判定可能没走全）。
有限、无环控制流中覆盖所有可行路径是很强的标准；存在循环时路径可能无限，实际通常覆盖选定的独立路径或限定循环次数。结构覆盖率高也不能证明需求和业务逻辑正确。
#### 考点20：测试阶段（顺序必记）

<span style="color:#c62828;font-weight:700">🔴【测试顺序】：单元测试 → 集成测试 → 系统测试 → 验收测试。</span>
<span style="color:#c62828;font-weight:700">🔴【单元测试】：测试最小模块（如函数、类、方法）。用例主要依据详细设计设计。</span>
<span style="color:#c62828;font-weight:700">🔴【集成测试（组装测试）】：逐步组合模块，重点检查接口、数据传递、调用关系和协作行为；可发现设计、实现及环境配置等多类缺陷，不能只说发现设计阶段错误。</span>
<span style="color:#c62828;font-weight:700">🔴【系统测试】：测试整个系统的功能和性能是否满足需求。</span>
<span style="color:#c62828;font-weight:700">🔴【验收测试】：用户确认系统是否满足需求。</span>
Alpha测试：通常在开发组织可控环境中，由内部人员或受邀用户测试，开发方可观察和支持。
Beta测试：发布给有限外部用户在真实或接近真实环境中试用并反馈。两者定义会因组织发布流程略有差异。
⚫️【坑点】：
集成测试测试的是模块接口，不是内部功能。
单元测试的用例依据详细设计设计，不是需求分析。
集合测试（集成测试）的主要目的是发现设计阶段的错误。
### 第七章：软件维护

#### 考点21：软件维护的类型（简答题必背）

<span style="color:#c62828;font-weight:700">🔴【维护特点】：维护通常贯穿软件交付后的较长时期，累计投入常很高；是否“花费最多”取决于系统寿命与统计口径。</span>
<span style="color:#1565c0;font-weight:700">🔵【四大维护类型】：</span>
改正性维护：识别和纠正已发现的错误（修复缺陷）。
适应性维护：使软件适应外部环境变化（平台、法规、接口等）。
完善性维护：根据新需求扩充功能、改善性能或可用性。
预防性维护：改善内部结构、可维护性或可靠性，以降低未来故障和修改风险。
⚫️【坑点】：
许多经典统计中完善性维护占比较大，但比例随项目和口径显著变化，题目未给教材数据时不应死记20/25/50/5。
改正性维护是修复Bug，适应性维护是适应外部环境变化。
### 第八章：软件项目管理

#### 考点21A：验证、确认与评审（高频辨析）

<span style="color:#c62828;font-weight:700">🔴【Verification，验证】：检查工作产品是否符合上一阶段规定和规格，常概括为“是否正确地构造产品”。评审、检查、静态分析不要求运行程序，也属于质量保证手段。</span>
<span style="color:#c62828;font-weight:700">🔴【Validation，确认】：检查最终系统是否满足用户真实需要和预期用途，常概括为“是否构造了正确的产品”。验收测试是重要确认活动。</span>
⚫️【坑点】：软件测试通常需要运行被测程序，静态评审不运行程序；二者都能发现缺陷，不能把“测试”与“评审”混为一种方法。
#### 考点21B：UML常用图（岗位考试高频）

<span style="color:#c62828;font-weight:700">🔴【用例图】：描述参与者与系统功能需求的关系，常用于需求分析；参与者是角色，不一定等同某个具体人。</span>
<span style="color:#c62828;font-weight:700">🔴【类图】：描述类、属性、操作及关联、聚合、组合、泛化等静态关系。</span>
<span style="color:#c62828;font-weight:700">🔴【顺序图】：按时间顺序描述对象之间的消息交互；【活动图】描述业务流程、控制流和并发分支；【状态机图】描述对象状态及事件触发的迁移。</span>
⚫️【坑点】：数据流图DFD是结构化分析工具，不是UML图；程序流程图描述算法控制流，也不等同于UML活动图。
#### 考点22：可行性研究

<span style="color:#c62828;font-weight:700">🔴【目的】：用最小的代价在尽可能短的时间内确定问题是否能够解决。</span>
<span style="color:#c62828;font-weight:700">🔴【四种可行性】：</span>
经济可行性：成本-效益分析。
技术可行性：从技术角度研究实现系统的可行性（对系统所要求的功能、性能及限制条件进行分析）。
操作可行性：系统的运行方式在用户组织中是否可行。
法律可行性：是否触犯法律或有侵权行为。
#### 考点23：软件质量与文档

<span style="color:#c62828;font-weight:700">🔴【软件文档的作用】：不仅是开发各阶段的依据，而且影响软件的可理解性。</span>
<span style="color:#c62828;font-weight:700">🔴【软件配置管理】：</span>
“检出（Check-out）”和“检入（Check-in）”用于受控获取、提交配置项版本，可辅助实现存取控制、版本管理和并行修改协调；现代分布式版本控制并不一定采用独占检出模式。
🔥 本章高频简答题汇总
1. 什么是软件危机？主要表现有哪些？
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：软件危机是在软件开发和维护中遇到的一系列问题。主要表现：进度滞后、成本超支、质量不可控、维护困难、文档缺乏。</span>
2. 简述软件生命周期的三大时期和主要阶段。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：</span>
软件定义期：问题定义、可行性研究、需求分析。
软件开发期：概要设计、详细设计、编码、测试。
软件运行期：维护（花费最多、时间最长）。
3. 比较瀑布模型和原型模型。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：</span>
瀑布：线性顺序、需求明确时适用、文档驱动、变更困难。
原型：边看边改、需求模糊时适用、快速构建、容易导致结构混乱。
4. 简述螺旋模型的核心特点。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：螺旋模型 =瀑布模型 + 原型模型 + 风险分析。每轮迭代都包含风险分析是其独有特征，适合大型高风险项目。</span>
5. 区分功能需求、非功能需求和约束需求。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：</span>
功能需求：系统必须做什么（具体功能）。
非功能需求：系统做得好不好（性能、安全、可靠性等）。
约束需求：开发的外部限制条件（必须使用XX语言/平台）。
6. 简述黑盒测试和白盒测试的区别。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：</span>
黑盒测试：不看内部代码，只测输入输出（功能测试）。方法：等价类划分、边界值分析。
白盒测试：看内部逻辑结构（结构测试）。方法：语句覆盖、判定覆盖、条件覆盖、路径覆盖。
7. 简述软件测试的四个阶段及其任务。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：</span>
单元测试：测试每个模块内部。
集成测试：测试模块之间的接口。
系统测试：测试整个系统。
验收测试：用户确认是否满足需求（Alpha = 开发环境，Beta = 真实环境）。
8. 简述软件维护的四种类型。
<span style="color:#1565c0;font-weight:700">🔵【踩分点】：</span>
改正性维护：修复已发现缺陷。
适应性维护：适应平台、法规、接口等环境变化。
完善性维护：扩充功能、改善性能或可用性。
预防性维护：改善内部质量，降低未来故障和修改风险。

## 信息安全

### 第一章：信息安全目标与风险管理

#### 考点1：安全目标

<span style="color:#c62828;font-weight:700">🔴【CIA 三要素】：保密性（Confidentiality）、完整性（Integrity）、可用性（Availability）是最通用的三项核心目标。</span>

<span style="color:#1565c0;font-weight:700">🔵保密性：信息只向获得授权的主体披露。</span>

<span style="color:#1565c0;font-weight:700">🔵完整性：防止信息被未经授权地修改或破坏，并能发现异常变更。</span>

<span style="color:#1565c0;font-weight:700">🔵可用性：授权主体在需要时能够访问和使用信息及系统。</span>

<span style="color:#c62828;font-weight:700">🔴【扩展目标】：真实性、可控性、可追溯/可审计性、不可否认性等可按不同标准加入，但不是所有教材都把它们和 CIA 并列为同一固定“五目标”。</span>

⚫️【概念区分】：不可否认性强调参与者不能否认已实施的发送、签署等行为；可审计性强调行为可记录、追踪和审查，二者相关但不是同义词。

#### 考点2：威胁、脆弱性与风险

<span style="color:#c62828;font-weight:700">🔴【基本关系】：资产具有价值；威胁可能利用脆弱性造成影响；风险通常综合考虑发生可能性与后果。漏洞本身不等于一次已经发生的攻击。</span>

<span style="color:#1565c0;font-weight:700">🔵【风险处置】：规避、降低/缓解、转移、接受。安全措施应覆盖预防、检测、响应和恢复，不能只依赖单一产品。</span>

<span style="color:#c62828;font-weight:700">🔴【最小权限】：用户、程序和服务只获得完成任务所需的最少权限；【纵深防御】：在身份、终端、网络、应用、数据等多层部署相互补充的控制。</span>

### 第二章：恶意代码与社会工程

#### 考点3：病毒、蠕虫与木马

<span style="color:#c62828;font-weight:700">🔴【计算机病毒】：通常指编制或插入计算机程序中，破坏功能或数据、影响使用并能自我复制的一组指令或程序代码。病毒一般依附于宿主文件、引导区或文档宏等传播。</span>

<span style="color:#c62828;font-weight:700">🔴【常见特征】：传染/复制、寄生、潜伏、隐蔽、可触发和破坏等是教材常见归纳，但“固定七大特征”并非所有标准的唯一分类，选择题要结合题干来源。</span>

<span style="color:#c62828;font-weight:700">🔴【蠕虫】：可独立运行并利用网络或系统漏洞自我传播，通常不需要寄生传统宿主程序。</span>

<span style="color:#c62828;font-weight:700">🔴【特洛伊木马】：伪装为正常程序或功能诱导用户执行，常用于窃密、远控或下载其他恶意代码；其定义重点不是自我复制。</span>

<span style="color:#c62828;font-weight:700">🔴【勒索软件】：通过加密文件、锁定系统或窃取数据进行勒索；它是一种攻击目的/恶意软件类型，可能借助木马、漏洞或蠕虫能力传播，不能把所有勒索软件都归为蠕虫。</span>

<span style="color:#c62828;font-weight:700">🔴【逻辑炸弹】：隐藏在程序中，在特定时间或条件满足时触发恶意操作；它不同于用大量流量使服务不可用的 DoS/DDoS。</span>

<span style="color:#c62828;font-weight:700">🔴【宏病毒】：利用 Office 等应用的宏机制传播。现代 Office 格式中，含宏文档常见 `.docm/.xlsm`，旧格式也可能含宏；不能只背 `.doc/.xls/.dot`。</span>

#### 考点4：传播与防护

<span style="color:#1565c0;font-weight:700">🔵【常见入口】：钓鱼邮件和链接、恶意附件、盗版或被篡改软件、漏洞利用、弱口令、移动介质、供应链与远程服务。</span>

⚫️【设备表述】：显示器本身通常不存储并执行恶意代码；但扫描仪、打印机等现代设备可能接入网络、使用固件或配套软件，相关驱动、扫描文件或服务也可能存在安全风险。因此“使用扫描仪绝对不会感染”过于绝对。

<span style="color:#1565c0;font-weight:700">🔵【防护清单】：及时修补系统和软件；从可信来源安装软件；启用终端防护和防火墙；对附件、宏和链接保持警惕；使用多因素认证；重要数据按 3-2-1 思路备份，并保留离线或不可变副本；定期验证恢复能力。</span>

⚫️【杀毒局限】：安全软件可检测和阻断大量已知及部分未知威胁，但存在误报、漏报和更新滞后，不能承诺查杀所有恶意代码。

### 第三章：身份、访问与网络安全

#### 考点5：认证、授权与审计

<span style="color:#c62828;font-weight:700">🔴【认证 Authentication】：确认“你是谁”；【授权 Authorization】：确定“你能做什么”；【审计 Accounting/Auditing】：记录“你做了什么”。三者不能混淆。</span>

<span style="color:#c62828;font-weight:700">🔴【认证因素】：所知道的（密码）、所持有的（令牌/手机/安全密钥）、所具有的（指纹/人脸）。多因素认证要求来自不同因素类别，两个密码不算真正的双因素。</span>

<span style="color:#1565c0;font-weight:700">🔵【密码安全】：使用长且唯一的口令；不同站点不复用；借助可信密码管理器；服务端采用带盐的专用口令哈希算法存储，不应明文保存。</span>

#### 考点6：防火墙与入侵检测

<span style="color:#c62828;font-weight:700">🔴【防火墙】：依据安全策略控制不同网络区域或主机之间的通信，可实现包过滤、状态检测、应用层代理等功能。</span>

<span style="color:#c62828;font-weight:700">🔴【局限】：防火墙不能保证阻断所有攻击；对绕过防火墙的路径、合法账号滥用、允许流量中的攻击和加密流量存在可见性限制。</span>

⚫️【绝对化陷阱】：现代下一代防火墙可集成入侵防御、恶意代码检测等能力，路由器也可集成访问控制；因此“防火墙绝对不能防病毒”“防火墙绝对不是路由设备”都不严谨。设备名称不如实际功能和部署策略重要。

<span style="color:#c62828;font-weight:700">🔴【IDS 与 IPS】：IDS 侧重发现、告警；IPS 通常串联部署并可实时阻断。误报是把正常行为判为攻击，漏报是未发现真实攻击。</span>

#### 考点7：常见网络攻击

<span style="color:#c62828;font-weight:700">🔴【网络钓鱼】：冒充可信主体诱骗用户交出凭据、付款或执行恶意程序；鱼叉式钓鱼面向特定个人或组织。</span>

<span style="color:#c62828;font-weight:700">🔴【DoS/DDoS】：消耗带宽、连接或计算资源，使服务对正常用户不可用；DDoS 使用多个分布式来源。邮件炸弹可视为资源滥用方式之一，但逻辑炸弹不是 DoS 的同义词。</span>

<span style="color:#c62828;font-weight:700">🔴【中间人攻击】：攻击者介入通信并窃听或篡改数据；正确验证证书并使用 TLS 可降低风险。</span>

<span style="color:#c62828;font-weight:700">🔴【SQL 注入】：把恶意输入拼入 SQL 语句；核心防护是参数化查询/预编译、输入校验与最小数据库权限，不能只靠前端过滤。</span>

<span style="color:#c62828;font-weight:700">🔴【XSS】：恶意脚本在受害者浏览器执行；核心防护包括按输出上下文编码、内容安全策略和安全 Cookie 等。</span>

### 第四章：密码学、哈希与数字签名

#### 考点8：对称与非对称密码

<span style="color:#c62828;font-weight:700">🔴【对称加密】：加密和解密使用同一密钥或可相互推导的共享密钥，速度快，适合大量数据；难点是密钥安全分发与管理。常见算法有 AES。</span>

<span style="color:#c62828;font-weight:700">🔴【非对称加密】：使用公钥和私钥对，便于密钥分发和身份认证，但计算开销较大。常见体系有 RSA、ECC。</span>

<span style="color:#1565c0;font-weight:700">🔵【混合加密】：通常用非对称机制协商或保护会话密钥，再用对称算法加密大量通信数据。HTTPS/TLS 的具体流程随协议版本和密码套件变化，不能简化成唯一固定算法。</span>

#### 考点9：哈希、签名与证书

<span style="color:#c62828;font-weight:700">🔴【密码散列/哈希】：把任意长度数据映射为固定长度摘要，具有单向性和抗碰撞等目标；哈希不是可逆加密。</span>

<span style="color:#c62828;font-weight:700">🔴【数字签名基本流程】：发送者用私钥相关的签名算法对消息摘要签名，接收者用相应公钥验证。</span>

<span style="color:#1565c0;font-weight:700">🔵【签名提供】：来源认证、完整性校验和一定条件下的不可否认性；数字签名本身不保证消息保密，也不防止病毒入侵。</span>

<span style="color:#c62828;font-weight:700">🔴【数字证书】：由可信 CA 等签发，把主体身份与公钥等信息绑定；验证证书时还要检查有效期、域名/用途、签发链及吊销状态。</span>

⚫️【加密与签名方向】：为接收者保密常使用接收者公钥加密、接收者私钥解密；签名使用签名者私钥生成、签名者公钥验证。二者目标不同。

### 第五章：数据、个人信息与法律规范

#### 考点10：数据与个人信息保护

<span style="color:#c62828;font-weight:700">🔴【数据全生命周期】：收集、存储、使用、加工、传输、提供、公开、删除等环节都需落实安全与合规要求。</span>

<span style="color:#c62828;font-weight:700">🔴【个人信息】：以电子或其他方式记录的、与已识别或可识别自然人有关的各种信息，不包括匿名化处理后的信息。</span>

<span style="color:#1565c0;font-weight:700">🔵【个人信息处理原则】：合法、正当、必要和诚信；目的明确、合理并与处理目的直接相关；采取对个人权益影响最小的方式；公开透明；保证质量；采取安全保护措施。</span>

⚫️【同意不是唯一基础】：取得个人同意是重要处理依据，但《个人信息保护法》还规定履行合同、履行法定职责等其他法定情形；不能写成“任何处理都必须一律同意”。处理敏感个人信息通常要求特定目的、充分必要、严格保护，并依法取得单独同意等。

#### 考点11：现行法律框架与考试边界

<span style="color:#c62828;font-weight:700">🔴【高频法律】：</span>

《中华人民共和国网络安全法》是网络安全领域基础性法律；2025 年修正文本自 2026 年 1 月 1 日起施行。

《中华人民共和国数据安全法》规范数据处理活动，建立数据分类分级保护等制度。

《中华人民共和国个人信息保护法》保护自然人个人信息权益并规范个人信息处理活动。

《中华人民共和国著作权法》《计算机软件保护条例》保护计算机软件著作权；保护的是符合法律条件的表达，不是把思想、算法或功能概念一概垄断。

<span style="color:#c62828;font-weight:700">🔴【“5000/500”必须写全条件】：两高关于利用信息网络实施诽谤等刑事案件的司法解释中，对“利用信息网络诽谤他人”的“情节严重”列有数量标准：同一诽谤信息实际被点击、浏览 5000 次以上，或者被转发 500 次以上。它不是“所有网络谣言达到次数就自动构成犯罪”的通用公式，还要满足捏造事实诽谤他人、主观认识及刑法规定等构成条件，并结合案件事实认定。</span>

⚫️【时效提醒】：法律法规可能修订；答题时先看题干给出的法规名称、年份和行为条件。仅凭“违法/合法”标签，不分析主体、行为、主观状态、后果和适用条款，通常不够严谨。

<span style="color:#1565c0;font-weight:700">🔵【网络文明与职业伦理】：尊重知识产权和个人隐私；不越权访问；不传播违法有害信息；不制作、传播恶意代码；发现漏洞应按授权范围测试并负责任披露。</span>
