# 讲义、三色回忆、刷题模块说明

## 一、三个模块分别解决什么问题

### 1. 讲义：负责从零学懂

讲义解释定义、特点、组成、作用和常见区别，并补充“为什么—如何实现—什么条件下成立”。计算保留子网划分和公式直接代入等基础任务，不展开长推导。

### 2. 三色回忆：负责练简答

- 先看问题，口述或默写后再揭示答案。
- 🔴 最小答案：必须答出的核心结论。
- 🔵 得分关键词：简答题的展开顺序、要点和必要步骤。
- ⚫ 判断陷阱：成立条件、相近概念和典型错误说法。

三色回忆不重复讲义的长篇解释。答不出时返回讲义，理解后再合上答案重答。

### 3. 刷题：负责判断与纠错

题库用于选择、判断、填空、简答自评和基础计算，支持错题、收藏和到期复习。简答题先自行作答，再核对参考得分点并自评；系统不用字面完全相同作为判分标准。

## 二、统一的基础应用命题框架

1. **一步计算**：子网划分、串并联、容抗、速率、容量和单位换算，要求写出公式、代入和结论。
2. **场景选型**：根据业务操作选数据结构、协议、软件功能或电路，并说明关键特性。
3. **机制解释**：回答TCP为什么可靠有序、进程如何同步、事务如何保持一致等“怎样保证”问题。
4. **比较说明**：比较进程与程序、TCP与UDP、AM与FM、数组与链表，回答共同点、差异和适用条件。
5. **过程描述**：按顺序说明模拟量数字化、网络排障、中断处理、事件响应等流程及每步作用。

## 三、当前内容方向

15门课程统一按事业单位基础专业知识组织：

| 课程组 | 重点掌握 |
|---|---|
| Office、教学论、多媒体 | 组件与功能、常用操作、教学设计要素、媒体特点和格式区别 |
| 编程、数据结构 | C与Python基础概念、简单程序阅读、三种结构、常见数据结构的特点和用途 |
| 操作系统、组成原理 | 五大管理、进程线程、存储与I/O、五大部件、存储层次和常见接口 |
| 数据库、软件工程 | DB/DBMS/DBS区别、关系模型、SQL基本功能、生命周期、模型、测试和维护 |
| 计算机网络、信息安全 | 分层与协议作用、设备、常用命令、TCP/IP、CIA、攻击防护、备份和等保基础 |
| 电路、模电、数电、通信 | 元件和物理量、串并联、器件状态、反馈、逻辑门和触发器、通信方式与调制 |

基础识记与基础应用并重。子网划分、串并联、容抗、可靠度、简单数制等常见计算保留；步骤过长的调度、范式推导、状态机和综合电路计算不作为默认训练重点。

## 四、是否掌握的判断标准

1. 不看原文能说出定义和三到五个关键词。
2. 能区分一组相近概念，并说明判断依据。
3. 简答题能按“结论—原因或机制—适用边界”完整表达。
4. 判断题能指出错在哪里，并把错误表述改正确。
5. 必要的一步计算能写公式、代入数值、带单位得出结论。

## 五、推荐使用顺序

1. 用讲义学懂一个小节。
2. 打开三色回忆，只看问题先口述或默写。
3. 做对应科目的客观题、简答自评和基础应用题。
4. 错题按“概念不清、条件漏看、记忆混淆、基础计算失误”归因。
5. 到期后优先重答错题和三色回忆卡，而不是从头重读整篇讲义。
