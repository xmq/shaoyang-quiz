# 讲义、三色笔记、习题模块说明

## 一、为什么必须拆成三个模块

### 1. 讲义：负责“学懂”

讲义面向零基础，保留概念解释、原理推导、配图、典型例题和完整计算过程。第一次学习某章时应使用讲义，不能只背结论。

### 2. 三色笔记：负责“主动提取并核对”

- 问题与提示始终可见，学生必须先口述、默写、画图或说步骤，再点击揭示答案。
- 🔴 最小答案：必须直接写进答案的定义、定律、判据和核心结论。
- 🔵 输出方法：计算公式的用法、算法流程、操作顺序和主观题骨架。
- ⚫ 成立边界：适用条件、反例、版本口径和常见误判。

三色笔记不承担从零解释、长篇推导、完整例题或成套练习。第一次答不出是返回讲义的信号，而不是立刻把整页答案展开重读。

### 3. 习题：负责“检验与纠错”

习题模块按科目、章节、题型和来源筛选，承担闭卷回忆、计算训练、错题复习和模拟测试。答案解析集中放在题库，不再塞进三色速记正文。

## 二、当前完整性结论

拆分前的 `notes.html` 是“讲义+三色速记+随堂题”的混合体，不能算严格意义上的独立三色笔记；不同课程详略也不一致。

拆分后已建立15门独立三色笔记，并按“普通本科期末主干范围”组织以下主线：

| 课程组 | 已覆盖的大学期末主线 | 必须按本校确认的差异 |
|---|---|---|
| Office、教学论、多媒体 | Microsoft Office 与 WPS 的长文档、表格、演示文稿共通操作；教学设计与评价；媒体数字化与容量/压缩 | Office/WPS具体版本、课程标准版本、指定制作软件 |
| 编程语言 | C 与 Python 的基础语法、控制结构、函数、数组/容器、C 指针、程序阅读与调试 | C 标准/编译器、Python 3 小版本及题型比例 |
| 操作系统、数据库 | 调度/PV/死锁/分页/置换；SQL/范式/事务/并发/恢复 | 教材采用的算法细节、SQL方言和题型比例 |
| 数据结构、组成原理 | 复杂度、树图查找排序；补码、存储、Cache、指令、CPU、流水线、I/O | 是否考算法代码、微程序控制器或更深硬件设计 |
| 计算机网络、软件工程 | 子网、ARP/IP/TCP、路由与排障；过程、需求、设计、测试、维护与项目管理 | 协议细节、UML范围、成本估算模型 |
| 电路、模电、数电、通信 | 电路定理与动态/交流；器件与放大反馈；逻辑与时序；抽样调制、香农和高频接收 | 教师指定器件模型、教材章节、是否拆成两门考试 |
| 信息安全 | CIA、风险、密码与PKI、访问控制、网络/Web安全、备份应急 | 法律、等保、TLS和算法版本会更新 |

这些内容用于建立通用主干，不能据此宣称覆盖任意学校、任意教师的全部试卷。仍需用本校教学大纲、教师划重点、作业和往年卷校准范围；校准后缺少的考点应先补讲义，再补回忆卡和题目。

## 三、判断一门三色笔记是否真的完整

每门笔记按章节提供回忆卡。满足以下四点才算掌握：

1. 基本定义能不看答案说出，并能区分相近概念。
2. 高频计算题能写公式、代入、单位和结论。
3. 分析/设计题能按标准步骤展开，而不是只记最终选项。
4. 能指出公式条件和至少一个典型陷阱。

若清单中有一项做不到，应回到讲义对应章节学习，再进入习题模块练习。

## 四、推荐使用顺序

1. **讲义**：零基础逐章学习，理解概念、图和例题。
2. **三色笔记**：保持答案遮挡，根据问题先输出；再用红色核对答案、蓝色核对方法、黑色核对成立条件。
3. **习题训练**：先做“大学期末改编”和“大学期末原创”，再做对应章节题和错题；改编题答题后可查看题型参考与改编说明。
4. **模拟小卷**：限时完成，按“概念不清、公式不会、条件漏看、计算失误”归因。
